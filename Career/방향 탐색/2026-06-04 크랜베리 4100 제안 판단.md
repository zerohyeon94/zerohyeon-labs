## 1. 결론

이번에 새로 확인된 조건을 반영하면 판단은 더 명확해진다.

```text
크랜베리:
- 연봉 4,100만 원
- 개발직
- React/Go 플랫폼 개발
- 정규직 성격의 개발 경력으로 남을 가능성

넥스트러너스:
- 계약직
- 개발직이 아님
- 조교/운영/자료 관리 중심
- 정규직 전환과 역할 확장 여부가 불확실
```

따라서 현재 기준의 객관적 판단은 아래다.

```text
헬스케어에 관심이 있더라도,
지금은 넥스트러너스 계약직 비개발직보다
크랜베리에서 개발직 경력을 유지하는 쪽이 미래성이 더 크다.
```

단, 이 선택의 의미는 `React/Go 개발자로 전향한다`가 아니다.

```text
정규직 개발 경력을 복구하고,
AIoT/AIOps 플랫폼 경험을 얻은 뒤,
iOS + 헬스케어 AI 방향으로 다시 연결한다.
```

## 2. 왜 크랜베리 쪽 점수가 올라갔는가

### 1. 연봉 4,100만 원은 현재 경력 상태에서 방어 가능한 제안이다

현재 상태는 아래처럼 이력서 해석이 복잡하다.

```text
녹십자MS 3개월: 개발보다 다른 업무 비중이 큼
공백 1년
iOS 개발경력 1년 6개월
Python/AI 학습 약 1년
현재 넥스트러너스 계약직 조교
```

이 상태에서 4,100만 원 개발직 제안은 단순히 돈만의 문제가 아니다.

```text
정규직 개발자 시장으로 다시 진입하는 티켓
개발 직무명 회복
이후 이직 시 연봉 기준선 확보
계약직/비개발직으로 경력이 더 흐려지는 것을 막는 장치
```

### 2. 넥스트러너스는 헬스케어와 가깝지만 개발 경력은 약하다

넥스트러너스는 AI 헬스케어 과정 안에 있다는 장점이 있다.
하지만 현재 업무는 개발이 아니라 교육 운영/조교 업무에 가깝다.

```text
장점:
- AI 헬스케어 과정 이해
- Python/AI 학습 유지
- 수강생 피드백과 교육 콘텐츠 경험

한계:
- 개발직 경력으로 강하게 인정받기 어렵다
- 계약직이다
- 정규직 전환 기준이 불확실하다
- 전환되어도 바로 개발자가 아니라 교육 운영/콘텐츠 쪽일 가능성이 크다
```

헬스케어에 관심이 있다는 이유만으로 개발직을 포기하면, 1-2년 뒤 이력서에서 `개발자`보다 `교육 운영/조교`가 더 커질 수 있다.

### 3. 크랜베리는 헬스케어와 직접 연결되지는 않지만 개발자 경력을 살린다

크랜베리의 React/Go는 영현님의 최종 목표 스택은 아니다.
하지만 `개발직 유지`라는 면에서는 강하다.

```text
React:
- 관제/관리자 화면
- 데이터 시각화
- 제품 UI

Go:
- API 서버
- 이벤트/로그/상태 데이터
- 장비/센서 데이터 흐름
- 백엔드 구조 이해
```

이 경험은 나중에 헬스케어 모바일로 돌아갈 때 아래처럼 연결할 수 있다.

```text
AIoT/AIOps 플랫폼에서 센서/이벤트/알림 데이터를 다룬 개발자
헬스케어 앱에서 디바이스/센서/AI 분석 결과를 제품으로 연결할 수 있는 모바일 개발자
```

## 3. 선택지 비교

| 기준 | 크랜베리 | 넥스트러너스 |
|---|---|---|
| 직무명 | 개발직 | 계약직 조교/운영 |
| 연봉/기준선 | 4,100만 원 | 계약직 기준, 향후 기준선 약함 |
| 개발 경력 유지 | 강함 | 약함 |
| 헬스케어 직접성 | 약함 또는 간접 | 강함 |
| iOS 직접성 | 없음 | 없음 |
| 2-3년 후 이직 설명 | 플랫폼 개발 + AIoT 데이터 경험 | 교육 운영 + 콘텐츠 보조 |
| 시장에서의 방어력 | 상대적으로 좋음 | 개발직 이직에는 약함 |
| 리스크 | React/Go 일반 웹으로 굳어질 위험 | 비개발 경력으로 굳어질 위험 |

핵심은 둘 다 iOS 직접 경력은 아니라는 점이다.

그래서 더 중요한 질문은 아래다.

```text
둘 중 어느 선택이 1년 뒤 개발자로 다시 설명하기 쉬운가?
```

이 질문에 대한 답은 크랜베리다.

## 4. 크랜베리를 선택할 때 반드시 지켜야 할 조건

크랜베리를 선택하더라도 아래를 지키지 않으면 방향이 흐려진다.

```text
1. 내 정체성을 React/Go 개발자로 바꾸지 않는다.
2. iOS 코드를 주 2회 이상 반드시 유지한다.
3. 회사 업무는 AIoT/AIOps 데이터 플랫폼 경험으로 기록한다.
4. 6개월 안에 iOS + 헬스케어 + AI 포트폴리오를 병행한다.
5. 1년 뒤 이력서 제목은 "플랫폼 개발자"가 아니라 "헬스케어 AI 모바일 개발자 후보"로 만든다.
```

크랜베리에서 기록해야 할 업무 표현:

```text
센서/이벤트/로그 데이터 흐름
관제 대시보드
Go API 서버
실시간 알림
장비 상태 관리
AI 탐지 결과를 사용자 화면에 연결한 경험
```

피해야 할 표현:

```text
React 화면 개발
Go CRUD API
웹사이트 개발
관리자 페이지 유지보수
```

## 5. 1년 뒤 목표 이력서 문장

크랜베리를 잘 활용했을 때 1년 뒤 이력서 문장은 아래가 되어야 한다.

```text
iOS 개발 경험과 AIoT/AIOps 플랫폼 개발 경험을 바탕으로,
센서/이벤트 데이터를 제품 기능으로 연결할 수 있는 헬스케어 AI 모바일 개발자입니다.
```

이 문장이 만들어지면, 이후 목표 회사는 아래로 잡을 수 있다.

```text
디지털 헬스케어 iOS 개발자
의료기기/웨어러블 연동 모바일 개발자
On-device AI 모바일 개발자
AI 헬스케어 앱 개발자
헬스케어 데이터 플랫폼을 이해하는 모바일 개발자
```

## 6. 최종 판단

현재 정보만 놓고 보면 크랜베리 선택이 더 현실적이다.

```text
이유:
1. 4,100만 원은 현재 경력 상태에서 개발직 기준선을 회복하는 제안이다.
2. 넥스트러너스는 헬스케어와 가깝지만 현재 업무는 개발직이 아니다.
3. 계약직 비개발 업무가 길어지면 개발자 경력 복구가 더 어려워진다.
4. React/Go라도 실제 개발직이면 1년 뒤 이직 시장에서 설명할 수 있다.
5. 단, iOS를 병행하지 않으면 React/Go 일반 플랫폼 개발자로 굳어질 위험이 있다.
```

한 줄 결론:

```text
크랜베리로 가는 것이 더 낫다.
다만 목표는 React/Go 전향이 아니라, 개발직 경력 복구 후 iOS + 헬스케어 AI로 재연결하는 것이다.
```
