## 1. 결론

2-3년 후 목표 포지션은 아래로 잡는 것이 가장 좋다.

```text
헬스케어 AI 모바일 제품 개발자
```

조금 더 구체적으로는:

```text
iOS를 중심으로,
On-device AI, Core ML, Vision, Apple Foundation Models,
헬스케어 데이터/API, 의료기기/센서 연동을 이해하는 제품 개발자
```

React/Go는 이 목표의 중심이 아니다. 보조 경험이다.

---

## 2. 왜 이 포지션인가

현재 강점은 아래 조합이다.

```text
의료기기 경험
헬스케어 앱 경험
iOS 개발경력 1년 6개월
Kotlin/Android 경험
Python/AI 헬스케어 학습
교육/멘토링 경험
```

이 조합을 살리면 경쟁자가 적어진다.

반대로 React/Go 웹 풀스택으로 가면 경쟁자가 많아진다.

```text
React/Go 웹 풀스택:
- 경쟁자 많음
- 기존 헬스케어/iOS 경력 희석
- 2-3년 후에도 범용 개발자 포지션

iOS + On-device AI + 헬스케어:
- 경쟁자 적음
- 기존 경력 연결 가능
- 의료기기/헬스케어 도메인 차별화 가능
```

---

## 3. 최종 포지션 이름 후보

이력서와 채용시장에 맞춰 아래 중 하나로 포지셔닝한다.

```text
1순위: iOS 헬스케어 AI 개발자
2순위: On-device AI 모바일 개발자
3순위: 헬스케어 모바일 제품 개발자
4순위: AI 헬스케어 앱 개발자
5순위: 모바일 + AI 제품 개발자
```

가능하면 피해야 할 포지션명:

```text
React 개발자
Go 개발자
웹 풀스택 개발자
교육 운영 담당자
조교
```

이 이름들은 현재 경력의 희소성을 약하게 만든다.

---

## 4. React/Go를 어떻게 써야 하는가

React/Go를 완전히 버릴 필요는 없다. 다만 목적이 달라야 한다.

```text
목적지: React/Go 개발자
→ 비추천

수단: 제품 구조, API, 백엔드, 관리자 대시보드 이해
→ 가능
```

React 경험은 아래에만 연결한다.

```text
1. 헬스케어 관리자 대시보드
2. 의사용/운영자용 웹 화면
3. iOS 앱과 연결되는 백오피스
4. React Native 확장 가능성
```

Go 경험은 아래에만 연결한다.

```text
1. 실시간 관제 이벤트 처리
2. 센서/장비 상태 API
3. 알림/로그/이벤트 파이프라인
4. 제품 서버 구조 이해
```

그 외 단순 웹 화면과 CRUD라면, 2-3년 후 목표 포지션에 큰 도움이 되지 않는다.

## 5. 2-3년 로드맵

### 0-6개월

목표:

```text
정규직 개발 경력 또는 교육 콘텐츠 산출물 중 하나를 확실히 만든다.
iOS 감각을 잃지 않는다.
```

해야 할 일:

```text
1. iOS 주 2회 코드 작성
2. SwiftUI/Core ML/Vision 작은 기능 1개 구현
3. Python/FastAPI로 헬스케어 API 1개 만들기
4. 현재 업무에서 배운 것을 헬스케어 제품 관점으로 기록
```

### 6-12개월

목표:

```text
iOS + AI + 백엔드가 연결된 포트폴리오 1개를 만든다.
```

포트폴리오 예시:

```text
헬스케어 집중/피로도 분석 앱
사용자 상태 입력 + 온디바이스 분석 + 서버 저장 + 리포트
SwiftUI + Core ML/Vision/Foundation Models + FastAPI
```

### 1-2년

목표:

```text
헬스케어 AI 모바일 개발자로 이직 가능한 이력서 구조를 만든다.
```

필요한 증거:

```text
1. iOS 앱 출시 또는 TestFlight 수준 앱
2. On-device AI 기능
3. 헬스케어 데이터/API 연동
4. 의료기기/센서/인허가 이해를 설명할 수 있는 문서
5. 기술 블로그 5개 이상
```

### 2-3년

목표 포지션:

```text
iOS 헬스케어 AI 개발자
On-device AI 모바일 개발자
헬스케어 모바일 제품 개발자
```

지원할 회사 유형:

```text
1. 디지털 헬스케어 스타트업
2. 의료기기 + 앱 연동 회사
3. 병원/검진/원격의료/건강관리 플랫폼
4. AI 헬스케어 제품 회사
5. 모바일 SDK/센서/웨어러블 회사
```

## 6-1. 이직할 때 검색해야 할 키워드

크랜베리 근무 후 이직을 검색할 때 `플랫폼개발자` 하나로 검색하면 너무 넓다. 이 키워드는 백엔드 플랫폼, 데이터 플랫폼, 인프라 플랫폼, 사내 플랫폼, 웹 서비스 플랫폼까지 모두 섞인다.

기본 검색축은 아래처럼 잡는다.

### 1순위 검색어

```text
iOS 개발자 헬스케어
iOS 개발자 AI
iOS Core ML
iOS Vision
On-device AI iOS
모바일 AI 개발자
헬스케어 모바일 개발자
디지털 헬스케어 iOS
의료기기 앱 개발자
웨어러블 iOS 개발자
```

### 2순위 검색어

```text
AI 모바일 앱 개발자
AI 헬스케어 앱 개발자
SwiftUI Core ML
Swift Vision
Apple Intelligence 개발자
Foundation Models iOS
모바일 SDK 개발자 iOS
센서 데이터 모바일 개발자
```

### 크랜베리 경험을 살릴 때 검색어

```text
AIoT 플랫폼 개발자
AIOps 플랫폼 개발자
관제 플랫폼 개발자
센서 데이터 플랫폼 개발자
실시간 이벤트 플랫폼 개발자
IoT 플랫폼 개발자
디지털 헬스케어 플랫폼 개발자
```

단, 이 검색어들은 `메인`이 아니라 보조축이다. 검색 결과에서 아래 조건이 있어야 영현님에게 맞다.

```text
1. 모바일 앱 또는 iOS와 연결된다.
2. 헬스케어/의료기기/센서/웨어러블 도메인이 있다.
3. AI 모델 결과를 제품에 적용한다.
4. 단순 웹 관리자 페이지가 아니다.
```

### 피해야 할 검색어

```text
React 개발자
Go 개발자
웹 프론트엔드 개발자
웹 풀스택 개발자
일반 플랫폼 개발자
운영 플랫폼 개발자
사내 시스템 개발자
```

이 검색어들은 영현님의 기존 차별점을 약하게 만든다.
