## 1. 자기소개 (60초)

```text
저는 의료기기 펌웨어 개발로 시작해 iOS/Android 헬스케어 앱 개발로 전환한
모바일 개발자 조영현입니다. 총 4년 8개월의 경력이 있습니다.

녹십자엠에스에서는 혈당기, HbA1c 측정기의 펌웨어 유지보수와 BLE 연동
모바일 앱을 개발했고, 이미지 처리 알고리즘으로 자가진단 키트 결과를 자동

이후 스마트사운드에서는 BLE 청진기와 연동되는 실시간 건강 모니터링 iOS 앱을
메인으로 개발했습니다. 실시간 센서 데이터 수신·처리·시각화와 AI 분석 API
연동까지 경험했습니다.

현재는 AI 헬스케어 과정에서 Python 기반 데이터 분석과 AI 모델링을 학습하며
역량을 확장하고 있습니다.

크랜베리에서는 실시간 센서 데이터를 기반으로 이상을 감지하고 제품화하는
흐름에 관심이 생겨 지원했습니다.
```

## 2. 예상 질문 & 준비된 답변

### Q1. 스마트사운드에서 왜 나오셨나요?

```text
스마트사운드는 헬스케어 분야의 좋은 경험이었고,
BLE 청진기 연동부터 AI 분석 결과 화면 구성까지 앱 전체 사이클을 경험했습니다.

다만 1년 7개월 근무 후 회사 사정으로 계약이 마무리되었고,
이후 AI 헬스케어 과정을 통해 Python과 데이터 분석, AI 모델링 역량을
보완하는 시간을 가졌습니다.

이제 지금까지의 센서 데이터 처리 경험과 AI 역량을 결합해서
실제 제품으로 이어지는 데이터 분석 시스템에 기여하고 싶습니다.
```

> **주의**: 부정적 이직 사유(연봉, 갈등 등)는 언급하지 않는다.

### Q2. BLE/IoT 연동 경험을 구체적으로 말해주세요.

```text
두 회사에서 총 4개 이상의 BLE 연동 기기를 다뤘습니다.

녹십자엠에스에서는:
- 혈당기 BLE 연동 앱 개발 (CoreBluetooth, Swift/Java)
- 실시간 측정값 수신 및 그래프 시각화
- 자가진단 키트 앱에서 이미지 처리 알고리즘으로 결과 자동 판독

스마트사운드에서는:
- 사람용 BLE 청진기 연동 — 실시간 청진 데이터 수신 및 처리
- AES-128 CCM 암호화된 데이터 복호화 처리 (민감 의료 데이터 보안)
- 동물용 청진기 앱의 DFU(펌웨어 업데이트) 기능 개발

특히 청진 데이터는 실시간성이 중요해서, 수신→복호화→시각화→AI 분석
요청까지의 흐름을 지연 없이 처리하는 데 집중했습니다.

크랜베리의 CCTV 음원 데이터도 실시간 수집과 처리가 핵심일 텐데,
기기에서 데이터를 받아 분석 파이프라인으로 넘기는 흐름은 제가
경험한 BLE 센서 처리와 구조적으로 유사하다고 생각합니다.
```

### Q3. AI 분석 API 연동 경험이 있나요?

```text
스마트사운드에서 청진 데이터를 AI 서버로 전송하고,
분석 결과(건강 평가 등급, 이상 감지 여부)를 받아 화면에 표현하는
전체 흐름을 구현했습니다.

AI 모델 자체를 개발하기보다는 모델의 결과를 사용자 경험으로
연결하는 역할을 했고, 분석 결과 데이터 구조가 바뀔 때
CoreData 마이그레이션으로 앱 안정성을 유지한 경험도 있습니다.

현재 AI 헬스케어 과정에서는 Python으로 직접 모델을 학습하고,
CoreML로 변환해서 앱에 탑재하는 On-Device AI까지 경험하고 있어
AI와 제품 사이의 연결을 더 깊게 이해하고 있습니다.
```

### Q4. 이미지 처리나 신호 처리 경험이 있나요?

```text
녹십자엠에스에서 자가진단 키트 자동 판독 앱을 개발할 때,
CoreImage 기반으로 키트 이미지를 분석하여 결과를 자동 판독하는
알고리즘을 구현했습니다. 이 기능으로 사용자 오입력 오류율을
크게 줄였습니다.

음원 신호처리는 이번 크랜베리에서 처음 접하는 영역이지만,
센서 데이터에서 패턴을 찾아 분류하는 기본 개념은 이미지 처리와
구조적으로 유사하고, Python/AI 과정에서 시계열 데이터 분석도
다뤘기 때문에 빠르게 적응할 수 있다고 생각합니다.
```

### Q5. Python 역량이 어느 정도인가요?

```text
AI 헬스케어 과정을 통해 Pandas, NumPy, Scikit-learn, 시각화 라이브러리를
활용한 데이터 분석과 모델링을 학습했습니다.

실제로 의료 데이터를 기반으로 분류 모델을 만들고, 결과를 평가하는
실습 프로젝트를 진행했으며, FastAPI로 간단한 API 서버를 구성한 경험도 있습니다.

현재 수준은 데이터 전처리, 모델 학습 및 평가, API 연동 정도이며
음원 데이터나 시계열 이상 탐지 쪽은 이 포지션에서 더 깊게 배우고 싶습니다.
```

### Q6. 이 포지션에 지원한 이유가 무엇인가요?

```text
단순히 AI 모델을 학습하는 것보다,
실시간으로 수집되는 센서 데이터에서 이상을 감지하고
실제 운영자나 사용자가 바로 반응할 수 있는 시스템으로
이어지는 흐름에 관심이 있습니다.

지금까지 BLE 청진기, 혈당기, 진단 키트 등 현장 기기에서
데이터를 받아 처리하고 사용자에게 의미 있는 화면으로 연결하는
경험을 쌓았는데, 크랜베리의 CCTV 음원 분석 시스템도 비슷한
흐름을 가지고 있다고 생각했습니다.

특히 조달청 우수제품으로 선정된 음원+영상 멀티모달 관제 시스템이
실제 공공/B2B 현장에 납품되는 제품이라는 점이 흥미로웠습니다.
```

### Q7. 약점 or 부족한 부분이 있다면?

```text
음원 신호처리나 오디오 도메인은 이번이 처음입니다.
FFT, MFCC 같은 음향 특성 분석 기술은 아직 깊게 다루지 않았습니다.

다만 센서 데이터 처리, 실시간 데이터 파이프라인, AI 결과 연동은
경험이 있어서 도메인 지식은 빠르게 채울 수 있다고 생각합니다.
입사 전후로 오디오 신호처리 기초부터 스스로 준비하겠습니다.
```

## 5. 이력서에서 크랜베리와 연결되는 강점 요약

면접관이 보는 시각으로 정리한 나의 강점:

| 내 경험 | 크랜베리 업무와의 연결 |
|---------|----------------------|
| BLE 청진기 실시간 데이터 수신·처리 | 실시간 음원 센서 데이터 처리 구조 이해 |
| 이미지 처리 알고리즘으로 자동 판독 | 신호에서 패턴 탐지 → 이상 감지 개념 연결 |
| AI 분석 API 연동 및 결과 표현 | AI 모델 결과를 시스템에 연동하는 경험 |
| AES 암호화 처리 | 보안 요구 있는 시스템 경험 |
| Python + FastAPI + Docker | 데이터 파이프라인/백엔드 기초 역량 |
| 의료기기 인허가 (DHF, DMR, GMP) | 공공/B2B 규제 환경 이해 (조달청 제품) |
