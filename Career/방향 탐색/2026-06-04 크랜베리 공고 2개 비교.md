## 1. 비교 대상

### A. React/Go 플랫폼 개발

```text
React 기반 웹/플랫폼 화면
Go 기반 백엔드/API
관제/관리자/데이터 표시 화면
백엔드 서버로부터 데이터 수신
```

### B. Python/AI/R&D 소프트웨어 개발

```text
음원 및 비디오 데이터 전처리 파이프라인 개발
음향/영상 기반 이상 감지 시스템 유지보수 및 기능 개발
AI 모델 개발, 학습, 검증 및 실증 테스트
오디오/비디오 데이터 수집, 정제 및 관리
임베디드 환경 기반 데이터 처리 및 시스템 연동
R&D 과제 제안서/보고서/기술문서 작성 지원
특허 출원 관련 기술자료 정리
```

---

## 2. 결론

영현님의 2-3년 후 미래 가능성만 보면 `B. Python/AI/R&D 소프트웨어 개발`이 더 좋다.

이유:

```text
1. Python/AI/데이터/음향/영상/임베디드는 React/Go 웹 플랫폼보다 희소성이 높다.
2. 영현님의 Python/AI 학습과 더 직접 연결된다.
3. 의료기기/헬스케어/센서/On-device AI 방향으로 확장하기 쉽다.
4. iOS + AI + 헬스케어 포지션으로 돌아갈 때 설명할 소재가 많다.
5. 단순 웹 개발자가 아니라 AI 제품화 경험자로 포지셔닝할 수 있다.
```

단, 조건이 있다.

```text
실제 업무의 60% 이상이 Python 개발, 데이터 파이프라인, AI 모델 테스트, 시스템 연동이어야 한다.
문서/정부과제/특허/출장 지원이 대부분이면 미래 가능성은 크게 떨어진다.
```

---

## 3. 영현님에게 더 유리한 이유

현재 목표 포지션은 아래다.

```text
iOS 헬스케어 AI 개발자
On-device AI 모바일 개발자
헬스케어 모바일 제품 개발자
의료기기/웨어러블 연동 모바일 개발자
```

이 목표와의 연결성을 보면:

| 기준 | React/Go 플랫폼 | Python/AI/R&D |
|---|---|---|
| 개발직 유지 | 좋음 | 좋음 |
| AI 직접성 | 약함 | 강함 |
| 데이터 파이프라인 | 보통 | 강함 |
| 음향/영상/센서 | 약함 | 강함 |
| 임베디드/디바이스 | 약함 | 있음 |
| 헬스케어 확장성 | 간접 | 강함 |
| iOS 복귀 연결성 | 보조 | Core ML/Vision/On-device AI로 연결 가능 |
| 시장 희소성 | 보통 | 높음 |
| 단기 적응 난이도 | 낮거나 보통 | 높음 |
| 문서/과제 리스크 | 낮음 | 높음 |

## 4. 리스크

### React/Go 플랫폼 개발 리스크

```text
일반 웹/플랫폼 개발자로 굳어질 수 있다.
React 공급이 많아 차별화가 약하다.
Go를 깊게 하지 못하면 백엔드 경력으로도 애매해질 수 있다.
iOS/헬스케어/AI와의 연결을 별도로 만들어야 한다.
```

### Python/AI/R&D 리스크

```text
AI 모델 개발이라고 쓰여 있지만 실제로는 데이터 정리/문서/보고서가 많을 수 있다.
정부과제/제안서/특허 지원이 많으면 개발 경력으로 약해질 수 있다.
경력 2년 이상 조건이라 입사 후 학습 부담이 클 수 있다.
iOS 코딩을 별도로 유지하지 않으면 모바일 개발 감각이 떨어질 수 있다.
```

## 5. 면접 또는 입사 전 확인 질문

Python/AI/R&D 공고를 선택하려면 아래 질문의 답을 받아야 한다.

```text
1. 실제 업무에서 Python 코드 작성 비중은 어느 정도인가요?
2. AI 모델을 직접 학습/검증/튜닝하나요, 아니면 결과만 테스트하나요?
3. 음원/영상 데이터 전처리 파이프라인은 직접 설계하고 구현하나요?
4. 임베디드 시스템 연동은 어떤 장비/환경에서 이루어지나요?
5. 정부과제 문서/보고서/특허 지원 비중은 전체 업무의 몇 %인가요?
6. 출장 업무는 월 몇 회 정도인가요?
7. 입사 후 3개월 동안 맡게 될 첫 기능 또는 과제는 무엇인가요?
```

판단 기준:

```text
Python/AI/데이터/시스템 연동 60% 이상 → Python/AI/R&D 선택
문서/보고서/출장/과제지원 40% 이상 → 신중
문서/보고서/출장/과제지원 60% 이상 → React/Go 플랫폼이 더 안전
```
