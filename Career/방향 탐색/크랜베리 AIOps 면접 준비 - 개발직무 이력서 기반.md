## 1. 면접의 목적

이번 면접은 단순히 합격 여부를 확인하는 자리가 아니다.

가장 중요한 목적은 아래 하나다.

```text
크랜베리의 AIOps 포지션이
"AI/IoT/센서 데이터 기반 제품을 개발하는 일"인지,
아니면 "관제 운영/고객사 기술지원/모니터링 업무"인지 확인한다.
```

합류를 긍정적으로 볼 수 있는 방향:

```text
AI 기반 이상 탐지
음원/영상/센서 데이터 처리
실시간 이벤트 분석
엣지 AI 또는 클라우드 AI 제품 개발
데이터 파이프라인/백엔드/API 개발
제품 기능으로 남는 개발 산출물
```

조심해야 하는 방향:

```text
단순 관제 모니터링
고객사 장애 접수/전달
AI 모델은 다루지 않고 설정만 변경
데이터 라벨링/엑셀 정리 중심
야간/장애 대응 중심 운영직
```

## 2. 나의 면접 포지셔닝

크랜베리에서 나를 `순수 iOS 개발자`로만 소개하면 강점이 좁아진다.

더 좋은 포지셔닝은 아래다.

```text
의료기기와 센서 기반 데이터를 다뤄봤고,
BLE/펌웨어/모바일 앱/AI 분석 API/헬스케어 모델링을 연결해본 개발자
```

한 문장으로는 이렇게 말할 수 있다.

```text
저는 현장 장비에서 발생하는 데이터를 앱과 서버, AI 분석 결과까지 연결해본 경험이 있는 개발자입니다.
```

크랜베리와 연결되는 나의 강점:

```text
1. BLE 의료기기 데이터 수신/처리 경험
2. AES-128 CCM 복호화 등 민감 데이터 보안 처리 경험
3. 실시간 데이터 시각화와 사용자 화면 구현 경험
4. AI 분석 API 연동과 결과 화면/UX 개선 경험
5. 펌웨어 유지보수, Calibration, 생산/반품 분석 경험
6. Python 기반 AI 모델링과 FastAPI 백엔드 경험
7. Celery/Redis, Docker, Locust 등 비동기 처리와 성능 테스트 경험
8. Core ML/Vision 기반 On-device AI 앱 구현 경험
```

## 3. 자기소개 답변

### 30초 버전

```text
안녕하세요. 저는 의료기기 펌웨어와 헬스케어 모바일 앱 개발을 경험한 조영현입니다.
녹십자엠에스에서는 혈당기, 자가진단 키트, 측정기 펌웨어와 생산/품질 문서를 다뤘고,
스마트사운드에서는 BLE 청진기와 연동되는 iOS 앱에서 실시간 데이터 수신, 암호화 데이터 복호화, AI 분석 결과 화면을 구현했습니다.
최근에는 AI 헬스케어 과정에서 XGBoost 모델링, FastAPI, Celery/Redis, Docker 기반 서비스를 경험했고,
Core ML과 Vision을 활용한 iOS 집중도 분석 앱도 구현하고 있습니다.
현장 장비에서 발생하는 데이터를 AI 분석과 사용자 서비스로 연결하는 흐름에 관심이 있어 이번 포지션에 관심을 갖게 되었습니다.
```

### 1분 버전

```text
저는 첫 경력을 의료기기 펌웨어와 생산/품질 업무에서 시작했습니다.
측정기 Calibration, 부품 변경 검증, 반품 원인 분석, GMP/인허가 문서 관리까지 경험하면서
현장 장비와 데이터 품질이 실제 서비스 신뢰도에 얼마나 중요한지 배웠습니다.

이후 모바일 개발로 전환해 BLE 혈당기, BLE 청진기, 자가진단 키트 앱을 개발했습니다.
특히 스마트사운드에서는 CoreBluetooth 기반 실시간 청진 데이터 수신, AES-128 CCM 복호화,
Charts 기반 시각화, AI 분석 API 연동, 병원/연구용 데이터 업로드 기능을 구현했습니다.

최근에는 AI 헬스케어 과정을 통해 Python 모델링과 FastAPI 백엔드도 경험했습니다.
XGBoost/SHAP 기반 건강 위험도 예측, Celery/Redis AI Worker, Docker Compose, Locust 성능 테스트를 다뤘고,
iOS 쪽으로는 Vision/Core ML 기반 집중도 분석 앱을 만들며 On-device AI 흐름을 학습하고 있습니다.

그래서 저는 단순 앱 개발자라기보다, 센서/장비 데이터가 수집되고 AI 분석을 거쳐 제품 기능으로 이어지는 흐름을 이해하는 개발자로 기여할 수 있다고 생각합니다.
```

## 4. 지원 동기 답변

```text
크랜베리가 AI와 IoT를 결합한 관제/안전 솔루션을 만든다는 점에 관심이 갔습니다.
제가 해왔던 의료기기와 헬스케어 앱 개발도 결국 현장 장비에서 발생한 데이터를 정확히 수집하고,
사용자가 이해할 수 있는 결과로 전달하는 일이었습니다.

크랜베리의 음원/영상/센서 기반 이상 탐지나 실시간 관제 솔루션도
데이터 수집, 분석, 판단, 사용자 전달이라는 흐름이 중요하다고 생각했습니다.

저는 BLE 기기 데이터 처리, 암호화 데이터 복호화, AI 분석 결과 연동, Python 기반 모델링과 백엔드 경험이 있어서
AIoT 제품이 실제 현장에서 동작하는 과정에 기여하고 싶습니다.
```

주의:

```text
"iOS 시장이 작아져서 AIOps를 보게 됐다"라고 말하지 않는다.
"현장 데이터와 AI 제품화에 관심이 생겼다"로 말한다.
```

## 5. 프로젝트별 연결 답변

### 1. 스마트사운드 - 실시간 BLE 청진 앱

면접에서 가장 먼저 꺼낼 프로젝트다.

```text
스마트사운드에서는 BLE 청진기와 연동되는 iOS 앱을 개발했습니다.
CoreBluetooth로 청진 데이터를 실시간 수신하고, AES-128 CCM으로 암호화된 데이터를 복호화해 Charts로 시각화했습니다.
또 AI 분석 API와 연동해 사용자가 건강 상태를 확인할 수 있는 화면을 구현했습니다.

이 경험을 통해 센서 데이터는 수집 자체보다 통신 안정성, 데이터 보안, 실시간 처리, 사용자에게 보여주는 방식이 함께 중요하다는 것을 배웠습니다.
크랜베리의 음원/센서 기반 관제 시스템도 비슷하게 데이터 수집부터 분석 결과 전달까지의 안정성이 중요할 것이라고 생각합니다.
```

강조 키워드:

```text
BLE
실시간 데이터
암호화/복호화
AI 분석 API
데이터 시각화
의료/헬스케어 도메인
```

### 2. 녹십자엠에스 - 펌웨어/생산/Calibration

크랜베리가 IoT/현장 장비를 다룬다면 이 경험이 중요하다.

```text
녹십자엠에스에서는 HbA1c와 LIPID 측정기의 펌웨어 유지보수, Calibration, 부품 변경 검증, 반품 원인 분석을 담당했습니다.
현장 장비는 소프트웨어만 안정적이어서는 부족하고, 센서/부품/펌웨어/생산 품질이 함께 맞아야 신뢰할 수 있다는 것을 배웠습니다.

크랜베리도 AIoT 관제 솔루션을 다룬다면 하드웨어와 소프트웨어, 현장 설치 환경을 함께 이해하는 관점이 필요할 것이라고 생각합니다.
```

강조 키워드:

```text
펌웨어
센서/측정기
Calibration
반품 원인 분석
현장성
품질 문서
```

### 3. MyHealthBuddy - AI 모델 + 백엔드

AIOps가 Python/백엔드/데이터 파이프라인이라면 이 프로젝트를 말한다.

```text
MyHealthBuddy 프로젝트에서는 건강검진 수치 기반 심혈관 위험도 예측 서비스를 만들었습니다.
XGBoost 모델과 SHAP 해석 결과를 FastAPI 서버에서 제공하고,
Celery와 Redis로 AI Worker를 분리해 비동기 처리 구조를 만들었습니다.
Docker Compose 환경 구성과 Locust 성능 테스트도 진행했습니다.

특히 EC2 환경에서 XGBoost predict_proba 이후 무한 대기 문제가 발생했는데,
단계별 로깅으로 블로킹 지점을 찾고 OMP_NUM_THREADS=1 설정으로 OpenMP fork deadlock을 해결했습니다.
이 경험을 통해 AI 모델은 학습만큼이나 배포 환경, 프로세스 모델, 성능 테스트가 중요하다는 것을 배웠습니다.
```

강조 키워드:

```text
XGBoost
SHAP
FastAPI
Celery/Redis
Docker
Locust
성능 테스트
배포 환경 디버깅
```

### 4. ZipJoong - Vision/Core ML On-device AI

크랜베리가 엣지 AI/멀티모달 AI를 강조할 때 쓴다.

```text
ZipJoong 프로젝트에서는 iPhone 카메라로 눈 상태를 분석해 집중도를 측정하는 앱을 만들고 있습니다.
처음에는 Core ML 모델만 사용하려 했지만 실제 책상 환경에서 오분류가 발생해,
Vision Framework 기반 EAR 분석과 Core ML 모델을 결합하는 하이브리드 구조로 바꿨습니다.

이 경험을 통해 모델 정확도가 데이터셋 안에서만 좋아서는 부족하고,
실제 사용 환경에서의 오분류 원인을 분석해 판단 기준을 재설계해야 한다는 것을 배웠습니다.
```

강조 키워드:

```text
Vision
Core ML
실제 환경 오분류
하이브리드 판단
On-device AI
```

## 6. 내가 반드시 물어볼 질문

### 우선순위 1: 역할 정의

```text
회사에서 말하는 AIOps는 IT 인프라 운영 자동화에 가까운가요,
아니면 IoT/관제 데이터 기반 이상 탐지와 제품 개발에 가까운가요?
```

얻어야 할 확신:

```text
내가 맡을 일이 단순 운영/모니터링이 아니라 개발과 데이터/AI 제품화에 가까운지 확인한다.
```

### 우선순위 2: 실제 하루 업무

```text
이 포지션의 하루 업무 비중을 개발, 데이터 분석, 모델 운영, 고객사 대응, 관제 운영으로 나누면 어느 정도인가요?
```

얻어야 할 확신:

```text
개발 비중이 최소 50% 이상인지 확인한다.
고객사 대응/장애 처리 중심이면 신중하게 본다.
```

### 우선순위 3: 다루는 데이터

```text
주로 다루는 데이터는 로그/메트릭인가요, 아니면 음원/영상/센서 데이터인가요?
제가 실제로 데이터 전처리나 모델 입력 구조를 다루게 되는지도 궁금합니다.
```

얻어야 할 확신:

```text
내 BLE/센서/의료 데이터 경험과 연결되는 데이터인지 확인한다.
```

### 우선순위 4: AI 모델 관여도

```text
AI 모델은 직접 학습/평가/개선하나요, 아니면 이미 만들어진 모델을 운영하고 설정하는 역할에 가까운가요?
```

얻어야 할 확신:

```text
모델 성능 개선, 데이터 파이프라인, 배포/모니터링 중 하나라도 실제로 맡을 수 있는지 확인한다.
```

### 우선순위 5: 제품 산출물

```text
제가 맡게 되는 작업이 실제 제품 기능이나 고객에게 제공되는 솔루션에 반영되나요?
아니면 내부 운영/기술지원 성격이 더 강한가요?
```

얻어야 할 확신:

```text
이력서에 쓸 수 있는 제품 개발 산출물이 남는지 확인한다.
```

### 우선순위 6: 기술 스택

```text
현재 팀에서 주로 사용하는 기술 스택은 무엇인가요?
Python, 백엔드, 클라우드, 엣지 디바이스, 프론트엔드 중 어떤 영역을 가장 많이 다루게 되나요?
```

얻어야 할 확신:

```text
Python/FastAPI/데이터/IoT 경험을 쌓을 수 있는지 확인한다.
```

### 우선순위 7: 현장/고객사 업무

```text
고객사 현장 구축이나 장애 대응 업무가 있다면, 개발자도 어느 정도 참여하나요?
야간 대응이나 긴급 출동 같은 운영 부담이 있는지도 궁금합니다.
```

얻어야 할 확신:

```text
운영 부담이 커서 개발 경험이 흐려지지 않는지 확인한다.
```

### 우선순위 8: 6개월 뒤 기대 역할

```text
입사 후 3개월, 6개월 시점에 이 포지션에서 기대하는 성과는 무엇인가요?
```

얻어야 할 확신:

```text
내가 배울 것과 만들어낼 것이 구체적인지 확인한다.
```

### 우선순위 9: 내 배경의 활용 가능성

```text
제가 의료기기 펌웨어, BLE 헬스케어 앱, AI 헬스케어 모델링을 경험했는데,
이 배경이 크랜베리의 AIOps/AIoT 포지션에서 강점으로 작동할 수 있는 지점이 있을까요?
```

얻어야 할 확신:

```text
회사가 나를 어떻게 쓰려는지 확인한다.
```

### 우선순위 10: 회사 안정성

```text
공개 자료상 매출 성장률이 큰 편으로 보였는데,
현재 확정 수주나 유지보수/구독 매출 비중은 어느 정도인지 궁금합니다.
```

얻어야 할 확신:

```text
성장률은 높지만 영업적자/현금흐름 리스크가 관리되고 있는지 확인한다.
```

## 8. 예상 질문과 준비 답변

### Q1. iOS 개발 경험이 많은데 왜 AIOps에 관심이 있나요?

```text
제 관심은 iOS 자체에서 끝나기보다, 현장 데이터가 사용자에게 가치 있는 결과로 전달되는 전체 흐름에 있습니다.
의료기기 회사에서 BLE 혈당기와 청진기 데이터를 앱으로 수신하고, AI 분석 결과를 사용자 화면으로 연결한 경험이 있습니다.
최근에는 Python 기반 AI 모델링과 FastAPI 백엔드도 경험하면서,
센서 데이터와 AI 분석이 실제 제품으로 이어지는 흐름에 더 관심이 생겼습니다.
크랜베리의 AIOps/AIoT 포지션이 음원, 영상, 센서 데이터 기반의 이상 탐지와 제품화에 가깝다면 제 경험을 확장할 수 있다고 생각했습니다.
```

### Q2. 본인의 가장 강한 프로젝트는 무엇인가요?

```text
스마트사운드의 BLE 청진기 연동 iOS 앱입니다.
실시간 청진 데이터를 CoreBluetooth로 수신하고, AES-128 CCM으로 암호화된 데이터를 복호화한 뒤 Charts로 시각화했습니다.
AI 분석 API와도 연동해 사용자가 건강 상태를 확인할 수 있는 화면을 구현했습니다.

이 프로젝트는 단순 앱 화면이 아니라 기기, 통신, 보안, 데이터 시각화, AI 분석 결과가 하나로 이어진 경험이었기 때문에
크랜베리의 AIoT/관제 솔루션과도 연결되는 부분이 있다고 생각합니다.
```

### Q3. AI 경험은 어느 정도인가요?

```text
AI 헬스케어 과정에서 건강검진 데이터를 활용한 심혈관 위험도 예측 서비스를 만들었습니다.
XGBoost 모델과 SHAP 해석 결과를 FastAPI 서버에서 제공했고,
Celery/Redis로 AI Worker를 분리해 비동기 처리 구조를 만들었습니다.
또 스트레스 지수 예측 대회에서는 XGBoost와 CatBoost 앙상블, 5-Fold CV, Optuna 실험을 경험했습니다.

아직 대규모 운영 환경의 ML 엔지니어 경험은 부족하지만,
모델링, 해석, API화, 비동기 처리, 성능 테스트까지 한 번의 서비스 흐름으로 연결해본 경험이 있습니다.
```

### Q4. 운영/장애 상황을 다뤄본 경험이 있나요?

```text
전문적인 AIOps 운영 경험은 아직 많지 않습니다.
다만 실무와 프로젝트에서 문제를 추적하고 원인을 찾아 해결한 경험은 있습니다.

예를 들어 MyHealthBuddy 프로젝트에서 EC2 환경에서만 XGBoost predict_proba 이후 프로세스가 멈추는 문제가 있었는데,
단계별 로깅으로 블로킹 지점을 찾고 OMP_NUM_THREADS=1 설정으로 OpenMP fork deadlock을 해결했습니다.
또 BLE 청진기 앱에서는 구버전 펌웨어와 신버전 암호화 기기가 혼재되어 파형이 표시되지 않는 문제를
기기 버전 조회 후 복호화 분기 처리로 해결했습니다.

이런 식으로 증상을 쪼개고 재현 조건을 좁혀 원인을 찾는 방식은 AIOps 업무에서도 중요하다고 생각합니다.
```

### Q5. 본인이 크랜베리에 기여할 수 있는 부분은 무엇인가요?

```text
첫째, 센서/장비 데이터가 앱과 서비스로 연결되는 흐름을 이해합니다.
둘째, 의료기기와 헬스케어 앱을 경험해서 데이터 신뢰도와 사용자 안전성의 중요성을 알고 있습니다.
셋째, Python 기반 AI 모델링과 FastAPI/Celery/Redis 같은 백엔드 흐름을 경험했습니다.

그래서 음원, 영상, 센서 데이터가 실제 관제 제품에서 어떻게 수집되고 분석되고 사용자에게 전달되는지 이해하며,
개발과 제품 관점 모두에서 기여할 수 있다고 생각합니다.
```

### Q6. 부족한 점은 무엇이고 어떻게 보완할 건가요?

```text
AIOps라는 이름의 운영 자동화, 로그/메트릭 기반 관제, 대규모 인프라 모니터링 경험은 아직 깊지 않습니다.
하지만 센서 데이터 처리, 실시간 앱, AI 모델 API화, 비동기 처리, 성능 테스트 경험은 가지고 있습니다.

입사하게 된다면 회사의 실제 데이터 흐름과 운영 지표를 빠르게 익히고,
Python/데이터 파이프라인/모니터링 도구를 중심으로 부족한 부분을 보완하겠습니다.
```
