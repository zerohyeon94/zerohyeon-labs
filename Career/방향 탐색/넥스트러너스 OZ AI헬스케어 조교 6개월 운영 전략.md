---

## 1. 현재 상황

현재 역할은 넥스트러너스에서 OZ 코딩스쿨 AI헬스케어 과정의 조교/운영 보조에 가깝다.

다만 실제로 해보니 단순 보조만 있는 것이 아니라, Python 문제를 만들고, 직접 학습해서 설명하고, 수강생이 막히는 지점을 파악하는 과정에서 뿌듯함이 있다.

이 감각은 중요하다. 개발자 커리어에서 멀어지는 신호라기보다, `기술을 구조화해서 전달하는 능력`이 실제로 작동한다는 신호이기 때문이다.

---

## 2. 이 방향의 핵심 포지셔닝

넥스트러너스에서 정규직 전환을 목표로 한다면 목표 이름은 `조교 정규직`이 아니어야 한다.

추천 포지션은 아래에 가깝다.

```text
AI 헬스케어 교육 콘텐츠를 만들고,
수강생 성과를 개선하는 학습 시스템을 설계하며,
필요하면 직접 도구까지 만들 수 있는 테크니컬 에듀케이터
```

더 짧게 표현하면:

```text
AI 헬스케어를 설명할 수 있고,
모바일/개발 경험으로 실무 연결까지 해주는 교육형 개발자
```

---

## 3. 6개월 동안 만들면 좋은 산출물

정규직 전환 가능성을 높이려면 `열심히 했다`보다 `회사 안에서 재사용 가능한 자산을 만들었다`가 중요하다.

### 1개월차: 문제 풀이와 피드백 기준 정리

- Python 기초 문제별 `핵심 개념 1개` 정리
- 수강생이 자주 막히는 지점 목록화
- 공통 피드백 템플릿 작성
- 예시 답안과 설명용 답안 분리

남길 산출물:

```text
문제 요구사항
최소 풀이
설명용 풀이
수강생이 막히는 지점
공통 피드백 문장
```

### 2개월차: 수업자료 재사용 구조 만들기

- 주차별 개념 흐름표
- 문제 난이도 기준표
- 필수/선택/이월 문제 분류 기준
- 수강생 질문 유형별 답변 매뉴얼

목표는 다음 기수에서도 그대로 쓸 수 있는 구조를 만드는 것이다.

### 3개월차: AI헬스케어 도메인 연결

- Python 문법을 헬스케어 데이터 예제로 연결
- 난임, 진단, 예측, 환자 데이터, 지표 같은 도메인 예시 정리
- 분류/회귀/평가지표를 수강생 눈높이로 설명하는 자료 제작

예시:

```text
단순 Python 리스트 문제
→ 환자 검사 수치 리스트
→ 평균/최댓값/이상값 확인
→ 데이터 전처리의 첫 단계로 연결
```

### 4개월차: 학습 운영 자동화 후보 찾기

- 반복 질문 모음 정리
- 자동 채점 가능 문제와 불가능 문제 분류
- AI 피드백 프롬프트 초안 작성
- 수강생 진도/오답/질문을 추적할 간단한 시트 또는 대시보드 구조 제안

이 단계부터는 조교가 아니라 `교육 운영을 개선하는 사람`으로 보일 수 있다.

### 5개월차: 콘텐츠/커리큘럼 제안

- 다음 기수 커리큘럼 개선안
- 과제량 조절 기준
- 헬스케어 프로젝트형 과제 후보
- 취업/포트폴리오 연결 과제 후보

### 6개월차: 정규직 전환 제안 자료

정규직 전환을 원한다면 아래 자료를 하나로 묶는다.

```text
1. 내가 만든 재사용 자료 목록
2. 수강생 피드백/성과 변화
3. 운영에서 줄인 반복 업무
4. 다음 기수 개선 제안
5. 내가 맡을 수 있는 정규직 역할안
```

---

## 4. 정규직 전환 시 노려볼 역할

### 1순위: AI헬스케어 커리큘럼/콘텐츠 개발자

가장 현실적이고 지금 경험과 잘 이어진다.

할 수 있는 일:

- Python/AI 과제 설계
- 헬스케어 데이터 기반 실습 설계
- 수강생 피드백 기준 수립
- 강사/멘토용 가이드 제작
- 다음 기수 커리큘럼 개선

### 2순위: 테크니컬 에듀케이터

개발자 경험을 교육으로 번역하는 역할이다.

할 수 있는 일:

- 수업자료 제작
- 실습 문제 설계
- 라이브 세션 보조 또는 일부 진행
- 기술 블로그/자료집 제작
- 수강생 프로젝트 리뷰

### 3순위: 교육 제품/운영 개선 담당

에듀테크 회사 안에서 더 돈이 되는 방향은 여기에 가깝다.

할 수 있는 일:

- 자동 피드백 도구 기획
- 수강생 진도 관리 구조 설계
- AI 튜터/채점 프롬프트 설계
- 학습 데이터 기반 개선안 제안
- LMS/내부 운영 도구 개선 아이디어 제안

---

## 5. 이 방향이 돈이 되려면 필요한 조건

에듀테크 자체가 돈이 되는 것이 아니라, 아래 조건이 붙을 때 미래성이 커진다.

```text
1. 수강생 성과를 개선한다.
2. 회사 안에서 반복 사용 가능한 콘텐츠를 만든다.
3. 운영 비용을 줄이거나 자동화한다.
4. B2B/B2G 교육 프로그램으로 확장 가능하다.
5. AI/헬스케어처럼 차별화된 도메인을 가진다.
```

OZ AI헬스케어 과정은 공식 페이지에서도 실무 연계, 포트폴리오, 인턴십/채용 연계를 강조하고 있다. 따라서 조교 역할도 단순 질의응답보다 `실무형 과제와 포트폴리오를 만드는 운영자` 쪽으로 확장할 여지가 있다.

---

## 6. 조심해야 할 위험

### 반복 업무에 갇히는 경우

```text
- 과제 채점만 반복
- 질문 답변만 반복
- Python 기초만 계속 설명
- 내가 만든 자료가 다음 기수에 재사용되지 않음
```

이 경우 6개월 뒤 이력서에 남는 말이 약해진다.

### 개발자 정체성이 흐려지는 경우

조교 업무를 하더라도 iOS/모바일 감각은 주 2-3회 유지해야 한다.

추천 최소 루틴:

```text
주 2회: Swift/iOS 코드 읽기 또는 작은 기능 구현
주 1회: AI헬스케어 교육 내용을 모바일 앱 기능으로 연결해보기
월 1회: 교육자료 또는 회고를 블로그 초안으로 정리
```

---

## 7. 다음 주부터 바로 할 일

1. Python 과제마다 `핵심 개념 1개`를 붙인다.
2. 수강생 질문을 `문법 / 문제 이해 / 디버깅 / 검색 부족 / 자료구조 선택`으로 분류한다.
3. 공통 피드백 문장 10개를 만든다.
4. 내가 만든 자료가 다음 기수에 재사용될 수 있도록 제목과 형식을 통일한다.
5. 매주 금요일에 `이번 주에 회사 자산으로 남긴 것`을 3줄로 기록한다.

---

## 8. 정규직 전환 판단 기준

6개월 뒤 아래 5개 중 3개 이상이면 정규직 전환을 진지하게 검토할 가치가 있다.

```text
1. 내가 만든 자료가 회사 안에서 재사용된다.
2. 수강생 피드백/성과가 실제로 좋아진다.
3. Python 학습이 AI헬스케어 이해를 강화한다.
4. 교육 운영을 자동화하거나 도구화할 기회가 있다.
5. 역할이 조교를 넘어 커리큘럼/콘텐츠/제품 쪽으로 확장된다.
```

2개 이하라면 조교 경험은 포트폴리오 소재로 정리하고, iOS/모바일 개발자 루트로 다시 무게를 옮긴다.

---

## 9. 결론

넥스트러너스/OZ AI헬스케어 조교 계약직은 잘 쓰면 좋은 브릿지다.

다만 핵심은 `조교로 오래 버티기`가 아니라, 6개월 동안 아래 증거를 만드는 것이다.

```text
나는 AI헬스케어 교육자료를 만들 수 있다.
나는 수강생이 막히는 지점을 구조화할 수 있다.
나는 교육과 개발 경험을 연결할 수 있다.
나는 필요하면 학습 운영을 도구화할 수 있다.
```

이 증거가 쌓이면 정규직 전환도 의미가 있고, 설령 전환하지 않더라도 `AI 헬스케어를 설명할 수 있는 모바일 개발자`라는 커리어 서사로 남길 수 있다.
