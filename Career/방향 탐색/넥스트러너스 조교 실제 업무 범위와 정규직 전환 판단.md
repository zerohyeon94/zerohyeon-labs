# 넥스트러너스 조교 실제 업무 범위와 정규직 전환 판단

## 1. 현재 업무의 실제 성격

현재 조교 업무는 단순히 수업 중 질문에 답하는 역할이라기보다, `KDT 교육 운영 + 학습자료 배포 + 출결/증빙 관리 + 과제/녹화/훈련일지 관리`가 결합된 운영형 역할이다.

핵심 업무 축은 아래와 같다.

```text
1. 교육 운영
   - 입실/퇴실 QR 공지
   - 오전/오후 출석 확인
   - 중간출결 관리
   - 결석/지각/휴가 인원 확인
   - Slack 출결보고와 마감보고

2. 학습자료 운영
   - 강의자료 Notion 업데이트
   - 학습콘텐츠 업로드
   - 과제 업로드 및 안내
   - 강사 변경이나 자료 변경 시 최신 자료 반영

3. 증빙/행정 관리
   - 녹화 시작/종료
   - 녹화 파일 Drive 업로드
   - 녹화 영상 링크를 Google Sheet와 관리자 페이지에 반영
   - 훈련일지 작성

4. 수강생 관리
   - 출석 확인전화
   - 연락 안 되는 인원 처리
   - 헬프센터 접수 인원 확인
   - 운영 만족도/학습 만족도 조사 안내

5. 업무 기록
   - 매일 업무일지 작성
   - 오늘 해야 할 일, 전달 사항, 반복 일정 관리
```

---

## 2. 이 역할의 현재 위치

현재 역할은 `교육 콘텐츠 개발자`라기보다는 `교육 운영 조교`에 가깝다.

다만 이 역할을 어떻게 수행하느냐에 따라 6개월 뒤의 의미가 달라진다.

```text
그냥 수행하면:
KDT 과정 운영 보조, 출결/자료/녹화/행정 처리 경험

잘 구조화하면:
AI 헬스케어 부트캠프 운영 프로세스 개선 경험
학습자료/과제/피드백 체계화 경험
교육 콘텐츠 기획자로 넘어갈 수 있는 근거

더 확장하면:
교육 운영 자동화, 학습 데이터 기반 개선, 커리큘럼 개선 제안 경험
```

즉, 현재 일 자체는 운영 비중이 높지만, 기록 방식과 산출물 관리에 따라 `운영 담당자`에서 `교육 시스템을 개선하는 사람`으로 포지션을 바꿀 수 있다.

## 3. 정규직 전환 시 예상 역할

현재 공유한 내용 기준으로 보면, 정규직 전환이 되더라도 바로 정식 강의를 맡기보다는 아래 단계로 갈 가능성이 높다.

```text
1단계: 교육 운영 담당
   - 기존 조교 업무를 더 안정적으로 운영
   - 과정 운영 체크리스트와 업무 표준화
   - 자료/출결/녹화/훈련일지 누락 방지

2단계: 학습자료/콘텐츠 보조 제작
   - 기존 강의자료 정리
   - 과제 설명 보강
   - 수강생 질문 기반 FAQ 작성
   - 강사/멘토가 쓰는 보조자료 제작

3단계: 커리큘럼 개선 참여
   - 과제량 조절
   - 난이도 조정
   - 수강생이 많이 막히는 구간 개선
   - AI 헬스케어 프로젝트 흐름 제안

4단계: 일부 세션 또는 보충 강의
   - 정식 수업 전, 보충 설명이나 실습 세션부터 시작
   - Python/AI 기초/과제 풀이 중심 가능성
```

정식 수업까지 1년 이상 걸릴 수 있다는 판단은 현실적이다. 교육 회사에서는 강의를 맡기는 것보다 먼저 `운영 안정성`, `자료 제작 역량`, `수강생 대응`, `커리큘럼 이해도`를 보기 때문이다.

---

## 4. 이 경험의 장점

```text
1. 교육 서비스가 실제로 어떻게 운영되는지 배울 수 있다.
2. KDT/국비 과정의 행정, 출결, 증빙, 훈련일지 흐름을 이해하게 된다.
3. 수강생이 언제, 어디서, 왜 막히는지 가까이서 볼 수 있다.
4. 학습자료와 과제 운영 흐름을 알게 되어 콘텐츠 기획으로 확장할 수 있다.
5. 정규직 전환 시 커리큘럼 개선과 자료 제작에 참여할 수 있는 길이 있다.
6. AI 헬스케어 과정 안에서 Python/AI/데이터 학습을 병행할 수 있다.
```

---

## 5. 이 경험의 한계

```text
1. 현재 업무만 보면 개발 직무 경험으로 강하게 남기는 어렵다.
2. 출결, 녹화, 업로드, 보고 업무가 많아 행정/운영 비중이 높다.
3. 정식 강의까지는 시간이 오래 걸릴 수 있다.
4. 자료 제작 기회가 없다면 커리큘럼/콘텐츠 경력으로도 약해질 수 있다.
5. 매일 바쁘지만, 결과물이 남지 않으면 6개월 뒤 설명하기 어렵다.
6. iOS/모바일 개발 감각은 별도로 유지해야 한다.
```

## 6. 업무일지를 커리어 자산으로 바꾸는 방법

매일 업무일지는 단순 체크리스트로 끝내면 행정 기록이 된다. 하지만 아래 4개 항목을 추가하면 정규직 전환 자료가 된다.

```text
1. 오늘 운영에서 막힌 지점
   - 예: 출결 누락, 자료 업로드 지연, 과제 안내 혼선

2. 수강생이 많이 헷갈린 지점
   - 예: Python 문법, 과제 요구사항, 환경설정, 제출 방식

3. 오늘 만든 재사용 가능 자료
   - 예: 안내문, FAQ, 과제 설명, 피드백 문장, 체크리스트

4. 다음 기수에 개선하면 좋은 점
   - 예: 공지 타이밍, 과제량, 자료 위치, 출결 안내 방식
```

추천 업무일지 추가 템플릿:

```markdown
## 오늘 운영 리스크
- 

## 수강생 혼란 포인트
- 

## 오늘 회사 자산으로 남긴 것
- 

## 다음 기수 개선 아이디어
- 
```

이 4개가 6개월 동안 쌓이면 `나는 업무를 수행했다`가 아니라 `교육 운영을 개선했다`고 말할 수 있다.

## 7. 6개월 동안 반드시 남겨야 할 산출물

### 1. 운영 체크리스트 고도화

현재 체크리스트를 아래처럼 정리한다.

```text
매일 반복 업무
요일별 특수 업무
강의 시작 전 업무
강의 중 업무
점심 전후 업무
강의 종료 후 업무
마감 보고 업무
누락 시 리스크가 큰 업무
```

### 2. 출결/보고 문구 템플릿

반복되는 Slack 보고 문구를 표준화한다.

```text
입실보고
중간출결보고
퇴실보고
마감보고
휴가/결석/지각 예외 처리 문구
```

### 3. 학습자료 업로드 기준

자료 업로드 시 확인할 기준을 만든다.

```text
오늘 과목
강사 변경 여부
자료 최신성
Notion 반영 여부
Drive 반영 여부
수강생 안내 여부
과제 업로드 여부
```

### 4. 수강생 질문/혼란 데이터

질문을 아래처럼 분류한다.

```text
문법 이해 부족
과제 요구사항 이해 부족
환경설정 문제
제출 방식 문제
디버깅 문제
개념은 알지만 적용을 못하는 문제
```

### 5. 콘텐츠 개선 제안

매주 금요일 아래 형식으로 3줄만 남긴다.

```text
이번 주 가장 많이 막힌 개념:
다음 기수에서 줄이거나 보강할 과제:
새로 만들면 좋은 자료:
```

## 8. 정규직 전환 판단 기준

정규직 전환을 긍정적으로 볼 수 있는 경우:

```text
1. 자료 제작에 실제로 참여하게 된다.
2. 과제/피드백/FAQ가 다음 기수에 재사용된다.
3. 커리큘럼 개선 회의나 과정 설계에 참여하게 된다.
4. 수강생 질문 데이터를 바탕으로 학습 흐름을 개선할 권한이 생긴다.
5. 운영 자동화나 학습도구 개선을 제안할 수 있다.
6. 1년 안에 보충 세션, 풀이 세션, 미니 강의 기회가 열린다.
```

신중해야 하는 경우:

```text
1. 6개월 내내 출결/녹화/업로드/보고만 반복한다.
2. 자료 제작이나 커리큘럼 참여 기회가 없다.
3. 조교 역할이 정규직이 되어도 운영 행정 중심으로만 고정된다.
4. Python/AI 학습은 늘지만 개발자로서의 산출물이 줄어든다.
5. 회사가 정규직 전환 기준을 명확히 말해주지 않는다.
```

## 9. 크랜베리 React/Go 플랫폼 개발과 비교

| 항목 | 넥스트러너스 조교/교육 운영 | 크랜베리 React/Go 플랫폼 |
|---|---|---|
| 현재 업무 성격 | 교육 운영, 자료 관리, 수강생 관리 | 웹/백엔드 플랫폼 개발 |
| 기술 개발 경험 | 낮음~중간. 직접 만들면 올라감 | 중간~높음. Go 백엔드 범위에 따라 달라짐 |
| AI 연결성 | AI 헬스케어 교육 콘텐츠 | AIoT 탐지 결과를 플랫폼으로 연결 |
| 모바일/iOS 연결성 | 별도 유지 필요 | 직접 연결성은 낮지만 API/플랫폼 이해에는 도움 |
| 정규직 전환 가능성 | 운영/콘텐츠/커리큘럼 쪽 | 제품/플랫폼 개발자 쪽 |
| 장기 장점 | 교육 콘텐츠, 커리큘럼, 사람 성장 경험 | 제품 개발, 백엔드, 관제 데이터 경험 |
| 핵심 리스크 | 행정 업무에 갇힘 | React 화면/SI 대응에 갇힘 |

## 10. 최종 판단

```text
현재 넥스트러너스 조교 업무는 개발직무라기보다 교육 운영직무에 가깝다.
하지만 업무일지를 잘 설계하고, 학습자료/FAQ/피드백/개선안을 남기면 교육 콘텐츠와 커리큘럼 기획으로 확장될 수 있다.
정규직 전환 후에도 바로 강의자가 되기보다는, 최소 1년 이상 운영과 자료 제작을 거쳐 보충 세션 또는 일부 강의로 넘어갈 가능성이 높다.
```

따라서 핵심 전략은 아래와 같다.

```text
매일 업무를 끝내는 사람이 아니라,
매일 운영 데이터를 모아 다음 기수를 더 좋게 만드는 사람이 되어야 한다.
```

이 방향으로 가면 넥스트러너스 경험은 단순 조교 경력이 아니라 `AI 헬스케어 교육 운영과 콘텐츠 개선 경험`으로 남길 수 있다.
