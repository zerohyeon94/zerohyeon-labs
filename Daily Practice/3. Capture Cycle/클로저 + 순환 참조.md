## 🧪 예제 프로젝트 구성: 클로저 + 순환 참조 실습

아래는 다음을 모두 체험할 수 있는 **간단한 iOS 프로젝트 예제**입니다:

1. ✅ 클로저가 `self`를 캡처할 때 메모리가 해제되지 않는 문제
    
2. ✅ `[weak self]`를 사용해 해결하는 방법
    
3. ✅ `deinit`이 호출되는지 확인

---

### 📦 프로젝트 구성 요약

|파일|설명|
|---|---|
|`MainViewController.swift`|버튼을 눌러 `Worker`를 생성하고 작업을 시작|
|`Worker.swift`|클로저를 통해 작업 실행. 순환 참조 여부 실험|
|`Console`|`deinit` 확인용 로그 출력|

---
## ✅ 상황 요약: 순환 참조가 발생한 상태에서 `Start Task` 버튼을 5번 눌렀을 때

### 로그:

```swift
🚀 Starting task
👷‍♂️ Task A initialized
... (5번 반복)
```

---

## 🔍 이게 의미하는 것

1. 매번 버튼을 누를 때마다 `Worker(name: "Task A")` 인스턴스가 생성됨  
    → 총 **5개의 Worker 인스턴스**가 생깁니다.
    
2. 그런데...
    
    - `worker = Worker(...)`로 새 인스턴스를 만들면, 이전 `worker`는 **참조가 끊겨야 함**
        
    - 그런데 **클로저가 `self`를 강하게 캡처**하고 있으므로,  
        이전 인스턴스는 여전히 메모리 상에 남아 있습니다 ❌
        
3. 그래서 각 인스턴스의 `deinit`이 호출되지 않고,  
    **5개 전부 메모리에서 살아있는 상태가 됩니다.**

---

## 📦 메모리적으로 어떤 일이 발생하나?

|인스턴스|해제됨?|왜?|
|---|---|---|
|Worker #1|❌|클로저가 self를 강하게 참조 중|
|Worker #2|❌|위와 동일|
|...|...|...|
|Worker #5|❌|역시 동일|

→ 즉, 메모리에 **사용되지 않는 인스턴스가 계속 남아서 누수(memory leak)** 상태가 됩니다.

---

## 🧠 이런 상황의 문제점

- 메모리 사용량이 계속 늘어남 → 앱 성능 저하
    
- 인스턴스에 들어 있는 리소스들(네트워크 핸들러, 타이머 등)도 정리되지 않음
    
- 일정 수 이상이 되면 **앱이 느려지거나 심하면 죽을 수도 있음 (Crash / Termination)**

---

## 🩹 해결 방법 요약

- 클로저 내에서 `self`를 **약한 참조 (weak / unowned)**로 처리
    
- `deinit` 로그가 잘 나오는지 꼭 확인할 것

---

## ✅ 순환참조를 해결

### 해결 코드 :

```swift
func startTask() {
	task = { [weak self] in 
		guard let self = self else { return } 
		print("✅ \(self.name) is working") 
	} 
}
```

### 로그:

```swift
🚀 Starting task
👷‍♂️ Task A initialized
🚀 Starting task
👷‍♂️ Task A initialized
💀 Task A deinitialized
🚀 Starting task
👷‍♂️ Task A initialized
💀 Task A deinitialized
```

> 로그에 `💀 Task A deinitialized`가 찍힌 시점은  
> **이전(첫 번째) `Worker` 인스턴스가 메모리에서 해제된 것**을 의미합니다.

---

## 🔍 왜 그럴까요?

### 코드 요약:

```swift
var worker: Worker?

@objc func didTapStart() {
    worker = Worker(name: "Task A")
    worker?.startTask()
}
```

이 코드에서 `worker`는 **ViewController의 프로퍼티**입니다.  
→ 즉, 오직 **하나의 인스턴스만 참조**할 수 있어요.

---

### 🚨 그래서 버튼을 눌렀을 때 내부에서 일어나는 일:

1. `worker = Worker(...)` 실행  
    → 기존 `worker`가 **새로운 Worker로 덮어쓰기**됨  
    → 기존 Worker에 대한 강한 참조가 끊어짐
    
2. 만약 기존 `Worker`가 **클로저로 인해 자기 자신을 캡처하지 않고 있다면** (즉, `[weak self]` 사용)  
    → 참조 카운트가 0이 됨  
    → `deinit` 호출됨  
    → 💀 로그 출력

---

## 🧠 정리: `💀` 로그가 의미하는 것

|로그 시점|해제된 객체|
|---|---|
|2번째 클릭 직후 `💀`|**1번째 Worker** 인스턴스 해제|
|3번째 클릭 직후 `💀`|**2번째 Worker** 인스턴스 해제|
|...|...|

---

## 📌 보너스: 순환 참조가 있을 때는?

같은 상황에서 `[weak self]` 없이 아래처럼 코드를 작성했다면:

```swift
task = {
    print("🔁 \(self.name) is working") // 강한 self 캡처
}
```

→ `worker = nil`이 되어도 참조 카운트가 줄지 않기 때문에  
→ `💀` 로그는 **절대 출력되지 않습니다.**
