AI 모델을 학습시킬 때 가장 주의해야 할 점은 모델이 주어진 학습 데이터에만 너무 익숙해져서, **처음 보는 새로운 데이터에 대해서는 엉망으로 예측하는 현상**입니다. 이를 방지하고 객관적으로 평가하기 위해 데이터를 나누어 학습합니다.

- **과소적합 (Underfitting):** 모델이 덜 학습되어서 규칙을 제대로 찾지 못한 상태
    
- **과적합 (Overfitting):** 모델이 학습 데이터의 정답을 통째로 외워버려서 새로운 문제에 대응하지 못하는 상태
    
- **데이터 분할:** 평가를 위해 전체 데이터를 훈련용(Train), 검증용(Validation), 테스트용(Test)으로 나누는 필수 과정

### 📖 상세 설명

**1. 과소적합과 과적합의 이해**

- **과소적합:** 모델이 너무 단순하거나 학습 시간이 너무 짧아, 훈련 데이터조차 제대로 맞추지 못하는 상태입니다. (비유: 수학의 덧셈 공식조차 아직 이해하지 못한 학생)
    
- **과적합:** 모델이 훈련 데이터에 있는 노이즈(오류)나 불필요한 특징까지 모조리 암기해버린 상태입니다. 훈련 데이터에 대한 점수는 100점이지만, 처음 보는 실전 데이터에서는 점수가 크게 떨어집니다. (비유: 수학 문제집의 정답 번호만 달달 외워서, 실제 시험에서 숫자가 조금만 바뀌어도 전부 틀리는 학생)

**2. 과적합을 방지하고 모델을 평가하는 방법: 데이터 분할**

가장 기본적이고 중요한 평가 방법은 가진 데이터를 전부 학습에 쓰지 않고, 목적에 따라 세 가지로 나누는 것입니다.

- **Train Data (훈련 데이터):** 모델을 학습시키는 데 사용하는 교재 (수학 문제집)
    
- **Validation Data (검증 데이터):** 학습 도중 모델이 잘 학습하고 있는지, 과적합이 발생하지 않는지 중간 점검하는 데이터 (모의고사)
    
- **Test Data (테스트 데이터):** 모든 학습이 끝난 후, 모델의 최종 성능을 객관적으로 평가하기 위해 마지막에 딱 한 번 사용하는 데이터 (수능 시험)

**3. 대표적인 과적합 방지 기법**

- **조기 종료 (Early Stopping):** 훈련 데이터로 학습을 진행하며 검증 데이터로 계속 중간 평가를 하다가, 훈련 점수는 오르는데 검증 점수가 떨어지기 시작하는 순간(과적합 시작점)에 학습을 강제로 멈추는 기법입니다.
    
- **데이터 증강 (Data Augmentation):** 가진 데이터가 부족할 때 이미지의 좌우를 반전시키거나, 회전, 자르기 등을 통해 가상의 데이터를 늘려 모델이 훨씬 다양한 케이스를 경험하게 만드는 방법입니다.

### 🆚 비교: 과소적합 vs 과적합

|**구분**|**과소적합 (Underfitting)**|**과적합 (Overfitting)**|
|---|---|---|
|**발생 원인**|모델이 너무 단순함, 학습량 부족|모델이 너무 복잡함, 학습 데이터 암기|
|**훈련 데이터 점수**|낮음|**매우 높음 (거의 100%)**|
|**테스트 데이터 점수**|낮음|**낮음**|
|**해결 방법**|모델 복잡도 증가, 학습 시간 늘리기|조기 종료, 데이터 추가 확보, 모델 단순화|

### ❓ 확인 질문 (면접 대비용)

**Q. AI 모델을 학습시켰더니 훈련 데이터에 대한 정확도는 99%가 나왔지만, 새로운 테스트 데이터를 넣었을 때는 정확도가 60%로 크게 떨어졌습니다. 현재 모델의 상태는 무엇이며, 이를 해결하기 위한 방법 한 가지를 제시해 보세요.**

> **A.** 현재 모델은 **과적합(Overfitting)** 상태입니다. 주어진 훈련 데이터에만 과도하게 최적화되어 새로운 데이터의 패턴을 일반화하지 못하고 있습니다. 이를 해결하기 위해 검증 데이터의 오차가 커지기 시작하는 시점에 학습을 멈추는 **조기 종료(Early Stopping)** 기법을 적용하거나, 모델이 더 다양한 패턴을 익힐 수 있도록 **데이터 증강**을 통해 학습 데이터의 양을 늘려야 합니다.
