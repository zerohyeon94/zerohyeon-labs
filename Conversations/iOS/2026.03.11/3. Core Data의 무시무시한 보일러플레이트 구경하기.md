**🗣️ 가볍게 시작하기 (Simple): "데이터베이스 공장 짓기"**

Core Data를 쓰려면 앱이 켜질 때 제일 먼저 **"데이터베이스 공장(Container)"**을 짓고, 그 공장 안에 **"작업대(Context)"**를 펼쳐두는 작업이 필수였습니다.

이걸 세팅하기 위해 예전에는 `Persistence.swift`라는 파일을 따로 만들어서 무려 30줄이 넘는 코드를 작성해야 했습니다.

---

**🤿 딥하게 들어가기 (Deep: ❌ 기존 Core Data 세팅 코드)**

아래 코드가 바로 수많은 iOS 개발자들을 숨 막히게 했던 전형적인 Core Data 세팅 코드(보일러플레이트)입니다.

**1. `Persistence.swift` (공장 짓기)**

```Swift
import CoreData

// 데이터베이스를 관리하는 싱글턴(공유) 객체
struct PersistenceController {
    static let shared = PersistenceController()

    // 공장(Container) 본체
    let container: NSPersistentContainer

    init(inMemory: Bool = false) {
        // 1. "MyAppData"라는 이름의 파일(데이터베이스)을 찾아서 공장을 세워라!
        container = NSPersistentContainer(name: "MyAppData")
        
        // (프리뷰용 메모리 세팅 등 생략)
        
        // 2. 실제 디스크에 있는 데이터베이스 파일을 읽어와라!
        container.loadPersistentStores(completionHandler: { (storeDescription, error) in
            if let error = error as NSError? {
                // 에러 나면 앱을 강제로 죽임 (끔찍하죠?)
                fatalError("Unresolved error \(error), \(error.userInfo)")
            }
        })
        
        // 3. 데이터가 바뀔 때 화면을 자동으로 업데이트하기 위한 추가 설정
        container.viewContext.automaticallyMergesChangesFromParent = true
    }
}
```

**2. `App.swift` (작업대를 화면에 던져주기)** 공장을 다 지었으면, 앱이 시작될 때 이 공장 안의 **'작업대(viewContext)'**를 끄집어내서 SwiftUI 화면 전체에 환경 변수로 뿌려줘야 했습니다.

```Swift
import SwiftUI

@main
struct MyApp: App {
    // 1. 방금 만든 그 길고 긴 파일에서 공장 객체를 가져옵니다.
    let persistenceController = PersistenceController.shared

    var body: some Scene {
        WindowGroup {
            ContentView()
                // 2. 공장 안의 '작업대(viewContext)'를 통째로 뷰 계층에 주입합니다!
                .environment(\.managedObjectContext, persistenceController.container.viewContext)
        }
    }
}
```

**🌟 다시 보는 ✅ SwiftData 세팅 코드**

위에서 보신 저 길고 복잡한 `NSPersistentContainer` 세팅, 에러 핸들링, 환경 변수 주입 과정이... SwiftData로 오면서 **전부 다 사라졌습니다.**

```Swift
import SwiftUI
import SwiftData

@main
struct MyApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
        // 끝. 이 한 줄이 위의 모든 코드를 대신합니다!
        .modelContainer(for: FocusSession.self) 
    }
}
```

---

**💡 핵심 요약**

- **Core Data:** `NSPersistentContainer`를 수동으로 만들고, 에러 처리를 하고, `viewContext`를 뽑아서 억지로 주입해야 했습니다.
    
- **SwiftData:** `.modelContainer(for:)` 한 줄만 쓰면, 시스템이 알아서 내부적으로 저 복잡한 공장을 짓고 작업대를 연결해 줍니다.
