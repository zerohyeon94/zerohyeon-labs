**1. Create (데이터 추가하기)**

새로운 데이터를 만들어서 데이터베이스에 밀어 넣는 과정입니다.

- **❌ Core Data:** 새 객체를 만들 때부터 반드시 '작업대(`viewContext`)'를 명시해서 태어나게 해야 합니다. 그리고 마지막에 무조건 `save()`를 호출하며 에러 처리(`try?`)를 해줘야 디스크에 저장됩니다.
    
    ```Swift
    // 1. 객체를 생성할 때 컨텍스트를 주입해야 함
    let newItem = Item(context: viewContext)
    newItem.title = "새로운 메모"
    newItem.date = Date()
    
    // 2. 명시적으로 저장 명령을 내려야 함
    do {
        try viewContext.save()
    } catch {
        print("저장 실패!")
    }
    ```
    
- **✅ SwiftData:** 그냥 평범한 Swift 클래스 만들듯이 쿨하게 인스턴스를 생성한 뒤, 작업대(`modelContext`)에 "이거 넣어줘!"(`insert`)라고 던져주면 끝납니다. **자동 저장(Autosave)** 기능이 기본으로 켜져 있어서 굳이 `save()`를 부를 필요도 없습니다.
    
    ```Swift
    // 1. 아주 자연스러운 순수 Swift 객체 생성
    let newItem = Item(title: "새로운 메모", date: Date())
    
    // 2. 컨텍스트에 밀어 넣기 (자동 저장됨!)
    modelContext.insert(newItem)
    ```

---

**2. Read (데이터 읽어오기)**

저장된 데이터를 화면에 리스트로 뿌려주기 위해 가져오는 과정입니다.

- **❌ Core Data:** 가져올 때 에러가 나지는 않을지, 정렬 기준(`NSSortDescriptor`)은 무엇인지 아주 길고 복잡하게 적어줘야 했습니다.
    
    ```Swift
    // FetchRequest라는 무거운 프로퍼티 래퍼와 긴 정렬 코드
    @FetchRequest(
        sortDescriptors: [NSSortDescriptor(keyPath: \Item.date, ascending: false)],
        animation: .default
    )
    private var items: FetchedResults<Item>
    ```
    
- **✅ SwiftData:** Swift의 최신 문법인 KeyPath(`\Item.date`)를 사용해서 정말 직관적이고 짧게 읽어옵니다. 배열(`[Item]`) 형태로 바로 튀어나와서 쓰기도 편합니다.
    
    ```Swift
    // Query 하나로 깔끔하게 끝!
    @Query(sort: \Item.date, order: .reverse) 
    private var items: [Item]
    ```

---

**3. Update (데이터 수정하기)**

리스트에 있는 특정 메모의 제목을 바꾸는 상황입니다.

- **❌ Core Data:** 값을 바꾼 뒤에, 또다시 `viewContext.save()`를 호출해서 디스크에 덮어씌우는 과정을 거쳐야 합니다. 까먹으면 앱 껐다 켰을 때 수정 전으로 돌아가 있습니다.
    
    ```Swift
    item.title = "수정된 메모"
    
    // 또 명시적으로 저장 (반복되는 보일러플레이트)
    try? viewContext.save()
    ```
    
- **✅ SwiftData:** 그냥 값을 바꾸면 끝입니다. `@Model` 매크로가 이 객체를 지켜보고 있다가, 값이 변하면 화면도 갱신해 주고 **알아서 백그라운드에서 디스크에 저장까지 마무리합니다.**
    
    ```Swift
    // 값을 변경하기만 하면 끝 (알아서 자동 저장됨)
    item.title = "수정된 메모"
    ```

---

**4. Delete (데이터 삭제하기)**

리스트에서 항목을 밀어서 삭제(Swipe to Delete)하는 상황입니다.

- **❌ Core Data:** 컨텍스트에 삭제 명령을 내리고, **또** `save()`를 해야 합니다.
    
    ```Swift
    viewContext.delete(item)
    try? viewContext.save()
    ```
    
- **✅ SwiftData:** 컨텍스트에 삭제 명령만 내리면 알아서 처리됩니다.
    
    ```Swift
    modelContext.delete(item)
    // 끝!
    ```

---

**💡 오늘의 요약 처방**

가장 큰 차이는 두 가지입니다.

1. **순수 Swift 객체처럼 다룬다:** SwiftData는 객체를 만들고 수정할 때 프레임워크의 눈치를 볼 필요 없이 일반 클래스 다루듯 아주 자연스럽습니다.
    
2. **자동 저장(Autosave)의 마법:** SwiftData는 뷰의 렌더링 루프에 맞춰 알아서 저장을 수행하기 때문에, 개발자가 실수로 `save()`를 빼먹어서 데이터가 날아갈 일이 획기적으로 줄어들었습니다.
