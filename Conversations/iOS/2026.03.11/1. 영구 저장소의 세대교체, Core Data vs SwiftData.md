**🗣️ 가볍게 시작하기 (Simple): "기억 상실증에 걸린 앱 치료하기"**

지금까지 우리가 다룬 `@State`나 뷰모델의 데이터는 앱을 완전히 끄면(메모리에서 내려가면) 깨끗하게 지워집니다. 면접 카드 앱에서 기껏 외운 단어장이나, 열심히 적어둔 지출 내역이 다음 날 초기화된다면 유저들이 패닉에 빠지겠죠?

이때 아이폰 내부의 저장 공간(디스크)에 데이터를 예쁘게 표(Table) 형태로 정리해 주고 관리해 주는 애플의 공식 데이터베이스 프레임워크가 바로 Core Data와 SwiftData입니다.

---

**🤿 딥하게 들어가기 (Deep: 두 프레임워크의 세계관 차이)**

**1. Core Data (전통의 강호)**

- **역사:** Objective-C 시절부터 무려 10년 넘게 iOS 생태계를 지배해 온 터줏대감입니다.
    
- **특징:** `.xcdatamodeld`라는 별도의 시각적 에디터(Visual Editor) 파일을 열어서 마우스로 엔티티(표)를 그리고, 관계(화살표)를 이어줘야 합니다.
    
- **단점:** 세팅해야 할 보일러플레이트(기본) 코드가 너무 길고 복잡합니다. `NSManagedObjectContext`, `NSPersistentContainer` 같은 무시무시한 이름의 클래스들을 다뤄야 하죠. 러닝 커브가 꽤 가파른 편입니다.

**2. SwiftData (떠오르는 신성, iOS 17+)**

- **역사:** 애플이 "SwiftUI 시대에 Core Data는 너무 낡고 무겁다!"며 2023년에 발표한 혁명적인 프레임워크입니다. (사실 내부적으로는 Core Data 엔진을 그대로 쓰지만, 겉모습을 엄청나게 쓰기 편하게 포장했습니다.)
    
- **특징:** 시각적 에디터가 아예 사라졌습니다! 그냥 평소에 짜던 Swift 코드(Class) 앞에 **`@Model`**이라는 마법의 단어(매크로) 하나만 붙이면 끝납니다.
    
- **장점:** SwiftUI와 미칠 듯이 궁합이 좋습니다. 데이터가 바뀌면 화면이 자동으로 스르륵 다시 그려집니다.

---

**💻 실전 코드 체감하기 (얼마나 편해졌을까?)**

집중한 '세션 기록'을 저장하는 상황을 상상해 보겠습니다.

**❌ Core Data 방식 (과거):** 마우스로 에디터에서 `FocusSession` 엔티티를 만들고, 타입을 지정하고, 코드를 자동 생성(Codegen)한 뒤, 엄청나게 긴 세팅 코드를 `AppDelegate`나 `SceneDelegate`에 집어넣어야 했습니다.

**✅ SwiftData 방식 (현재):** 그냥 일반적인 클래스를 하나 만들고 앞에 `@Model`만 딱 붙여줍니다.

```Swift
import SwiftData
import Foundation

// 1. 이게 끝입니다. 데이터베이스 테이블 생성 완료!
@Model
class FocusSession {
    var id: UUID
    var durationMinutes: Int
    var date: Date
    var focusScore: Double // CoreML로 뽑아낸 점수 저장!
    
    init(durationMinutes: Int, date: Date, focusScore: Double) {
        self.id = UUID()
        self.durationMinutes = durationMinutes
        self.date = date
        self.focusScore = focusScore
    }
}
```

그리고 화면(View)에서 이 데이터를 불러와서 리스트로 뿌려주는 코드도 충격적으로 짧습니다.

```Swift
import SwiftUI
import SwiftData

struct SessionHistoryView: View {
    // 2. 데이터베이스에서 데이터 가져오기 (정렬까지 한 방에!)
    @Query(sort: \FocusSession.date, order: .reverse) var sessions: [FocusSession]
    
    var body: some View {
        List(sessions) { session in
            Text("\(session.durationMinutes)분 집중! (점수: \(session.focusScore))")
        }
    }
}
```

단 두 줄(`@Model`과 `@Query`)로 데이터베이스 설계와 읽기가 끝나버렸습니다! 🤯
