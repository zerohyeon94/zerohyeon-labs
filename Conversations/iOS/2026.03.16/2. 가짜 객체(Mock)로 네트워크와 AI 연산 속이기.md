**🗣️ 가볍게 시작하기 (Simple): "영화 촬영장의 스턴트맨(대역 배우)"**

우리의 뇌(ViewModel)가 톰 크루즈라면, 네트워크 통신이나 CoreML 연산은 절벽에서 뛰어내리는 '위험하고 오래 걸리는 액션씬'입니다.

리허설(테스트)을 할 때마다 진짜로 뛰어내릴 수는 없겠죠? 그래서 우리는 톰 크루즈와 옷차림만 똑같이 입은 **스턴트맨(Mock Object, 가짜 객체)**을 고용해서, "뛰어내렸다고 치고 결과만 빨리 알려줘!"라고 속이는 작전을 씁니다.

---

**🤿 딥하게 들어가기 (Deep: 프로토콜과 의존성 주입)**

스턴트맨을 고용하려면 아주 중요한 전제 조건이 있습니다. 뷰모델이 "나는 무조건 진짜 `QuoteNetworkManager`랑만 일할 거야!"라고 고집을 부리면 안 됩니다.

대신 **"명언을 가져올 줄 아는 사람(Protocol)이면 누구든 상관없어!"**라고 마음을 열어야 합니다. 이것을 전문 용어로 **의존성 주입(Dependency Injection)**이라고 부릅니다.

**💻 실전 코드: FocusSense 앱에 스턴트맨 고용하기**

**Step 1: 자격증(Protocol) 만들기** 진짜든 가짜든, 이 자격증만 있으면 뷰모델과 일할 수 있습니다.

```Swift
// "명언을 비동기로 가져오는 기능이 있어야 함" 이라는 자격증
protocol QuoteServiceProtocol {
    func fetchQuote() async throws -> String
}
```

**Step 2: 진짜 객체와 가짜 객체(Mock) 만들기**

```Swift
// 1. 진짜 객체 (실제 앱 구동 시 사용)
class RealQuoteManager: QuoteServiceProtocol {
    func fetchQuote() async throws -> String {
        // 실제 URLSession 통신 진행 (3초 걸림, 인터넷 끊기면 에러남)
        return "진짜 서버에서 온 명언"
    }
}

// 2. 가짜 객체 - 스턴트맨 (테스트 코드에서만 사용 ⭐️)
class MockQuoteManager: QuoteServiceProtocol {
    // 우리가 테스트하고 싶은 결과를 마음대로 조작할 수 있는 리모컨 변수!
    var mockResult: String = "가짜 명언"
    var shouldFail: Bool = false
    
    func fetchQuote() async throws -> String {
        if shouldFail {
            throw URLError(.notConnectedToInternet)
        }
        // 0.001초 만에 하드코딩된 결과 리턴!
        return mockResult 
    }
}
```

**Step 3: 뷰모델 리모델링 (마음 열기)** 이제 뷰모델은 초기화(`init`)될 때 밖에서 서비스 객체를 주입받습니다.

```Swift
class FocusViewModel: ObservableObject {
    @Published var todayQuote: String = ""
    
    // 특정 클래스가 아니라 '프로토콜'을 타입으로 가집니다.
    private let quoteService: QuoteServiceProtocol
    
    // 앱을 켤 때는 진짜를 넣고, 테스트할 때는 가짜를 넣을 수 있게 구멍(init)을 뚫어줍니다.
    init(quoteService: QuoteServiceProtocol) {
        self.quoteService = quoteService
    }
    
    func loadQuote() async {
        do {
            let quote = try await quoteService.fetchQuote()
            self.todayQuote = quote
        } catch {
            self.todayQuote = "에러 발생"
        }
    }
}
```

### 🚀 대망의 테스트 코드 (Mock 활용하기)

이제 테스트 파일에서 스턴트맨(`MockQuoteManager`)을 투입해 보겠습니다. 인터넷 연결이나 서버 상태는 전혀 상관없는 완벽한 테스트가 탄생합니다.

```Swift
import XCTest
@testable import FocusSense

final class FocusViewModelTests: XCTestCase {
    
    var sut: FocusViewModel!
    var mockService: MockQuoteManager! // 스턴트맨 준비

    override func setUpWithError() throws {
        try super.setUpWithError()
        mockService = MockQuoteManager() 
        // ⭐️ 뷰모델에 가짜 객체(스턴트맨)를 주입합니다!
        sut = FocusViewModel(quoteService: mockService) 
    }

    // 테스트 1: 네트워크 통신이 "성공"했을 때 뷰모델이 데이터를 잘 담는지 검사
    func test_loadQuote_success_updatesTodayQuote() async {
        // Given (준비)
        mockService.mockResult = "집중력은 근육이다" // 서버가 이렇게 줬다고 '치자'
        
        // When (실행)
        await sut.loadQuote()
        
        // Then (검증)
        XCTAssertEqual(sut.todayQuote, "집중력은 근육이다")
    }
    
    // 테스트 2: 네트워크 통신이 "실패"했을 때 뷰모델이 에러 처리를 잘하는지 검사
    func test_loadQuote_failure_showsErrorMessage() async {
        // Given (준비)
        mockService.shouldFail = true // 이번엔 통신이 터졌다고 '치자'
        
        // When (실행)
        await sut.loadQuote()
        
        // Then (검증)
        XCTAssertEqual(sut.todayQuote, "에러 발생")
    }
}
```

**💡 오늘의 요약**

> 네트워크 통신이나 CoreML 같이 외부 요인에 흔들리는 무거운 작업들은, **Protocol과 의존성 주입(DI)**을 통해 언제나 똑같은 결과를 0초 만에 뱉어내는 **가짜 객체(Mock)**로 둔갑시켜 테스트해야 합니다!

이 방식을 사용하면 CoreML 모델이 카메라 영상을 어떻게 분석하는지도 똑같이 테스트할 수 있습니다. `MockMLService`를 만들어서 "눈 깜빡임 5회 감지됨" 이라는 가짜 결과값을 뷰모델에 던져주면, 뷰모델이 점수를 정확히 깎는지 확인할 수 있죠!
