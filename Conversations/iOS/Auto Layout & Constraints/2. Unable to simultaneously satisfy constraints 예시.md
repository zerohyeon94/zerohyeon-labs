### Auto Layout 충돌

**🗣️ 가볍게 시작하기: 시스템은 왜 제약 조건을 부쉈을까?**

로그의 마지막 줄을 보면 시스템이 이렇게 말하고 있습니다.

> _"어휴, 너희가 준 조건 도저히 다 못 맞추겠어! 앱 터지는 것보단 나으니까, 내가 임의로 **밑바닥 여백(bottom == UIView... - 8) 조건 하나 깰게!**"_

왜 이런 일이 발생했는지, 로그 안에 있는 **'핵심 스모킹 건(결정적 단서)'** 두 가지를 찾아보겠습니다.

---

**🤿 딥하게 들어가기: 범인은 `fittingSizeHTarget`**

로그를 찬찬히 뜯어보면 두 세력 간의 팽팽한 줄다리기가 보입니다.

**세력 A: "나는 무조건 이 높이가 필요해!" (내부 뷰들)**

- `<NSLayoutConstraint... SpeechBubbleView...height == 52>`: 말풍선 뷰는 무조건 높이가 **52**여야 해!
    
- `<NSLayoutConstraint... V:[UIView]-(20)-[UIView]>`: 뷰 사이에 간격은 무조건 **20**이어야 해!
    
- 그 외 각종 StackView의 위아래 여백(8, 3 등)

**세력 B: "아니, 지금 내 너비가 0인데 높이가 어떻게 나와?" (외부 컨테이너)**

- `<NSLayoutConstraint... 'fittingSizeHTarget' HaruUp.HomeHeaderView...width == 0>`: **🚨 (핵심)** 시스템이 `HomeHeaderView`의 적정 크기를 계산하려고 임시로 **너비를 0**으로 줘봤습니다.
    
- `<NSLayoutConstraint... HaruUp.HomeHeaderView...height == 1.264 * width>`: 그런데 이 뷰는 **높이가 너비의 1.264배**가 되어야 한다는 비율(Aspect Ratio) 조건이 있네요.

**💥 충돌 발생!** 시스템의 임시 계산 과정(가상 시뮬레이션)에서 너비가 0이 들어오니, 비율에 따라 **전체 높이도 0**이 되어야 합니다. 그런데 뷰 내부에는 높이가 52인 말풍선과 여백들이 꽉꽉 들어차 있죠?

- `전체 높이 0` vs `내부 높이 다 합치면 최소 100 이상` 이 둘이 충돌해서 시스템이 비명을 지른 것입니다.

_(참고: 이 `fittingSizeHTarget width == 0` 현상은 뷰가 화면에 완전히 그려지기 전, 혹은 TableView/CollectionView의 헤더 크기를 시스템이 미리 계산해 볼 때 아주 흔하게 발생하는 일시적인 현상입니다.)_

### 💊 해결 처방: 마법의 숫자 '999' (Priority)

이 문제는 우리가 방금 배웠던 **우선순위(Priority)**를 살짝 비틀어주면 마법처럼 해결됩니다.

시스템이 임시로 크기를 계산할 때 에러를 뿜지 않고 부드럽게 넘어가게 하려면, **내부의 수직 제약 조건 중 하나의 우선순위를 1000(Required)에서 999(High)로 낮춰주면 됩니다.**

**어떻게 고치나요?** 로그에서 시스템이 깨버리겠다고 지목한 불쌍한 제약 조건이 하나 있죠? `UIStackView:0x10636b400.bottom == UIView:0x10b33ed80.bottom - 8`

1. 스토리보드나 xib를 사용 중이시라면, 해당 StackView의 **Bottom 제약 조건(여백 8)**을 클릭합니다.
    
2. 우측 인스펙터 창에서 **Priority 값을 1000에서 999로 변경**합니다.
    
    - (코드로 레이아웃을 짜셨다면 `.priority = .init(999)` 또는 `.priority = .required - 1` 처리를 해줍니다.)

**왜 999로 하나요?**

- **1000:** "하늘이 두 쪽 나도 여백 8을 지켜!" -> (너비 0일 때) "공간이 없는데 어떻게 지켜!! (에러 쾅💥)"
    
- **999:** "웬만하면 여백 8을 지키는데, 진짜 공간이 없어서 찌그러져야 하면 **잠깐 포기할게.**" -> 시스템이 임시 계산(width=0)을 할 때 에러 없이 조용히 넘어갑니다. 그리고 나중에 실제 너비(예: 390)가 들어오면 다시 999의 강력한 힘으로 원래 여백 8을 예쁘게 그려냅니다.
