**🗣️ 가볍게 시작하기 (Simple): "나 태어났어!" vs "나 나갈게!"**

화면(View)은 메모리에 올라가서 유저의 눈에 보이고, 다시 사라질 때까지 각 단계마다 시스템에 "보고"를 합니다. 우리는 그 보고가 올라오는 타이밍을 가로채서 필요한 작업(데이터 불러오기, UI 업데이트 등)을 지시하면 됩니다.

---

**🤿 딥하게 들어가기 (Deep: 1. UIKit의 클래식한 생명주기)**

UIKit의 `UIViewController`는 아주 섬세하고 세분화된 보고 체계를 가지고 있습니다.

- **`viewDidLoad` (탄생):** "저 방금 메모리에 태어났어요!"
    
    - **특징:** 뷰 컨트롤러가 생겨날 때 **단 한 번만** 호출됩니다.
        
    - **사용처:** UI 컴포넌트들을 뷰에 올리거나, 한 번만 세팅하면 되는 초기 설정들을 여기서 합니다.
        
- **`viewWillAppear` (등장 직전):** "저 이제 유저 눈앞에 나가기 1초 전이에요!"
    
    - **특징:** 화면이 띄워지거나(Present), 다른 화면에 갔다가 뒤로 가기로 다시 돌아올 때마다 **매번** 호출됩니다.
        
    - **사용처:** 가계부 앱에서 지출 내역을 추가하고 메인 화면으로 돌아왔을 때, 새롭게 업데이트된 이번 달 잔액을 갱신해서 보여주기 딱 좋은 위치입니다.
        
- **`viewDidDisappear` (퇴장):** "저 화면에서 완전히 사라졌어요."
    
    - **특징:** 화면이 완전히 닫히거나 다른 화면에 덮였을 때 호출됩니다.
        
    - **사용처:** 돌아가고 있던 타이머를 일시 정지하거나, 카메라 세션을 잠시 끄는 등 불필요한 리소스 낭비를 막을 때 씁니다.

---

**🤿 더 딥하게 (Deep+: 2. SwiftUI의 모던한 생명주기)**

SwiftUI로 넘어오면서 이 복잡했던 체계가 아주 단순하고 강력하게 압축되었습니다.

- **`init()` (구조체 생성):** * SwiftUI의 뷰는 가벼운 구조체(Struct)라서 상태가 변할 때마다 수십 번씩 다시 만들어집니다.
    
    - **🚨 절대 주의:** 여기서 무거운 네트워크 통신이나 CoreML 모델 로딩을 하면 앱이 심각하게 버벅거립니다!
        
- **`.onAppear` (화면에 붙음!):** * UIKit의 `viewWillAppear`와 `viewDidAppear`를 합쳐놓은 느낌입니다. 뷰가 화면 계층에 추가되는 순간 호출됩니다.
    
    - 가벼운 애니메이션을 시작하거나, 간단한 로그를 찍을 때 사용합니다.
        
- **`.task` (비동기의 마법사 ⭐️⭐️⭐️⭐️⭐️):**
    
    - SwiftUI 생명주기의 **꽃**입니다. `.onAppear`와 비슷하게 화면이 뜰 때 실행되지만, **비동기(`async/await`) 작업**을 위해 태어났습니다.
        
    - 집중도 측정 앱을 켤 때 무거운 CoreML 모델을 백그라운드에서 비동기로 슥~ 준비시키거나, 네트워크에서 데이터를 받아올 때 완벽합니다.
        
    - **최고의 장점:** 만약 데이터를 다 받아오기도 전에 유저가 성격이 급해서 화면을 닫아버리면? `.task`는 그걸 귀신같이 알아채고 **진행 중이던 네트워크 요청이나 무거운 작업을 알아서 취소(Cancel)해 줍니다.** (메모리 누수 원천 차단!)

**💻 SwiftUI 실전 비교 코드**

```Swift
struct MyFocusView: View {
    @State private var score: Int = 0

    var body: some View {
        Text("현재 점수: \(score)")
            // ❌ 옛날 방식 (클로저 안에서 Task를 또 열어야 함)
            .onAppear {
                Task {
                    score = await fetchScore()
                }
            }
            // ✅ 요즘 방식 (깔끔하고, 화면 닫히면 알아서 취소됨)
            .task {
                score = await fetchScore()
            }
    }
}
```

---

**💡 오늘의 요약**

> UIKit은 `viewDidLoad`와 `viewWillAppear`로 섬세하게 타이밍을 쪼개서 컨트롤했다면, SwiftUI는 무거운 작업은 무조건 `.task`에 던져놓고 시스템이 알아서 관리하게 놔두는 아주 스마트한 방식을 취하고 있습니다.
