**🗣️ 가볍게 시작하기 (Simple): "완성된 빵 대신, 빵 굽는 레시피만 공유하기"**

- **과거의 방식:** 팀원들과 협업할 때, 우리는 `FocusSense.xcodeproj`라는 '완성된 프로젝트 파일' 자체를 GitHub에 올려서 공유했습니다.
    
- **Tuist 방식:** 프로젝트 파일은 절대 공유하지 않습니다! 대신 "이 앱은 이런 모듈들로 이루어져 있어"라는 **'레시피(Project.swift)'**만 텍스트로 적어서 공유합니다. 그리고 각자 자기 컴퓨터에서 `tuist generate`라는 마법의 주문을 외우면, 레시피를 바탕으로 새하얀 프로젝트 파일이 1초 만에 딱 구워져 나옵니다.

---

**🤿 딥하게 들어가기 (Deep: 도대체 왜 이런 귀찮은 짓을 할까?)**

iOS 개발자들이 Tuist에 열광하는 단 하나의 절대적인 이유가 있습니다. 바로 **`project.pbxproj` 충돌 지옥**에서 벗어나기 위해서입니다.

**🚨 pbxproj 충돌이란?** Xcode는 우리가 파일을 하나 새로 만들거나 폴더를 옮길 때마다, 보이지 않는 곳에서 `project.pbxproj`라는 수만 줄짜리 설정 파일에 그 기록을 난수(알 수 없는 영문과 숫자 조합)로 빽빽하게 적어둡니다. 만약 팀원 A가 '타이머 화면' 파일을 만들고, 팀원 B가 '통계 화면' 파일을 만든 뒤 GitHub에서 합치려고(Merge) 하면? 이 수만 줄짜리 파일이 서로 엉키면서 끔찍한 충돌(Conflict)이 발생합니다. 이걸 손으로 푸는 건 거의 불가능에 가깝죠.

**✅ Tuist의 해결책** Tuist는 이 골치 아픈 `project.pbxproj` 파일을 GitHub에 아예 안 올리게 만들어줍니다. (gitignore 처리) 대신 순수한 Swift 코드로 작성된 레시피 파일만 공유하니까 충돌이 날 일도 없고, 나더라도 그냥 Swift 코드 줄 맞춤 하듯이 쉽게 고칠 수 있습니다.

---

**💻 실전 코드: Tuist 레시피 작성해 보기**

모듈화 아키텍처를 적용한 앱의 레시피(`Project.swift`)는 대략 이렇게 생겼습니다. 우리가 평소에 짜던 Swift 문법과 100% 똑같아서 아주 친숙합니다.

```Swift
import ProjectDescription

// 1. 집중 타이머 앱 프로젝트 생성
let project = Project(
    name: "FocusSense",
    targets: [
        // 2. 앱 본체 타겟 (껍데기)
        Target(
            name: "FocusSenseApp",
            platform: .iOS,
            product: .app,
            bundleId: "com.zerohyeon.focussense",
            infoPlist: .default,
            sources: ["Targets/App/Sources/**"],
            // 3. ⭐️ 이 앱은 아래 두 개의 모듈(프레임워크)을 조립해서 씁니다!
            dependencies: [
                .target(name: "TimerFeature"),
                .target(name: "FocusMLKit")
            ]
        ),
        
        // 4. 타이머 화면 모듈 (프레임워크)
        Target(
            name: "TimerFeature",
            platform: .iOS,
            product: .framework,
            bundleId: "com.zerohyeon.timerfeature",
            infoPlist: .default,
            sources: ["Targets/TimerFeature/Sources/**"],
            // 타이머 모듈은 ML 모듈을 가져다 씁니다.
            dependencies: [
                .target(name: "FocusMLKit")
            ]
        ),
        
        // 5. ML 분석 모듈 (프레임워크)
        Target(
            name: "FocusMLKit",
            platform: .iOS,
            product: .framework,
            bundleId: "com.zerohyeon.focusmlkit",
            infoPlist: .default,
            sources: ["Targets/FocusMLKit/Sources/**"],
            dependencies: []
        )
    ]
)
```

이 파일을 작성하고 터미널에서 **`tuist generate`**를 치면? Xcode가 스르륵 열리면서 저 모듈들이 레고 블록처럼 완벽하게 연결된 프로젝트 세팅이 끝납니다!
