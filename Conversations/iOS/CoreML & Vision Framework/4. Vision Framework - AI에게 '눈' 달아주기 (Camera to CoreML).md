**🗣️ 가볍게 시작하기 (Simple)**

"카메라 영상을 한 프레임씩 캡처해서 AI 모델에 넣고 돌린다." 말은 쉽지만, 이걸 직접 구현하려면 **이미지 크기 조절(Resizing), 회전(Rotation), 포맷 변환(RGB/YUV)** 등 골치 아픈 전처리 과정이 수두룩합니다.

**Vision Framework**는 이 귀찮은 작업을 애플이 대신 해주는 도구입니다.

- 개발자: "야 Vision, 이 카메라 프레임(`PixelBuffer`) 줄 테니까, 아까 만든 `FocusSense` 모델 좀 돌려봐."
    
- Vision: "ㅇㅇ 알았어. (알아서 크기 맞추고 회전시키고 모델에 넣어서 결과 리턴)"

**🤿 딥하게 들어가기 (Deep: CVPixelBuffer와 파이프라인)**

펌웨어(C언어)를 하셨으니 **버퍼(Buffer)** 개념이 익숙하시죠? iOS 카메라가 뱉어내는 데이터의 핵심은 **`CVPixelBuffer`**입니다.

**1. 데이터의 흐름 (Pipeline)**

1. **Camera (AVFoundation):** `AVCaptureVideoDataOutput`이 매 순간(초당 30~60장) **`CMSampleBuffer`**라는 패킷을 뱉어냅니다.
    
2. **Extract (Raw Data):** 여기서 알맹이인 **`CVPixelBuffer`**(실제 메모리에 박힌 YUV/RGB 픽셀 데이터)를 꺼냅니다.
    
3. **Vision Request (`VNCoreMLRequest`):** "이 모델을 써라"라고 요청서를 작성합니다.
    
4. **Vision Handler (`VNImageRequestHandler`):** "자, 이 버퍼(이미지)랑 요청서 처리해!" 하고 던집니다.

**2. 실전 코드 (Delegate)**

카메라 델리게이트 메서드 안에서 이 모든 일이 벌어집니다.

```Swift
// AVCaptureVideoDataOutputSampleBufferDelegate
func captureOutput(_ output: AVCaptureOutput, didOutput sampleBuffer: CMSampleBuffer, from connection: AVCaptureConnection) {
    
    // 1. Raw 픽셀 버퍼 꺼내기 (C 포인터 같은 녀석)
    guard let pixelBuffer = CMSampleBufferGetImageBuffer(sampleBuffer) else { return }
    
    // 2. 모델 준비 (미리 로딩해두는 게 좋음)
    guard let model = try? VNCoreMLModel(for: FocusSense_Int8().model) else { return }
    
    // 3. 요청서 작성 (결과 받으면 실행할 클로저)
    let request = VNCoreMLRequest(model: model) { (request, error) in
        // 5. 결과 처리 (비동기로 옴)
        guard let results = request.results as? [VNClassificationObservation] else { return }
        
        // "눈 깜빡임: 98%" 같은 결과를 여기서 받음!
        print(results.first?.identifier) 
    }
    
    // 4. 요청 실행 (Handler)
    // 여기서 Vision이 알아서 이미지 리사이징/크롭 다 해줍니다. 꿀이죠? 🍯
    let handler = VNImageRequestHandler(cvPixelBuffer: pixelBuffer, options: [:])
    try? handler.perform([request])
}
```

---

**⚠️ 주의할 점 (펌웨어 개발자의 직감)**

1. **발열 관리:** 위 코드는 초당 30번(30fps) 실행됩니다. 무거운 모델을 매 프레임마다 돌리면 폰이 **손난로**가 됩니다.
    
    - **팁:** `throttle`을 걸어서 0.1초에 한 번만 추론하게 하거나, 모델을 더 가볍게 만들어야 합니다.
        
2. **좌표계의 함정 (Coordinate System):**
    
    - **UIKit:** 왼쪽 위가 (0, 0)입니다.
        
    - **Vision:** **왼쪽 아래가 (0, 0)**입니다. (수학 그래프처럼)
        
    - 나중에 얼굴 위치(Bounding Box)를 화면에 그릴 때, **Y축을 뒤집어주지 않으면** 네모 상자가 엉뚱한 곳에 그려집니다.
