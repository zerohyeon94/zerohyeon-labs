**🗣️ 가볍게 시작하기 (Simple) 상태 변화와 춤추는 UI**

과거 UIKit 시절에는 뷰를 몇 픽셀 옮기고 투명도를 조절하는 복잡한 수학 계산을 직접 해야 했습니다. 하지만 SwiftUI에서는 "나는 상태(State)만 바꿀 테니, 중간 애니메이션은 네가 알아서 예쁘게 그려줘!"라고 프레임워크에 통보만 하면 됩니다.

- **Animation (애니메이션)** 이미 화면에 있는 뷰의 크기, 색상, 위치가 바뀔 때 부드럽게 이어주는 효과입니다.
    
- **Transition (트랜지션)** 화면에 없던 뷰가 새로 나타나거나, 있던 뷰가 완전히 사라질 때(생성과 소멸) 보여주는 등장/퇴장 효과입니다.

---

**🤿 딥하게 들어가기 (Deep 1) 부드러운 움직임, Animation**

집중(ZipJoong) 앱에서 타이머 시작 버튼을 누를 때, 버튼의 모양이 바뀌고 점수 텍스트가 커지는 것을 부드럽게 만들어보겠습니다. 방법은 크게 두 가지가 있습니다.

**1. 암시적 애니메이션 (Implicit) - 뷰에 찰딱 붙이기** 뷰의 끝에 `.animation()` 수식어를 달아줍니다. "이 값이 변하면 무조건 애니메이션을 줘!"라는 뜻입니다.

```Swift
struct ZipJoongTimerView: View {
    @State private var isRunning = false
    
    var body: some View {
        VStack {
            Text("집중 점수: 100")
                // isRunning이 바뀔 때 텍스트 크기가 부드럽게 커졌다 작아집니다.
                .scaleEffect(isRunning ? 1.5 : 1.0)
                .animation(.spring(response: 0.5, dampingFraction: 0.5), value: isRunning)
            
            Button("타이머 시작") {
                isRunning.toggle()
            }
        }
    }
}
```

**2. 명시적 애니메이션 (Explicit) - 행동을 감싸기 (⭐️ 실무 국룰)** 현업에서 훨씬 많이 쓰는 방식입니다. 뷰에 수식어를 다는 대신, 값을 바꾸는 행동 자체를 `withAnimation` 블록으로 감싸버립니다.

```Swift
struct ZipJoongTimerView: View {
    @State private var isRunning = false
    
    var body: some View {
        VStack {
            // 여기에 .animation 수식어를 달 필요가 없습니다!
            Text("집중 점수: 100")
                .scaleEffect(isRunning ? 1.5 : 1.0)
            
            Button("타이머 시작") {
                // 이 블록 안에서 일어나는 모든 상태 변화는 자동으로 애니메이션 처리됩니다.
                withAnimation(.easeInOut(duration: 0.3)) {
                    isRunning.toggle()
                }
            }
        }
    }
}
```

**🤿 딥하게 들어가기 (Deep 2) 뷰의 탄생과 죽음, Transition**

애니메이션이 '모양 변화'라면, 트랜지션은 '등장과 퇴장'입니다. 집중이 끝나고 최종 점수 결과창이 화면 아래에서 스르륵 올라오게 만들어보겠습니다.

트랜지션이 작동하려면 반드시 **`if`문으로 뷰가 생겼다 없어지는 조건**이 있어야 합니다!

```Swift
struct ZipJoongResultView: View {
    @State private var showResult = false
    
    var body: some View {
        ZStack {
            VStack {
                Button("집중 종료") {
                    // 뷰를 등장시킬 때도 반드시 withAnimation을 써야 효과가 보입니다!
                    withAnimation(.spring()) {
                        showResult = true
                    }
                }
            }
            
            // ⭐️ if문으로 뷰의 탄생과 죽음을 제어합니다.
            if showResult {
                VStack {
                    Text("최종 점수: 95점!")
                        .font(.largeTitle)
                }
                .frame(width: 300, height: 200)
                .background(Color.blue.cornerRadius(20))
                // 등장할 때는 아래에서 올라오고, 퇴장할 때는 투명해지며 사라집니다.
                .transition(.asymmetric(insertion: .move(edge: .bottom), removal: .opacity))
            }
        }
    }
}
```

**💡 오늘의 핵심 요약**

> SwiftUI의 시각적 마법은 **상태(State) 변화**에서 시작됩니다. 이미 있는 뷰의 모양을 부드럽게 바꿀 때는 **`withAnimation`**을 사용하고, 뷰를 화면에 새롭게 띄우거나 없앨 때는 `if`문과 함께 **`.transition`**을 사용합니다.

코드를 몇 줄만 추가해도 화면이 생동감 넘치게 변하는 걸 보면 코딩하는 맛이 확 살아나실 겁니다! 이 애니메이션들을 조합하면 복잡한 커스텀 로딩 화면이나 팝업창도 뚝딱 만들어낼 수 있습니다.
