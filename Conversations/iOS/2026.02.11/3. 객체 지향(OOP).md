## 객체 지향(OOP), C언어와 어떻게 다를까?

### **1. 절차 지향 (Procedural Programming) - C언어**

> **"동사(행동)가 중심인 세계"**

C언어를 하실 때를 떠올려보세요. 데이터(`struct`)와 함수(`function`)가 따로 놉니다.

- **상황:** LED를 켜야 한다.
    
- **코드:** `turnOnLed(myLed)`
    
- **해석:** `turnOnLed`라는 함수에게 `myLed`라는 데이터를 **매개변수로 던져줍니다.**
    
- **특징:** 함수가 주인공이고, 데이터는 그저 함수가 처리하는 재료일 뿐입니다. "데이터를 함수에 집어넣는다"는 느낌이죠.

---
### **2. 객체 지향 (Object-Oriented Programming) - Swift/UIKit**

> **"명사(객체)가 중심인 세계"**

OOP에서는 데이터와 함수를 하나로 묶어서 **'객체(Object)'**라고 부릅니다. 이제 데이터가 수동적으로 끌려다니는 게 아니라, **스스로 행동하는 주체**가 됩니다.

- **상황:** LED를 켜야 한다.
    
- **코드:** `myLed.turnOn()`
    
- **해석:** `myLed`라는 객체에게 **"너 켜져!"라고 명령(메시지)을 보냅니다.**
    
- **특징:** 주어(객체)가 있고, 그 주어가 스스로 행동(메서드)을 수행합니다.

---

### 🧱 펌웨어 개발자를 위한 비유: "공장 vs 로봇"

이 차이를 하드웨어로 비유하면 이해가 빠릅니다.

- **C언어 (절차 지향):** **공장 라인**입니다.
    
    - 컨베이어 벨트(함수)가 있고, 그 위에 부품(데이터)이 지나가면서 조립됩니다.
        
    - 부품은 가만히 있고, 기계가 와서 납땜하고 조립합니다.
        
- **OOP (객체 지향):** **자율 주행 로봇**입니다.
    
    - 로봇(객체) 안에 배터리(데이터)도 있고, 모터(데이터)도 있고, 움직이는 기능(메서드)도 다 들어있습니다.
        
    - 우리는 그저 로봇에게 "앞으로 가!"라고 명령만 내리면 됩니다. 로봇 안에서 모터가 어떻게 도는지는 알 필요가 없죠.

### 📱 iOS(UIKit)에서의 실제 예시

UIKit이 왜 `Class` 덩어리인지 이제 이해가 되실 겁니다. 화면에 있는 버튼 하나하나를 **"스스로 기능하는 로봇(객체)"**으로 보기 때문입니다.

```swift
// C언어 스타일 (만약 이렇게 짰다면...)
struct ButtonData {
    int x, y;
    char* title;
};

void setButtonTitle(struct ButtonData* btn, char* title) {
    btn->title = title;
    redraw(btn); // 함수가 데이터를 받아서 그림
}

// --------------------------------------------------

// 객체 지향 스타일 (실제 iOS 코드)
class UIButton {
    var title: String? // 데이터(속성)
    
    func setTitle(_ title: String) { // 행동(메서드)
        self.title = title
        self.setNeedsDisplay() // "나 다시 그려줘!"라고 스스로 갱신
    }
}

let myButton = UIButton()
myButton.setTitle("로그인") // "버튼아, 네 이름 바꿔!"
```

**결론:** 객체 지향 프로그래밍이란, 프로그램을 **"명령어의 목록"**으로 보는 게 아니라, **"역할을 가진 객체들의 협력"**으로 보는 것입니다.

- "뷰 컨트롤러야, 화면 띄워."
    
- "버튼아, 눌리면 나한테 알려줘."
    
- "테이블뷰야, 데이터 리로드해."

이렇게 객체끼리 서로 일을 시키고 대화하는 방식이라서 **"객체 지향(Object-Oriented)"**이라고 부르는 거죠.
