**🗣️ 가볍게 시작하기 (Simple): "종류별로 묶을까, 세트 메뉴로 묶을까?"**

옷장을 정리할 때 '바지는 바지칸에, 셔츠는 셔츠칸에' 넣는 방식이 있고, '출근용 세트, 운동용 세트'로 묶어두는 방식이 있죠? 앱 폴더링도 정확히 똑같습니다.

---

**🤿 딥하게 들어가기 (Deep: 실전 폴더 구조 비교)**

**1. 종류별 분리 (Group by Type / Layer-First)** 초보자들이 MVVM을 처음 배울 때 가장 많이 쓰는 직관적인 방식입니다.

- 📁 `Views`
    
    - `TimerView.swift`
        
    - `SettingsView.swift`
        
- 📁 `ViewModels`
    
    - `TimerViewModel.swift`
        
    - `SettingsViewModel.swift`
        
- 📁 `Models`
    
    - `Session.swift`
        
    - `User.swift`
        
- 📁 `Services`
    
    - `MLService.swift`
        
    - `NetworkService.swift`
        
- **장점:** 아키텍처의 계층(Layer)이 한눈에 보입니다.
    
- **단점:** 타이머 기능을 하나 수정하려고 하면, `Views` 폴더 열어서 파일 고치고, 스크롤 한참 내려서 `ViewModels` 폴더 열고, 또 `Models` 폴더를 열어야 합니다. (파일이 수백 개면 스크롤 지옥이 펼쳐집니다.)

---

**2. 기능별 분리 (Group by Feature / Domain-Driven) ⭐️ 현업 표준** 앱이 조금만 커져도 무조건 이 방식으로 넘어갑니다. 관련된 파일들을 '세트 메뉴'처럼 한 폴더에 몰아넣는 방식입니다. 집중력을 추적하고 기록하는 앱을 만든다고 가정해 볼까요?

- 📁 `App` (앱의 시작점)
    
    - `MyApp.swift`
        
- 📁 `Core` (앱 전체에서 공통으로 쓰는 도구들)
    
    - 📁 `Network` (`APIClient.swift`)
        
    - 📁 `ML` (`FocusMLManager.swift`)
        
    - 📁 `DesignSystem` (`CustomColors.swift`, `CustomFonts.swift`)
        
    - 📁 `Extensions` (`Date+Ext.swift`)
        
- 📁 **`Features` (핵심! 기능별 세트 메뉴)**
    
    - 📁 `FocusTimer`
        
        - `FocusTimerView.swift`
            
        - `FocusTimerViewModel.swift`
            
    - 📁 `History` (기록 탭)
        
        - `HistoryListView.swift`
            
        - `HistoryViewModel.swift`
            
    - 📁 `Settings` (설정 탭)
        
        - `SettingsView.swift`
            
        - `SettingsViewModel.swift`
            
- 📁 `Models` (공통 데이터 구조체)
    
    - `FocusSession.swift`
        
- **장점:** '집중 타이머' 기능을 고칠 때는 오직 `Features/FocusTimer` 폴더 안에서만 움직이면 됩니다. 응집도가 엄청나게 높아져서 작업 속도가 쾌속이 됩니다!

### 🚨 현업 꿀팁: "폴더는 물리적인 경로를 의미하지 않을 수도 있다?"

과거 Xcode에서는 노란색 폴더(Group)와 실제 컴퓨터 하드디스크의 파란색 폴더(Folder)가 달라서 파일 위치가 꼬이는 끔찍한 일이 잦았습니다. 하지만 **최신 Xcode에서는 폴더를 만들면 실제 Mac의 디렉토리와 1:1로 완벽하게 동기화**되니 안심하고 팍팍 폴더를 쪼개셔도 됩니다.

특히 `Core` 폴더 안에 나만의 디자인 시스템(버튼 스타일, 색상 등)이나 ML 매니저를 잘 모아두면, 나중에 가계부 앱이나 단어장 앱 같은 완전히 새로운 프로젝트를 시작할 때 저 `Core` 폴더만 쏙 복사해서 재사용하기가 기가 막히게 좋습니다.
