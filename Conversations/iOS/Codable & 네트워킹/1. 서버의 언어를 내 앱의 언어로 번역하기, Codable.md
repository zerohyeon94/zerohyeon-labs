**🗣️ 가볍게 시작하기 (Simple): "붕어빵 틀과 밀가루 반죽"**

- **서버의 언어 (JSON):** 서버는 보통 데이터를 텍스트 형태인 **JSON**으로 줍니다. 이건 그냥 형태가 없는 '밀가루 반죽(Raw Data)'입니다.
    
- **내 앱의 언어 (Swift Struct):** 우리는 이 반죽을 FocusSense 앱에서 쓰기 편하게 이름표가 딱딱 붙은 **'붕어빵(Struct)'**으로 만들어야 합니다.

이 반죽을 붕어빵 틀에 넣고 촥! 찍어내는 기계가 바로 **`JSONDecoder`**이고, "나는 붕어빵 틀이 될 준비가 되었어!"라고 구조체에 달아주는 자격증이 바로 **`Codable`**입니다.

---

**🤿 딥하게 들어가기 (Deep: 실전 번역 과정)**

서버에서 '오늘의 집중 명언'을 방금 받아왔다고 상상해 보겠습니다.

**1. 서버가 보낸 밀가루 반죽 (JSON)** 보통 웹 서버는 단어 사이를 밑줄로 잇는 **스네이크 케이스(snake_case)**를 좋아합니다.

```JSON
{
  "quote_text": "집중력은 근육과 같다. 쓸수록 강해진다.",
  "author_name": "Zerohyeon"
}
```

**2. 붕어빵 틀 만들기 (Swift Struct)** 이 데이터를 받을 구조체를 만듭니다. 이름 뒤에 `: Codable`만 붙여주면 끝입니다. (참고로 `Codable`은 데이터를 해독하는 `Decodable`과 데이터를 생성하는 `Encodable`이 합쳐진 말입니다.)

```Swift
struct QuoteResponse: Codable {
    let quoteText: String
    let authorName: String
    
    // ⭐️ 마법의 번역 사전 (CodingKeys)
    // 서버의 "quote_text"를 내 앱의 "quoteText"로 매칭시켜 줍니다.
    enum CodingKeys: String, CodingKey {
        case quoteText = "quote_text"
        case authorName = "author_name"
    }
}
```

*💡 iOS 개발자는 낙타 등처럼 대문자로 띄어쓰기를 구분하는 **카멜 케이스(camelCase)*_를 쓰기 때문에, `CodingKeys`라는 사전을 만들어서 이름의 짝을 지어주는 것이 실무에서 아주 중요합니다._

**3. 기계 작동! (Decoding)** 지난 수다에서 짰던 `URLSession` 코드와 합쳐볼까요?

```Swift
func fetchQuote() async throws -> QuoteResponse {
    let url = URL(string: "https://api.example.com/quote")!
    
    // 1. 네트워크에서 반죽(Data)을 받아옵니다.
    let (data, _) = try await URLSession.shared.data(from: url)
    
    // 2. 붕어빵 기계(JSONDecoder)를 돌려서 반죽을 틀(QuoteResponse)에 맞춰 찍어냅니다!
    let decodedQuote = try JSONDecoder().decode(QuoteResponse.self, from: data)
    
    return decodedQuote
}
```

### 🚨 현업에서 가장 많이 마주치는 멘붕 (Optional의 함정)

서버 개발자분들과 협업하다 보면, 가끔 `author_name` 데이터가 아예 안 오거나(null) 빠져서 올 때가 있습니다.

이때 내 구조체에 `let authorName: String`이라고 **무조건 값이 있다**고 선언해 두면, `JSONDecoder` 기계가 붕어빵을 찍어내다가 멈춰버리고 앱에 에러가 뜹니다(Decoding Error).

그래서 "서버가 이 값을 안 줄 수도 있겠다" 싶은 데이터는 반드시 **옵셔널(`?`)**로 처리해 주어야 앱이 죽지 않고 유연하게 넘어갈 수 있습니다.

```Swift
struct QuoteResponse: Codable {
    let quoteText: String
    let authorName: String? // "이름은 없을 수도 있어요!" 라고 미리 방어막 치기
}
```
