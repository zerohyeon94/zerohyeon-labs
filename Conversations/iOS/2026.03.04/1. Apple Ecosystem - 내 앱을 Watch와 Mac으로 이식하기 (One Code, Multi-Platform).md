**🗣️ 가볍게 시작하기 (Simple)**

예전(UIKit 시절)에는 아이폰 앱, 맥 앱, 워치 앱을 만들려면 코드를 거의 새로 짜다시피 했습니다. 언어도 프레임워크도 미묘하게 달랐거든요.

하지만 **SwiftUI** 세상에서는 **"Learn Once, Apply Anywhere"**가 가능합니다.

- **핵심 로직(ViewModel, Model, Network):** 99% 공유합니다. (복사-붙여넣기 아님!)
    
- **UI(View):** 80% 이상 공유 가능하지만, 각 기기의 특성(화면 크기, 입력 방식)에 맞춰 조금씩 다듬어줍니다.

즉, 지금 만드시는 `TimerManager` 클래스는 아이폰, 아이패드, 맥, 워치에서 **그대로 쓸 수 있습니다.**

---

**🤿 딥하게 들어가기 (Deep: Target과 모듈화)**

Xcode에서 이걸 어떻게 구현하는지 아키텍처 관점에서 보겠습니다.

**1. 타겟(Target) 추가하기** Xcode 프로젝트 설정에 가서 `+` 버튼을 누르고 **watchOS App**이나 **macOS App**을 추가하면 됩니다. 그러면 프로젝트 안에 폴더가 생기는데, 이때 중요한 건 **"파일을 복사하지 않는 것"**입니다.

**2. Target Membership (코드 공유의 핵심)** `TimerManager.swift` 파일을 클릭하고 우측 인스펙터(File Inspector)를 보세요. `Target Membership`이라는 체크박스 목록이 있을 겁니다.

- [v] iOS App
    
- [v] watchOS App
    
- [v] macOS App

이렇게 체크만 해주면, 하나의 파일이 세 가지 앱 모두에 컴파일되어 들어갑니다. 로직을 수정하면 3개 앱에 동시에 반영되죠! 🍯

### ⌚️ watchOS 개발 꿀팁 (펌웨어 감성)

사용자님은 펌웨어 경험이 있으시니 워치 개발이 아주 재밌으실 겁니다. 워치는 배터리와 성능 제약이 심해서 **최적화**가 중요하거든요.

1. **Always On Display (AOD):** 워치는 손목을 내리면 화면이 어두워집니다. 이때 타이머 숫자를 0.1초 단위로 갱신하면 배터리가 녹습니다. AOD 모드일 때는 1분 단위로 갱신하거나, 특정 UI 요소를 숨기는 처리를 해줘야 합니다 (`Environment(\.isLuminanceReduced)` 사용).
    
2. **Digital Crown:** 아이폰의 스크롤과 달리, 워치는 용두(Crown)를 돌려서 값을 조절하죠? SwiftUI에서는 `.digitalCrownRotation()` 수식어 하나로 이 물리적인 입력을 아주 쉽게 처리할 수 있습니다.
    
3. **Complication (컴플리케이션):** 워치 페이스 구석에 작은 아이콘이나 숫자를 띄우는 기능입니다. 타이머 앱이라면 "남은 시간"을 페이스에 띄워주는 게 필수겠죠? (`WidgetKit` 사용)

### 🖥️ macOS 개발 꿀팁 (Mac Catalyst vs Native)

맥 앱을 만드는 방법은 두 가지가 있습니다.

1. **Mac Catalyst:** "아이패드 앱을 맥에서 그대로 돌리는 기술"입니다. 체크박스 하나만 켜면 되지만, 버튼이나 UI가 좀 '모바일스러운' 느낌이 납니다.
    
2. **Native macOS (SwiftUI):** 맥 전용 타겟을 만들어서 뷰를 따로 짜는 겁니다. 메뉴바(Menu Bar)에 타이머를 띄우거나, 키보드 단축키(Command+S 등)를 적극적으로 활용할 수 있어서 훨씬 '맥 앱다운' 퀄리티가 나옵니다.
