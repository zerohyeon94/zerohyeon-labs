**🗣️ 가볍게 시작하기 (Simple): 와이파이(Wi-Fi) 공유기 달기**

SwiftUI의 `@EnvironmentObject`는 집에 **'공유기'**를 설치하는 것과 완벽히 똑같습니다.

- **과거 (릴레이 지옥):** 안방에 있는 D에게 인터넷을 연결해주려고, 거실(A) ➔ 주방(B) ➔ 옷방(C) ➔ 안방(D)으로 길고 거추장스러운 랜선을 억지로 끌고 왔습니다.
    
- **현재 (`@EnvironmentObject`):** 거실(A)에 와이파이 공유기를 딱 켜둡니다. 그러면 주방(B)이나 옷방(C)은 신경 쓸 필요 없이, 안방(D)에서 그냥 와이파이 비밀번호만 입력해서 바로 데이터를 꺼내 씁니다.

---

**🤿 딥하게 들어가기 (Deep: SwiftUI의 마법)**

코드로 보면 얼마나 허무할(?) 정도로 쉬운지 볼 수 있습니다.

**1. 공유기 켜기 (최상단 뷰)** 데이터를 `ObservableObject` 클래스로 만들고, 뷰 계층의 가장 꼭대기(또는 상위 뷰)에 `.environmentObject()`를 달아서 공중에 뿌려줍니다.

```Swift
// 앱의 시작점이나 최상단 부모 뷰
ContentView()
    .environmentObject(SharedDataManager()) // "내 밑으로 있는 애들은 다 이거 써라!"
```

**2. 와이파이 접속하기 (저 밑바닥 자식 뷰)** 중간 뷰들은 아무것도 안 해도 됩니다. 데이터를 쓸 뷰에서만 이렇게 선언합니다.

```Swift
struct DeepChildView: View {
    // "공중에 떠다니는 SharedDataManager 타입의 데이터를 내가 잡아서 쓸게!"
    @EnvironmentObject var dataManager: SharedDataManager
    
    var body: some View {
        Text("가져온 데이터: \(dataManager.someValue)")
    }
}
```

정말 깔끔하죠? 화면이 아무리 깊어져도 코드가 지저분해지지 않습니다.

---

### 🏛️ UIKit에서는 이걸 어떻게 해결했을까요?

UIKit에는 `@EnvironmentObject`처럼 프레임워크 자체에서 지원하는 완벽한 '와이파이' 기능이 내장되어 있지 않았습니다. 그래서 개발자들이 스스로 패턴을 만들어내며 생존해야 했죠. 주로 3가지 대응 방안을 썼습니다.

**1. 싱글턴 패턴 (Singleton Pattern)** 가장 쉽고, 가장 많이 쓰였지만, 가장 위험한 방법입니다.

```Swift
class DataManager {
    static let shared = DataManager() // 전역 변수처럼 사용
    var someValue: String = ""
}

// D 화면에서 호출할 때
let data = DataManager.shared.someValue
```

- **장점:** 어디서든 릴레이 없이 바로 접근 가능.
    
- **단점:** 누구나 접근할 수 있어서 언제, 누가 데이터를 바꿨는지 추적이 어렵고 메모리에 계속 남아있습니다.

**2. NotificationCenter (사내 방송)** 앱 전체에 방송을 때리는 방식입니다.

- A 화면: "데이터 바꼈다!! (방송 송출)"
    
- D 화면: "어? 내 번호 부르네? 데이터 업데이트 해야지. (방송 수신)" 이 방식은 릴레이를 없애주지만, 방송이 너무 많아지면 코드가 어디로 튀는지 알 수 없는 '스파게티 코드'가 되기 쉽습니다.

**3. Coordinator 패턴 + 의존성 주입 (Dependency Injection)** 현업 UIKit 개발자들이 최종적으로 정착한 가장 우아한 아키텍처입니다. 각 화면(ViewController)이 다음 화면을 직접 만들어서 띄우는 게 아니라, **'Coordinator(길잡이)'**라는 별도의 객체가 화면 전환과 데이터 주입을 전담합니다.

- Coordinator: "음, 지금 A 화면에서 D 화면으로 바로 가야 하네? 내가 D 화면 만들면서 데이터 딱 찔러(Inject) 넣어주고 띄워줄게." 이렇게 하면 뷰 컨트롤러 간의 결합도가 낮아지고 릴레이 지옥을 막을 수 있습니다.

---

**💡 오늘의 요약**

> UIKit에서는 전역 변수(Singleton)나 별도의 아키텍처(Coordinator)를 도입해서 억지로 릴레이 지옥을 피해야 했지만, SwiftUI는 `@EnvironmentObject`라는 강력한 내장 도구로 이 문제를 아주 우아하게 해결했습니다.
