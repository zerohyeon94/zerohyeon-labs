**🗣️ 가볍게 시작하기 (Simple): "딱 두 가지만 기억하세요"**

현재 IT 업계(특히 스타트업과 유니콘 기업들)에서 가장 사랑받는 iOS 자동 배포 조합은 명확합니다. 무거운 젠킨스를 직접 구축하기보다는, 클라우드 위에서 가볍고 우아하게 돌아가는 **GitHub Actions**와 **Fastlane**의 조합이 압도적인 표준입니다.

---

**🤿 딥하게 들어가기 (Deep: 무엇을, 왜, 어떻게 공부해야 할까?)**

**1순위: Fastlane (배달의 민족... 아니, 배포의 신) ⭐️⭐️⭐️⭐️⭐️** 이건 선택이 아니라 필수입니다. 애플의 복잡한 배포 과정을 마법처럼 줄여주는 도구입니다.

- **무엇을 배우나요?**
    
    - `match`: 현업에서 가장 골치 아파하는 '팀원 간 인증서 공유' 문제를 암호화된 GitHub 저장소를 통해 한 방에 해결하는 기능입니다. (면접에서 `match` 써봤다고 하면 무조건 합격 점수를 받습니다!)
        
    - `gym`: 앱을 빌드하고 `ipa` 파일(설치 파일)로 압축해 주는 기능입니다.
        
    - `deliver`: 앱스토어 스크린샷, 릴리즈 노트 등을 자동으로 업로드해 주는 기능입니다.
        
- **어떻게 학습하나요?** 터미널 환경에 익숙하시니, 프로젝트 폴더에서 `fastlane init`을 입력하고 `Fastfile`이라는 텍스트 파일에 스크립트를 짜보는 연습부터 하시면 됩니다.

**2순위: GitHub Actions (최강의 접근성) ⭐️⭐️⭐️⭐️** 우리가 이미 코드를 저장하고, 버전을 관리하는 그곳에서 CI/CD까지 한 번에 끝내는 기술입니다.

- **무엇을 배우나요?**
    
    - `.yml` (야믈) 파일 작성법: "누군가 main 브랜치에 코드를 Push하면 -> Mac 가상 서버를 빌려서 -> XCTest를 돌려라"라는 명령서를 작성하는 문법을 배웁니다.
        
    - Secrets 관리: API 키나 애플 인증서 비밀번호를 안전하게 숨겨서 서버에 전달하는 방법을 익힙니다.
        
- **어떻게 학습하나요?** 저장소에 `.github/workflows` 폴더를 만들고, 내가 코드를 올릴 때마다 이전에 짠 Unit Test 로봇들이 알아서 윙윙 돌아가게 만드는 '테스트 자동화'부터 구축해 보세요.

**3순위: Xcode Cloud (애플의 공식 무기) ⭐️⭐️⭐️** 2026년 현재, 꽤 많이 성숙해져서 애플 생태계를 벗어나지 않으려는 팀들이 적극적으로 도입하고 있습니다.

- **무엇을 배우나요?**
    
    - Xcode 프로그램 내부에서 클릭 몇 번으로 빌드, 테스트, TestFlight 배포를 연결하는 워크플로우 세팅법.
        
- **특징:** `.yml` 코드를 짤 필요 없이 UI가 너무 잘 되어 있지만, 1순위와 2순위만큼 커스텀이 자유롭지 못하다는 한계가 있어서 우선순위는 살짝 뒤로 미루셔도 좋습니다.

### 🚀 포트폴리오 프로젝트에 당장 적용하는 꿀팁!

면접관에게 어필할 수 있는 가장 멋진 구조는 다음과 같습니다.

1. **로컬 작업:** 내 컴퓨터에서 집중 タイ머 앱 기능 개발 및 테스트 코드 작성 ➔ **Git Push**
    
2. **CI (GitHub Actions 동작):** GitHub에 코드가 올라오자마자, 자동으로 가상 Mac 서버가 코드를 받고 Unit Test를 돌림. 실패하면 PR 병합(Merge) 차단!
    
3. **CD (Fastlane 동작):** `main` 브랜치에 코드가 무사히 합쳐지면, Fastlane이 가동되어 인증서를 입히고 TestFlight에 앱을 알아서 올려줌.

이 파이프라인만 포트폴리오 프로젝트(README.md 등)에 예쁜 아키텍처 다이어그램으로 그려두시면, **"아, 이 지원자는 입사 첫날부터 당장 실무 팀의 워크플로우에 녹아들 수 있겠구나!"**라는 엄청난 신뢰를 줄 수 있습니다.
