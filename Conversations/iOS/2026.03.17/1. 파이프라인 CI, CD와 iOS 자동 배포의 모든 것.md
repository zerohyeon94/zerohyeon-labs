**🗣️ 가볍게 시작하기 (Simple): "공장 조립 라인과 자동 택배 발송"**

- **CI (Continuous Integration - 지속적 통합):** 여러 명의 개발자가 각자 짠 코드를 메인 저장소(GitHub)에 합칠 때, **"이 코드가 기존 앱을 망가뜨리지 않는지" 자동으로 빌드하고 테스트(XCTest)해 보는 품질 검사소**입니다.
    
- **CD (Continuous Delivery/Deployment - 지속적 배포):** 검사를 무사히 통과한 코드를 실제 아이폰에 설치할 수 있는 앱 파일로 포장(Archive)해서, **테스터나 유저들에게 자동으로 배달해 주는 택배 시스템**입니다.

이 두 과정이 끊기지 않고 하나로 쭉 이어져서 물 흐르듯 진행된다고 하여 **'파이프라인(Pipeline)'**이라고 부릅니다.

**🤿 딥하게 들어가기 (Deep 1: iOS에서 CI가 하는 일)**

내가 오늘 하루 종일 '타이머 기능'을 만들고 GitHub에 PR(Pull Request)을 올렸다고 상상해 보겠습니다.

**❌ CI가 없을 때 (수동 지옥):** 팀장이 내 코드를 다운받아서, 본인 Mac에서 Xcode를 켜고, 시뮬레이터를 띄워서, `Cmd + U`를 눌러 테스트가 통과하는지 직접 눈으로 확인해야 합니다. "내 컴퓨터에선 되는데 팀장님 컴퓨터에선 안 되는데요?" 같은 악몽이 시작됩니다.

**✅ CI가 있을 때 (자동화 천국):**

1. 내가 코드를 GitHub에 올리는 순간, 클라우드 어딘가에 있는 **가상의 Mac 컴퓨터(Runner)**가 자동으로 켜집니다.
    
2. 그 Mac이 내 코드를 다운받고, 알아서 외부 라이브러리(Alamofire, RxSwift 등)를 설치합니다.
    
3. 화면 없는 백그라운드 상태에서 앱을 쫙 빌드하고, 우리가 짰던 수십 개의 Unit Test 로봇들을 일제히 실행시킵니다.
    
4. 3분 뒤, GitHub PR 화면에 **초록색 체크 마크(✅ All checks have passed)**를 띄워줍니다.
    
5. 팀장은 안심하고 합치기(Merge) 버튼만 누르면 됩니다!

**🤿 더 딥하게 (Deep 2: iOS 자동 배포(CD)의 험난한 여정)**

iOS 앱을 심사(App Store)에 올리거나 내부 테스터(TestFlight)에게 보내려면 정말 귀찮은 수작업이 필요합니다.

1. 개발자 인증서와 프로비저닝 프로파일 매칭하기
    
2. 앱 버전 번호(예: 1.0.1 -> 1.0.2) 올리기
    
3. 무거운 Xcode를 켜서 `Product ➔ Archive` 누르고 10분 넘게 기다리기
    
4. App Store Connect에 업로드하고 또 처리될 때까지 기다리기
    
5. 슬랙(Slack)에 "여러분 새 버전 올라갔어요~" 공지 쓰기

**🌟 CD 파이프라인이 이 모든 걸 해치웁니다!** 코드가 `main` 브랜치에 합쳐지는 순간, CD 로봇이 가동됩니다.

- **"버전 올리고, 인증서 서명해서 압축하고, 애플 서버로 전송한 다음, 슬랙에 배포 완료 메시지까지 쏴줘!"** 개발자는 코드를 합치고 커피 한 잔 마시고 오면, 이미 전 세계 유저들의 폰에 업데이트가 떠 있게 됩니다.

**🛠️ 현업에서 쓰는 핵심 도구 3대장**

1. **Fastlane (가장 중요 ⭐️⭐️⭐️):**
    
    - iOS 배포 자동화의 '신'입니다. 인증서 관리부터 앱스토어 스크린샷 캡처, TestFlight 업로드까지 스크립트 한 줄로 끝내주는 마법의 도구입니다. CI/CD를 구축한다면 무조건 Fastlane부터 배웁니다.
        
2. **GitHub Actions:**
    
    - 요즘 가장 대세인 무료/유료 CI/CD 플랫폼입니다. GitHub 안에서 `.yml` 파일 하나만 작성하면 가상의 Mac을 무료로 빌려 쓸 수 있어서 접근성이 엄청납니다.
        
3. **Xcode Cloud:**
    
    - 애플이 직접 내놓은 솔루션입니다. 설정이 기가 막히게 쉽고 애플 생태계와 완벽하게 연동되지만, 속도가 다소 느리고 요금이 비싸다는 단점이 있습니다.

**💡 오늘의 요약**

> **CI**는 내 코드가 똥(?)이 아닌지 서버가 대신 **검사(Test)**해 주는 것이고, **CD**는 통과된 코드를 **Fastlane** 같은 도구를 써서 TestFlight나 App Store로 알아서 **배달(Deploy)**해 주는 것입니다.
