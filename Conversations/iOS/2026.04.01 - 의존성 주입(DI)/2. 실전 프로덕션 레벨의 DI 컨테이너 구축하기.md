**🗣️ 가볍게 시작하기 (Simple): "공장과 창고의 차이"**

실전 DI 컨테이너는 객체를 단순히 보관하는 '창고'를 넘어서, 요청이 들어올 때마다 객체를 찍어내는 '공장(Factory)' 역할까지 해야 합니다.

- 뷰모델이 `FocusMLService`를 달라고 할 때마다 새 객체를 주면 메모리가 낭비되겠죠? 이런 건 **앱 전체에서 딱 1개만 공유(Singleton)**하도록 세팅합니다.
    
- 반면, 뷰모델 자체는 화면이 켜질 때마다 새로 만들어져야 하니까 **매번 새로 찍어내도록(Factory)** 세팅합니다.

---

**🤿 딥하게 들어가기 (Deep: 실전 코드 3단계)**

Swinject 같은 무거운 외부 라이브러리를 쓰지 않고, 순수 Swift로 구현하는 실무형 경량 DI 컨테이너입니다.

**Step 1: 진짜 실무형 DI Container 만들기** 핵심은 '객체'를 바로 저장하는 게 아니라, **'객체를 생성하는 클로저(함수)'**를 저장하는 것입니다.

```Swift
import Foundation

final class DIContainer {
    static let shared = DIContainer()
    private init() {}
    
    // 객체 생성 공장(클로저)들을 보관하는 금고
    private var factories: [String: () -> Any] = [:]
    
    // 싱글톤으로 공유할 객체들을 보관하는 금고
    private var sharedInstances: [String: Any] = [:]
    
    // 1. 등록하기 (공장 세팅)
    // isSingleton을 true로 주면 앱 전체에서 1개만 돌려씁니다.
    func register<T>(_ type: T.Type, isSingleton: Bool = false, factory: @escaping () -> T) {
        let key = String(describing: type)
        
        if isSingleton {
            // 싱글톤이면 아예 생성해서 금고에 평생 보관!
            sharedInstances[key] = factory()
        } else {
            // 아니면 공장(클로저)만 등록해 둡니다.
            factories[key] = factory
        }
    }
    
    // 2. 꺼내 쓰기 (주문하기)
    func resolve<T>() -> T {
        let key = String(describing: T.self)
        
        // 싱글톤 금고에 있으면 그거 주고, 없으면 공장에서 새로 하나 찍어냅니다!
        if let shared = sharedInstances[key] as? T {
            return shared
        } else if let factory = factories[key] {
            return factory() as! T
        } else {
            fatalError("\(key)가 컨테이너에 등록되지 않았습니다! Crash!")
        }
    }
}
```

**Step 2: 앱이 켜질 때 모든 공장 등록하기 (`App` 껍데기)** 앱이 실행되는 가장 첫 진입점(`@main`)에서 멀티탭에 모든 코드를 꽂아줍니다.

```Swift
import SwiftUI

@main
struct FocusSenseApp: App {
    
    init() {
        setupDependencies()
    }
    
    var body: some Scene {
        WindowGroup {
            // 메인 뷰를 띄웁니다!
            FocusTimerView()
        }
    }
    
    // ⭐️ 여기가 핵심입니다. 앱의 모든 의존성 조립 설명서!
    private func setupDependencies() {
        let container = DIContainer.shared
        
        // 1. CoreML 서비스는 엄청 무거우니까 앱 전체에서 1개만 공유 (isSingleton: true)
        container.register(FocusMLProtocol.self, isSingleton: true) {
            return FocusMLService() 
        }
        
        // 2. 뷰모델은 화면 켤 때마다 새로 만들어야 하니까 Factory로 등록
        container.register(FocusTimerViewModel.self, isSingleton: false) {
            // 뷰모델을 만들 때 필요한 ML 서비스를 자기 자신이 다시 resolve 해서 꽂아줍니다! (DI의 마법)
            let mlService: FocusMLProtocol = container.resolve()
            return FocusTimerViewModel(mlService: mlService)
        }
    }
}
```

**Step 3: 화면(View)과 뇌(ViewModel)에서 우아하게 꺼내 쓰기** 이제 `TimerFeature` 모듈 내부에서는 객체를 어떻게 만드는지, 진짜 객체인지 가짜(Mock)인지 전혀 신경 쓸 필요가 없습니다.

```Swift
// 📁 TimerFeature 내부

class FocusTimerViewModel: ObservableObject {
    @Published var score: Int = 0
    private let mlService: FocusMLProtocol
    
    // 밖에서 누군가(컨테이너가) 주입해 주기를 기다립니다.
    init(mlService: FocusMLProtocol) {
        self.mlService = mlService
    }
    
    func start() {
        mlService.startTracking()
    }
}

struct FocusTimerView: View {
    // ⭐️ 화면을 그릴 때, 내가 직접 뷰모델을 생성하지 않고 컨테이너에 "하나 만들어줘!" 라고 요청합니다.
    @StateObject private var viewModel: FocusTimerViewModel = DIContainer.shared.resolve()
    
    var body: some View {
        Button("집중 시작") {
            viewModel.start()
        }
    }
}
```

### 🚨 이렇게 짰을 때의 엄청난 시너지 (가짜 데이터 테스트)

나중에 Unit Test를 짤 때 코드가 얼마나 예뻐지는지 보세요. 앱 코드는 1줄도 안 건드리고, 테스트 파일에서 컨테이너의 부품만 살짝 갈아 끼우면 됩니다!

```Swift
// 테스트 코드 파일 내부
override func setUp() {
    // 앱이 켜질 때 등록했던 진짜 ML 서비스 대신, 
    // 가짜(Mock) ML 서비스를 컨테이너에 덮어씌워 버립니다!
    DIContainer.shared.register(FocusMLProtocol.self, isSingleton: true) {
        return MockFocusMLService(mockScore: 100)
    }
    
    // 이제 뷰모델을 resolve하면 자연스럽게 가짜 객체가 꽂혀서 나옵니다.
    sut = DIContainer.shared.resolve() 
}
```

**💡 오늘의 핵심 요약**

> 실전 DI 컨테이너는 `@main` (앱 진입점)에서 **어떤 프로토콜(돼지코)에 어떤 클래스(플러그)를 꽂을지 한 번에 싹 다 정의(Register)**해 두고, View나 ViewModel에서는 **컨테이너를 통해 꺼내 쓰는(Resolve)** 완벽한 분업 시스템입니다.
