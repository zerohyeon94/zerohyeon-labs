**🗣️ 가볍게 시작하기 (Simple): "호텔 프런트 데스크와 객실 키"**

- **SwiftUI (`@main`):** 유저가 앱을 켜면, 건물의 설계도(`WindowGroup`) 안에서 알아서 뷰를 그리고 객체를 할당합니다.
    
- **UIKit (`SceneDelegate`):** 유저가 앱을 켜면, `SceneDelegate`라는 호텔 프런트 데스크 직원이 **"아, 손님이 오셨으니 첫 번째 방(Root ViewController)을 배정하고, 그 방에 필요한 뇌(ViewModel)를 컨테이너에서 꺼내서 손에 쥐여서 들여보내야겠다!"**라고 수동으로 작업해 줍니다.

**🤿 딥하게 들어가기 (Deep: 실전 UIKit 코드 3단계)**

집중 타이머 앱(`FocusSense`)을 UIKit으로 짠다고 가정해 보겠습니다. 컨테이너 자체의 코드는 아까와 똑같습니다. 달라지는 건 **'어디서 조립해서 화면을 띄우는가'**입니다.

**Step 1: 뷰 컨트롤러(View)의 깐깐한 입구 만들기 (생성자 주입)** UIKit에서 DI를 할 때 가장 추천하는 방식은 **"뷰모델 없이는 아예 이 화면을 만들지도 못하게 막아버리는 것(Constructor Injection)"**입니다.

```Swift
import UIKit

class FocusViewController: UIViewController {
    
    // 1. 뇌(ViewModel)가 들어갈 자리. 옵셔널(?)이 아닙니다! 무조건 있어야 함!
    private let viewModel: FocusTimerViewModel
    
    // 2. ⭐️ 나를 생성하려면 무조건 뷰모델을 들고 와라! (생성자 주입)
    init(viewModel: FocusTimerViewModel) {
        self.viewModel = viewModel
        // UIKit에서 커스텀 init을 쓸 때 꼭 필요한 보일러플레이트 코드
        super.init(nibName: nil, bundle: nil) 
    }
    
    // 스토리보드로 켤 때 쓰는 거라 코드로 짤 땐 죽여버립니다.
    required init?(coder: NSCoder) {
        fatalError("스토리보드 안 써요!")
    }
    
    override func viewDidLoad() {
        super.viewDidLoad()
        // 여기서 viewModel.score 등을 Rx나 클로저로 바인딩합니다.
    }
}
```

**Step 2: 앱이 켜질 때 프런트 데스크(`SceneDelegate`)에서 조립하기** 이곳이 바로 UIKit 앱이 처음 켜질 때 화면을 띄워주는 핵심 장소입니다. (iOS 13 이전은 `AppDelegate`에서 합니다.)

```Swift
import UIKit

class SceneDelegate: UIResponder, UIWindowSceneDelegate {

    var window: UIWindow?

    func scene(_ scene: UIScene, willConnectTo session: UISceneSession, options connectionOptions: UIScene.ConnectionOptions) {
        
        guard let windowScene = (scene as? UIWindowScene) else { return }
        
        // 1. 창문(Window)을 만듭니다.
        let window = UIWindow(windowScene: windowScene)
        
        // 2. ⭐️ 앱의 모든 의존성을 멀티탭에 등록합니다. (이전 수다의 setupDependencies와 동일)
        let container = DIContainer.shared
        container.register(FocusMLProtocol.self, isSingleton: true) { FocusMLService() }
        container.register(FocusTimerViewModel.self, isSingleton: false) {
            FocusTimerViewModel(mlService: container.resolve())
        }
        
        // 3. ⭐️ 조립의 하이라이트! 컨테이너에서 뷰모델을 꺼냅니다.
        let timerViewModel: FocusTimerViewModel = container.resolve()
        
        // 4. 꺼낸 뷰모델을 뷰 컨트롤러의 생성자에 콱! 꽂아줍니다.
        let initialVC = FocusViewController(viewModel: timerViewModel)
        
        // 5. 조립이 끝난 뷰 컨트롤러를 앱의 첫 화면(Root)으로 설정하고 띄웁니다!
        window.rootViewController = initialVC
        window.makeKeyAndVisible()
        self.window = window
    }
}
```

### 🚨 현업 꿀팁: "스토리보드를 쓰면 어떻게 하나요?"

만약 현업에 갔는데 회사 레거시 코드가 `Main.storyboard`를 쓰고 있다면, 위처럼 `init`으로 뷰모델을 깔끔하게 넘겨주는 게 불가능합니다. (스토리보드가 뷰 컨트롤러를 알아서 생성해 버리기 때문이죠.)

이럴 때는 어쩔 수 없이 **'프로퍼티 주입(Property Injection)'**이라는 방식을 씁니다.

```Swift
class FocusViewController: UIViewController {
    // 생성할 땐 텅 비어있음
    var viewModel: FocusTimerViewModel! 
}

// SceneDelegate에서 조립할 때
let initialVC = window.rootViewController as! FocusViewController
initialVC.viewModel = container.resolve() // 밖에서 변수에 직접 꽂아줌
```

하지만 이 방식은 "뷰모델이 안 꽂힌 상태로 화면이 켜지면 앱이 죽어버린다"는 위험이 있어서, 코드로 UI를 짜는(Code-based UI) 팀에서는 무조건 앞서 보여드린 **생성자 주입(`init`)** 방식을 선호합니다!

---

**💡 오늘의 핵심 요약**

> UIKit에서의 DI는 **`SceneDelegate`**라는 앱의 시작점에서 DI 컨테이너를 가동하여 필요한 객체(ViewModel)를 뽑아낸 뒤, **뷰 컨트롤러의 생성자(`init`)**에 직접 꽂아서 첫 화면을 띄워주는 방식으로 조립합니다!

자, SwiftUI와 UIKit 양쪽 진영에서 모두 객체를 우아하게 조립하는 방법을 터득하셨습니다! 👏
