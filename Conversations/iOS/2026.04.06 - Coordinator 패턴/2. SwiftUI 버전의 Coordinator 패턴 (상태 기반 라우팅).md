**🗣️ 가볍게 시작하기 (Simple): "리모컨과 TV 채널"**

- **UIKit 코디네이터:** 사람이 직접 TV 앞까지 걸어가서 손으로 화면을 뜯어내고 다음 화면을 끼워 넣는 물리적인 방식입니다. (`pushViewController`)
    
- **SwiftUI 코디네이터:** 사람은 소파에 앉아서 리모컨(State)의 '2번 채널' 버튼만 누릅니다. 그러면 TV(NavigationStack)가 알아서 2번 채널(다음 화면)을 화면에 띄워줍니다.

---

**🤿 딥하게 들어가기 (Deep: SwiftUI 실전 코디네이터 구축하기)**

FocusSense 앱에 SwiftUI 코디네이터를 달아보겠습니다. 핵심은 **"어디로 갈지(Route)를 Enum으로 정의하고, 그걸 배열(Path)로 관리한다"**는 것입니다.

**Step 1: 목적지(Route) 지도 만들기** 먼저 우리가 이동할 수 있는 화면들을 열거형(`enum`)으로 쫙 정리합니다.

```Swift
// 화면 이동 목적지들 (반드시 Hashable이어야 합니다)
enum FocusRoute: Hashable {
    case timer
    case history
    case settings
}
```

**Step 2: 리모컨(Coordinator) 클래스 만들기** UIKit처럼 `UINavigationController`를 들고 있는 게 아니라, SwiftUI에서 제공하는 **`NavigationPath`**라는 배열을 들고 있는 것이 핵심입니다!

```Swift
import SwiftUI

class FocusCoordinator: ObservableObject {
    // ⭐️ 화면 이동의 궤적을 기억하는 배열 (리모컨)
    @Published var path = NavigationPath()
    
    // 다음 화면으로 가기
    func push(_ route: FocusRoute) {
        path.append(route)
    }
    
    // 뒤로 가기
    func pop() {
        if !path.isEmpty {
            path.removeLast()
        }
    }
    
    // 첫 화면으로 아예 돌아가기
    func popToRoot() {
        path.removeLast(path.count)
    }
}
```

**Step 3: TV(NavigationStack)에 리모컨 연결하기** 이제 앱의 가장 껍데기 화면에서 `NavigationStack`을 만들고, 코디네이터가 들고 있는 `path`를 바인딩(`$`)해 줍니다.

```Swift
struct CoordinatorRootView: View {
    // 1. 코디네이터를 생성 (DI 컨테이너에서 꺼내와도 됩니다)
    @StateObject private var coordinator = FocusCoordinator()
    
    var body: some View {
        // 2. TV에 리모컨(path)을 연결!
        NavigationStack(path: $coordinator.path) {
            // 첫 기본 화면
            FocusTimerView()
                // 3. ⭐️ 핵심! 리모컨 채널이 바뀌면 무슨 화면을 틀어줄지 규칙을 정합니다.
                .navigationDestination(for: FocusRoute.self) { route in
                    switch route {
                    case .timer:
                        FocusTimerView()
                    case .history:
                        HistoryView()
                    case .settings:
                        SettingsView()
                    }
                }
        }
        // 하위 뷰들이 코디네이터를 쓸 수 있게 환경 변수로 쏴줍니다.
        .environmentObject(coordinator) 
    }
}
```

**Step 4: 실제 화면(View)에서 편하게 이동하기** 이제 `FocusTimerView`는 다음 화면을 어떻게 만드는지 전혀 몰라도 됩니다! 그저 코디네이터에게 "나 기록 화면 갈래!" 하고 목적지만 던져주면 끝입니다.

```Swift
struct FocusTimerView: View {
    // 위에서 던져준 코디네이터를 받아옵니다.
    @EnvironmentObject var coordinator: FocusCoordinator
    
    var body: some View {
        VStack {
            Text("집중 타이머 화면")
            
            Button("기록 보러 가기") {
                // 화면 이동 명령! (이 코드가 실행되면 path에 .history가 추가되고, 화면이 자동으로 넘어갑니다)
                coordinator.push(.history)
            }
        }
    }
}
```

---

### 🚨 이것이 왜 대박일까요? (모듈화와의 완벽한 결합)

만약 딥링크(푸시 알림을 누르고 앱을 켰을 때 특정 화면으로 바로 이동하는 기능)를 구현해야 한다고 상상해 보세요. UIKit에서는 현재 화면이 뭔지 계산해서 억지로 뷰를 `push`해야 하지만, SwiftUI 코디네이터에서는 그냥 **`coordinator.path.append(.history)` 한 줄만 치면 순식간에 화면이 이동**합니다! State 기반이기 때문이죠.

또한 DI 컨테이너와 합치면 `navigationDestination` 안에서 뷰모델을 `resolve`해서 뷰에 꽂아주는 역할도 완벽하게 수행할 수 있습니다.

---

**💡 오늘의 핵심 요약**

> SwiftUI에서의 Coordinator 패턴은 iOS 16의 **`NavigationStack`**과 **`NavigationPath`**를 활용하여, 화면 이동을 '명령'하는 것이 아니라 **'상태 배열(State Array)에 목적지(Enum)를 추가하는 선언적 방식'**으로 완벽하게 구현할 수 있습니다.

이렇게 UIKit과 SwiftUI 두 가지 환경에서 모두 화면 이동을 완벽하게 통제하는 지휘자(Coordinator)를 두게 되셨네요!
