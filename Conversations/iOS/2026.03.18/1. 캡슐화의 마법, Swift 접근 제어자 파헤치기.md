**🗣️ 가볍게 시작하기 (Simple): "우리 집 방문 열쇠 등급"**

앱 전체를 커다란 집이라고 생각해 보겠습니다.

- **거실:** 누구나 들어와서 TV를 볼 수 있습니다.
    
- **안방:** 우리 가족만 들어올 수 있습니다.
    
- **내 일기장:** 오직 나 혼자만 볼 수 있습니다.

코드도 마찬가지입니다. 어떤 변수나 함수는 다른 파일에서 마음껏 써도 되지만, 어떤 데이터는 철저하게 숨겨야(은닉, 캡슐화) 안전합니다. 이 문단속 등급을 정해주는 것이 바로 접근 제어자입니다.

---

**🤿 딥하게 들어가기 (Deep: 5가지 문단속 등급)**

Swift에는 가장 널널한 것부터 가장 빡빡한 것까지 5단계의 자물쇠가 있습니다.

**1. `open` (대문 개방 & 인테리어 공사 허용)**

- 가장 개방적인 상태입니다. 다른 모듈(다른 프로젝트나 프레임워크)에서도 쓸 수 있고, 심지어 내 클래스를 **상속(Override)받아서 마음대로 뜯어고칠 수도** 있습니다.
    
- (주로 애플이 UIKit 같은 프레임워크를 만들 때 씁니다. 우리가 뷰 컨트롤러를 상속받아 쓸 수 있는 이유죠!)

**2. `public` (대문 개방 & 구경만 허용)**

- `open`처럼 밖에서 자유롭게 가져다 쓸 수는 있지만, **상속해서 뜯어고치는 건 불가능**합니다.
    
- (내가 만든 타이머 기능을 남들이 쓸 수 있게 라이브러리로 배포할 때 씁니다.)

**3. `internal` (우리 가족 전용) ⭐️ 기본값**

- **아무것도 안 적으면 자동으로 적용되는 기본 등급입니다.**
    
- 같은 모듈(내 앱 프로젝트 전체) 안에서는 어디서든 자유롭게 쓸 수 있습니다. 타이머 앱 안이라면 A 화면이든 B 화면이든 서로 호출할 수 있죠.

**4. `fileprivate` (같은 방 안에서만)**

- 파일(File) 단위로 문을 잠급니다.
    
- 오직 **같은 `.swift` 파일 안에서만** 서로 부를 수 있습니다. 파일이 다르면 절대 접근 불가능합니다.

**5. `private` (내 일기장) ⭐️ 제일 중요!**

- 가장 빡빡한 자물쇠입니다. 자기가 속한 괄호 `{ }` (클래스나 구조체) 밖으로는 절대 나갈 수 없습니다.

### 🚀 실무 적용: MVVM에서 `private`이 필수인 이유!

앞서 우리가 뷰모델(ViewModel)을 만들었었죠? 이때 `private`을 안 쓰면 끔찍한 일이 벌어집니다.

**❌ `private`을 안 썼을 때의 대참사**

```Swift
class FocusViewModel: ObservableObject {
    // 자물쇠가 없으면(internal) View에서 이걸 마음대로 바꿀 수 있습니다!
    @Published var score: Int = 0 
}

struct FocusView: View {
    @StateObject var viewModel = FocusViewModel()
    
    var body: some View {
        Button("꼼수 버튼") {
            // 뷰가 로직을 무시하고 점수를 마음대로 조작해버림! (MVVM 원칙 파괴)
            viewModel.score = 9999 
        }
    }
}
```

**✅ `private(set)`의 마법 (현업 국룰)** 점수를 읽는 건 뷰(View)가 마음대로 해도 되지만, 점수를 바꾸는 건 오직 뷰모델(ViewModel) 내부에서만 허용하고 싶을 때 쓰는 기가 막힌 문법입니다.

```Swift
class FocusViewModel: ObservableObject {
    // "읽는 건 누구나 해, 하지만 수정(set)은 나(private)만 할 거야!"
    @Published private(set) var score: Int = 0
    
    // 외부에서 점수를 올리고 싶으면 이 정해진 통로(함수)만 이용해!
    func addScore() {
        self.score += 10
    }
}

struct FocusView: View {
    @StateObject var viewModel = FocusViewModel()
    
    var body: some View {
        Text("점수: \(viewModel.score)") // ✅ 읽기는 허용됨!
        
        Button("정상 버튼") {
            // viewModel.score = 9999 // ❌ 컴파일 에러! (수정 불가)
            viewModel.addScore() // ✅ 정해진 함수로만 접근 가능
        }
    }
}
```

> 접근 제어자는 코드를 꽁꽁 숨기기 위해 쓰는 게 아닙니다. 남들이(혹은 미래의 내가) **"이 변수는 마음대로 건드리면 안 되는구나!"** 하고 설계자의 의도를 명확하게 알 수 있도록 **가이드라인**을 그어주는 친절한 표지판입니다.
