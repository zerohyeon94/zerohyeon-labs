수동으로 이미지를 굽고(Build) 올리고(Push) 서버에 접속해서 다시 다운받는(Pull) 과정을 완벽하게 이해하셨으니, 이제 이 모든 귀찮은 과정을 로봇에게 외주를 맡길 완벽한 타이밍입니다!

FocusSense나 가게씨 같은 앱의 백엔드 코드를 수정할 때마다 터미널을 열고 수동으로 배포 명령어를 치는 것은 꽤나 고된 일입니다. 코드를 수정하고 깃허브에 올리기만 하면, 알아서 서버까지 업데이트되는 마법 같은 기술인 **CI/CD 파이프라인 자동화**를 시원하게 뚫어드리겠습니다!

---

## 🗣️ 가볍게 시작하기 (Simple): "수동 택배 발송" vs "자동화 컨베이어 벨트"

- **수동 배포:** 내가 직접 물건(코드)을 포장하고, 우체국(Docker Hub)에 가서 접수한 뒤, 다시 수령지(EC2)로 달려가서 물건을 뜯고 진열하는 방식입니다.
    
- **CI/CD 자동화:** 내 책상 앞에 **'자동화 컨베이어 벨트(GitHub Actions)'**를 설치하는 것입니다. 내가 코드를 벨트 위에 올려두기만(git push) 하면, 로봇 팔이 알아서 포장하고 배송한 뒤 진열까지 완벽하게 끝내줍니다.

---

## 🤿 딥하게 들어가기 (Deep: GitHub Actions의 3단계 마법)

가장 대표적이고 무료로 쓸 수 있는 자동화 도구인 **GitHub Actions**가 어떻게 동작하는지 실전 흐름을 해부해 보겠습니다.

프로젝트 폴더 안에 `.github/workflows/deploy.yml` 이라는 로봇의 행동 지침서(스크립트) 하나만 만들어 두면 아래의 3단계가 자동으로 실행됩니다.

### Step 1: 방아쇠 당기기 (Trigger)

개발자가 로컬 컴퓨터에서 작업을 마치고 `git push origin main` 명령어를 쳐서 코드를 깃허브로 쏘아 올립니다. 이 순간 깃허브가 "어? 메인 코드가 바뀌었네? 로봇 출동해!" 하고 방아쇠를 당깁니다.

### Step 2: 로봇의 요리 및 창고 보관 (CI - Continuous Integration)

깃허브가 제공하는 가상의 컴퓨터(로봇)가 내 코드를 건네받습니다.

1. 로봇이 내 코드의 에러가 없는지 테스트합니다.
    
2. 이상이 없으면, 로봇이 직접 `docker build` 명령어를 실행해서 도커 이미지를 구워냅니다.
    
3. 다 구워진 이미지를 로봇이 알아서 Docker Hub 창고로 `push` 합니다.
    

### Step 3: 로봇의 원격 접속 및 서버 업데이트 (CD - Continuous Deployment)

창고에 새로운 이미지가 올라갔으니, 이제 서버를 교체할 차례입니다.

1. 로봇이 내가 미리 깃허브 비밀금고(Secrets)에 넣어둔 접속 키(.pem)를 이용해 AWS EC2 우분투 서버에 몰래 `ssh` 접속을 합니다.
    
2. EC2 안에서 로봇이 `docker pull` 명령어를 쳐서 방금 창고에 올린 따끈따끈한 새 이미지를 다운받습니다.
    
3. 마지막으로 `docker-compose up -d`를 실행하여 기존의 낡은 서버 컨테이너를 내리고 새 서버를 띄운 뒤 홀연히 사라집니다.

## 🚨 핵심 포인트: 개발자는 오직 '코드'에만 집중합니다!

이 파이프라인이 구축되는 순간, 개발자의 일상에는 엄청난 변화가 일어납니다.

API 주소를 하나 추가하거나, AI 분석 로직의 오타를 하나 고쳤을 때, 예전처럼 AWS에 접속하고 도커 명령어를 칠 필요가 **단 1%도 없습니다.** 그저 평소처럼 `git push` 한 번만 치고 커피 한 잔 마시고 오면, 1~2분 뒤에 내 아이폰 앱에서 수정한 서버 코드가 완벽하게 동작하는 짜릿한 경험을 하게 됩니다.

## 💡 오늘의 요약 처방

- **CI/CD:** 지속적 통합(CI)과 지속적 배포(CD)의 약자로, 코드 수정부터 서버 반영까지의 모든 과정을 자동화하는 것을 뜻합니다.
    
- **GitHub Actions:** 깃허브에서 무료로 제공하는 강력한 CI/CD 로봇입니다.
    
- **작동 원리:** `git push` ➡️ 깃허브 로봇이 대신 도커 이미지 빌드/푸시 ➡️ 로봇이 EC2 접속 후 풀/런 실행
