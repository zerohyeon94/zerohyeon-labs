🗣️ **가볍게 시작하기 (Simple): "CoreData(기본 장착) vs Realm(외부 설치)"**

- **기본 내장 (Built-in):** 앱을 만들 때 뼈대에 기본적으로 포함되어 있어 별도 설치 없이 바로 쓸 수 있는 CoreData나 SwiftData 같은 기능들이죠.
    
- **외부 설치 (Library):** 파이썬의 ORM은 마치 SPM이나 CocoaPods를 통해 외부에서 끌어와 설치하는 **Realm**이나 통신을 위한 **Alamofire** 같은 강력한 '외부 부품'에 해당합니다.

🤿 **딥하게 들어가기 (Deep: 그럼 어떻게 설치하고 쓰나요?)**

파이썬은 언어 자체를 아주 가볍게 유지하고, 필요한 기능은 패키지 관리자를 통해 골라 쓰는 생태계를 가지고 있습니다.

✅ **Step 1: 파이썬의 SPM, `pip`** 파이썬 생태계에서는 외부 라이브러리를 설치할 때 터미널에서 `pip`라는 명령어 도구를 사용합니다. 패키지 주소를 복잡하게 찾을 필요 없이 명령어 한 줄이면 끝납니다.

✅ **Step 2: 대장급 ORM 라이브러리 설치하기 (터미널)** 백엔드 세계에서 가장 대표적이고 강력한 ORM 라이브러리 중 하나인 **SQLAlchemy**(알케미)를 설치해 보겠습니다. 터미널을 열고 아래 마법의 주문을 외웁니다.

```Bash
# 터미널에 입력 (Mac/Windows 동일)
pip install sqlalchemy
```

✅ **Step 3: 내 코드에 부품 조립하기 (Python)** 설치가 끝났다면, 파이썬 파일 상단에 해당 라이브러리를 불러오겠다는 선언을 해줍니다.

```Python
# 파이썬 코드 상단
from sqlalchemy import Column, Integer, String
```

이제 비로소 지난 수다에서 보셨던 `Column`, `Integer` 같은 클래스 도구들을 마음껏 가져다 쓸 수 있게 되는 것입니다!

🚨 **핵심 포인트: 왜 굳이 라이브러리를 고르게 만들었을까?**

"그냥 파이썬 안에 기본으로 넣어주면 안 되나?" 싶을 수 있습니다. 하지만 백엔드의 세계는 너무나 방대합니다. 어떤 프로젝트는 웹 풀스택 프레임워크인 **Django**를 쓰면서 그 안에 내장된 전용 ORM을 통째로 쓰고, 어떤 프로젝트는 가볍고 빠른 백엔드를 위해 **FastAPI**와 방금 설치한 **SQLAlchemy** 조합을 선택합니다.

즉, 개발자에게 **'내 프로젝트의 성격에 가장 완벽하게 맞는 통역사(ORM)를 직접 고를 자유'**를 주기 위해 라이브러리 형태로 분리되어 있는 것입니다.

💡 **오늘의 요약 처방**

- 파이썬에서 ORM을 쓰려면 터미널에서 `pip install` 명령어로 **'외부 라이브러리(SQLAlchemy 등)를 설치'**해야 합니다.
    
- 파이썬 파일 상단에 `from ~ import ~` 문법을 통해 설치한 부품을 끌어와서 ORM 클래스 모델을 완성합니다.
