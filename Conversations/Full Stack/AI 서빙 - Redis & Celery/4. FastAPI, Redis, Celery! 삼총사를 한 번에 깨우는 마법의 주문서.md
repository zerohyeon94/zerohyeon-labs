드디어 가장 웅장하고 멋진 백엔드 아키텍처를 하나의 파일로 엮어낼 시간입니다!

개별적으로 `docker run` 명령어를 세 번이나 치고, 네트워크를 일일이 연결해야 하는 고통스러운 과정을 **단 한 번의 엔터 키**로 끝내버리는 마법, `docker-compose.yml`의 실전 세팅을 시원하게 해부해 드리겠습니다.

---

## 🗣️ 가볍게 시작하기 (Simple): "건물 전체 통째로 짓기"

- **개별 Docker:** 카운터 직원(FastAPI) 방 따로 만들고, 메모장(Redis) 방 따로 만들고, 주방장(Celery) 방 따로 만들어서 서로 문을 열어주는 복잡한 공사입니다.
    
- **Docker Compose:** "여기는 카운터, 여기는 주방, 가운데는 메모장 둬!"라고 적어둔 **'건축 설계도'**입니다. 이 도면만 실행하면 세 개의 방이 동시에 지어지고 서로 소통할 수 있는 내부 복도까지 완벽하게 깔립니다.

---

## 🤿 딥하게 들어가기 (Deep: 실전 docker-compose.yml 작성)

프로젝트 폴더 최상단(Dockerfile이 있는 곳)에 `docker-compose.yml` 파일을 만들고, 세 명의 일꾼을 아래와 같이 배치합니다.

```YAML
version: '3.8'

services:
  # 1. 게시판이자 메모장 (Redis)
  redis:
    image: redis:alpine
    ports:
      - "6379:6379"

  # 2. 카운터 직원 (FastAPI 서버)
  web:
    build: .
    ports:
      - "8000:8000"
    command: uvicorn main:app --host 0.0.0.0 --port 8000
    depends_on:
      - redis

  # 3. 뒤에서 일하는 주방장 (Celery Worker)
  worker:
    build: .
    command: celery -A celery_worker.celery_app worker --loglevel=info
    depends_on:
      - redis
    # 주방장은 외부에서 직접 접속할 필요가 없으므로 ports(문)를 뚫지 않습니다!
```

### 코드 완벽 해부

- **`build: .`**: 현재 폴더에 있는 `Dockerfile` 하나를 재사용해서, `web` 방과 `worker` 방을 똑같은 파이썬 환경으로 구워냅니다.
    
- **`command`**: 컨테이너가 켜질 때 실행할 명령어입니다. `web`은 서버를 켜고, `worker`는 Celery 일꾼을 깨웁니다.
    
- **`depends_on`**: 실행 순서를 결정합니다. 당연히 메모장(`redis`)이 먼저 깔려 있어야 직원들이 일을 할 수 있으므로, 항상 Redis부터 켜지도록 강제합니다.
    
- **`ports` 생략**: `worker`(Celery)는 철저히 백그라운드에서만 일합니다. 모바일 앱에서 직접 Celery로 들어갈 일이 없기 때문에 외부로 통하는 문(`ports`)을 아예 만들지 않아 보안을 높입니다.

## 🚨 핵심 포인트: 더 이상 'localhost'가 아닙니다!

Docker Compose로 묶인 컨테이너들은 자기들만의 **거대한 내부 와이파이(네트워크)**를 공유하게 됩니다. 이게 엄청난 마법을 부립니다.

기존에 로컬에서 개발하실 때는 FastAPI나 Celery 파이썬 코드 안에 레디스 주소를 이렇게 적으셨을 겁니다. 
```yml
redis_url = "redis://localhost:6379/0"
```

하지만 Docker Compose 안에서는 각 방의 이름(`services` 아래에 적은 이름)이 곧 주소가 됩니다! 따라서 파이썬 코드를 아래와 같이 수정해야 합니다. 
```yml
redis_url = "redis://redis:6379/0"
```

이렇게 하면 FastAPI(web)가 `redis`라는 이름만 부르면 알아서 Redis 방으로 정확히 찾아가게 됩니다.

## 💡 오늘의 요약 처방

- **`docker-compose.yml`**: 여러 개의 도커 컨테이너를 한 번에 띄우는 오케스트라 지휘자 문서입니다.
    
- **3개의 서비스**: `redis` (메모장), `web` (서버), `worker` (AI 일꾼)를 나누어 역할을 명확히 분리합니다.
    
- **내부 통신**: Compose로 묶인 방들은 IP 주소가 아닌 **'서비스 이름(redis)'**만으로 서로 완벽하게 통신합니다.
