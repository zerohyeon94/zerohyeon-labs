## 🗣️ 가볍게 시작하기 (Simple): "한국어, 영어, 스페인어로 똑같은 햄버거 주문받기"

- **한국어 알바생 (Python):** "치즈버거 하나 추가요!"라고 들으면 주방에 주문을 넣습니다.
    
- **영어 알바생 (JavaScript):** "One cheeseburger, please!"라고 들으면 주방에 주문을 넣습니다.
    
- **핵심:** 알바생이 어떤 언어를 쓰든, 결국 손님(모바일 앱)은 **'치즈버거(JSON 데이터)'**를 완벽하게 똑같은 모양으로 받아 가게 됩니다.

## 🤿 딥하게 들어가기 (Deep: 언어별 코드 생김새 비교)

모든 예시는 유저의 집중 시간 데이터를 요청하는 **`GET /users/1/focus-time`** 이라는 동일한 API 주소를 만들었을 때의 모습입니다.

### 1. Python 백엔드 (FastAPI)

가장 직관적이고 코드가 짧습니다. 함수 위에 `@` 기호(데코레이터)를 붙여서 주소와 방식을 지정합니다.

```Python
from fastapi import FastAPI

app = FastAPI()

# GET 방식으로 주소를 열어둡니다.
@app.get("/users/{user_id}/focus-time")
def get_focus_time(user_id: int):
    # DB에서 데이터를 찾았다고 가정합니다.
    data = {"userId": user_id, "focusTime": 45, "status": "completed"}
    return data # 딕셔너리를 반환하면 알아서 JSON으로 변환해 줍니다!
```

### 2. JavaScript/TypeScript 백엔드 (Node.js + Express)

React 프론트엔드 생태계와 가장 찰떡궁합인 백엔드입니다. `app.get`이라는 함수 안에 주소와, 응답을 처리할 코드를 집어넣습니다.

```JavaScript
const express = require('express');
const app = express();

// GET 방식으로 주소를 열어둡니다.
app.get('/users/:user_id/focus-time', (req, res) => {
    const userId = req.params.user_id;
    
    // DB에서 데이터를 찾았다고 가정합니다.
    const data = { userId: parseInt(userId), focusTime: 45, status: "completed" };
    res.json(data); // 데이터를 JSON 형태로 손님에게 던져줍니다!
});
```

### 3. Swift 백엔드 (Vapor)

놀랍게도 모바일에서 쓰는 Swift로도 백엔드 서버를 만들 수 있습니다! 문법이 완전히 익숙하실 겁니다.

```Swift
import Vapor

func routes(_ app: Application) throws {
    
    // GET 방식으로 주소를 열어둡니다.
    app.get("users", ":user_id", "focus-time") { req -> [String: Int] in
        guard let userId = req.parameters.get("user_id", as: Int.self) else {
            throw Abort(.badRequest)
        }
        
        // DB에서 데이터를 찾았다고 가정합니다.
        let data = ["userId": userId, "focusTime": 45]
        return data // 알아서 JSON으로 변환되어 날아갑니다!
    }
}
```

### 4. Java 백엔드 (Spring Boot) 

전 세계 기업용 서버 시장을 꽉 잡고 있는 전통의 강자입니다. 코드가 다소 길고 규칙이 엄격하지만, 그만큼 거대한 시스템을 튼튼하고 안정적으로 굴리는 데 최적화되어 있습니다.

```Java
@RestController
public class UserController {
    
    @GetMapping("/users/{user_id}/focus-time")
    public Map<String, Object> getFocusTime(@PathVariable("user_id") int userId) {
        // DB에서 데이터를 찾았다고 가정합니다.
        Map<String, Object> data = new HashMap<>();
        data.put("userId", userId);
        data.put("focusTime", 45);
        data.put("status", "completed");
        
        return data; // Spring Boot가 알아서 JSON으로 변환해 줍니다!
    }
}
```

### 5. Kotlin 백엔드 (Spring Boot) 

자바의 복잡한 문법을 현대적이고 세련되게 다듬은 언어입니다. 스위프트를 다루시는 분들이 코틀린 백엔드 코드를 보면 변수 선언이나 함수 구조가 너무 비슷해서 아주 친숙하게 느끼실 겁니다. 최근 국내 대기업들이 백엔드 언어를 자바에서 코틀린으로 활발하게 전환하고 있는 추세입니다.

```Kotlin
@RestController
class UserController {
    
    @GetMapping("/users/{user_id}/focus-time")
    fun getFocusTime(@PathVariable("user_id") userId: Int): Map<String, Any> {
        // DB에서 데이터를 찾았다고 가정합니다.
        val data = mapOf(
            "userId" to userId,
            "focusTime" to 45,
            "status" to "completed"
        )
        
        return data // 마찬가지로 완벽한 JSON으로 변환되어 응답합니다!
    }
}
```

## 🚨 핵심 포인트: 결국 '라우팅(Routing)'과 'JSON 반환'의 반복입니다!

세 가지 언어의 코드를 쭉 보시니 공통점이 보이시나요? RESTful API를 만드는 백엔드 개발자의 진짜 업무는 언어가 무엇이든 결국 아래의 두 가지 작업으로 귀결됩니다.

1. **라우팅(길 뚫어주기):** 프론트엔드가 접속할 주소(URL)와 행동(GET/POST)을 정의한다.
    
2. **응답(JSON 포장):** DB에서 꺼낸 데이터를 언어의 기본 자료구조(딕셔너리 등)에 담아서 반환하면, 프레임워크가 알아서 세계 공용어인 JSON으로 번역해 프론트엔드로 쏴준다.

이 구조만 머릿속에 잡혀있다면, 나중에 파이썬이 아닌 다른 언어로 백엔드를 짜게 되더라도 며칠 만에 금방 적응하실 수 있습니다.

## 💡 오늘의 요약 처방

- RESTful API 코드는 어떤 언어든 **"특정 URL로 요청이 오면 함수를 실행해라"**라는 구조로 짜여 있습니다.
    
- **FastAPI(파이썬), Express(자바스크립트), Vapor(스위프트)** 모두 각자의 문법으로 주소를 열고 데이터를 JSON 형태로 뱉어냅니다.
    
- 언어의 문법보다 API 주소를 어떻게 깔끔하게 설계하느냐가 풀스택 개발자의 진짜 실력입니다.
