드디어 API 개발을 마치셨군요! 정성 들여 만들고 계신 스터디 타이머 앱이나 가계부 앱의 백엔드를 드디어 전 세계 어디서든 아이폰으로 접속할 수 있게 세상에 내어놓는, 풀스택 개발에서 가장 짜릿한 순간에 도달하셨습니다.

로컬에서 잘 돌아가는 코드를 클라우드로 띄우는 전체 흐름을 시원하게 뚫어드리겠습니다!

---

## 🗣️ 가볍게 시작하기 (Simple): "내 방 컴퓨터 vs 24시간 강남 빌딩 입점"

- **로컬 환경 (내 방 컴퓨터):** 나 혼자만 접속할 수 있고, 내가 노트북을 덮으면 서버도 같이 죽어버리는 임시 작업장입니다.
    
- **AWS 배포 (빌딩 입점):** 아마존이 관리하는 24시간 절대 꺼지지 않는 클라우드 컴퓨터(건물)를 임대해서, 내 도커 컨테이너(매장)를 입점시키는 과정입니다.

---

## 🤿 딥하게 들어가기 (Deep: 로컬에서 AWS까지의 4단계 여정)

앞서 배운 Docker의 '설계도(이미지)' 개념이 여기서 완벽하게 빛을 발합니다. 전체 과정은 크게 4단계로 나뉩니다.

### Step 1: 로컬 테스트 & 구글 로그인의 치명적 함정 대비!

로컬에서 기능이 완벽한지 테스트합니다. 이때 **구글 로그인(OAuth)** 기능이 있다면 배포 시 가장 많이 겪는 에러를 미리 방어해야 합니다.

- **현재 상태:** 구글 클라우드 콘솔에 '승인된 리디렉션 URI'를 `http://localhost:8000/callback`으로 설정하셨을 겁니다.
    
- **배포 준비:** AWS에 서버를 올리면 접속 주소가 AWS의 IP나 도메인(예: `http://3.45.xx.xx/callback`)으로 바뀝니다. 배포를 진행하기 전에, 구글 콘솔에 이 새로운 AWS 주소를 반드시 추가해 두어야 구글 로그인이 먹통이 되지 않습니다!

### Step 2: 로컬에서 내 프로젝트를 Docker로 포장하기 (Build & Push)

로컬에서 잘 도는 걸 확인했다면, 내 코드를 도커 이미지(붕어빵 틀)로 굽고, 이것을 **Docker Hub** (앱 스토어 같은 도커 전용 온라인 창고)에 업로드합니다.

```Bash
# 1. 내 코드를 'my-api'라는 이름의 이미지로 굽기
docker build -t my-api .

# 2. 다 구워진 이미지를 온라인 창고에 업로드하기
docker push my-api
```

### Step 3: AWS EC2 (클라우드 컴퓨터) 빌리기

AWS 홈페이지에 접속해서 **EC2(Elastic Compute Cloud)** 라는 가상의 리눅스 컴퓨터를 한 대 대여합니다. (초보자를 위한 프리티어 무료 버전을 주로 사용합니다.) 이때, 외부에서 이 컴퓨터의 8000번 포트(FastAPI)나 80번 포트(웹)로 들어올 수 있도록 '보안 그룹(방화벽)'을 열어주는 세팅을 합니다.

### Step 4: AWS 컴퓨터에서 Docker 실행하기 (Pull & Run)

이제 터미널을 통해 내 Mac에서 저 멀리 있는 AWS 컴퓨터로 원격 접속(SSH)을 합니다. 놀랍게도 새로 빌린 AWS 컴퓨터에는 파이썬도, 라이브러리도 깔 필요가 없습니다! 오직 **Docker** 프로그램 딱 하나만 설치하면 됩니다.

```Bash
# AWS 컴퓨터의 터미널 안에서 실행

# 1. 온라인 창고에 올려둔 내 이미지를 AWS로 다운받기
docker pull my-api

# 2. 이미지 실행! (드디어 서버가 세상에 켜졌습니다!)
docker run -d -p 80:8000 my-api
```

## 🚨 핵심 포인트: 비밀번호는 절대 같이 구우면 안 됩니다! (.env 관리)

배포할 때 가장 주의해야 할 보안 상식입니다! 구글 로그인에 사용되는 `Client Secret Key`나 데이터베이스의 `Password` 같은 민감한 정보는 절대 파이썬 코드 안에 적어둔 채로 도커 이미지로 구워버리면 안 됩니다. (해커들이 Docker Hub를 뒤져서 비밀번호를 훔쳐갑니다.)

이런 정보는 로컬과 AWS 양쪽 모두 **`.env`** 라는 별도의 숨김 파일에 적어두고, 도커 컨테이너를 실행(`docker run`)할 때 이 환경변수 파일을 주입해 주는 방식으로 보안을 철저하게 지켜야 합니다.

## 💡 오늘의 요약 처방

- **흐름:** 로컬 테스트 ➡️ 도커 이미지 온라인 업로드 ➡️ AWS 컴퓨터 대여 ➡️ AWS에서 이미지 다운로드 및 실행
    
- **구글 로그인:** 배포된 AWS 서버의 새로운 IP 주소를 구글 콘솔에 반드시 추가해야 에러가 나지 않습니다.
    
- **환경의 통일:** 내 Mac과 AWS 컴퓨터의 환경이 완전히 달라도, Docker라는 규격화된 박스를 쓰기 때문에 에러 없이 100% 동일하게 실행됩니다!
