## 🗣️ 가볍게 시작하기 (Simple): "개별 전원 스위치 vs 스마트홈 일괄 제어"

- **개별 실행 (docker run):** 방에 들어갈 때마다 전등 켜고, 보일러 켜고, 가습기를 따로따로 켜는 방식입니다. 컨테이너가 하나일 땐 괜찮지만, 늘어날수록 매번 명령어를 치기가 매우 귀찮아집니다.
    
- **일괄 제어 (docker-compose):** "외출 모드" 버튼 하나만 누르면 모든 전자기기가 알아서 착착 세팅되는 스마트홈 시스템입니다. 텍스트 문서 하나에 규칙을 다 적어두고 명령어 한 번만 치면 전체 인프라가 동시에 켜집니다.

## 🤿 딥하게 들어가기 (Deep: 오케스트라 지휘는 어떻게 할까?)

Gagae-C 같은 가계부 앱이나 ZipJoong 타이머 앱의 실전 백엔드를 돌리려면, **FastAPI(서버)** 컨테이너와 **PostgreSQL(데이터베이스)** 컨테이너가 반드시 짝을 이루어 동시에 켜져야 합니다. 이 두 개의 방을 한 번에 띄우기 위해 `docker-compose.yml`이라는 지휘자 문서를 작성합니다.

### Step 1: 지휘자 문서(docker-compose.yml) 작성하기

프로젝트 폴더 최상단에 파일을 만들고, 어떤 컨테이너들을 띄울지 명단을 작성합니다.

```YAML
version: '3.8'

services:
  # 첫 번째 방: 데이터베이스 (PostgreSQL)
  db:
    image: postgres:15
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: password
      POSTGRES_DB: mydatabase
    ports:
      - "5432:5432"

  # 두 번째 방: 백엔드 서버 (FastAPI)
  web:
    build: .  # 현재 폴더에 있는 Dockerfile을 읽어서 이미지를 구워라!
    ports:
      - "8000:8000"
    depends_on:
      - db  # db 방이 먼저 켜진 다음에 web 방을 켜라!
```

### Step 2: 마법의 지휘봉 휘두르기 (터미널 명령어)

문서를 다 작성했다면, 복잡하게 긴 명령어를 칠 필요 없이 터미널에 딱 한 줄만 입력합니다.

```Bash
docker-compose up -d
```

이 주문을 외우는 순간, Docker Compose가 문서를 읽고 **"PostgreSQL 방부터 만들어서 켜고, 그다음 Dockerfile을 읽어서 FastAPI 서버 방을 구운 다음에 켜!"**라는 복잡한 과정을 알아서 순서대로 실행해 줍니다. 끌 때는 `docker-compose down` 한 줄이면 모든 방이 깔끔하게 철거됩니다.

## 🚨 핵심 포인트: 서로 다른 컨테이너가 어떻게 통신할까?

각각 독립된 컨테이너 공간(방)에 갇혀 있는 FastAPI와 PostgreSQL은 어떻게 서로 데이터를 주고받을 수 있을까요?

Docker Compose의 가장 강력한 마법 중 하나는, 한 문서 안에 묶여서 실행된 컨테이너들끼리 **'자동으로 같은 가상 공유기(네트워크)'**에 묶인다는 것입니다. 그래서 FastAPI 코드에서 DB 주소를 적을 때 복잡한 IP 주소를 찾을 필요 없이, 그냥 위 문서에 적어둔 이름인 `db`라고만 적어주면 알아서 데이터베이스 방으로 통신이 연결됩니다!

---

## 💡 오늘의 요약 처방

- **`docker-compose`**는 여러 개의 도커 컨테이너를 하나의 그룹으로 묶어서 동시에 관리하고 실행하는 도구입니다.
    
- `docker-compose.yml` 파일에 세팅을 적어두면 매번 복잡한 명령어를 칠 필요가 없습니다.
    
- 묶인 컨테이너들끼리는 자동으로 내부 네트워크가 연결되어 이름만으로 서로 통신할 수 있습니다.
