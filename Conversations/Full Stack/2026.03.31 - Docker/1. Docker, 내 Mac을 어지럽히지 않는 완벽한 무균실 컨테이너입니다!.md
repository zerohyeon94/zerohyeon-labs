# 🗣️ 가볍게 시작하기 (Simple): 원룸에 직접 가구 조립하기 vs 풀옵션 컨테이너 통째로 배달받기

### 로컬 직접 설치:
- 내 소중한 Mac(원룸) 바닥에 PostgreSQL 부품을 늘어놓고 직접 조립하는 겁니다. 
- 나중에 지우고 싶어도 찌꺼기가 남고, 다른 버전이 필요해지면 방 전체가 엉망진창이 됩니다. 
### Docker 설치: 
- 이미 PostgreSQL이 완벽하게 세팅된 '풀옵션 컨테이너 박스'를 내 방 한구석에 툭 내려놓는 겁니다. 
- 필요 없으면 박스째로 내다 버리면 그만이라 내 시스템은 항상 깨끗한 무균실 상태를 유지합니다.

# 🤿 딥하게 들어가기 (Deep: 백엔드 개발에 Docker가 필수인 이유)

iOS 개발을 하실 때는 Xcode 하나만 설치하면 끝났지만, 풀스택 개발은 다릅니다. Python 백엔드, PostgreSQL 데이터베이스, 나중에는 AI 모델을 돌릴 환경까지 각각 띄워야 하죠. 이 모든 걸 시스템에 직접 깔지 않게 해주는 구원자가 바로 Docker입니다.

## ✅ Step 1: 
Image(이미지)와 Container(컨테이너)의 이해 Docker를 이해하는 핵심은 두 단어입니다. 
### 이미지: 
- 붕어빵 틀이자 완벽한 설계도입니다. 
- "Ubuntu 리눅스 위에 PostgreSQL 15 버전을 깔아둔 상태" 자체를 얼려둔 파일입니다. 
### 컨테이너: 
- 그 이미지를 바탕으로 내 Mac에서 실제로 실행된 상태(구워진 붕어빵)입니다. 
- 컨테이너 여러 개를 띄우면 내 컴퓨터 안에 독립된 꼬마 컴퓨터 여러 대가 돌아가는 것과 같습니다.

## ✅ Step 2: 
장비를 넘나드는 마법의 치트키 맥북 프로와 아이맥을 오가며 개발을 하다 보면 가장 끔찍한 에러가 "어제 랩탑에서는 잘 됐는데 데스크탑에서는 왜 안 되지?" 입니다. 
운영체제나 자잘한 환경 설정이 다르기 때문이죠. 
Docker를 사용하면 이 환경 설정에 구애받지 않습니다. 
Docker 설정 파일 하나만 Git으로 공유하면 기기가 무엇이든, 심지어 나중에 배포할 실제 서버 컴퓨터에서든 100% 똑같은 환경의 컨테이너가 실행됩니다. 세팅 지옥에서 영원히 해방되는 것입니다.

## ✅ Step 3:
Docker Compose로 오케스트라 지휘하기 집중 타이머나 가계부 앱의 실전용 백엔드를 돌리려면, FastAPI 서버 컨테이너와 PostgreSQL 컨테이너가 동시에 켜져서 서로 데이터를 주고받아야 합니다. 
이때 docker-compose.yml 이라는 지휘자 문서를 작성하고 터미널에 docker-compose up 이라는 마법의 주문을 치면, 여러 개의 컨테이너가 한 번에 짠! 하고 실행되어 완벽한 인프라가 구성됩니다.

# 🚨 핵심 포인트: 내 Mac 시스템에는 오직 Docker 프로그램 하나만 깔립니다!

터미널을 열고 복잡하게 DB를 설치할 필요가 없습니다. 
Docker Desktop이라는 프로그램 하나만 설치해두면, 그 위에서 수십 가지의 서버와 DB를 컨테이너 형태로 자유롭게 띄웠다 지웠다 할 수 있습니다.

# 💡 오늘의 요약 처방

Docker는 복잡한 백엔드 환경(DB, 서버 등)을 내 시스템과 격리된 '컨테이너' 안에서 실행해 주는 프로그램입니다. 어떤 컴퓨터에서든 동일한 개발 환경을 1초 만에 구성할 수 있게 해주는 풀스택 개발의 필수 무기입니다. 설치 찌꺼기가 남지 않아 내 컴퓨터를 항상 쾌적하게 유지할 수 있습니다.
