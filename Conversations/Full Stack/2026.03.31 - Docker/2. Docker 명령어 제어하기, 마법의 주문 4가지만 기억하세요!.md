# 🗣️ 가볍게 시작하기 (Simple): 앱 스토어 다운로드 vs 터미널 주문 외우기

### 앱 스토어 방식: 
Docker Desktop이라는 예쁜 프로그램을 켜서 마우스로 재생 버튼, 정지 버튼을 누르는 아주 직관적인 방법입니다. 
### 터미널 방식: 
개발자들의 진짜 무기입니다. 키보드로 마법의 주문(명령어)을 쳐서 컨테이너를 자유자재로 다룹니다. 
처음엔 어색하지만 손에 익으면 마우스보다 훨씬 빠릅니다.

# 🤿 딥하게 들어가기 (Deep: 백엔드 개발자가 매일 치는 4대 마법 주문)

가계부 앱이나 집중 타이머 앱의 백엔드를 띄우기 위해 매일 아침 터미널에서 치게 될 핵심 명령어들입니다.

## Step 1: 붕어빵 틀 가져오기 (이미지 다운로드) 
- 명령어: docker pull postgres 
- 설명: Docker 창고(Hub)에서 PostgreSQL 최신 버전의 완벽한 세팅 도면(이미지)을 내 Mac으로 다운로드해 달라는 뜻입니다. 앱 스토어에서 앱을 다운받는 것과 같습니다.

## Step 2: 붕어빵 굽고 실행하기 (컨테이너 띄우기) 
- 명령어: docker run --name my-db -p 5432:5432 -e POSTGRES_PASSWORD=1234 -d postgres 
- 설명: 가장 중요하고 긴 주문입니다! 
	- `name` 옵션: 이 꼬마 컴퓨터(컨테이너)의 이름을 my-db로 지어줍니다. 
	- `p` 옵션 (포트): 내 Mac의 5432번 문과 컨테이너의 5432번 문을 연결해 줍니다. 그래야 파이썬 백엔드가 DB에 들어갈 수 있습니다. 
	- `e` 옵션 (환경변수): DB의 비밀번호를 1234로 설정합니다. 
	- `d` 옵션 (백그라운드): 터미널을 꺼도 뒤에서 계속 실행되게 만듭니다.

## Step 3: 돌아가는 꼬마 컴퓨터들 확인하기 (상태 보기) 
- 명령어: docker ps 
- 설명: 현재 내 Mac 안에서 어떤 Docker 컨테이너들이 윙윙 돌아가고 있는지 리스트를 보여줍니다. FastAPI 서버와 DB가 잘 켜져 있는지 확인할 때 수시로 칩니다.

## Step 4: 컴퓨터 끄고 치우기 (정지 및 삭제) 
- 명령어: docker stop my-db (멈추기) 
- 명령어: docker rm my-db (삭제하기) 
- 설명: 오늘 개발을 마치고 꼬마 컴퓨터를 끌 때는 stop을 치고, 아예 환경을 초기화하고 싶을 때는 rm을 쳐서 깔끔하게 지워버립니다. 찌꺼기가 전혀 남지 않죠!

# 🚨 핵심 포인트: 이 명령어들을 하나로 합쳐주는 궁극의 마법!

컨테이너가 하나일 때는 이렇게 치면 되지만, 나중에 앱을 실전 배포할 때 DB, 백엔드 서버, AI 모델까지 3~4개를 동시에 띄워야 한다면 어떨까요? 
이때 사용하는 것이 지난 수다에서 잠깐 언급한 Docker Compose입니다. 
텍스트 파일 하나에 위 명령어들의 규칙을 다 적어두고, docker-compose up 이라는 단 한 줄의 주문만 치면 오케스트라 지휘자처럼 모든 컨테이너가 알아서 순서대로 착착 켜집니다!

# 💡 오늘의 요약 처방

이미지 다운로드는 pull, 컨테이너 실행은 run 명령어를 사용합니다. 
현재 켜져 있는 상태 확인은 ps, 끄고 지울 때는 stop과 rm을 씁니다. 
실무에서는 이 명령어들을 매번 치기 귀찮아서 docker-compose라는 자동화 지휘자를 사용합니다.
