### Closure 지옥 탈출기: `async/await`의 등장

비디오 스트림에서 눈 깜빡임(Blink Rate)과 머리 움직임(Head Stability) 데이터를 비동기로 가져와서, 최종 집중도 점수(Focus Score)를 계산해 화면에 업데이트하는 상황을 가정해 보겠습니다.

#### 😱 1. 과거의 방식: Closure (Completion Handler)

```swift
func updateFocusScore() {
    // 1. 눈 깜빡임 데이터 가져오기
    fetchBlinkRate { [weak self] blinkRate in
        guard let self = self else { return }
        
        // 2. 머리 흔들림 데이터 가져오기 (1번이 끝나야 실행 가능)
        self.fetchHeadStability { [weak self] stability in
            guard let self = self else { return }
            
            // 3. 두 데이터를 조합해 최종 점수 계산하기
            self.calculateFocusScore(blinkRate: blinkRate, stability: stability) { [weak self] score in
                guard let self = self else { return }
                
                // 4. UI 업데이트 (메인 스레드에서 실행해야 함)
                DispatchQueue.main.async {
                    self.scoreLabel.text = "현재 집중도: \(score)점"
                }
            }
        }
    }
}
```

- **문제점:**
    
    - 코드가 오른쪽으로 계속 파고들어 갑니다. (들여쓰기 지옥)
        
    - 매 단계마다 `[weak self]`와 `guard let self`를 도배해야 합니다. (메모리 누수 방어)
        
    - 중간에 에러가 났을 때 예외 처리(`catch`)를 하기가 굉장히 까다롭고, 개발자가 실수로 `completion()`을 호출하지 않으면 앱이 그대로 멈춰버립니다(Hang).

#### 😍 2. 현재의 방식: `async/await` (Swift 5.5+)

이제 마법 같은 일이 벌어집니다. 위와 **정확히 똑같이 동작하는 코드**를 `async/await`로 바꿔보겠습니다.

```swift
func updateFocusScore() async {
    do {
        // 1 & 2. 데이터 가져오기 (await를 만나면 작업이 끝날 때까지 여기서 잠깐 '대기' 합니다)
        let blinkRate = try await fetchBlinkRate()
        let stability = try await fetchHeadStability()
        
        // 3. 점수 계산
        let score = try await calculateFocusScore(blinkRate: blinkRate, stability: stability)
        
        // 4. UI 업데이트 (@MainActor 등을 사용하면 DispatchQueue도 생략 가능)
        self.scoreLabel.text = "현재 집중도: \(score)점"
        
    } catch {
        print("점수 계산 중 에러 발생: \(error)")
    }
}
```

- **해결된 점:**
    
    - **직관적인 흐름:** 코드가 그냥 위에서 아래로 똑바로 흐릅니다. (마치 동기 코드처럼 보이죠!)
        
    - **안전성:** 컴파일러가 알아서 작업 완료와 반환을 보장하므로, 개발자가 콜백을 빼먹을 일이 없습니다.
        
    - **`[weak self]` 해방:** 클로저를 중첩해서 열지 않기 때문에, 강한 순환 참조(Retain Cycle)의 늪에서 자연스럽게 빠져나올 수 있습니다. 에러 처리도 일반적인 `do-catch`문으로 아주 깔끔해졌죠.
