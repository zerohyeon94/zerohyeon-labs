# 3. SampleBuffer에서 Vision과 Core ML까지 연결하기

> Daily: [[Daily/2026-05-17]]
> 분야: iOS Mentoring
> 대주제: AVFoundation 실시간 카메라 파이프라인
> 멘토링 시작 문장: "iOS 멘토링을 시작하겠습니다. 질문: SampleBuffer에서 Vision/Core ML까지 이어지는 흐름을 CameraService 코드에서 어떻게 추적하면 좋을까요?"

---

## 오늘 이 주제를 보는 이유

지난 흐름에서는 `AVCaptureSession`, `AVCaptureVideoDataOutput`, `CMSampleBuffer`가 카메라 프레임을 앱 코드로 가져오는 구조를 잡았다.

오늘은 그 다음 단계다. 실시간 카메라 앱에서 진짜 중요한 지점은 `CMSampleBuffer`로 들어온 한 프레임이 `CVPixelBuffer`로 꺼내지고, 이 데이터가 `Vision` 또는 `Core ML` 요청으로 이어지는 순간이다.

## 학습 목표

- `CMSampleBuffer`와 `CVPixelBuffer`의 역할을 다시 구분한다.
- `VNImageRequestHandler` 또는 Core ML 입력으로 프레임이 넘어가는 지점을 찾는다.
- CameraService 분석 시 "프레임 수신"과 "모델 추론"을 분리해서 읽는다.
- 실시간 처리에서 매 프레임 추론이 왜 부담이 되는지 설명한다.

## 핵심 개념

### 1. SampleBuffer는 한 프레임의 컨테이너

`captureOutput(_:didOutput:from:)`에서 받는 `CMSampleBuffer`는 카메라 한 프레임의 포장된 데이터다.

```swift
func captureOutput(
    _ output: AVCaptureOutput,
    didOutput sampleBuffer: CMSampleBuffer,
    from connection: AVCaptureConnection
) {
    guard let pixelBuffer = CMSampleBufferGetImageBuffer(sampleBuffer) else {
        return
    }
}
```

이 코드에서 분석 포인트는 `CMSampleBufferGetImageBuffer`다. 이 줄이 "AVFoundation 프레임"이 "이미지 처리 입력"으로 바뀌는 경계다.

### 2. Vision은 PixelBuffer를 해석하는 분석 레이어

Vision을 사용한다면 보통 `CVPixelBuffer`를 `VNImageRequestHandler`에 넘긴다.

```swift
let handler = VNImageRequestHandler(
    cvPixelBuffer: pixelBuffer,
    orientation: .up,
    options: [:]
)

try? handler.perform([request])
```

여기서 `request`는 얼굴 인식, 포즈 인식, 객체 감지, Core ML 모델 실행 같은 분석 작업을 의미한다.

### 3. Core ML은 모델 추론 레이어

Core ML 모델을 직접 쓰거나 Vision을 통해 감싸서 쓸 수 있다.

```text
CMSampleBuffer
-> CVPixelBuffer
-> VNImageRequestHandler
-> VNCoreMLRequest
-> MLModel
-> 결과
```

CameraService를 볼 때는 이 흐름 중 어느 단계까지 Service가 책임지는지 확인해야 한다.

- Service가 PixelBuffer까지만 넘기는가?
- Vision/Core ML 요청까지 직접 수행하는가?
- 결과를 ViewModel 또는 UI 계층으로 전달하는가?

### 4. 프레임 수신과 추론은 분리해서 읽기

초당 30프레임이 들어오는데 매 프레임마다 모델 추론을 하면 쉽게 밀릴 수 있다.

분석할 때는 아래 질문을 남긴다.

- `captureOutput` 안에서 바로 추론하는가?
- 별도 inference queue가 있는가?
- 이미 추론 중일 때 다음 프레임을 건너뛰는가?
- UI 업데이트만 main queue로 보내는가?

## 실무 연결

CameraService 메모는 아래 6줄이면 충분하다.

```text
1. SampleBuffer 수신 위치:
2. PixelBuffer 추출 위치:
3. Vision/Core ML 요청 생성 위치:
4. 추론 queue:
5. 프레임 드랍 또는 중복 추론 방지 정책:
6. UI/ViewModel 전달 위치:
```

오늘은 이 6줄을 실제 프로젝트 코드 옆에 적어보는 것을 목표로 한다.
