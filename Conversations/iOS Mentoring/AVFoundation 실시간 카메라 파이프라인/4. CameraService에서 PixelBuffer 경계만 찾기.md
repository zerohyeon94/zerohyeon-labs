# 4. CameraService에서 PixelBuffer 경계만 찾기

---

## 오늘 이 주제를 보는 이유

2026-05-18은 수강생 과제 피드백/풀이 업무가 본격적으로 시작되는 첫 월요일이다. 그래서 iOS 학습은 새 범위를 크게 열기보다, 전날 만든 [[Conversations/iOS Mentoring/AVFoundation 실시간 카메라 파이프라인/3. SampleBuffer에서 Vision과 Core ML까지 연결하기]]의 범위를 아주 작게 줄인다.

오늘 목표는 Vision/Core ML 전체 흐름을 다 이해하는 것이 아니라, CameraService 코드에서 아래 두 줄을 채우는 것이다.

```text
1. SampleBuffer 수신 위치:
2. PixelBuffer 추출 위치:
```

## 학습 목표

- `captureOutput(_:didOutput:from:)` 또는 유사한 프레임 콜백 위치를 찾는다.
- `CMSampleBufferGetImageBuffer`처럼 `CMSampleBuffer`에서 `CVPixelBuffer`를 꺼내는 경계를 찾는다.
- 프레임 수신 책임과 이미지 분석 책임을 분리해서 읽는다.
- 오늘은 Vision/Core ML 요청 생성까지 무리해서 추적하지 않는다.

## 핵심 개념

### 1. SampleBuffer 수신 위치는 카메라 프레임이 앱으로 들어오는 입구

`CMSampleBuffer`는 AVFoundation이 넘겨주는 한 프레임의 컨테이너다. CameraService에서 이 값이 처음 등장하는 함수가 있다면, 그 함수가 실시간 카메라 파이프라인의 입구다.

찾을 키워드:

```swift
captureOutput
CMSampleBuffer
AVCaptureVideoDataOutputSampleBufferDelegate
```

### 2. PixelBuffer 추출 위치는 이미지 처리 입력으로 바뀌는 경계

`CVPixelBuffer`는 Vision/Core ML이 실제 이미지 입력으로 다루기 좋은 형태다. 아래 호출을 찾으면 AVFoundation 영역에서 이미지 분석 영역으로 넘어가는 경계를 찾은 것이다.

```swift
CMSampleBufferGetImageBuffer(sampleBuffer)
```

이 줄을 찾으면 바로 다음 코드를 무리해서 모두 따라가지 말고, "누가 이 pixelBuffer를 가져가는지"만 확인한다.

### 3. 오늘은 두 위치만 적어도 충분하다

업무가 있는 평일에는 완성된 이해보다 끊기지 않는 추적이 더 중요하다. 아래 4줄 중 앞의 2줄만 채워도 오늘 학습은 성공으로 본다.

```text
1. SampleBuffer 수신 위치:
2. PixelBuffer 추출 위치:
3. PixelBuffer를 넘겨받는 다음 객체/함수:
4. 아직 모호한 지점:
```

## 실무 연결

실시간 카메라 코드는 한 번에 읽으면 금방 복잡해진다. 특히 AVFoundation, Vision, Core ML, UI 업데이트가 한 파일에 섞이면 "프레임을 받는 코드"와 "분석하는 코드"가 헷갈린다.

오늘처럼 경계를 먼저 표시하면 다음 단계에서 아래 질문을 더 정확히 할 수 있다.

- 추론은 CameraService 안에서 하는가, 별도 객체로 넘기는가?
- 매 프레임 추론하는가, 일부 프레임만 처리하는가?
- 결과는 ViewModel로 바로 가는가, delegate/closure/Combine stream으로 가는가?
