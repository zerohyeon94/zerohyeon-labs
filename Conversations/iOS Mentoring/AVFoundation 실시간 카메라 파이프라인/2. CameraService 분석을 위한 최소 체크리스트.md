## 오늘 이 주제를 보는 이유

어제는 첫 출근과 늦은 퇴근으로 학습을 진행하지 못했다. 그래서 오늘은 새로운 개념을 넓게 추가하지 않고, 어제 만들었던 [[1. CaptureSession과 SampleBuffer 흐름]]을 실제 코드 읽기 순서로 바꾸는 데 집중한다.

`CameraService.swift` 같은 파일은 처음부터 모든 라인을 이해하려고 하면 부담이 크다. 대신 "세션 생성 → 입력 설정 → 출력 설정 → 프레임 처리 → 생명주기" 순서로 보면 AVFoundation의 구조와 성능 병목을 동시에 잡을 수 있다.

## 학습 목표

- `CameraService.swift`를 읽을 때 확인해야 할 지점을 체크리스트로 정리한다.
- `AVCaptureSession`, `AVCaptureVideoDataOutput`, `CMSampleBuffer`가 코드 안에서 어디에 등장하는지 찾을 수 있다.
- 실시간 카메라 처리에서 병목이 생길 수 있는 위치를 말로 설명할 수 있다.
- 오늘 퇴근이 늦어도 20분 안에 읽고 질문 1개를 남길 수 있게 한다.

## 핵심 개념

### 1. 세션 소유권 확인

먼저 `AVCaptureSession`이 어디에서 생성되고, 어떤 객체가 소유하는지 확인한다.

```swift
private let session = AVCaptureSession()
```

확인할 질문:
- 세션은 singleton처럼 오래 살아 있는가?
- 화면 진입/이탈과 함께 만들어지고 해제되는가?
- `startRunning()`과 `stopRunning()`이 어디에서 호출되는가?

### 2. 입력 장치 설정 확인

카메라 입력은 보통 `AVCaptureDevice`와 `AVCaptureDeviceInput`으로 구성된다.

```swift
let device = AVCaptureDevice.default(.builtInWideAngleCamera, for: .video, position: .back)
let input = try AVCaptureDeviceInput(device: device)
```

확인할 질문:
- 전면/후면 카메라 선택 기준이 있는가?
- 권한이 없을 때 흐름이 끊기지 않는가?
- 입력 추가 실패 시 에러 처리가 있는가?

### 3. 출력과 delegate queue 확인

실시간 분석 앱에서 가장 중요한 부분은 `AVCaptureVideoDataOutput`이다.

```swift
let output = AVCaptureVideoDataOutput()
output.setSampleBufferDelegate(self, queue: videoQueue)
```

확인할 질문:
- delegate queue가 메인 큐가 아닌가?
- 프레임이 밀릴 때 오래된 프레임을 버리는 정책이 있는가?
- 픽셀 포맷 설정이 Vision/Core ML 처리에 맞는가?

### 4. 프레임 처리 경로 확인

`captureOutput(_:didOutput:from:)` 안에서 프레임이 어디로 넘어가는지 본다.

```swift
func captureOutput(
    _ output: AVCaptureOutput,
    didOutput sampleBuffer: CMSampleBuffer,
    from connection: AVCaptureConnection
) {
    // Vision/Core ML 분석 경로 확인
}
```

확인할 질문:
- `CMSampleBuffer`에서 `CVPixelBuffer`를 꺼내는가?
- 매 프레임마다 모델 추론을 하는가?
- UI 업데이트는 메인 스레드로 안전하게 넘기는가?

## 실무 연결

코드 분석 메모는 아래 5줄만 남겨도 충분하다.

```text
1. 세션 생성 위치:
2. 입력 설정 위치:
3. 출력 delegate queue:
4. 프레임 처리 함수:
5. 병목 의심 지점:
```

오늘은 이 5줄을 채우는 것이 목표다. 구현이나 리팩토링은 하지 않는다.
