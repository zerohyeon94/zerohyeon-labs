## 오늘 이 주제를 보는 이유

첫 출근일에는 무리해서 새로운 구현을 많이 하기보다, 기존 포트폴리오 방향과 연결되는 핵심 개념 하나를 안정적으로 잡는 것이 좋다. 5/8 Daily Note에서 ZipJoong의 `CameraService.swift` 분석과 실시간 비전 처리 병목 탐색이 미완료로 남아 있었기 때문에, 오늘은 AVFoundation 카메라 파이프라인의 흐름을 먼저 정리한다.

이 주제는 iOS + On-device AI 방향에서 특히 중요하다. Core ML이나 Vision 모델을 앱에 붙이더라도 결국 입력 데이터는 카메라 프레임이고, 프레임이 어떻게 들어오고 어느 스레드에서 처리되며 언제 버려지는지를 모르면 성능 병목을 설명하기 어렵다.

## 학습 목표

- `AVCaptureSession`이 카메라 입력과 출력의 조정자 역할을 한다는 점을 이해한다.
- `AVCaptureDeviceInput`, `AVCaptureVideoDataOutput`, `CMSampleBuffer`의 역할을 구분한다.
- 실시간 프레임 처리에서 CPU, 메모리, 스레드 병목이 생기는 지점을 설명할 수 있다.
- ZipJoong 같은 자세 분석 앱에서 카메라 프레임이 Vision/Core ML 파이프라인으로 넘어가는 흐름을 말로 설명할 수 있다.

## 핵심 개념

### AVCaptureSession

`AVCaptureSession`은 카메라 입력과 출력 사이의 실행 파이프라인이다. 앱은 세션에 입력 장치와 출력 객체를 붙이고, 세션을 시작하면 프레임이 계속 흘러간다.

```swift
let session = AVCaptureSession()
session.sessionPreset = .high

let input = try AVCaptureDeviceInput(device: camera)
session.addInput(input)

let output = AVCaptureVideoDataOutput()
session.addOutput(output)
```

### AVCaptureVideoDataOutput

`AVCaptureVideoDataOutput`은 카메라 프레임을 앱 코드로 직접 받을 때 사용한다. 일반 촬영 앱은 사진/영상 출력이 중요하지만, 실시간 AI 앱은 매 프레임을 분석해야 하므로 이 출력이 중요하다.

```swift
output.setSampleBufferDelegate(self, queue: videoQueue)
```

여기서 중요한 점은 delegate queue다. 메인 스레드에서 무거운 분석을 하면 UI가 끊긴다. 그래서 카메라 프레임 처리는 별도 queue에서 받고, UI 업데이트만 메인으로 되돌리는 구조가 일반적이다.

### CMSampleBuffer

`CMSampleBuffer`는 카메라에서 들어온 한 프레임의 컨테이너다. 이미지 픽셀 데이터뿐 아니라 시간 정보, 포맷 정보도 함께 가진다.

```swift
func captureOutput(
    _ output: AVCaptureOutput,
    didOutput sampleBuffer: CMSampleBuffer,
    from connection: AVCaptureConnection
) {
    // sampleBuffer -> CVPixelBuffer -> Vision/Core ML
}
```

## 실무 연결

ZipJoong의 `CameraService.swift`를 볼 때는 아래 순서로 읽으면 좋다.

1. 세션 생성 위치: `AVCaptureSession`을 어디에서 만들고 소유하는가
2. 입력 설정: 전면/후면 카메라 선택이 어디에서 일어나는가
3. 출력 설정: `AVCaptureVideoDataOutput` delegate queue가 어떻게 잡혀 있는가
4. 프레임 처리: `CMSampleBuffer`를 Vision/Core ML로 넘기는 경로가 어디인가
5. 생명주기: 화면 진입/이탈 시 세션 start/stop이 정확히 호출되는가

성능 병목은 보통 세 군데에서 생긴다.

- 프레임을 너무 자주 분석해서 모델 추론이 밀리는 경우
- `CMSampleBuffer` 또는 `CVPixelBuffer`를 불필요하게 복사하는 경우
- 분석 결과를 매 프레임 메인 스레드에 반영해서 UI 업데이트가 과도해지는 경우
