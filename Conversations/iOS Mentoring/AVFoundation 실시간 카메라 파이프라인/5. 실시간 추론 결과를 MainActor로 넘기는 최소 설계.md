# 5. 실시간 추론 결과를 MainActor로 넘기는 최소 설계

> Daily: [[Daily/2026-06-05]]
> 분야: iOS Mentoring
> 대주제: AVFoundation 실시간 카메라 파이프라인
> 멘토링 시작 문장: "iOS 멘토링을 시작하겠습니다. 질문: 실시간 카메라 분석 결과를 MainActor로 안전하게 넘기려면 어떤 경계를 먼저 세워야 하나요?"

## 오늘 이 주제를 보는 이유

지난 학습에서는 `SampleBuffer`, `PixelBuffer`, `Vision/Core ML`까지 이어지는 입력 쪽 경계를 잡았다.

오늘은 그 다음 단계인 `분석 결과를 언제, 어떤 형태로, 어떤 스레드에서 UI 상태로 반영할 것인가`를 본다. 이 지점이 정리되지 않으면 카메라 앱은 금방 메인 스레드 병목, 상태 꼬임, 프레임 드랍으로 이어진다.

## 학습 목표

- `output queue`, `analysis queue`, `MainActor`의 책임을 구분한다.
- 추론 결과를 UI에 반영할 때 어떤 데이터만 넘겨야 하는지 정리한다.
- 메인 스레드에서 하면 안 되는 일과 반드시 메인에서 해야 하는 일을 나눈다.
- 실시간 카메라 앱에서 결과 전달 구조를 5줄 메모로 남긴다.

## 핵심 개념

### 1. 프레임 수신 큐와 추론 큐는 같은 책임이 아니다

`AVCaptureVideoDataOutputSampleBufferDelegate`가 호출되는 큐는 "프레임이 들어오는 입구"다. 여기서 오래 걸리는 Vision/Core ML 계산까지 전부 처리하면 프레임 수신이 바로 밀릴 수 있다.

최소 설계는 아래처럼 잡는다.

```text
output queue:
sampleBuffer를 받는다.

analysis queue:
pixelBuffer 추출, Vision/Core ML 추론, 후처리 수행

MainActor:
@Published 상태, SwiftUI 바인딩, 사용자에게 보이는 값 반영
```

### 2. MainActor에는 결과만 넘긴다

메인 액터는 계산 장소가 아니라 상태 반영 장소다.

좋은 경계는 아래와 같다.

```text
sampleBuffer 전체를 MainActor로 보내지 않는다.
추론에 필요한 계산을 MainActor에서 하지 않는다.
최종 결과 요약값만 MainActor로 보낸다.
```

예를 들면 아래처럼 정리할 수 있다.

```text
Vision/Core ML 결과
-> 집중도 점수
-> 얼굴 감지 여부
-> 경고 표시 여부
-> 마지막 업데이트 시각
```

### 3. 전달 데이터는 작고 의미가 분명해야 한다

실시간 파이프라인에서 UI 계층으로 넘기는 값은 `분석 결과 요약 struct` 수준이 좋다.

```text
AnalysisResult
- isFaceDetected
- focusScore
- shouldShowWarning
- updatedAt
```

이렇게 해야 CameraService나 분석 서비스가 UI 모델 타입에 과하게 묶이지 않는다.

### 4. 드랍 정책이 없으면 실시간성이 무너진다

카메라 앱은 모든 프레임을 완벽히 처리하는 것보다 "지금 화면에 의미 있는 최신 상태"를 유지하는 편이 더 중요하다.

오늘 메모에 아래 질문을 남긴다.

- 이미 추론 중일 때 다음 프레임을 건너뛰는가?
- `alwaysDiscardsLateVideoFrames` 외에 앱 레벨 스로틀링이 있는가?
- 결과 업데이트도 매 프레임마다 하는가, 간격을 두는가?

### 5. 오늘은 설계 문장 5개만 남기면 충분하다

실제 프로젝트 코드를 다 뜯지 않아도 아래 5줄을 적을 수 있으면 오늘 학습 목표는 충분하다.

```text
1. output queue:
2. analysis queue:
3. MainActor에서 바꾸는 상태:
4. MainActor로 넘기는 최소 데이터:
5. 프레임/결과 드랍 정책:
```

## 실무 연결

이 주제는 iOS 면접이나 포트폴리오 설명에도 그대로 연결된다.

`카메라 프레임을 받았습니다`보다 더 좋은 설명은 아래에 가깝다.

```text
AVFoundation으로 프레임을 받고,
백그라운드에서 Vision/Core ML 추론을 수행한 뒤,
요약 결과만 MainActor에서 상태로 반영해
실시간 UI를 안정적으로 유지했다.
```

즉, 오늘 파일은 WWDC26 전에 새 기술을 넓히는 용도가 아니라 `AI를 이해하는 iOS 개발자`라는 설명력을 복구하는 용도다.
