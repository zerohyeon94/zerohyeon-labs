# 1. Amplitude 이벤트와 캘린더 재진입 회귀 확인

> Daily: [[Daily/2026-06-19]]
> 분야: iOS Mentoring
> 대주제: 하루업 배포 전 회귀 확인
> 오늘의 멘토링 렌즈: 디버깅·성능 / 프로젝트 적용
> 멘토링 시작 문장: "iOS 멘토링을 시작하겠습니다. 질문: 하루업에서 Amplitude 이벤트 수신과 미션 완료 후 캘린더 재진입을 각각 어떤 관찰 지점과 성공 기준으로 확인하면, 코드 수정 완료와 배포 준비 완료를 구분할 수 있을까요?"

## 오늘 이 주제를 보는 이유

2026-06-18에 Amplitude 적용과 캘린더 미션 목록 표시 문제 수정은 끝났지만, 실제 이벤트 수신과 화면 재진입까지는 확인하지 못했다. 오늘은 새 기능을 늘리지 않고 두 흐름을 각각 한 번 재현해 `코드 수정 완료`를 `배포 준비 완료`에 가깝게 만든다.

## 학습 목표

- Amplitude 이벤트가 발생·전송·수신되는 흐름의 관찰 지점을 구분한다.
- 미션 완료 후 캘린더 재진입에서 기대 상태와 실제 상태를 비교한다.
- 성공, 실패, 추가 확인 필요를 두 줄로 남길 수 있는 기준을 만든다.

## 핵심 개념

### 1. 분석 이벤트 확인

이벤트 코드를 호출했다는 사실과 Amplitude에서 이벤트를 수신했다는 사실은 다르다.

확인 순서:

1. 검증할 사용자 행동 1개를 고른다.
2. 기대 이벤트명과 핵심 속성을 미리 적는다.
3. 앱에서 행동을 한 번 수행한다.
4. Amplitude의 실시간 또는 디버그 화면에서 이벤트 수신 여부를 확인한다.
5. 이벤트명, 사용자 식별 값, 핵심 속성이 기대와 맞는지 기록한다.

### 2. 캘린더 재진입 확인

화면이 한 번 정상 표시되는 것과 상태 변경 후 다시 진입했을 때도 정상인 것은 다르다.

확인 순서:

1. 미완료 미션이 보이는 초기 상태를 확인한다.
2. 미션을 완료한다.
3. 캘린더 화면을 벗어났다가 다시 진입한다.
4. 완료한 미션이 기대 위치와 상태로 표시되는지 확인한다.
5. 앱 재실행까지 필요한지는 오늘 결과를 본 뒤 결정한다.

### 3. 오늘의 기록 형식

| 흐름 | 기대 결과 | 실제 결과 | 판단 |
| --- | --- | --- | --- |
| Amplitude 이벤트 1건 | 이벤트명과 핵심 속성이 수신됨 |  | 성공 / 실패 / 추가 확인 |
| 미션 완료 → 캘린더 재진입 | 완료 미션 목록이 다시 표시됨 |  | 성공 / 실패 / 추가 확인 |

## 실무 연결

- 이벤트 수신 확인은 배포 후 사용자 행동을 추적할 수 있는지 검증하는 작업이다.
- 재진입 회귀 확인은 화면 전환과 데이터 갱신 경계에서 버그가 다시 발생하지 않는지 확인하는 작업이다.
- 두 결과가 정상이면 다음 작업은 `다른 미션 추천 기능` 또는 `프롬프트 앱설치 추천 문구 제거` 중 하나만 고른다.
