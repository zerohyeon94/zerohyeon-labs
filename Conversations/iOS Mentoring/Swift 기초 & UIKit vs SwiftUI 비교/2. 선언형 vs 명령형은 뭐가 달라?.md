# **Q.** SwiftUI는 선언형, UIKit은 명령형이라던데?  
### **A.** **명령형(UIKit)** 은 “어떻게 그릴지”를 단계별로 지시. **선언형(SwiftUI)** 은 “무엇을 보여줄지” 상태에 따라 **결과를 선언**하면 프레임워크가 그리는 방법을 결정.

```swift
// UIKit(명령형)
let label = UILabel()
label.text = "Hello"
label.textColor = .label

// SwiftUI(선언형)
Text("Hello").foregroundStyle(.primary)
```

### 명령형 & 선언형

- **명령형(Imperative)**: “이 라벨을 만들고 → 글자 넣고 → 색을 이렇게 바꿔라”처럼 **단계별 지시**(상태를 직접 조작).
    
- **선언형(Declarative)**: “상태가 이러하면, **이런 모양의 UI가 되어야 한다**”를 **관계/규칙로 선언**(그리는 절차는 프레임워크가 결정).

### 명령형 예 (UIKit)

```swift
let label = UILabel()
label.text = "Hello"
label.textColor = .label
stackView.addArrangedSubview(label)

// 상태가 바뀌면, 내가 직접 찾아서 다시 지시
func updateTitle(_ text: String) {
    label.text = text
    label.textColor = .label
}
```

- “무엇을 보여주라는 규칙”이 아니라, **“어떻게 바꿀지” 절차**를 매번 수행.

### 선언형 예 (SwiftUI)

```swift
struct TitleView: View {
  @State var title = "Hello"

  var body: some View {
    Text(title)                 // "상태(title)"를 보여주는 규칙을 선언
      .foregroundStyle(.primary)
  }
}

// 상태만 바꾸면 규칙에 따라 UI가 자동 갱신됨
// title = "Welcome"
```

- “텍스트 색은 기본(시스템) 스타일을 따른다”는 **규칙**을 적고, **상태만 바꾸면** 프레임워크가 알아서 그려줌.

### 선언형은 “함수로 꺼내는 것”과 다릅니다

```swift
// UIKit에서 '검은색 라벨'을 만드는 헬퍼 함수
func makeBlackLabel(_ text: String) -> UILabel {
  let l = UILabel()
  l.text = text
  l.textColor = .black
  return l
}
```

- 보기엔 간단해졌지만, 여전히 **라벨 생성/속성 변경**이라는 **절차**를 수행 → **명령형 추상화**일 뿐, 선언형은 아님.

> 명령형인 UIKit은 '어떻게'를 단계별로 직접 지시하는 것이고, 선언형인 SwiftUI는 '무엇을'만을 규칙으로 적고 어떻게 해야할지는 프레임워크/엔진이 결정한다.
