## 오늘의 주제: 값 타입과 참조 타입의 메모리 동작 원리

오늘의 첫 번째 주제는 iOS 개발자로서 애플리케이션 성능 최적화의 가장 근간이 되는 메모리 할당 방식(Stack과 Heap)에 대한 내용입니다. Swift에서 우리가 매일 사용하는 구조체(Struct)와 클래스(Class)가 메모리 단에서 어떻게 다르게 동작하는지 CS 지식과 엮어 파헤쳐 보겠습니다.

### 1. 핵심 CS 지식 연결: 스택(Stack)과 힙(Heap)

프로그램이 실행될 때 운영체제(OS)는 RAM에 메모리 공간을 할당하며, 이 중 우리가 직접 데이터를 올리며 제어하게 되는 핵심 영역이 스택과 힙입니다.

- **스택(Stack):** LIFO(Last-In-First-Out) 형태의 자료구조 원리로 동작하는 메모리 영역입니다. 함수가 호출될 때 생성되는 지역 변수나 값 타입(Value Type) 데이터가 이곳에 차곡차곡 쌓입니다.
    
- **힙(Heap):** 동적으로 할당되는 넓은 메모리 영역입니다. 개발자가 필요에 의해 생성하는 참조 타입(Reference Type) 데이터가 저장됩니다. 컴파일 타임에 크기를 알 수 없는 데이터들이 이곳에 배치되며, 메모리 공간을 직접 탐색하여 할당하고 사용 후 해제하는 별도의 관리가 필요합니다.

### 2. 복잡도 분석 (Big-O Notation)

두 메모리 영역은 동작 방식의 차이로 인해 시간 복잡도에서 극명한 차이를 보입니다.

- **스택의 시간 복잡도 ($O(1)$):** 스택은 스택 포인터(Stack Pointer)라는 레지스터를 단순히 위아래로 이동시키는 연산만 수행합니다. 따라서 메모리 할당과 해제의 시간 복잡도는 $O(1)$이며, 속도가 매우 빠릅니다.
    
- **힙의 시간 복잡도:** 힙은 스택처럼 순서대로 쌓이는 것이 아니라, 비어있는 적절한 크기의 메모리 블록을 탐색해야 합니다. 탐색 알고리즘에 따라 가변적이며, 멀티 스레드 환경에서는 여러 스레드가 동시에 힙에 접근하는 것을 막기 위한 동기화(Locking) 처리 오버헤드가 발생합니다. 더불어 Swift에서는 ARC(Automatic Reference Counting)가 참조 횟수를 계산하고 메모리를 관리하는 비용까지 추가됩니다.

### 3. 실무자 입장에서 반드시 알아야 하는 이유 (성능 최적화)

실무에서 카메라를 통해 사용자의 상태 변화를 실시간으로 감지하고 분석하거나, 심혈관 건강 지표처럼 대량의 데이터를 연속적으로 계산하여 UI를 갱신해야 하는 상황을 가정해 보겠습니다.

이러한 고빈도 데이터를 모두 클래스로 모델링하면 초당 수십에서 수백 번의 힙 할당과 해제가 발생합니다. 백그라운드 스레드와 메인 스레드가 힙 메모리에 동시 접근할 때 발생하는 동기화 오버헤드로 인해, 결국 앱에 버벅임이 생기거나 프레임 드롭(Frame Drop)이 발생할 위험이 커집니다.

따라서 실시간 추론 결과나 상태값처럼 수명이 짧고 잦은 변경이 일어나는 데이터 모델은 가급적 구조체로 설계하여 스택 메모리의 빠른 할당/해제 이점을 챙기는 것이 핵심적인 성능 최적화 기법입니다.

### 4. Swift 코드 구현 예시

```Swift
import Foundation

// Stack에 할당되는 값 타입 (매우 빠른 생성 및 해제)
struct StateData {
    let confidenceScore: Double
    let isActive: Bool
    let timestamp: TimeInterval
}

// Heap에 할당되는 참조 타입 (상대적으로 무거운 생성 및 관리)
class SessionManager {
    var sessionID: String
    var activeStates: [StateData] = []
    
    init(sessionID: String) {
        self.sessionID = sessionID
    }
    
    func recordState(score: Double, active: Bool) {
        // 고빈도로 발생하는 상태 기록에 구조체를 활용하여 Heap 할당 오버헤드 방지
        let newState = StateData(
            confidenceScore: score, 
            isActive: active, 
            timestamp: Date().timeIntervalSince1970
        )
        activeStates.append(newState)
    }
}
```
