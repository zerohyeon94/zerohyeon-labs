# 1. Amplitude 적용 화면 확정

> Daily: [[Daily/2026-06-16]]
> 분야: iOS Mentoring
> 대주제: 하루업 iOS Amplitude 적용 범위
> 오늘의 멘토링 렌즈: 프로젝트 적용
> 멘토링 시작 문장: "iOS 멘토링을 시작하겠습니다. 질문: 하루업에서 Amplitude를 추가 적용할 화면을 어떻게 고르고, 이벤트명/속성/확인 방법을 어떤 순서로 확정하면 빠른 배포에 도움이 될까요?"

## 오늘 이 주제를 보는 이유

하루업의 이번 주 목표는 기능을 넓히는 것이 아니라 빠른 배포 전에 필요한 관측 지점을 선명하게 만드는 것이다.

Amplitude는 배포 후 사용자가 어떤 화면과 행동을 거치는지 확인하는 데 중요하지만, 기능 자체를 완성하는 작업은 아니다. 그래서 오늘은 이벤트 코드를 바로 늘리기보다 화면 목록, 이벤트명 후보, 속성 후보, 확인 방법을 먼저 고정한다.

## 학습 목표

1. Amplitude 추가 적용 후보 화면을 `필수 / 선택 / 보류`로 나눈다.
2. 각 후보에 대해 이벤트명, 속성, 확인 방법을 최소 단위로 적는다.
3. 금요일 QA 전에 실제 구현이 필요한 항목과 지금은 보류해도 되는 항목을 분리한다.

## 핵심 개념

### 화면 이벤트와 행동 이벤트

- 화면 이벤트: 사용자가 특정 화면에 도달했는지 확인한다.
- 행동 이벤트: 사용자가 버튼 탭, 미션 재추천, 완료 미션 확인 같은 의미 있는 행동을 했는지 확인한다.
- 빠른 배포에서는 모든 화면을 다 잡기보다 배포 리스크가 큰 흐름부터 잡는다.

### 오늘의 우선순위 기준

| 기준 | 질문 |
| --- | --- |
| 배포 후 꼭 확인해야 하는가 | 이 이벤트가 없으면 배포 후 사용자 흐름을 판단하기 어려운가 |
| iOS/Backend가 같이 맞아야 하는가 | 응답 변화나 실패 케이스를 함께 확인해야 하는가 |
| QA에서 바로 재현 가능한가 | 테스트 기기에서 이벤트 발생 여부를 확인할 수 있는가 |
| 없어도 첫 배포가 가능한가 | 기능 안정화 뒤 다음 배포로 넘겨도 되는가 |

## 실무 연결

오늘 [[Projects/Haru Up/하루업 API 점검 및 문서화 계획]]에 아래 표를 채우면 충분하다.

| 화면/행동 | 왜 필요한가 | 이벤트명 후보 | 속성 후보 | 확인 방법 | 우선순위 |
| --- | --- | --- | --- | --- | --- |
| 오늘의 추천 미션 화면 표시 | 회귀 확인 | `mission_recommendation_viewed` | `mission_id`, `source` | 화면 진입 후 Amplitude DebugView 확인 | 필수 |
| 다른 미션 추천 버튼 탭 | 이번 주 Backend 리스크 확인 | `mission_reroll_tapped` | `current_mission_id`, `entry_point` | 버튼 탭 후 이벤트 발생 확인 | 필수 |
| 다른 미션 추천 결과 표시 | 응답 성공 여부 확인 | `mission_reroll_result_viewed` | `mission_id`, `result_status` | 성공/실패 케이스 각각 확인 | 필수 |
| 나의 기록 완료 미션 화면 진입 | 완료 미션 표시 기준 확인 | `completed_missions_viewed` | `completed_count` | 완료 기록 화면 진입 후 확인 | 선택 |
| 챗봇/프롬프트 응답 확인 | 앱 설치 문구 제거 QA | `chatbot_response_viewed` | `contains_app_install_copy` | 문구 미노출 여부 수동 확인 | 보류 또는 선택 |

## 오늘 남길 산출물

- `필수` 화면은 오늘 확정한다.
- `선택` 화면은 금요일 QA 전에 여력이 있으면 붙인다.
- `보류` 화면은 이유를 한 줄로 남기고 이번 주 범위에서 빼도 된다.

## 다음 질문 후보

- Amplitude 이벤트명은 화면 기준으로 잡을까요, 사용자 행동 기준으로 잡을까요?
- 다른 미션 추천 실패 케이스는 어떤 속성으로 남겨야 QA에 도움이 될까요?
- 완료 미션 화면은 첫 배포에서 필수 이벤트로 봐야 할까요, 선택 이벤트로 봐도 될까요?
