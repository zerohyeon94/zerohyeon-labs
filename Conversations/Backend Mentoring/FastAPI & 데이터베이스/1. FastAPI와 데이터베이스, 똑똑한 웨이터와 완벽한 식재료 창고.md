🗣️ 가볍게 시작하기 (Simple): 똑똑한 웨이터 vs 엑셀 창고

FastAPI (웹 프레임워크): 모바일 앱(손님)이 데이터를 달라고 요청하면, 주방(서버)에서 아주 빠르게 데이터를 요리해서 가져다주는 똑똑한 웨이터입니다.

데이터베이스 (DB): 웨이터가 요리를 만들기 위해 재료를 꺼내오는 식재료 창고입니다. 종류에 따라 창고를 정리하는 방식이 완전히 다릅니다.

🤿 딥하게 들어가기 (Deep: 왜 써야 하고, 뭘 골라야 할까요?)

1. FastAPI, 도대체 왜 써야 할까요? iOS 생태계에 익숙하시다면 FastAPI는 백엔드 진입을 위한 최고의 선택지입니다. 스위프트처럼 변수의 타입(String, Int 등)을 엄격하게 지정할 수 있어서 코드가 아주 안전하고 에러를 잡기 쉽습니다. 가장 강력한 장점은 코드를 짜기만 하면 모바일 앱에서 바로 테스트해볼 수 있는 API 설명서(Swagger UI)를 자동으로 예쁘게 만들어준다는 점입니다. 백엔드를 처음 구축하고 프론트엔드와 연결할 때 이보다 더 완벽한 프레임워크는 없습니다.
    
2. 데이터베이스 시스템의 종류와 선택 데이터베이스는 크게 두 가지 파벌로 나뉩니다.

관계형 DB (SQL): MySQL, PostgreSQL, SQLite 등이 있습니다. 데이터를 엑셀 표처럼 아주 엄격한 줄과 칸에 맞춰 저장합니다. 비관계형 DB (NoSQL): MongoDB 등이 있습니다. 데이터를 하나의 큰 문서(JSON 형태)로 뭉텅이로 저장하며 구조가 아주 유연합니다.

어떤 것을 골라야 할까요? 가계부 앱의 금전 기록이나 집중 타이머의 시간 기록처럼, 데이터의 구조가 명확하고 통계가 중요하며 절대 숫자가 틀리면 안 되는 서비스에는 무조건 관계형 DB를 선택하셔야 합니다!

3. 각 시스템의 설치 방향성 설치는 어떻게 진행될까요? 각 파트별로 설치하는 공간과 역할이 다릅니다.

FastAPI 설치 (파이썬 환경) 터미널을 열고 파이썬의 패키지 매니저인 pip를 이용해 파이썬 환경 안에 라이브러리로 설치합니다. 터미널 입력: pip install fastapi uvicorn 여기서 uvicorn은 작성한 파이썬 코드를 실제 웹 서버로 띄워주는 필수 엔진 역할을 합니다.

데이터베이스 설치 (운영체제 환경) 1단계 (가볍게 시작): 파이썬에는 SQLite라는 가벼운 파일 기반의 관계형 DB가 기본으로 내장되어 있습니다. 처음 로컬에서 개발할 때는 아무것도 설치할 필요 없이 바로 지난번 수다에서 나눈 알케미(SQLAlchemy)와 연결해서 쓰면 됩니다. 2단계 (본격적인 출시): 실제 앱을 런칭하고 트래픽을 감당할 때는 성능이 뛰어난 PostgreSQL을 추천합니다. 이것은 파이썬 라이브러리가 아니라 Mac 컴퓨터 시스템 자체에 설치하는 독립적인 서버 프로그램입니다. Mac 터미널에서 Homebrew를 통해 설치(brew install postgresql)하고, 파이썬에서는 이 DB 서버와 통신하기 위한 드라이버를 추가로 설치하여 연결하게 됩니다.

💡 오늘의 요약 처방

FastAPI는 스위프트 개발자에게 가장 친숙하고, 문서화까지 알아서 해주는 빠르고 훌륭한 API 제작 도구이므로 적극 추천합니다. 데이터베이스는 정확성이 생명인 관계형 DB로 방향을 잡고, 처음엔 내장된 SQLite로 개발하다가 나중에 PostgreSQL로 넘어가는 것이 정석입니다. 설치는 파이썬 라이브러리(FastAPI)와 Mac 시스템 프로그램(PostgreSQL)을 명확히 나누어서 진행하게 됩니다.
