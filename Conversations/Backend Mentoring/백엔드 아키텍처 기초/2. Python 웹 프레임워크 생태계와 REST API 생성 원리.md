## 1. Python 환경에서 REST API를 생성하기 위해서는 FastAPI 사용?

네, 맞습니다. 파이썬(Python) 환경에서 **REST API 서버를 구축하고 클라이언트와 통신하기 위해 사용하는 가장 현대적이고 대표적인 도구가 바로 FastAPI**입니다.

파이썬에는 웹 프레임워크가 여러 가지(Django, Flask 등) 있지만, 그중에서도 FastAPI는 이름처럼 속도가 매우 빠르고 **비동기 처리(Async)** 와 **타입 힌트(Type Hint)** 를 기본적으로 지원하여 최근 백엔드 및 AI 서빙 분야에서 표준처럼 자리 잡고 있습니다.

## 2. 핵심 CS 지식: 소켓(Socket) 통신과 직렬화(Serialization)

왜 굳이 FastAPI 같은 '프레임워크'를 사용할까요? 이를 이해하려면 네트워크 통신의 가장 밑바닥에 있는 **소켓(Socket)** 개념을 알아야 합니다.

- **소켓과 HTTP 파싱:**
    
    원래 컴퓨터 간에 통신(TCP/IP)을 하려면 운영체제(OS)가 제공하는 '소켓'을 열고, 클라이언트가 보낸 날것의 바이트(Byte) 텍스트 데이터(`GET /predict HTTP/1.1...`)를 직접 읽어서 해석(Parsing)해야 합니다. FastAPI는 이러한 복잡하고 반복적인 네트워크 통신과 HTTP 프로토콜 해석 과정을 완벽하게 추상화하여, 개발자가 비즈니스 로직(AI 모델 실행, DB 저장 등)에만 집중할 수 있게 해줍니다.
    
- **직렬화(Serialization)와 역직렬화:**
    
    네트워크를 통해 데이터를 주고받을 때는 주로 문자열 형태인 **JSON**을 사용합니다. 클라이언트가 보낸 JSON 문자열을 파이썬 내부에서 사용할 수 있는 객체(Dictionary나 Class)로 변환하는 것을 역직렬화, 반대로 파이썬 객체를 JSON으로 변환해 클라이언트로 보내는 것을 직렬화라고 합니다.
    

## 3. 타 직군 협업 포인트: 데이터 스펙(Spec) 맞추기

이 직렬화/역직렬화 과정이 프론트엔드(클라이언트)와 백엔드가 협업하는 가장 핵심적인 접점입니다.

- **Swift `Codable`과의 매핑:**
    
    iOS 앱에서 서버와 통신할 때 데이터 구조를 `Struct`로 정의하고 `Codable` 프로토콜을 채택하여 JSON을 파싱하는 것처럼, FastAPI 백엔드에서는 **`Pydantic`** 이라는 라이브러리를 사용해 동일한 형태의 데이터 모델을 정의합니다.
    
- 프론트엔드와 백엔드는 개발 전 단계에서 이 모델의 '키(Key) 이름'과 '데이터 타입(String, Int, Array 등)'을 명확히 맞추는 API 명세 작업(예: Swagger/OpenAPI)을 반드시 거쳐야 합니다.
    

## 4. 실무 최적화 및 복잡도 분석

FastAPI가 데이터를 검증하고 직렬화하는 과정은 성능에 직결됩니다.

- **시간 복잡도와 Pydantic V2:**
    
    과거 파이썬 기반 서버들은 넘어온 JSON 데이터의 타입(예: 나이가 문자열인지 숫자인지)을 검증할 때 순수 파이썬 로직을 사용하여 $O(N)$ (N은 필드 수)의 시간이 걸렸고, 데이터가 커질수록 오버헤드가 발생했습니다.
    
- 하지만 최신 FastAPI(Pydantic V2 기반)는 이 직렬화/데이터 검증 코어 로직이 시스템 프로그래밍 언어인 **Rust**로 작성되어 있습니다. 덕분에 C 언어에 준하는 속도로 데이터를 파싱하여, CPU 오버헤드를 극단적으로 줄이고 초당 요청 처리량(RPS)을 극대화하는 실무 최적화를 기본적으로 제공합니다.
    

## 5. 코드 예시: 데이터 모델 정의 (Pydantic)

클라이언트가 iOS에서 JSON 형태로 회원의 신체 정보(키, 몸무게)를 보내면, FastAPI가 이를 어떻게 파이썬 객체로 안전하게 변환하는지 보여주는 코드입니다.

```Python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

# 1. Pydantic을 활용한 데이터 모델 정의 (iOS의 Codable Struct 역할)
class UserHealthData(BaseModel):
    user_id: str
    height: float
    weight: float
    is_active: bool = True # 기본값 설정 가능

# 2. 클라이언트가 POST 메서드로 JSON 데이터를 보내면, 
# FastAPI가 자동으로 UserHealthData 형식에 맞는지 검증(Validation)합니다.
@app.post("/health/record")
async def record_health_data(data: UserHealthData):
    # 만약 iOS에서 height에 "180cm"라는 문자열(String)을 보내면, 
    # Float 타입이 아니므로 FastAPI가 자동으로 422 Error(Unprocessable Entity)를 반환합니다.
    
    bmi = data.weight / ((data.height / 100) ** 2)
    
    return {
        "message": "데이터가 성공적으로 저장되었습니다.",
        "calculated_bmi": round(bmi, 2)
    }
```
