## 🗣️ 가볍게 시작하기 (Simple): "단어장 앱과 마법의 주문"

- **파이썬 (FastAPI):** 레디스에게 "이거 30분 뒤에 지워", "제일 큰 순서대로 10명만 불러와"라고 마법의 주문(함수)을 내리는 지휘관입니다.
    
- **레디스 (Redis):** 파이썬의 주문을 알아듣고, 메모리 안에서 눈 깜짝할 새에 데이터를 지우거나 줄을 세우는 아주 똑똑하고 빠른 비서입니다.

## 🤿 딥하게 들어가기 (Deep: 생각하신 시나리오의 파이썬 코드 구현)

백엔드 서버에 `redis` 라이브러리를 설치했다고 가정하고, FastAPI에서 어떻게 레디스를 조종하는지 살펴보겠습니다.

### 준비 운동: 레디스 비서와 연결하기

가장 먼저 파이썬 파일 상단에 레디스와 통신할 수 있는 다리를 놓아줍니다.

```Python
import redis
from fastapi import FastAPI

app = FastAPI()

# 레디스 비서와 연결하는 다리 (보통 로컬호스트의 6379번 포트를 씁니다)
redis_db = redis.Redis(host='localhost', port=6379, db=0, decode_responses=True)
```

### 시나리오 1: 로그아웃 토큰 검문소 (TTL 설정)

유저가 로그아웃할 때 해당 토큰을 레디스에 넣고, 유효기간(예: 30분)이 지나면 알아서 지워지도록 폭파 시간을 설정합니다.

```Python
@app.post("/logout")
def logout(token: str):
    # set 명령어: 토큰을 레디스에 저장합니다.
    # ex=1800: 핵심입니다! 1800초(30분) 뒤에 레디스가 알아서 이 데이터를 지워버립니다.
    redis_db.set(token, "blacklisted", ex=1800)
    
    return {"message": "로그아웃 성공! 토큰이 30분 뒤 폭파됩니다."}

@app.get("/secure-data")
def get_data(token: str):
    # exists 명령어: 레디스 검문소에 이 토큰이 존재하는지 0.001초 만에 확인합니다!
    if redis_db.exists(token):
        return {"error": "로그아웃된 토큰입니다. 삐빅! 접근 금지!"}
        
    return {"data": "환영합니다! 안전한 데이터입니다."}
```

### 시나리오 2: 건강 정보 '키' 실시간 랭킹 (Sorted Set)

100명의 유저가 키를 입력했을 때, 메인 DB에 전체 정보를 저장하는 동시에 레디스에는 랭킹을 위한 핵심 정보(키, 유저ID)만 던져줍니다.

```Python
@app.post("/health-record")
def add_health_record(user_id: str, height: float):
    # (여기에 원래 PostgreSQL 등 메인 DB에 모든 건강 정보를 저장하는 코드가 들어갑니다)
    
    # zadd 명령어: 'tallest_users'라는 랭킹 보드에 {유저명: 키(점수)} 형태로 던집니다.
    # 레디스는 이걸 받는 즉시 알아서 키 순서대로 줄을 쫙 세웁니다!
    redis_db.zadd("tallest_users", {user_id: height})
    
    return {"message": "건강 정보가 저장되고 실시간 랭킹이 업데이트되었습니다."}

@app.get("/top-10-tallest")
def get_top_10():
    # zrevrange 명령어: 랭킹 보드에서 점수가 높은 순(역순)으로 0등부터 9등(10명)까지 뽑아옵니다.
    # 메인 DB를 뒤질 필요 없이 0.01초 만에 최상위 10명을 가져옵니다.
    top_users = redis_db.zrevrange("tallest_users", 0, 9, withscores=True)
    
    return {"top_10": top_users}
```

## 🚨 핵심 포인트: 코드가 정말 짧고 직관적입니다!

파이썬 백엔드 코드에서 직접 보시니 어떠신가요? 복잡하게 데이터를 정렬하는 알고리즘(Sort) 코드를 짜거나, 시간을 계산해서 데이터를 지우는 타이머를 백엔드 서버에 직접 만들 필요가 전혀 없습니다.

우리는 그저 **`ex=1800` (1800초 뒤에 지워!), `zadd` (알아서 줄 세워!), `zrevrange` (위에서 10명만 줘!)** 라는 아주 짧고 명확한 명령어를 레디스에게 던지기만 하면 됩니다. 서버 코드는 믿을 수 없을 만큼 깔끔해지고, 속도는 상상을 초월하게 빨라집니다.

---

## 💡 오늘의 요약 처방

- **TTL 구현:** `set` 명령어에 `ex=(초)` 옵션을 주면 레디스가 알아서 시한폭탄을 작동시킵니다.
    
- **토큰 검증:** `exists` 명령어로 해당 데이터가 존재하는지 메모리에서 빛의 속도로 검사합니다.
    
- **자동 랭킹:** `zadd`로 점수를 넣고 `zrevrange`로 최상위 데이터를 아주 쉽게 뽑아냅니다.
