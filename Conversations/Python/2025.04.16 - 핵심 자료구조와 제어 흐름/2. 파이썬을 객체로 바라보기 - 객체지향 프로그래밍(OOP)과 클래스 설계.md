## 📌 핵심 요약 (객체지향의 본질)

- **패러다임의 전환:** 코드를 위에서 아래로 실행되는 명령어의 나열(절차적)로 보지 않고, 데이터와 그 데이터를 처리하는 함수를 하나로 묶은 **'객체(Object)'**들의 상호작용으로 봅니다.
    
- **왜 필요한가?:** 백엔드 API 서버나 복잡한 AI 파이프라인처럼 규모가 커지는 프로젝트에서 코드의 재사용성을 높이고 유지보수를 쉽게 만들기 위해 필수적입니다.
    
- **설계도와 결과물:** **'클래스(Class)'**는 속성과 기능을 정의한 붕어빵 틀(설계도)이고, 이 설계도를 바탕으로 메모리에 찍혀 나와 실제로 동작하는 결과물이 **'객체(인스턴스)'**입니다.

## 📖 상세 설명 (OOP의 4가지 핵심 기둥)

### **Step 1. 캡슐화 (Encapsulation): 안전하게 감추기**

- 데이터와 기능을 하나의 클래스 안에 묶고, 외부에서 함부로 내부 데이터를 수정하지 못하게 숨기는(은닉) 기법입니다.
    
- 타이머 앱을 구동할 때 '남은 시간'이라는 변수를 외부에서 임의로 마이너스로 조작하지 못하게 막고, 오직 `decrease_time()` 같은 정해진 메서드(함수)를 통해서만 변경하도록 통제하여 버그를 예방합니다.

### **Step 2. 상속 (Inheritance): 물려받아 확장하기**

- 이미 만들어진 부모 클래스의 기능과 데이터를 자식 클래스가 그대로 물려받아 사용하는 것입니다.
    
- 기본 `AI_Model` 클래스에 데이터 전처리, 모델 로드 기능 등을 구현해 두고, 이를 상속받아 `Drowsiness_Detection_Model`(졸음 인식)이나 `Focus_Analysis_Model`(집중도 분석)처럼 특화된 하위 모델을 코드 중복 없이 쉽게 찍어낼 수 있습니다.

### **Step 3. 다형성 (Polymorphism): 같은 지시, 다른 동작**

- 부모 클래스에서 물려받은 메서드를 자식 클래스에서 자신의 상황에 맞게 덮어쓰기(Overriding)하여 사용하는 것입니다.
    
- `predict()`라는 동일한 함수를 호출해도, 현재 메모리에 올라가 있는 객체가 어떤 모델이냐에 따라 각기 다른 예측 알고리즘이 유연하게 돌아가게 만듭니다.

### **Step 4. 추상화 (Abstraction): 핵심만 뽑아내기**

- 복잡한 내부 구현은 숨기고, 사용자가 알아야 할 핵심적인 인터페이스만 노출하는 것입니다.
    
- 백엔드 API에서 데이터베이스에 데이터를 저장할 때, 내부적으로 복잡한 SQL 쿼리문이 어떻게 도는지 몰라도 `db.save(user_data)` 한 줄만 호출하면 알아서 저장되도록 설계하는 것과 같습니다.

## ❓ 확인 질문 (면접 대비용 최종 보스)

**Q. 백엔드 서버에서 여러 종류의 AI 모델(예: 영상 기반 졸음 인식 모델, 시선 추적 모델 등)을 서빙하여 클라이언트에게 결과를 반환해야 합니다. 향후 새로운 종류의 AI 모델이 계속 추가될 수 있는 상황일 때, 객체지향 프로그래밍의 특징(특히 상속과 다형성)을 활용하여 이 시스템을 어떻게 설계할지 설명해 보세요.**

**A. >** 1. 먼저 모든 AI 모델이 공통적으로 가져야 할 속성(모델 버전, 가중치 경로 등)과 행동(데이터 전처리, 예측하기)을 추상화하여 기본이 되는 `BaseModel` 클래스를 정의하겠습니다.

2. 졸음 인식 모델이나 시선 추적 모델은 이 `BaseModel`을 **상속(Inheritance)**받아 구현함으로써, 공통된 코드를 다시 짤 필요 없이 개발 속도와 재사용성을 높이겠습니다.

3. **다형성(Polymorphism)**을 적극 활용하여 부모 클래스의 `predict()` 메서드를 각 하위 모델 클래스의 특성에 맞게 오버라이딩(재정의)하겠습니다.

4. 이렇게 설계하면 백엔드 서버에서 클라이언트의 분석 요청이 들어왔을 때, 구체적으로 어떤 모델인지 신경 쓰지 않고 `model.predict(user_data)`라는 동일한 인터페이스만 호출해도 각각 알맞은 예측 결과를 도출할 수 있습니다. 결과적으로 새로운 모델이 추가되더라도 기존 서버 코드를 거의 수정할 필요 없이 유연한 확장이 가능합니다.
