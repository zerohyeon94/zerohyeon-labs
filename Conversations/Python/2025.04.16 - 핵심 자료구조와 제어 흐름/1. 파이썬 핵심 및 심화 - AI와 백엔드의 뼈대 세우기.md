## 📌 핵심 요약 (파이썬 데이터 핸들링의 본질)

- **목적의 분리:** 
	- 데이터를 순서대로 모을 것인가(리스트)
	- 변하지 않게 고정할 것인가(튜플)
	- 키워드로 찾을 것인가(딕셔너리)
	- 중복을 없앨 것인가(세트)
	명확히 구분해야 합니다.
    
- **파이썬다운 코드(Pythonic):** 단순히 기능만 동작하는 코드를 넘어, 리스트 컴프리헨션 등을 활용해 데이터를 빠르고 간결하게 전처리하는 것이 AI/백엔드 성능의 핵심입니다.
    
- **메모리 안정성:** 함수와 변수의 유효 범위(Scope)를 철저히 통제하여, 24시간 돌아가는 서버나 무거운 AI 모델 학습 시 발생할 수 있는 메모리 누수와 버그를 사전에 차단합니다.

## 📖 상세 설명 (실무에서 쓰이는 방식)

### **Step 1. 무엇에 데이터를 담을 것인가? (핵심 자료구조)**

- 데이터를 순차적으로 쌓아서 AI 모델에 던져줄 때 $\rightarrow$ **리스트 (List)**
    
- 데이터베이스 접속 정보처럼 절대 중간에 바뀌면 안 되는 값을 저장할 때 $\rightarrow$ **튜플 (Tuple)**
    
- 클라이언트(iOS 앱 등)와 서버가 주고받는 JSON 데이터를 다룰 때 $\rightarrow$ **딕셔너리 (Dictionary)**
    
- 수만 건의 로그 데이터 중 중복 없는 고유 사용자만 빠르게 추려낼 때 $\rightarrow$ **세트 (Set)**

### **Step 2. 데이터를 어떻게 빠르고 우아하게 가공할까? (리스트 컴프리헨션)**

- 기존의 `for`문과 `if`문을 여러 줄 쓰지 않고, 단 한 줄로 새로운 리스트를 생성해 냅니다.
    
- 데이터베이스에서 꺼낸 수백 개의 행(Row) 데이터를 API 응답 포맷으로 변환하거나, 이미지 픽셀 값들을 AI 모델에 맞게 정규화(Normalization)하는 전처리 단계에서 속도와 가독성을 극대화합니다.

### **Step 3. 데이터는 어디까지 살아있는가? (함수와 유효 범위)**

- **지역 변수 (Local Scope):** 함수 안에서 잠시 쓰이고 사라지는 데이터입니다. 불필요한 메모리 차지를 막습니다.
    
- **전역 변수 (Global Scope):** 프로그램 전체에서 공유되는 데이터입니다. 남용하면 어떤 함수가 이 값을 바꿨는지 추적하기 힘들어져 치명적인 서버 장애를 유발할 수 있습니다.

## 🆚 상황별 자료구조 선택 가이드 (Cheatsheet)

| **해결하려는 문제 상황**                        | **필요한 파이썬 자료구조**         |
| -------------------------------------- | ------------------------ |
| "클라이언트가 보낸 회원가입 요청(JSON)을 파싱해야 해."     | **딕셔너리 (Dictionary)** 적용 |
| "모델 학습을 위해 이미지 10,000장의 경로를 모아둬야 해."   | **리스트 (List)** 적용        |
| "이 변수에는 절대 바뀌면 안 되는 서버 포트 번호를 넣을 거야."  | **튜플 (Tuple)** 적용        |
| "오늘 우리 서비스 API를 호출한 중복 없는 IP 목록이 필요해." | **세트 (Set)** 적용          |

## ❓ 확인 질문 (면접 대비용 최종 보스)

**Q. 새로운 백엔드 API를 개발하게 되었습니다. 클라이언트(iOS 앱 등)로부터 사용자의 상태 데이터(예: 집중 여부, 자세 등)를 JSON 형태로 받아, 이를 AI 모델이 분석할 수 있는 형태로 변환하여 넘겨주는 과정을 구현해야 합니다. 지금까지 배운 파이썬의 자료구조를 종합하여 데이터를 어떻게 처리할지 설명해 보세요.**

**A. >**

1. 먼저 클라이언트로부터 들어오는 JSON 형태의 요청은 **딕셔너리(Dictionary)**로 받아 키(Key)-값(Value) 쌍으로 안전하고 직관적으로 파싱하겠습니다.
    
2. 이후 AI 모델에는 정해진 순서의 데이터 배열이 들어가야 하므로, 딕셔너리에서 필요한 수치 값들만 추출하여 **리스트(List)** 형태로 변환하겠습니다. 이 과정에서 코드를 간결하고 빠르게 만들기 위해 **리스트 컴프리헨션**을 적극 활용하겠습니다.
    
3. 모델이 결과를 반환할 때 사용하는 여러 개의 고정된 상태 임계값(Threshold) 등은 데이터 오염을 막기 위해 **튜플(Tuple)**로 관리하여 백엔드 로직의 안정성을 높이겠습니다.
