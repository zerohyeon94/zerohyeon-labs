## 📌 핵심 요약 (파이썬 심화 기술의 본질)

- **데코레이터 (Decorator)**: 함수 앞뒤에 공통적으로 들어가는 로직(예: 실행 시간 측정, 로그인 확인 등)을 분리하여 코드 재사용성을 극대화합니다.
    
- **제너레이터 (Generator)**: 수만 장의 이미지나 대용량 데이터를 한 번에 메모리에 올리지 않고, 필요할 때마다 하나씩 꺼내 쓰도록 만들어 메모리 효율을 극대화합니다.
    
- **비동기 프로그래밍 (Async/Await)**: 네트워크 요청이나 데이터베이스 응답을 기다리는 동안 멈춰있지 않고 다른 작업을 먼저 처리하여 빠르고 효율적인 API 서버를 만듭니다.

## 📖 상세 설명 (실무에서 쓰이는 방식)

### Step 1. 코드의 포장지, 데코레이터 (Decorator)

API를 개발하다 보면 여러 함수에 '사용자 인증'이나 '실행 시간 로그 남기기' 같은 동일한 코드가 반복됩니다. 데코레이터를 사용하면 핵심 로직과 부가 기능을 깔끔하게 분리할 수 있습니다. 기존 함수를 수정하지 않고도 기능을 덧붙일 수 있어 유지보수에 매우 유리합니다.

```Python
import time

# 데코레이터 정의: 함수의 실행 시간을 측정하는 포장지
def time_logger(func):
    def wrapper(*args, **kwargs):
        start_time = time.time()
        result = func(*args, **kwargs)
        end_time = time.time()
        print(f"[{func.__name__}] 실행 시간: {end_time - start_time:.4f}초")
        return result
    return wrapper

# 데코레이터 적용 (@ 태그 사용)
@time_logger
def train_ai_model():
    print("AI 모델 학습 시작...")
    time.sleep(2)  # 학습에 2초가 걸린다고 가정
    print("AI 모델 학습 완료!")

train_ai_model() 
# 출력 결과에 시간 측정 로그가 자동으로 추가됩니다.
```

### Step 2. 메모리 다이어트, 제너레이터 (Generator)

`return` 대신 `yield`를 사용하는 제너레이터는 데이터를 한 번에 모두 만들어 리스트로 반환하는 것이 아니라, 데이터를 생성하는 '규칙'만 가지고 있다가 다음 값을 요청할 때마다 하나씩 내어줍니다.

```Python
# 1. 일반적인 방식 (결과를 모두 메모리에 적재)
def normal_list_func(n):
    result = []
    for i in range(n):
        result.append(i)
    return result

# 2. 제너레이터 방식 (현재 상태만 기억하고 필요할 때 생성)
def generator_func(n):
    for i in range(n):
        yield i  # 값을 하나씩 내보내고 함수 상태를 일시 정지

# 수백만 개의 데이터를 처리할 때 메모리 차이가 극명하게 나타납니다.
huge_data_gen = generator_func(1000000)
print(next(huge_data_gen)) # 0 반환 후 대기
print(next(huge_data_gen)) # 1 반환 후 대기
```

### Step 3. 기다림의 미학, 비동기 프로그래밍 (Async/Await)

FastAPI와 같은 최신 파이썬 웹 프레임워크의 핵심입니다. 데이터베이스에서 집중도 분석 결과나 사용자 기록을 가져올 때, DB의 응답을 기다리는 시간 동안 서버가 멈추지 않고 다른 사용자의 요청을 동시에 처리할 수 있게 해줍니다.

```Python
import asyncio

# 비동기 함수 정의 (async 키워드 사용)
async def fetch_database(user_id):
    print(f"User {user_id} 데이터베이스 조회 요청...")
    await asyncio.sleep(2)  # DB 응답을 2초간 기다림 (이 동안 다른 작업 수행 가능)
    print(f"User {user_id} 조회 완료!")
    return {"status": "success"}

async def main():
    # 여러 비동기 작업을 동시에 실행하여 시간을 크게 단축
    await asyncio.gather(
        fetch_database("A"), 
        fetch_database("B")
    )

# 실제 환경에서는 asyncio.run(main()) 으로 실행합니다.
```

## ❓ 확인 질문 (면접 대비용 최종 보스)

**Q. 이미지 기반의 AI 모델을 훈련하기 위해 로컬 스토리지에 있는 100만 장의 이미지를 읽어오는 전처리 파이프라인을 구축해야 합니다. 또한, 처리 속도가 너무 느려서 각 전처리 단계(함수)마다 시간이 얼마나 걸리는지 로그를 남기고 싶습니다. 지금까지 배운 파이썬 심화 개념을 활용하여 이 시스템을 어떻게 구현할지 설명해 보세요.**

**A. >**

> 1. 먼저, 100만 장의 이미지를 한 번에 리스트(List)에 담으면 메모리 초과(OOM)가 발생할 수 있으므로, 이미지를 순차적으로 하나씩 읽어오는 **제너레이터(Generator)** 함수를 만들겠습니다. `yield`를 사용해 메모리 사용량을 최소화합니다.
>     
> 2. 각 전처리 단계(예: 이미지 리사이징, 흑백 변환 등) 함수의 실행 시간을 측정하기 위해, 시작 시간과 종료 시간을 기록하는 **데코레이터(Decorator)**를 작성하겠습니다.
>     
> 3. 작성한 데코레이터를 전처리 함수들 위에 `@time_logger`와 같이 붙여주면, 기존 전처리 로직을 직접 수정하지 않고도 모든 함수의 실행 시간을 깔끔하고 효율적으로 로깅할 수 있습니다.
