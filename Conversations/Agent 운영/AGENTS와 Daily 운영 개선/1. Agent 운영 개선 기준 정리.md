# 1. Agent 운영 개선 기준 정리

> Daily: [[Daily/2026-06-12]]
> 분야: Agent 운영
> 대주제: AGENTS와 Daily 운영 개선
> 멘토링 시작 문장: "Agent 운영 멘토링을 시작하겠습니다. 질문: 지금 zerohyeon-labs에서 Codex 운영을 개선하려면 무엇을 먼저 정리해야 하나요?"

## 오늘 이 주제를 보는 이유

최근 멘토링 주제, 프로젝트 회의, 리젝 대응, Go/React 전환 준비가 한꺼번에 쌓이면서 Codex가 매일 어떤 기준으로 움직여야 하는지 더 명확히 할 필요가 생겼다. 오늘은 기능을 늘리기보다 운영 기준을 줄이고 선명하게 만드는 것이 목표다.

## 학습 목표

- `AGENTS.md`가 담당해야 할 규칙과 Daily Note가 담당해야 할 계획을 분리한다.
- 멘토링 시작 문장을 분야별로 안정화한다.
- Agent 개선을 한 번에 크게 바꾸지 않고 하루 실험 단위로 운영하는 법을 익힌다.

## 핵심 개념

### AGENTS.md의 역할

- 장기 운영 원칙
- Codex가 해도 되는 일과 하지 말아야 할 일
- Daily Note 생성 규칙
- 멘토링 파일 생성 위치
- 트리거 문장과 응답 방식

### Daily Note의 역할

- 오늘 실제로 할 일
- 오늘 읽을 파일
- 오늘 시작할 멘토링
- 저녁에 확인할 산출물
- 내일로 넘길 후보

### Agent 개선의 단위

좋은 개선은 큰 선언이 아니라 작은 규칙 하나다.

예:

- "코드 저장소는 명시 요청 없이는 열지 않는다."
- "평일 필수 멘토링은 1개만 둔다."
- "멘토링 리셋 요청이 있으면 미완료 이월보다 리셋을 우선한다."
- "Agent 개선은 Daily에 완료 기준을 남긴다."

## 실무 연결

다음 주부터 Go/React 프로젝트가 시작되면 Codex가 코드까지 바로 들어가고 싶어질 수 있다. 하지만 `zerohyeon-labs`는 학습/운영 공간이므로, 여기서는 먼저 계획, 질문, 체크리스트, 멘토링 흐름을 정리한다. 코드 수정은 사용자가 명시적으로 요청한 별도 작업으로 분리한다.
