## 오늘 이 주제를 보는 이유

핵심 목표는 Swift/iOS 기반 모바일 개발자로 집중하는 것이다. 다만 모바일 개발자로 커리어를 넓게 가져가려면 Android, Kotlin, Flutter, React Native가 어떤 문제를 해결하는 기술인지 정도는 비교할 수 있어야 한다.

오늘은 첫 출근일이므로 깊게 파고들기보다 앞으로의 확장 지도를 만든다. 이 파일은 당장 학습할 필수 과제가 아니라, 이후 Daily Note에서 모바일 확장 주제가 필요할 때 참고할 기준점이다.

## 학습 목표

- iOS 중심성을 유지하면서 다른 모바일 스택을 학습하는 이유를 정리한다.
- Android/Kotlin/Flutter/RN을 각각 어떤 관점에서 보면 되는지 구분한다.
- 지금 당장 깊게 공부할 것과 나중에 필요할 때 볼 것을 나눈다.

## 핵심 개념

### Android Mentoring

Android는 iOS와 같은 네이티브 모바일 축이다. Activity/Fragment 생명주기, 권한, 백그라운드 제약, Jetpack Compose를 iOS의 ViewController/SwiftUI와 비교하면 학습 효율이 높다.

### Kotlin Study

Kotlin은 Android의 주력 언어이면서 서버에서도 쓰인다. Swift 개발자 입장에서는 null safety, data class, sealed class, coroutine을 Swift의 Optional, struct, enum, async/await와 비교하면 좋다.

### Flutter Study

Flutter는 Dart와 자체 렌더링 엔진 기반의 크로스 플랫폼 UI 프레임워크다. SwiftUI 경험이 있으면 Widget tree와 상태 관리를 비교하며 이해할 수 있다.

### RN Study

React Native는 JavaScript/TypeScript와 네이티브 브리지를 연결하는 방식이다. iOS 개발자는 native module, bridge, JSI, 성능 병목을 이해하면 협업에서 강점이 생긴다.

## 실무 연결

현재 우선순위는 아래처럼 둔다.

1. iOS/Swift: 매일의 주 학습 축
2. On-device AI: iOS 포트폴리오와 연결될 때 학습
3. Android/Kotlin: 모바일 협업과 비교 관점 확보
4. Flutter/RN: 채용 공고나 프로젝트 필요가 생길 때 선택적으로 확장
