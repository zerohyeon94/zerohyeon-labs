> 멘토링 시작 문장: "Mobile 멘토링을 시작하겠습니다. 질문: 평일 근무를 병행하면서 iOS, Android, Kotlin, Flutter, RN을 어떤 우선순위로 가져가면 좋을까요?"

---

## 오늘 이 주제를 보는 이유

평일 근무가 시작되면 학습 시간보다 중요한 것은 우선순위다. 모든 모바일 스택을 동시에 깊게 파면 금방 지친다.

현재 핵심은 Swift/iOS 모바일 개발자로 집중하는 것이다. Android, Kotlin, Flutter, React Native는 지금 당장 모두 공부할 대상이 아니라, iOS 중심성을 강화하고 협업 시야를 넓히기 위한 확장 지도에 가깝다.

## 학습 목표

- 평일 근무 병행 상황에서 모바일 학습 우선순위를 정한다.
- iOS/Swift를 주축으로 유지하고, 나머지 스택은 비교 관점으로 둔다.
- 이번 주에 실제로 할 수 있는 최소 행동을 정한다.

## 핵심 개념

### 1. 1순위는 iOS와 Swift 유지

현재 커리어의 중심축은 iOS다. 따라서 매일 또는 격일로 이어갈 최소 루틴은 iOS/Swift여야 한다.

- iOS Mentoring: 프레임워크와 앱 구조 이해
- Swift Study: 언어, 동시성, 메모리, 성능 이해
- On-device AI: Vision/Core ML과 연결될 때만 확장

### 2. Android/Kotlin은 비교 학습

Android와 Kotlin은 네이티브 모바일의 반대편 축이다. 당장 앱을 새로 만들기보다, iOS 개념과 비교하면서 이해하는 방식이 효율적이다.

비교 포인트:

- SwiftUI vs Jetpack Compose
- Swift async/await vs Kotlin coroutine
- iOS 권한/백그라운드 제약 vs Android 권한/백그라운드 제약
- Swift Optional vs Kotlin null safety

### 3. Flutter/RN은 채용 공고와 프로젝트 필요에 따라 확장

Flutter와 React Native는 "언젠가 필요할 수 있는 모바일 확장 카드"다. 지금은 깊게 파기보다 아래 정도를 목표로 둔다.

- 어떤 문제를 해결하는 프레임워크인지 설명하기
- iOS 개발자가 협업에서 맡을 수 있는 native bridge/native module 역할 이해하기
- 채용 공고에서 요구될 때 빠르게 진입할 수 있게 키워드만 확보하기

## 실무 연결

근무 병행 기준으로는 아래처럼 둔다.

```text
평일 필수: iOS 또는 Swift 20분
평일 선택: 업무와 연결되는 Backend/AI 개념 10분
주말 선택: Android/Kotlin/Flutter/RN 비교 학습 1개
```

오늘은 일요일이므로, 다음 주를 위해 "이번 주 필수 축 1개"만 정하면 된다.
