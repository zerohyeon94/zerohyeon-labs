### 📌 핵심 요약

모델의 예측 결과를 실제 정답과 비교할 때, 단순히 맞고 틀린 것을 넘어 **'어떻게 맞췄고 어떻게 틀렸는지'**를 4가지 경우의 수로 나누어 평가합니다.

- **정확도 (Accuracy):** 전체 문제 중 모델이 정답을 맞춘 비율
    
- **정밀도 (Precision):** 모델이 '정답(Positive)'이라고 예측한 것 중, 실제 정답인 비율
    
- **재현율 (Recall):** 실제 '정답(Positive)'인 데이터 중, 모델이 제대로 찾아낸 비율
    
- **F1-Score:** 정밀도와 재현율의 조화평균 (둘 중 하나에 치우치지 않는지 평가)

### 📖 상세 설명

평가지표를 이해하려면 먼저 **오차 행렬(Confusion Matrix)**의 4가지 상태를 알아야 합니다. 카메라로 사용자의 상태를 분석해 **'졸음(Positive)'**을 감지하는 AI 모델을 예로 들어보겠습니다.

- **TP (True Positive):** 실제 졸음인데, 모델도 졸음이라고 **맞게** 예측함 (정답)
    
- **TN (True Negative):** 실제 정상인데, 모델도 정상이라고 **맞게** 예측함 (정답)
    
- **FP (False Positive):** 실제 정상인데, 모델이 졸음이라고 **틀리게** 예측함 (오경보)
    
- **FN (False Negative):** 실제 졸음인데, 모델이 정상이라고 **틀리게** 예측함 (놓침) 

이 4가지 요소를 조합하여 다양한 지표를 계산합니다.

**1. 정확도 (Accuracy)**

- **개념:** 전체 데이터 중 모델이 올바르게 예측한(TP + TN) 비율입니다.
    
- **수식:** $\frac{TP + TN}{TP + TN + FP + FN}$
    
- **한계:** 데이터가 불균형할 때 착시를 일으킵니다. 예를 들어 100시간 중 졸음 시간이 1시간뿐이라면, 모델이 무조건 "100% 정상입니다"라고만 찍어도 정확도는 99%가 되어버립니다.

**2. 정밀도 (Precision)**

- **개념:** 모델이 '졸음(Positive)'이라고 알람을 울린 횟수 중, 실제로 사용자가 졸고 있었던 비율입니다.
    
- **수식:** $\frac{TP}{TP + FP}$
    
- **중요한 상황:** 모델이 틀렸을 때(FP) 사용자가 느끼는 피로도나 비용이 클 때 중요합니다. (예: 스팸 메일 필터링 - 정상 메일을 스팸으로 분류해버리면 큰일 남)

**3. 재현율 (Recall / Sensitivity)**

- **개념:** 사용자가 실제로 '졸음(Positive)' 상태였던 전체 횟수 중, 모델이 놓치지 않고 알람을 울린 비율입니다.
    
- **수식:** $\frac{TP}{TP + FN}$
    
- **중요한 상황:** 모델이 정답을 놓쳤을 때(FN) 발생하는 위험이 매우 클 때 중요합니다. (예: 암 진단, 졸음 운전 감지 - 오경보가 울리더라도 진짜 위험을 절대 놓치면 안 됨)

**4. F1-Score**

- **개념:** 정밀도와 재현율은 서로 반비례 관계(Trade-off)를 가지는 경우가 많습니다. 둘 중 어느 한쪽으로 치우치지 않게 균형을 평가하기 위한 조화평균 지표입니다.
    
- **수식:** $2 \times \frac{Precision \times Recall}{Precision + Recall}$

### 🆚 비교: 정밀도 vs 재현율 (Trade-off)

|**구분**|**정밀도 (Precision)**|**재현율 (Recall)**|
|---|---|---|
|**관점**|**모델의 예측**이 얼마나 정확한가?|**실제 정답**을 얼마나 안 놓쳤는가?|
|**리스크**|확실할 때만 예측하려다 진짜 정답을 놓침|다 찾아내려다 오경보(가짜 정답)가 남발됨|
|**줄여야 할 오차**|**FP** (오경보 줄이기)|**FN** (놓침 줄이기)|
|**초점**|스팸 메일, 추천 시스템|암 진단, 재난 알림, 보안 침입 탐지|

### ❓ 확인 질문 (면접 대비용)

**Q. 카메라 영상으로 사용자가 자리를 비웠는지(이석) 판단하는 앱을 개발 중입니다. 이때 '정밀도'를 높게 세팅한 모델과 '재현율'을 높게 세팅한 모델은 실제 앱에서 각각 사용자에게 어떤 경험을 주게 될까요?**

> **A.** > **정밀도가 높은 모델**은 사용자가 완벽하게 카메라 밖으로 나갔을 때만 '이석'으로 판정합니다. 오작동(자리에 있는데 이석 처리되는 일)은 거의 없지만, 화면 모서리에 살짝 걸쳐 있거나 애매한 상황에서는 진짜 이석이 발생해도 시스템이 알람을 놓치는 빈도가 잦아집니다.
> 
> 반면 **재현율이 높은 모델**은 조금만 움직여서 얼굴이 화면 밖으로 살짝만 나가도 즉시 '이석'으로 판정합니다. 진짜 이석 상황을 절대 놓치지는 않지만, 사용자가 기지개를 켜거나 고개를 잠깐 숙이기만 해도 경고가 울리는 오경보(False Positive)가 잦아져 사용자 피로도를 유발할 수 있습니다.
