### 📌 핵심 요약

활성화 함수는 입력받은 데이터를 다음 층(Layer)으로 출력할지 말지, 출력한다면 **어떤 형태로 변환해서 내보낼지 결정하는 '문지기'** 역할입니다.

- **가장 중요한 역할:** 신경망에 **비선형성(Non-linearity)**을 부여합니다. 이 함수들이 없으면 층을 100개를 쌓아도 결국 단순한 직선 하나(단층 퍼셉트론)와 똑같아집니다.
    
- **대표적인 종류:** Sigmoid(시그모이드), ReLU(렐루), Softmax(소프트맥스) 등

### 📖 상세 설명

**1. 왜 꼭 비선형이어야 할까?**

신경망에서 가중치를 곱하고 편향을 더하는 작업($x \times w + b$)은 모두 선형 연산(직선)입니다. 선형 방정식은 아무리 여러 번 합성하고 중첩해도 결국 또 다른 하나의 선형 방정식으로 합쳐져 버립니다.

즉, 은닉층 사이에 '선을 구부려주는' 비선형 활성화 함수를 끼워 넣지 않으면 다층 신경망을 만드는 의미가 아예 없어집니다.

**2. 1세대 활성화 함수: Sigmoid (시그모이드)**

- **특징:** 어떤 값이 들어오든 무조건 **0과 1 사이의 부드러운 S자 곡선**으로 압축해서 변환합니다.
    
- **장점:** 출력값이 0~1 사이라서 확률처럼 해석하기 좋습니다. (예: 0.8이 나오면 "졸음일 확률이 80%구나!")
    
- **치명적인 단점:** 입력값이 너무 크거나 작으면 곡선의 양 끝에서 기울기가 거의 0에 가까워집니다. 이 때문에 층을 깊게 쌓을수록 데이터가 뒤쪽으로 제대로 전달되지 않고 희미해져 버리는 **기울기 소실(Vanishing Gradient)** 문제가 발생합니다.

**3. 딥러닝의 구세주: ReLU (렐루)**

- **특징:** 입력값이 0보다 작으면 무조건 0으로 만들고, 0보다 크면 입력값을 그대로 통과시킵니다. ($f(x) = \max(0, x)$)
    
- **장점:** 양수일 때는 기울기가 항상 일정하게 유지되므로 시그모이드의 치명적 단점인 **기울기 소실 문제를 획기적으로 해결**했습니다. 게다가 0 이하는 무시해 버리니 연산 속도도 엄청나게 빠릅니다. 현재 딥러닝 은닉층의 가장 표준적인 활성화 함수입니다.

**4. 다중 분류의 해결사: Softmax (소프트맥스)**

- **특징:** 주로 출력층에서 사용되며, 여러 개의 선택지가 있을 때 각각의 확률을 계산해 줍니다.
    
- **장점:** 모든 출력값의 합이 정확히 1(100%)이 되도록 만들어주기 때문에, "강아지 70%, 고양이 20%, 토끼 10%"처럼 결과를 직관적으로 해석할 수 있습니다.

### 🆚 비교: Sigmoid vs ReLU

|**구분**|**Sigmoid (시그모이드)**|**ReLU (렐루)**|
|---|---|---|
|**출력 범위**|0부터 1 사이|0 이상 (음수는 0, 양수는 무한대)|
|**형태**|부드러운 S자 곡선|0에서 꺾이는 직선|
|**기울기 소실 문제**|**발생함** (층이 깊어지면 학습 불가)|**발생하지 않음** (양수 영역에서 학습 원활)|
|**주요 사용 위치**|이진 분류 모델의 **최종 출력층**|거의 모든 딥러닝 모델의 **은닉층** (표준)|

### ❓ 확인 질문 (면접 대비용)

**Q. 딥러닝 모델의 은닉층(Hidden Layer)을 구성할 때, 과거에 주로 쓰던 시그모이드(Sigmoid) 함수 대신 현재는 ReLU 함수를 기본적으로 사용하는 이유는 무엇인가요?**

> **A.** 가장 큰 이유는 **'기울기 소실(Vanishing Gradient)' 문제를 해결하기 위해서**입니다. 시그모이드 함수는 입력값의 절대값이 커지면 미분값(기울기)이 0에 가까워져, 은닉층을 깊게 쌓았을 때 학습 내용이 앞쪽 층까지 전달되지 않는 문제가 있었습니다. 반면 ReLU 함수는 양수 영역에서 항상 일정한 기울기를 가져 층이 깊어져도 학습이 잘 이루어지며, 연산 구조가 단순해 학습 속도도 훨씬 빠르기 때문입니다.
