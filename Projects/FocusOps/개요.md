> 목적: Go/React 서버 배포와 실제 동작 구현을 확인하기 위한 iOS 기반 집중 서비스 프로젝트

## 왜 준비하는가

ZipJoong은 크랜베리 입사 전후로 Go 백엔드, PostgreSQL, React 대시보드, iOS 연동, Docker/Linux 배포 감각을 실제로 연결하기 위한 프로젝트로 둔다.

현재 프로젝트는 아직 본격적으로 진행되지 않았다.
따라서 지금 목표는 큰 제품을 바로 완성하는 것이 아니라, Go와 React로 서버/대시보드를 배포해보고 실제로 동작하는 최소 흐름을 확인하는 것이다.

## 우선 연결할 학습 축

- Go: REST API, handler/service/repository 구조, DB 연결, 배포
- PostgreSQL: 사용자, 장치, 집중 세션, 이벤트 로그, 통계 테이블 설계
- React: 세션 목록, 이벤트 로그, 일별 통계 대시보드
- Docker/Linux: Go API, PostgreSQL, React 앱을 실제 실행 환경으로 묶기
- iOS/Swift: 이후 ZipJoong iOS 앱에서 세션 시작/종료와 이벤트 전송을 연결

## 초기 MVP 후보

1. Go REST API
   - 장치 등록
   - 집중 세션 생성/조회
   - 집중 이벤트 생성/조회

2. PostgreSQL 저장소
   - users
   - devices
   - focus_sessions
   - focus_events

3. React 대시보드
   - 장치 목록
   - 집중 세션 목록
   - 이벤트 로그
   - 일별 통계 카드

4. iOS 연동
   - 시작 시간
   - 종료 시간
   - 집중 세션 결과
   - 이벤트 전송

## 당장 하지 않을 것

- 처음부터 iOS, AI, Go, React, PostgreSQL, gRPC를 모두 구현
- 복잡한 협업 기능
- AI 자동 계획
- 완전한 프로젝트 관리 도구
- 실시간 관제나 gRPC 시뮬레이터
