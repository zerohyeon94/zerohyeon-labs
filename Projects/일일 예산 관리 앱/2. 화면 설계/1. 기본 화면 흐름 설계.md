## ✅ 2단계 목표: "가계씨" 앱의 기본 화면 흐름 설계

---

### 🧭 전체 화면 구조 (초기 버전 MVP)

| 순서     | 화면 이름                                | 설명                         |
| ------ | ------------------------------------ | -------------------------- |
| 1️⃣    | **설정 화면** (`SetupViewController`)    | 월급, 월급일, 고정비 항목 입력         |
| 2️⃣    | **홈 화면** (`HomeViewController`)      | 오늘 쓸 수 있는 금액, 이월 금액, 소비 버튼 |
| 3️⃣    | **소비 기록 화면** (`SpendViewController`) | 오늘 쓴 금액을 입력하고 저장           |
| ✅ (옵션) | 소비 통계 화면                             | 추후 확장 가능                   |

---

### ✅ 기본 흐름 (Onboarding → 설정 → 홈)

```mathematica
On Launch
   ↓ (CoreData에 BudgetConfig 없으면)
SetupViewController  ← 월급, 월급일, 고정비 입력
   ↓ 저장 완료 시
HomeViewController ← 오늘의 예산 표시
   ↓
SpendViewController ← 소비 입력
```

---

## 🧱 화면 구성 방식

- UIKit + MVVM 구조 권장
- ViewController 이름은 기능 중심으로:
    - `SetupViewController`, `HomeViewController`, `SpendViewController`

## ✅ 다음 추천 작업

1.  `SetupViewController` 생성
2.  월급, 월급일, 고정비 항목 UI 구성 (`UITextField`, `UIDatePicker`, `UITableView`)
3.  저장 버튼 → CoreDataManager 통해 BudgetConfig 저장
4.  저장 완료 → `HomeViewController`로 화면 전환

---

## 🎁 시작 포인트 제안

- Navigation 기반 앱이면 `Setup → Home`은 **pushViewController**
- 모달로 구성하고 싶다면 **present → dismiss**
- `SceneDelegate.swift` 또는 `AppDelegate.swift`에서 **초기 진입 화면 결정**

```swift
if CoreDataManager.shared.fetchBudgetConfig() == nil {
    window.rootViewController = SetupViewController()
} else {
    window.rootViewController = HomeViewController()
}
```
