**UIKit에서 코드베이스로 ViewController를 생성할 때도 `super.init(nibName: nil, bundle: nil)`을 사용하는 것이 맞습니다.**  
이건 약간 이상하게 보일 수 있지만, **UIKit의 설계 특성상 필요한 초기화 방식**입니다.

---

## ✅ 결론부터 말씀드리면:

> ✅ **코드베이스에서도 `super.init(nibName: nil, bundle: nil)`을 사용하는 것이 정석입니다.**  
> ❌ `super.init()`은 존재하지 않으며 호출 시 컴파일 에러가 발생합니다.

---

## 📌 왜 그런가요?

UIKit의 `UIViewController`는 다음과 같은 초기화 메서드만 제공합니다:

```swift
public init(nibName nibNameOrNil: String?, bundle nibBundleOrNil: Bundle?)
```

즉, **UIKit의 ViewController는 항상 nibName/bundle을 받는 초기화자만 가지고 있고**,  
`init()`이라는 **매개변수 없는 생성자는 제공하지 않습니다.**

---

## ✅ 코드베이스 vs 스토리보드 차이

|구분|nibName|bundle|
|---|---|---|
|코드베이스|`nil`|`nil`|
|XIB 사용|`"MyView"`|`Bundle.main`|
|스토리보드 사용|해당 안 됨 (스토리보드 ID로 인스턴스 생성)||

---

## ✅ 예시: 코드베이스용 올바른 초기화

```swift
final class HomeViewController: UIViewController {
    override init(nibName nibNameOrNil: String? = nil, bundle nibBundleOrNil: Bundle? = nil) {
        super.init(nibName: nil, bundle: nil)  // ← 코드베이스는 항상 nil
    }

    required init?(coder: NSCoder) {
        fatalError("init(coder:) has not been implemented")
    }
}
```

## ❗ 실수 예시 (컴파일 에러 발생)

```swift
super.init() // ❌ 오류! 정의된 init() 없음
```

> `'super.init' has no accessible initializers`

## ✅ 결론 요약

|항목|답변|
|---|---|
|코드베이스에서도 `super.init(nibName:nil, bundle:nil)` 써야 하나요?|✅ 네, UIKit의 규칙입니다|
|`super.init()`만으로 가능한가요?|❌ 컴파일 에러 발생|
|스토리보드 안 쓸 건데도 왜 nibName이 필요한가요?|UIKit에서 ViewController 생성자는 항상 `nibName/bundle`을 받도록 되어 있음|

---

따라서 다음처럼 사용하는 것이 **정석이자 안전한 방식입니다**:

```swift
super.init(nibName: nil, bundle: nil)
```
