## 📅 2026년 6월 28일 (일) – SwiftUI 마이그레이션 후 MVP 기능 보강 (출시 준비)

> UIKit → SwiftUI 마이그레이션 및 돼지 캐릭터 테마 UI 재설계 이후,
> 빠른 출시를 위해 우선순위 기준으로 MVP 범위를 선정하고 핵심 기능을 마무리한 세션.

### 0. **MVP 범위 점검 및 선정**

- 기능별 구현률 점검: 온보딩 100% / 통계 95% / 홈 90% / 지출기록 85% / 설정 80%

- 핵심 컨셉인 **이월(carry-over)이 데이터·UI는 있으나 로직 미연결(항상 0)** 임을 발견

- MVP 포함: ① 이월 자동화 ② 지출 수정 / MVP 제외(추후): 백업·피드백·알림

### 1. **이월(Carry-over) 자동화 구현**

- `CoreDataManager.processDailyBudgets(upTo:)` 신규 추가

- 마지막 기록일~오늘까지 누락된 날을 순회하며 전날 잔액(`todayAvailable`)을 이월금으로 누적

- 규칙: **마이너스 이월 / 전부 소급 / 급여일 리셋 없음**, 그리고 **멱등성**(이미 있는 날짜는 건너뜀)

- 트리거: `HomeViewModel.fetchTodayBudget()` 진입 + `HomeView`의 `scenePhase == .active`

- → [[이월(Carry-over) 자동화 - 데이터 모델은 있는데 로직이 연결 안 된 문제]]

### 2. **지출 수정 기능 추가**

- `SpendViewModel`에 `editingRecordId` 기반 편집 모드(`beginEdit`/`cancelEdit`) 도입

- 지출 행에 수정 버튼 추가, 편집 중 헤더·저장버튼·하이라이트 전환

- `updateSpendingRecord`에 **날짜 변경 시 대상 `DailyBudget` 재연결** 로직 보강 (홈·통계 일관성)

### 3. **급여일 선택 Picker 가시성 버그 수정**

- 설정 화면에서 급여일 메뉴 Picker가 키보드에 가려지고 부모 `onTapGesture`와 탭 충돌
