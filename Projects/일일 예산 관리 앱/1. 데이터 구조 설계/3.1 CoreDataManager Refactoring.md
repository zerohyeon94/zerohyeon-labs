## ✅ **중복 코드 및 불필요 코드 정리 제안**

### 1. 🔁 `fetchBudgetModel()` vs `fetchBudgetConfig()`

#### 현재:

- `fetchBudgetModel()`은 `NSManagedObject`를 사용
    
- `fetchBudgetConfig()`는 실제 `BudgetConfig` 타입을 사용

#### 개선 제안:

- **둘 중 하나만 유지**하고, 사용하지 않는 방식 제거
    
- `NSManagedObject` 기반은 타입 안정성이 떨어지므로 `BudgetConfig` 타입을 사용하는 `fetchBudgetConfig()` 중심으로 통일하는 것이 좋습니다.

```swift
// ✅ fetchBudgetModel 제거 → fetchBudgetConfig 사용 일원화
```

---

### 2. 🔁 `save(budgetModel:)`와 `createBudgetConfig(salary:payday:)` 중복

#### 개선 제안:

- `createBudgetConfig`는 거의 사용되지 않고 `save(budgetModel:)`에 통합되어 있음
    
- 하나로 통일하고, 내부에서 FixedCost까지 포함 처리 가능하도록 `save(budgetModel:)`를 중심으로 유지

```swift
// ✅ createBudgetConfig 삭제
```

---

### 3. 🧼 불필요한 `setValue(_:forKey:)` 제거

#### 예:

```swift
config.setValue(NSDecimalNumber(value: salary), forKey: "salary")
```

#### 개선 제안:

```swift
config.salary = NSDecimalNumber(value: salary)
```

> CoreData의 클래스 기반 entity를 사용하는 경우에는 키-밸류 접근보다는 **속성 기반 접근**이 더 안전하고 가독성이 좋습니다.

---

### 4. 🧹 중복되는 context 참조 제거

```swift
let context = self.context
```

#### 개선 제안:

- 이미 `context`는 computed property로 정의되어 있으므로 대부분의 함수에서 별도 선언 없이 `context` 직접 사용 가능
    
- 중복 제거

---

### 5. 🔧 `fetchSpending(on:)`와 `saveSpending(...)` 중 DailyBudget fetch 중복

#### 현재:

`saveSpending` 안에서 또 DailyBudget을 fetch함

#### 개선 제안:

- `fetchDailyBudget`을 재활용하도록 구성하면 중복 제거 가능

```swift
let dailyBudget = fetchDailyBudget(on: day) ?? DailyBudget(context: context)
```

---

### 6. 🧪 Debug용 print 제거 or DebugLogger로 통일

현재 디버깅용 `print()`가 다수 있습니다. 예:

swift

`print("config.salary : \(config.salary)")`

→ 유지보수를 위해 모두 `DebugLogger`로 통일하거나, 필요 없는 경우 제거

---

### 7. 🔁 `insertFixedCost`, `updateFixedCost` vs `insertOrUpdateFixedCost` 도입 고려

#### 개선 제안:

- 기존 title로 검색해서 없으면 insert, 있으면 update하는 통합 함수 `insertOrUpdateFixedCost(_:)` 작성 가능

```swift
func insertOrUpdateFixedCost(_ model: FixedCostModel) {
    if let existing = fetchFixedCostEntity(named: model.title) {
        updateFixedCost(model, target: existing)
    } else {
        insertFixedCost(model)
    }
}
```

## ✨ 리팩토링 우선순위 추천

|우선순위|작업 항목|
|---|---|
|⭐️⭐️⭐️|`fetchBudgetModel()` 제거, `fetchBudgetConfig()` 중심으로 통일|
|⭐️⭐️⭐️|`createBudgetConfig` → `save(budgetModel:)`에 통합|
|⭐️⭐️|`setValue(_:forKey:)` 제거 및 속성 접근으로 전환|
|⭐️⭐️|Debug용 `print` 정리|
|⭐️|`insertOrUpdateFixedCost` 도입 (선택 사항)|
