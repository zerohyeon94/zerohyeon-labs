ㄷ## ✅ 1. SpendingRecord와 DailyBudget 관계 설정

### 📦 CoreData 관계 구성

- **DailyBudget → SpendingRecord**: **1:N 관계**
    
- 즉, 하루에 여러 개의 소비 기록이 연결됨

|Entity|관계|
|---|---|
|DailyBudget|`spendingRecords: NSSet` ← To-Many (inverse: `dailyBudget`)|
|SpendingRecord|`dailyBudget: DailyBudget?` ← To-One|

### 🌱 이 구조의 장점

- 어떤 날짜의 소비 내역을 **`.spendingRecords`** 로 바로 접근 가능
    
- **오늘 총 소비 금액 = spendingRecords의 amount 합계**로 계산 가능
    
- DailyBudget 화면에서 소비 리스트 바로 바인딩 가능

## ✅ 2. carryOverAmount 상세 출처 추적

지금처럼 단순히 `carryOverAmount = 어제 절약한 금액`으로만 저장하는 방식은  
✅ 오늘 총 금액 계산에는 충분하지만  
❌ “어느 날에 얼마나 절약했는가”라는 정보를 제공할 수 없습니다.

그래서 다음과 같은 방식으로 개선할 수 있습니다:

---

### 📦 CoreData Entity: `CarryOverSource`

| 속성       | 설명                                   |
| -------- | ------------------------------------ |
| `date`   | 절약했던 날짜                              |
| `amount` | 해당 날짜에서 이월된 금액                       |
| `toDate` | 이 금액이 이월된 대상 날짜 (예: 5월 27일)          |
| 관계       | `targetBudget: DailyBudget` ← To-One |

---

### 🔗 DailyBudget 연결

- DailyBudget ← `carryOverSources: NSSet` (to-many)
    
- carryOverAmount = `carryOverSources.map { $0.amount }.reduce(0, +)`

---

### 🌿 예시

**5월 27일 DailyBudget**

|fromDate|amount|
|---|---|
|5월 25일|3000|
|5월 26일|2000|

➡️ 총 carryOverAmount = 5,000  
➡️ 자세히 보기 클릭 시 위 데이터를 리스트로 노출

---

## ✅ 정리된 모델 구조

### 🧩 DailyBudget

- `date: Date`
    
- `availableAmount: Int`
    
- `spentAmount: Int`
    
- `carryOverSources: [CarryOverSource]`
    
- `spendingRecords: [SpendingRecord]`

### 🧩 SpendingRecord

- `title: String`
    
- `amount: Int`
    
- `date: Date`
    
- `dailyBudget: DailyBudget?` ← inverse

### 🧩 CarryOverSource

- `date: Date`
    
- `amount: Int`
    
- `toDate: Date` (또는 `targetBudget: DailyBudget`)

---

## ✅ 결론

|기능|구현 방식|
|---|---|
|오늘 소비 기록 조회|`DailyBudget.spendingRecords`|
|오늘 이월된 금액 계산|`carryOverSources.map { $0.amount }.reduce(0, +)`|
|이월 상세 보기|`carryOverSources`를 날짜순 정렬해서 보여줌|
