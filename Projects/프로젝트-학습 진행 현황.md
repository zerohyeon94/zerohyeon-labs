## 정리 기준

- 프로젝트 목표를 먼저 본다.
- 학습 주제는 프로젝트를 진행하기 위해 필요한 것만 우선 배치한다.
- 평일에는 준비/확인/정리 중심으로 작게 배치하고, 토요일/일요일에 프로젝트 산출물을 집중한다.
- 매주 월요일에는 이 노트를 보고 이번 주 핵심 목표 1개, 보조 목표 2-3개, 선택 목표 1개를 정한다.
- 일요일에는 이 노트를 기준으로 완료/이월/보류를 판단한다.

서비스 이름은 **ZipJoong**으로 변경되었다. Obsidian 폴더는 기존 링크 보존을 위해 일단 `Projects/FocusOps/`를 유지하되, 내용에서는 `ZipJoong (구 FocusOps)`로 표기한다.

Notion 기준:
- [FocusOps: 온디바이스 집중도 분석 관제 플랫폼](https://app.notion.com/p/375e5d4a0bac8194a944c8146c26b6ac)
- Notion의 기존 프로젝트명은 FocusOps지만, 현재 iOS 서비스명은 ZipJoong으로 본다.

현재 상태:
- 아직 실제 프로젝트 진행 전이다.
- 목표는 Go와 React로 서버/대시보드를 배포해보고 실제 동작과 구현을 확인하는 것이다.
- 처음부터 iOS, AI, Go, React, PostgreSQL, Docker를 모두 만들지 않고 작은 실행 단위로 시작한다.

권장 첫 범위:
1. Go REST API
2. PostgreSQL
3. React 대시보드
4. Docker Compose 로컬 실행
5. 이후 iOS 앱 연동

필요 학습:
- Go REST API 구조
- PostgreSQL 테이블 설계
- React + TypeScript API 연동
- Docker Compose 배포 흐름
- iOS 앱에서 서버 API 호출

## Daily 반영 방식

- Daily 생성 시 이 노트를 먼저 보고, 오늘 항목이 어느 프로젝트/학습 축에 연결되는지 표시한다.
- 하루업과 Jday는 2026-07 전까지 `최우선 프로젝트`로 둔다.
- ZipJoong은 아직 시작 전이므로, 당장 구현 과제로 크게 열지 않고 Go/React/DB 학습과 작은 실행 계획으로 연결한다.
- Go 멘토링은 하루업이 아니라 ZipJoong 서버 구현과 배포 실험에 연결한다.
- React 학습은 하루업 연결이 아니라 ZipJoong 대시보드 연결로 배치한다.
- iOS/Swift는 새 멘토링 파일을 계속 늘리기보다 프로젝트 수정/재배포/복습 파일 중심으로 둔다.
