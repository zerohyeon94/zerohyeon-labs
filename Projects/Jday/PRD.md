# 📄 Jday PRD — 화면별 기능 명세서

> PRD는 개발자가 이 문서만 보고 개발할 수 있어야 한다.
> 화면 목적 → UI 컴포넌트 → 동작 명세 → 예외 처리 순으로 작성.

---

## PRD-01 · 홈 화면

**화면 목적**
출근 직후 어제 업무 요약과 오늘 할 일을 한눈에 파악하고, 오늘 일정을 확인한다.

| 컴포넌트 | 동작 | 예외/엣지 케이스 |
|----------|------|-----------------|
| 상단 날짜/요일 | 오늘 날짜 자동 표시 | 날짜 변경 시 자정에 자동 갱신 |
| 숫자 카드 - 할 일 남음 | 미완료 DailyTask 개수 표시 | 0개면 "모두 완료! 🎉" 표시 |
| 숫자 카드 - 오늘 일정 | 오늘 Schedule 개수 표시 | 0개면 "일정 없음" 표시 |
| 숫자 카드 - 진행률 | 완료 할 일 / 전체 할 일 % | 할 일 0개면 카드 숨김 |
| 어제 미완료 섹션 | 전날 미완료 DailyTask 목록 | 없으면 섹션 전체 숨김 |
| 오늘로 옮기기 (개별) | 해당 Task의 date를 오늘로 변경 | — |
| 모두 오늘로 옮기기 버튼 | 어제 미완료 전체 date를 오늘로 변경 | 확인 Alert 없이 즉시 실행 |
| 오늘 할 일 체크리스트 | DailyTask 완료 토글 | 완료 시 completedAt 기록 + 취소선 + 하단 이동 |
| 오늘 할 일 스와이프 삭제 | 할 일 row 좌우 스와이프 시 삭제 액션 노출 | 삭제 후 Undo 토스트 제공, 완료 기록도 함께 제거 |
| 오늘 일정 타임라인 | 오늘 Schedule 시간순 정렬 | 지난 일정은 흐리게(opacity 0.4) 처리 |

**데이터 연결**
- `DailyTask` where date == today → 오늘 할 일
- `DailyTask` where date == yesterday AND isDone == false → 어제 미완료
- `Schedule` where startTime/endTime range overlaps today → 오늘 일정

**플랫폼별 차이**
- iOS: 세로 스크롤 단일 컬럼
- macOS: 왼쪽 메인(어제 미완료 + 오늘 할 일) + 오른쪽 패널(오늘 일정)

## PRD-02 · 빠른 추가 (바텀시트 / 팝오버)

**화면 목적**
현재 화면을 유지하면서 할 일 / 일정 / 이슈를 빠르게 입력한다.

| 컴포넌트 | 동작 | 예외/엣지 케이스 |
|----------|------|-----------------|
| 입력 유형 탭 (할 일/일정/이슈) | 탭 전환 시 입력 폼 변경 | 입력 중 탭 전환 시 입력값 초기화 + Alert 확인 |
| 할 일 - 제목 입력 | 텍스트 입력 | 빈 값이면 저장 버튼 비활성화 |
| 할 일 - 날짜 | 기본값: 오늘, 탭 시 DatePicker | — |
| 할 일 - 우선순위 | 낮음/중간/높음 선택, 기본값: 중간 | — |
| 일정 - 시작/종료 시간 | DatePicker, 기본값: 현재 시간 | 종료 < 시작이면 저장 버튼 비활성화 |
| 일정 - 장소 | 선택 입력 | — |
| 이슈 - 상세 내용 | 선택 입력, 멀티라인 | — |
| 이슈 - 알림 시간 | 30분 전 기본값, 변경 가능 | 과거 시간 설정 시 경고 표시 |
| 저장 버튼 | SwiftData에 저장 + CloudKit 동기화 트리거 | 저장 실패 시 Toast 에러 메시지 |

**플랫폼별 차이**
- iOS: 하단 바텀시트 (FAB 버튼 탭)
- macOS: 툴바 + 빠른 추가 버튼 클릭 시 팝오버

## PRD-03 · 캘린더 화면

**화면 목적**
월별/주간 뷰로 전체 일정을 조망하고, 날짜를 선택해 해당 일의 일정과 할 일을 확인한다.

| 컴포넌트 | 동작 | 예외/엣지 케이스 |
|----------|------|-----------------|
| 주간/월간 세그먼트 | 뷰 전환 | 선택 날짜 유지 |
| 월별 캘린더 그리드 | 날짜 탭 시 하단 리스트 갱신 | 오늘 날짜 파란 원 강조 |
| 날짜 이벤트 도트 | Schedule/DailyTask 존재 시 도트 표시 | 최대 3개 도트, 초과 시 ... 표시 |
| 기간 일정 바/칩 | Schedule startTime~endTime 범위가 겹치는 날짜에 표시 | 여러 날 일정은 시작~종료 날짜를 이어서 표시 |
| 이전/다음 월 이동 | 스와이프 or 화살표 버튼 | — |
| 선택일 일정 리스트 | 선택 날짜와 겹치는 Schedule 시간순 | 없으면 "일정 없음" 빈 상태 |
| 선택일 할 일 리스트 | 해당 날짜 DailyTask | 완료 항목 하단 정렬 |
| 완료 기록 타임라인 | completedAt이 선택 날짜에 포함되는 DailyTask 시간순 표시 | completedAt이 없으면 표시하지 않음 |

**플랫폼별 차이**
- iOS: 상단 캘린더 + 하단 리스트 (세로 레이아웃)
- macOS: 중앙 캘린더 그리드 + 우측 패널(일정+할 일)

## PRD-04 · 이슈 목록 화면

**화면 목적**
등록된 이슈를 한눈에 보고 상태별로 필터링한다.

| 컴포넌트 | 동작 | 예외/엣지 케이스 |
|----------|------|-----------------|
| 상단 미해결 카운트 | 미해결 Issue 개수 표시 | 0이면 "모든 이슈 해결됨 ✅" 표시 |
| 필터 탭 (전체/미해결/해결완료) | 탭 전환 시 리스트 필터링 | 해당 탭에 항목 없으면 빈 상태 |
| 이슈 카드 | 제목 + 등록일 + 상태 뱃지 | 미해결: 파란 뱃지 / 해결완료: 회색 뱃지 |
| 이슈 카드 탭 | 이슈 상세 화면으로 이동 | macOS는 우측 패널에 표시 |
| 스와이프 삭제 (iOS) | 왼쪽 스와이프 시 삭제 버튼 노출 | 삭제 확인 Alert 필요 |

## PRD-05 · 이슈 상세 화면

**화면 목적**
이슈 내용을 확인하고 해결 완료 처리한다.

| 컴포넌트 | 동작 | 예외/엣지 케이스 |
|----------|------|-----------------|
| 상태 뱃지 | 미해결/해결완료 표시 | 해결완료 시 뱃지 색상 회색으로 변경 |
| 이슈 제목 | 표시 전용 | — |
| 상세 내용 카드 | 표시 전용 | 내용 없으면 카드 숨김 |
| 메타 정보 | 등록일 / 알림 시간 / 상태 | — |
| 해결완료로 표시 버튼 | Issue.isResolved = true 저장 | 이미 해결완료면 버튼 숨김 |
| 알림 | 설정된 notifyAt 시간에 로컬 알림 발송 | 해결완료 시 예약 알림 자동 취소 |

## PRD-06 · 설정 화면 (macOS)

**화면 목적**
앱 동작 방식을 사용자가 커스터마이징한다.

| 설정 항목 | 기본값 | 동작 |
|-----------|--------|------|
| 푸시 알림 | ON | 이슈·일정 리마인더 활성화/비활성화 |
| 이메일 요약 | OFF | 매일 아침 요약 이메일 (2단계 기능) |
| 기본 알림 시간 | 30분 전 | 이슈 알림 기본값 변경 |
| 시작 화면 | 홈 | 앱 실행 시 첫 화면 선택 |
| 한 주 시작 요일 | 일요일 | 캘린더 그리드 시작 요일 |
| 완료 항목 자동 숨김 | ON | 체크 후 24시간 뒤 자동 숨김 |
| 프로필 관리 | — | iCloud 계정 정보 표시 |
| 로그아웃 | — | CloudKit 동기화 해제 |

## 공통 명세

**CloudKit 동기화 정책**
- 모든 CRUD 작업은 SwiftData 로컬 저장 후 CloudKit 자동 동기화
- 오프라인 시 로컬 저장 → 온라인 복귀 시 자동 sync
- 충돌 발생 시 최신 `updatedAt` 기준으로 덮어쓰기

**알림 정책**
- 이슈 알림: `notifyAt` 시간에 UserNotifications 로컬 알림
- 일정 알림: `startTime` 30분 전 로컬 알림 (설정에서 변경 가능)
- 해결완료 이슈: 예약 알림 즉시 취소
- 앱 권한 미허용 시: 설정 화면에서 권한 요청 배너 표시

**빈 상태 (Empty State) 정책**
- 어제 미완료 없음 → 섹션 숨김
- 오늘 할 일 없음 → "오늘 할 일을 추가해보세요 +" 버튼
- 오늘 일정 없음 → "일정 없음" 텍스트
- 이슈 없음 → "등록된 이슈가 없어요 ✅"
