# 🚀 Jday — App Store 업로드 체크리스트

> SwiftUI Multiplatform (iOS + macOS) 기준
> 번들 ID: com.j.jday | 사용 기능: SwiftData, CloudKit, UserNotifications

---

## 📦 1. 빌드 준비 (Xcode)

### 버전 관리
- [ ] `CFBundleShortVersionString` (마케팅 버전, e.g. 1.0.0) 확인
- [ ] `CFBundleVersion` (빌드 번호, e.g. 1) 이전 빌드보다 높은지 확인
- [ ] iOS target과 macOS target 모두 버전 일치 확인

### 코드 서명 & Capabilities
- [ ] Signing & Capabilities → Team 선택 확인
- [ ] Bundle Identifier `com.j.jday` 설정 확인
- [ ] **CloudKit** 기능 활성화 확인 (iCloud Container)
- [ ] **Push Notifications** 기능 활성화 확인 (CloudKit 원격 알림)
- [ ] **UserNotifications** 권한 선언 확인 (`NSUserNotificationUsageDescription`)
- [ ] Provisioning Profile: Distribution (App Store) 선택

### 빌드 설정
- [ ] Scheme → Release 모드로 빌드
- [ ] `DEBUG` 플래그 제거 확인 (`#if DEBUG` 블록 내 개발용 코드 확인)
- [ ] 하드코딩된 테스트 데이터 없는지 확인
- [ ] 실제 기기(iOS + macOS)에서 최종 실행 확인

## 📋 2. App Store Connect 메타데이터

### 앱 정보
- [ ] 앱 이름: `Jday` (12자 이내)
- [ ] 부제목 (선택): e.g. `하루 업무를 하나로`
- [ ] 카테고리: `생산성` (주), 보조 카테고리 선택
- [ ] 연령 등급: 설문 완료 후 등급 확인 (일반적으로 4+)

### 설명 & 키워드
- [ ] 앱 설명 (4000자 이내) 한/영 작성
- [ ] 홍보 텍스트 (170자 이내) — "새로운 소식" 위치
- [ ] 키워드 (100자 이내): `할일,업무관리,일정,이슈,생산성,다이어리` 등
- [ ] 지원 URL 입력 (GitHub Pages or 랜딩 페이지)
- [ ] 개인정보 처리방침 URL 입력 **(필수 — CloudKit 사용 시 의무)**

### 스크린샷 (필수 해상도)

**iOS**
- [ ] 6.9인치 (iPhone 16 Pro Max): 1320 × 2868px — **최소 1장, 최대 10장**
- [ ] 6.5인치 (iPhone 14 Plus): 1242 × 2688px (6.9인치로 대체 가능)
- [ ] 5.5인치 (iPhone 8 Plus): 1242 × 2208px

**macOS**
- [ ] macOS 스크린샷: 2560 × 1600px (또는 1280 × 800px)

**선택 (권장)**
- [ ] App Preview 동영상 (15~30초, 선택 사항)

## 🔒 3. 개인정보 & 법적 요구사항

### App Privacy (필수)
- [ ] App Store Connect → 앱 개인정보 보호 항목 작성
  - 수집 데이터: `CloudKit 사용자 식별자` (CloudKit 사용 시)
  - 로컬 전용 데이터: 수집 안 함 체크
  - 제3자 연동: 없음 (초기 버전)

### 개인정보 처리방침
- [ ] 앱에서 수집하는 데이터 명시 (CloudKit으로 동기화되는 데이터)
- [ ] 데이터 보관 기간, 삭제 방법 명시
- [ ] 웹에 업로드 후 URL 확보

### 권한 사용 설명 (`Info.plist`)
- [ ] `NSUserNotificationUsageDescription` — "이슈 및 일정 알림을 보내기 위해 사용합니다."
- [ ] `NSUbiquitousContainers` (CloudKit) — 컨테이너 설정 확인

## ☁️ 4. CloudKit 배포 설정 (중요)

> 개발 중 테스트는 Development 환경, 배포 시 Production으로 전환 필요

- [ ] CloudKit Dashboard → 컨테이너 Production 환경으로 배포(Deploy to Production)
- [ ] Production 환경에서 스키마 확인 (DailyTask, Schedule, Issue 레코드 타입)
- [ ] CloudKit 동기화 오프라인 → 온라인 전환 시나리오 실제 기기 테스트
- [ ] iCloud 미로그인 상태에서 앱 동작 확인 (로컬 전용 폴백 처리)

## 🧪 5. TestFlight 베타 테스트 (권장)

- [ ] 어제 미완료 → "모두 오늘로 옮기기" 동작 확인
- [ ] 앱 강제 종료 후 데이터 유지 확인

## 📤 6. 심사 제출

### 제출 전 최종 확인
- [ ] Xcode → Product → Archive → Validate App (심사 전 유효성 검사)
- [ ] Validate App 경고/오류 없음 확인
- [ ] Xcode → Distribute App → App Store Connect 업로드

### App Store Connect 제출 설정
- [ ] 빌드 선택 (TestFlight 검증 완료 빌드 권장)
- [ ] "수출 규정 준수" 질문 응답 (암호화 사용 여부 — CloudKit은 Apple 관리이므로 일반적으로 No)
- [ ] 심사 노트 (선택): 테스터 계정이 필요한 경우 기재 — iCloud 로그인 필요 명시
- [ ] 출시 방식 선택: 수동 출시 (심사 통과 후 직접 게시 권장)
