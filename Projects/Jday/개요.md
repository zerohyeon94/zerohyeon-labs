# 📋 Jday — 나만의 업무 관리 앱

> **한 줄 소개:** J의 하루를 흐름있게 — 할 일, 일정, 이슈를 하나로
> **타겟 유저:** 매일 업무 계획을 세우고 실행 흐름을 관리하고 싶은 직장인·개발자
> **핵심 가치:** 출근 시 계획 → 업무 중 실행 → 퇴근 후 회고까지 하나의 앱에서

---

## 앱 이름

- **Jday** — J(성 이니셜 + MBTI J Judging 감성) + day(하루 단위 업무 관리)
- **발음:** 제이데이 (직관적, 기억하기 쉬움)
- **번들 ID:** com.j.jday

---

## 1. 문제 정의

**핵심 페인포인트**
- 노션(업무) + 캘린더(일정) + 미리알림(할 일) 3개 앱을 매일 왔다갔다 해야 함
- 앱 전환마다 흐름이 끊겨 집중력과 시간이 낭비됨

| 타이밍 | 현재 행동 | 문제점 |
|--------|-----------|--------|
| 🌅 아침 출근 직후 | 노션 → 어제 업무 확인 → 캘린더 → 이슈 → 캘린더 재진입 | 3번 앱 전환으로 업무 파악에 시간 소요 |
| ⚡ 업무 중간중간 | 할 일·일정 생길 때마다 각 앱으로 이동해서 입력 | 업무 흐름 중단, 입력 누락 위험 |

**핵심 유저 니즈**
1. 출근 직후 → 어제 업무 요약 + 오늘 할 일을 한 화면에서 확인
2. 업무 중 → 오늘 일정 + 할 일 추가를 앱 전환 없이 즉시 입력

---

## 2. 핵심 기능

**[Must-Have] MVP**
- 어제 업무 요약 + 오늘 할 일 통합 뷰 ← 핵심 니즈 #1
- 오늘 일정 + 할 일 빠른 추가 (한 화면에서) ← 핵심 니즈 #2
- 이슈 사항 입력 + 로컬 알림 (UserNotifications)
- iCloud 동기화 (CloudKit) — 맥 ↔ 아이폰 실시간 반영

**[Nice-to-Have] 추후 확장**
- 맥 메뉴바 위젯 (오늘 업무 빠른 확인)
- 배운 점 / 회고 기록 기능
- Claude API 연동 — 하루 업무 자동 분석 및 다음날 제안

## 3. 사용자 시나리오

|--------|------|------|

## 4. 유저 플로우

**앱 실행 시 컨텍스트 어웨어 분기**
- 월요일 or 공휴일 다음날 → 월별 캘린더 홈 (주간 계획 모드)
- 그 외 평일 → 홈 대시보드 (어제 요약 + 오늘 할 일)

**하단 탭바 구성 (5탭)**

| 탭 | 화면 구성 |
|----|-----------|
| 🏠 홈 | 숫자 요약 카드 → 어제 미완료 → 오늘 할 일 → 오늘 일정 타임라인 |
| 📅 캘린더 | 월별 뷰 → 날짜 선택 → 일간/주간 뷰 전환 |
| ⚡ 빠른 추가 (FAB) | 바텀시트 → 할 일 / 일정 / 이슈 선택 → 저장 |
| ⚠️ 이슈 | 이슈 목록 → 상세 + 알림 설정 → 해결 완료 처리 |
| ⚙️ 설정 | 알림 설정 → 테마/표시 환경설정 |

**핵심 UX 결정 사항**
- 요일/날짜 기반 첫 화면 자동 전환 (Context-Aware UX)
- 가운데 탭은 항상 빠른 추가 FAB 버튼으로 고정
- 빠른 추가는 바텀시트로 열려 현재 화면 유지하며 입력

## 6. 기술 스펙

| 항목 | 선택 | 이유 |
|------|------|------|
| Platform | SwiftUI Multiplatform | 맥 + iOS 동시 지원 |
| Architecture | MVVM | SwiftUI 표준 패턴 |
| 로컬 저장 | SwiftData | CoreData 대체 최신 트렌드 |
| 동기화 | CloudKit (iCloud) | 무료, 서버 불필요 |
| 알림 | UserNotifications | iOS/macOS 공통 |
| 맥 전용 | MenuBar (WidgetKit) | 상단 메뉴바 빠른 확인 |

**플랫폼별 UI 분기 전략**

| iOS | macOS |
|-----|-------|
| 하단 탭바 | 왼쪽 NavigationSplitView 사이드바 |
| 전체화면 뷰 | 사이드바 + 메인 2단 레이아웃 |
| 바텀시트 추가 | 오른쪽 패널 or 팝오버 |
| FAB 버튼 | 툴바 버튼 |

**데이터 모델**
- `DailyTask` — 제목, 완료 여부, 날짜, 우선순위(높음/중간/낮음), 완료 시각(`completedAt`)
- `Schedule` — 제목, 시작/종료 시간, 장소, 기간 표시 기준
- `Issue` — 제목, 상세 내용, 생성일, 해결 여부, 알림 시간

**비용 구조**
- iCloud / CloudKit: 무료 (Apple 제공)
- 서버 운영: 불필요
- Apple Developer 계정: 연 $99 (App Store 배포 시만 필요)

## 7. 개발 로드맵

**1단계 — MVP**
- SwiftUI Multiplatform 프로젝트 생성
- SwiftData 모델 3개 구현 (DailyTask / Schedule / Issue)
- 홈 화면: 어제 요약 + 오늘 할 일 통합 뷰
- CloudKit 동기화 연결

**2단계 — 기능 확장**
- 월별 캘린더 화면
- 주간 뷰 추가
- 이슈 알림 (UserNotifications)

**3단계 — 고도화**
- 맥 메뉴바 위젯 (WidgetKit)
- 배운 점 / 회고 기록
- Claude API 연동 — 일일 업무 분석 자동화
