### Index(인덱스)는 무엇인가요?

**정답: 데이터베이스의 '검색 속도'를 비약적으로 높여주는 '색인(목차)'입니다.**

- **개념:** 두꺼운 전공서적에서 '스위프트'라는 단어를 찾으려고 할 때, 1페이지부터 1000페이지까지 전부 넘겨보는 것(Full Scan)은 너무 비효율적이죠? 그래서 책 맨 뒤에 있는 **'찾아보기(색인)'**를 펼쳐 'ㅅ' 부분으로 바로 갑니다. 데이터베이스의 인덱스도 이와 똑같습니다.
    
- **장단점 (트레이드오프):** 읽기(조회) 속도는 엄청나게 빨라집니다. 하지만 책에 새로운 내용을 추가(Insert)하거나 수정(Update)할 때마다 맨 뒤의 '찾아보기' 페이지도 계속 고쳐 써야 하므로, **쓰기/수정 속도는 오히려 약간 느려지고 저장 공간(메모리)을 추가로 차지**하게 됩니다.

**(면접용 팁):** "인덱스는 데이터베이스에서 원하는 데이터를 빠르게 찾기 위해 사용하는 색인입니다. 책의 맨 뒤에 있는 찾아보기와 같은 역할을 합니다. 검색(SELECT) 속도를 크게 향상시켜 주지만, 데이터가 추가되거나 수정될 때 인덱스도 함께 업데이트해야 하므로 쓰기 작업 시 성능 저하가 발생할 수 있다는 트레이드오프를 고려하여 신중하게 설정해야 합니다."

### ORM은 무엇인가요?

**정답: 객체 지향 언어(파이썬, 스위프트)와 데이터베이스(SQL) 사이의 '자동 통역사'입니다.**

- **개념:** ORM은 **O**bject-**R**elational **M**apping의 약자입니다. 파이썬이나 스위프트는 데이터를 '객체(Object/Class)'로 다루지만, 데이터베이스는 데이터를 엑셀 같은 '표(Table)'로 다룹니다. 둘의 언어가 완전히 다른 것이죠.
    
- **왜 쓸까요?:** ORM이 없다면 백엔드 개발자는 파이썬 코드 안에 `SELECT * FROM users WHERE id = 1` 같은 투박한 SQL 쿼리문(문자열)을 직접 적어야 합니다. 오타가 나도 실행 전엔 모릅니다. 하지만 ORM을 쓰면 `User.query.get(1)` 처럼 **우리가 늘 쓰던 익숙하고 안전한 객체 지향 코드로 DB를 조작**할 수 있게 해 줍니다.

**(면접용 팁):** "ORM은 객체 지향 프로그래밍의 '객체'와 관계형 데이터베이스의 '테이블'을 자동으로 매핑(연결)해 주는 기술입니다. 복잡한 SQL 쿼리문을 직접 작성하지 않고도 파이썬이나 자바 같은 언어의 코드로 데이터베이스를 다룰 수 있게 해 주어, 개발 생산성을 높이고 코드의 가독성을 크게 향상시켜 줍니다."

### 4. MySQL은 ORM인가요?

**정답: 아닙니다! MySQL은 데이터를 저장하는 '창고(DB)' 자체이고, ORM은 그 창고를 부리는 '도구'입니다.**

면접관이 지원자가 ORM과 DB의 개념을 정확히 분리해서 이해하고 있는지 찔러보기 위해 자주 던지는 함정 질문입니다.

- **MySQL (RDBMS):** 데이터를 실제로 영구 보관하는 물리적인 **관계형 데이터베이스 시스템** 그 자체입니다. (PostgreSQL, Oracle 등과 같은 부류입니다.)
    
- **ORM:** 이 MySQL에 접근해서 데이터를 넣고 빼기 위해, 파이썬이나 서버 코드 내에서 사용하는 **라이브러리(통역기)**입니다. (파이썬의 SQLAlchemy, Node.js의 Prisma 등이 ORM입니다.)

**(면접용 팁):** "아닙니다. MySQL은 데이터를 실제로 저장하고 관리하는 '관계형 데이터베이스 시스템(RDBMS)' 그 자체입니다. 반면 ORM은 백엔드 코드에서 이 MySQL과 원활하게 소통하기 위해 사용하는 '중간 번역 도구 혹은 라이브러리'입니다. 즉, ORM이라는 도구를 사용해서 MySQL이라는 창고를 조작하는 관계입니다."
