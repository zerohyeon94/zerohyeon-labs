### 시각화를 하는 방법은 무엇인가요?

**정답: 복잡한 데이터 숫자들을 사람이 한눈에 이해할 수 있도록 '그래프나 차트'로 그리는 작업입니다.**

- **개념:** 사용자가 집중한 시간 데이터를 `[45, 12, 120, 30]`처럼 숫자 배열로만 보여주면 추세를 알기 어렵습니다. 이것을 꺾은선 그래프나 막대그래프로 변환하여 시각적 인사이트를 제공하는 과정입니다.
    
- **영역별 시각화 도구 (이렇게 답변하시면 가산점을 받습니다)**
    
    - **AI 및 파이썬 백엔드:** 데이터 사이언티스트들은 주로 파이썬의 `Matplotlib`이나 `Seaborn` 라이브러리를 사용해 데이터를 분석하고 시각화합니다.
        
    - **프론트엔드 (웹):** React 같은 웹 환경에서는 `Chart.js`나 `D3.js`를 사용해 화려하고 반응형인 차트를 그립니다.
        
    - **모바일 (iOS):** 스위프트 환경에서는 애플이 기본 제공하는 `Swift Charts` 프레임워크를 사용하여 네이티브하고 부드러운 그래프를 구현할 수 있습니다.

**(면접용 팁)** "시각화는 원시 데이터를 그래프나 차트로 변환하여 사용자가 직관적으로 인사이트를 얻게 하는 과정입니다. 집중 타이머 앱에서 주간 집중 시간을 막대그래프로 보여주는 것이 그 예입니다. 백엔드(파이썬)에서는 Matplotlib을, 프론트엔드나 모바일 화면에서는 Chart.js나 Swift Charts 같은 라이브러리를 목적에 맞게 활용하여 구현합니다."

### Message Broker, Redis Stream, Pub/Sub, SSE를 활용한 비동기 분산 아키텍처

#### 1. Message Broker (메시지 브로커)

- **역할:** 카운터(FastAPI)와 주방장(AI 일꾼) 사이에서 주문을 안전하게 전달하는 '우체국'입니다. Redis, RabbitMQ, Kafka 등이 이 역할을 합니다. 서버 간에 직접 통신하지 않고 브로커를 거치게 해서, 한쪽 서버가 죽어도 전체 시스템이 뻗지 않게(디커플링) 막아줍니다.

#### 2. Redis의 두 가지 통신 방식 (Pub/Sub vs Stream)

브로커(Redis)가 메시지를 전달하는 방식에는 크게 두 가지가 있습니다.

- **Pub/Sub (라디오 생방송):** 발행자(Publisher)가 메시지를 던지면, 지금 당장 접속해 있는 구독자(Subscriber)들에게 실시간으로 쫙 뿌립니다(Broadcast). 엄청 빠르지만, 만약 주방장이 잠깐 화장실에 가서 접속이 끊겨있었다면 그 주문은 영영 날아가 버립니다. (휘발성)
    
- **Stream (유튜브 영상 / 로그북):** 주문 내역을 장부에 차곡차곡 기록해 둡니다. 주방장이 화장실에 다녀와도 장부를 보고 "아까 못 본 주문부터 다시 처리해야지"가 가능합니다. 중요한 AI 분석 요청이 절대 유실되지 않도록 보장하는 아주 튼튼한 구조입니다. 실무에서는 무거운 AI 연산을 넘길 때 Pub/Sub보다 Stream이나 Celery 큐를 더 선호합니다.

#### 3. SSE Response (Server-Sent Events)

- **역할:** 모바일 앱이 서버에 "다 됐어?"라고 3초마다 계속 물어보는 것(Polling)은 비효율적입니다. SSE는 프론트엔드와 서버 사이에 '단방향 파이프'를 연결해 두는 기술입니다.
    
- **작동 원리:** 아이폰은 처음에 연결만 딱 해두고 가만히 기다립니다. AI 모델의 분석이 끝나면, 백엔드가 이 파이프를 통해 아이폰으로 "1단계 완료", "분석 100% 완료!"라는 알림을 **알아서 밀어내 줍니다(Push).**

**(면접용 팁)** "대규모 AI 서빙 아키텍처에서 시스템의 안정성을 위해 Message Broker를 도입합니다. 중요한 작업이 유실되지 않도록 단순한 Pub/Sub보다는 내구성이 있는 Redis Stream(또는 큐)을 활용해 비동기 워커에 작업을 넘깁니다. 그리고 클라이언트가 결과를 받기 위해 무의미하게 Polling하는 리소스 낭비를 막기 위해, 서버에서 분석이 끝나는 즉시 클라이언트로 이벤트를 밀어주는 SSE(Server-Sent Events) 방식을 사용하여 효율적인 비동기 분산 아키텍처를 구성할 수 있습니다."
