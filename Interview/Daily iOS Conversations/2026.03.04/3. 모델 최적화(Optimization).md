**🗣️ 가볍게 시작하기 (Simple)**

서버용 GPU(엔비디아 H100 등)는 메모리가 엄청나게 커서 모델이 아무리 커도 상관없습니다. 하지만 우리 아이폰은 작고 소중하죠.

`cto` (CoreML Tools Optimizer)는 모델의 성능(정확도)은 거의 유지하면서, **모델의 크기를 1/2, 1/4, 심지어 1/8까지 줄여주는 압축 기술 모음**입니다.

---

**🤿 딥하게 들어가기 (Deep: Quantization - 양자화)**

이 모듈의 핵심 기능은 **양자화(Quantization)**입니다. 펌웨어 하셨으니 **데이터 타입** 이야기로 설명하면 바로 이해되실 겁니다.

1. **원본 (Float32):**
    
    - 보통 딥러닝 모델의 가중치(Weight, 신경망의 연결 강도)는 **32비트 부동소수점(`Float32`)**으로 저장됩니다.
        
    - 정밀하지만 용량을 많이 차지합니다. (4 bytes)
        
    - `0.12345678` 처럼 아주 디테일하죠.
        
2. **최적화 후 (Int8):**
    
    - `cto`를 사용하면 이걸 **8비트 정수(`Int8`)**로 변환해 버립니다.
        
    - 용량이 4분의 1로 줄어듭니다. (1 byte)
        
    - `0.12` 정도로 퉁쳐서 저장합니다.

**🤔 "정확도가 떨어지지 않을까요?"** 당연히 정밀도는 떨어집니다. 하지만 딥러닝 모델은 의외로 둔감해서, 소수점 아래 7자리까지 정확할 필요가 없는 경우가 많습니다. 약간 뭉뚱그려도 결과(개인지 고양이인지)는 똑같이 잘 맞춥니다.

**🚀 효과:**

- **앱 용량 감소:** 100MB짜리 모델이 **25MB**가 됩니다. (사용자가 좋아합니다)
    
- **속도 향상:** 데이터 전송량이 적으니 ANE(뉴럴 엔진)가 훨씬 빠르게 계산합니다.
    
- **배터리 절약:** 메모리를 적게 읽으니까 전기를 덜 먹습니다.

---

**💻 실전 코드 (어떻게 쓰나요?)**

방금 보신 변환 코드 뒤에 딱 한 단계만 더 추가하면 됩니다.

```Python
import coremltools as ct
import coremltools.optimize.coreml as cto # 👈 주인공 등장

# 1. 먼저 일반적인 변환을 합니다 (Float32 상태)
mlmodel = ct.convert(torch_model, inputs=[...])

# 2. 최적화 설정 (가중치를 8비트 선형 양자화로 압축해라!)
op_config = cto.OpLinearQuantizerConfig(
    mode="linear_symmetric",
    weight_threshold=512
)

# 3. 모델 압축 실행 (다이어트 시작!)
compressed_model = cto.linear_quantize_weights(mlmodel, config=op_config)

# 4. 저장 (용량이 확 줄어든 파일이 나옵니다)
compressed_model.save("FocusSense_Int8.mlpackage")
```
