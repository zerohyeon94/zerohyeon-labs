# **Q.** Swift가 함수형이라고?  

### **A.** Swift는 다중 패러다임 언어지만, **순수 함수·불변성·고차함수(map/filter/reduce)** 같은 함수형 도구를 지원해 **버그를 줄이고 테스트가 쉬운 코드**를 쓰게 해준다.

```swift
// Imperative - 명령
var evens: [Int] = []
for n in 1...10 { if n % 2 == 0 { evens.append(n * n) } } // [4, 16, 36, 64, 100]

// Functional - 함수
let evens = (1...10).filter { $0.isMultiple(of: 2) }.map { $0 * $0 } // [4, 16, 36, 64, 100]
```

### 다중 패러다임 (multi-paradigm)
- 한 언어가 여러가지 프로그래밍 스타일을 동시에 잘 지원하고, 필요에 따라 섞어 쓸 수 있게 해준다.

- 주요 패러다임
	- **명령형 / 절차적**: `var`, `for`, `if`, `guard`로 “어떻게”를 단계적으로 기술
	
	- **객체지향(OOP)**: `class`/상속/다형성으로 객체 간 협력 모델링
    
	- **프로토콜 지향(POP)**: `protocol` + `extension` + 값 타입 중심 합성(스위프트의 시그니처 강점)
    
	- **함수형(FP)**: 불변성 지향(`let`·값 타입·COW), 고차함수 `map/filter/reduce`
    
	- **제네릭/추상화**: `generic`, `associatedtype`으로 타입 안전한 재사용
    
	- **선언형(Declarative)**: SwiftUI(뷰 = **상태의 선언**), Result Builder(DSL)
    
	- **동시성/Actor 모델**: `async/await`, `Task`, `actor`로 데이터 경합을 모델링
    
	- **메타프로그래밍(매크로)**: 반복/보일러플레이트 제거(예: `@Observable` 등)

- 실전에서의 적용
	- **도메인 모델**: 값 타입(`struct`) + **프로토콜 지향**으로 합성 우선, 상속은 최소화
    
	- **데이터 흐름**: 컬렉션/옵셔널 변환은 **함수형 연산자**로 파이프라인
    
	- **UI**: SwiftUI면 **선언형 상태 바인딩**, UIKit이면 명령형 + Auto Layout(관계 선언)
    
	- **부수효과**: 네트워크/저장 등은 스트림/비동기 체인의 **끝단**에서만 처리
    
	- **동시성**: 공유 상태는 **actor**로 보호, 나머지는 `async/await`로 명료하게

```swift
// FP: 데이터 변환(불변·고차함수)
let emails = users.map(\.email).filter { !$0.isEmpty }

// POP: 공통 기능을 합성
protocol Cacheable { associatedtype Key: Hashable; func load(_ k: Key) -> Data? }
extension MemoryCache: Cacheable {}
extension DiskCache: Cacheable {}

// 선언형 UI(SwiftUI)
struct CounterView: View {
  @State private var n = 0
  var body: some View { VStack { Text("\(n)"); Button("Up") { n += 1 } } }
}

// 동시성 + Actor
actor ImageStore { private var cache: [URL: UIImage] = [:] }
```

> Swift는 함수형 기법을 잘 지원하는 다중 패러다임 언어다.
