**🗣️ 가볍게 시작하기 (Simple): 식당 진동벨의 마법**

네트워크 통신은 식당에서 음식을 주문하는 것과 같습니다. 서버(주방)에 요청을 보내고, 응답(음식)이 올 때까지 기다려야 하죠.

- **과거의 Closure 방식 (카운터 대기):** 음식이 나올 때까지 카운터 앞에서 계속 서서 기다립니다. 직원이 "나왔습니다!" 하고 부르면(콜백) 그때서야 음식을 받아서 자리로 갑니다. 코드가 계속 안으로 파고드는 지옥이 펼쳐집니다.
    
- **현재의 `async/await` 방식 (진동벨):** 주문을 하고 **진동벨(`await`)**을 받은 뒤 내 자리로 돌아가서 다른 일을 합니다. 징~ 울리면 그때 다시 가서 음식을 받아옵니다. 코드가 그냥 위에서 아래로 깔끔하게 떨어집니다.

**🤿 딥하게 들어가기 (Deep: 실전 코드 비교)**

서버에서 집중 관련 명언 데이터를 하나 가져오는 함수를 만든다고 상상해 보겠습니다.

**❌ 1. 과거의 Closure 방식 (읽기 힘든 콜백 지옥)**

예전 `URLSession.shared.dataTask`를 쓰면 코드가 이렇게 오른쪽으로 쑥쑥 들어갔습니다.

```Swift
func fetchQuote(completion: @escaping (String?, Error?) -> Void) {
    let url = URL(string: "https://api.example.com/quote")!
    
    // 1. 요청을 보냅니다. (이 괄호 안이 콜백 블록입니다)
    URLSession.shared.dataTask(with: url) { data, response, error in
        // 2. 에러가 있는지 확인합니다.
        if let error = error {
            completion(nil, error)
            return
        }
        
        // 3. 데이터가 있는지 확인합니다.
        guard let data = data else {
            completion(nil, nil)
            return
        }
        
        // 4. UI 업데이트를 위해 메인 스레드로 억지로 돌려보냅니다.
        DispatchQueue.main.async {
            let quote = String(data: data, encoding: .utf8)
            completion(quote, nil)
        }
    }.resume() // ⭐️ 수많은 주니어 개발자가 이 .resume()을 빼먹어서 밤을 새웠습니다.
}
```

**✅ 2. 현재의 `async/await` 방식 (직관적이고 우아한 흐름)**

이제는 함수 이름 뒤에 `async`를 붙이고, 데이터를 기다리는 곳에 `try await`만 적어주면 끝입니다.

```Swift
// 함수 이름 뒤에 async throws를 붙여 "시간이 걸리고, 에러가 날 수 있다"고 선언합니다.
func fetchQuote() async throws -> String {
    let url = URL(string: "https://api.example.com/quote")!
    
    // 1. 데이터를 요청하고, 올 때까지 이 줄에서 대기(잠시 멈춤)합니다.
    let (data, _) = try await URLSession.shared.data(from: url)
    
    // 2. 데이터가 도착하면 아래 코드가 마저 실행됩니다.
    let quote = String(data: data, encoding: .utf8) ?? "명언을 불러오지 못했습니다."
    
    return quote
}
```

### 🎨 SwiftUI 뷰에서 사용하기 (`.task`의 재등장!)

```Swift
struct QuoteView: View {
    @State private var todayQuote: String = "로딩 중..."
    
    var body: some View {
        Text(todayQuote)
            // 화면이 뜰 때 자동으로 네트워크 요청을 보냅니다!
            .task {
                do {
                    // 진동벨(await)을 들고 데이터를 기다립니다.
                    todayQuote = try await fetchQuote()
                } catch {
                    todayQuote = "에러 발생!"
                }
            }
    }
}
```

**💡 핵심 요약**

> `URLSession`에 `async/await`가 도입되면서,
> 
> 1. 골치 아픈 콜백(`completion`)이 사라졌고
>     
> 2. `DispatchQueue.main.async`로 스레드를 신경 쓸 필요가 줄었으며
>     
> 3. `.resume()`을 까먹어서 통신이 안 되는 어처구니없는 버그가 멸종했습니다!

이제 외부 서버와 어떻게 깔끔하게 통신하는지 뼈대가 잡히셨나요?

네트워크에서 받아온 이 데이터(`data`)는 보통 0과 1로 이루어진 날것의 데이터(JSON)입니다. 이걸 우리가 만든 Swift 구조체 모델로 예쁘게 번역해 주는 과정이 반드시 필요하죠!
