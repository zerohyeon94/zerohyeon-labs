**🗣️ 가볍게 시작하기 (Simple): "인테리어만 바꾸는 리모델링"**

Core Data로 저장해 둔 기존 유저들의 소중한 데이터(SQLite 파일)를 날려버릴 필요가 없습니다. 앱 코드를 수정해서 "이제부터는 이 파일을 SwiftData 방식으로 읽고 쓸게!"라고 선언만 해주면 됩니다.

---

**🤿 딥하게 들어가기 (Deep: 마이그레이션 3단계 핵심 루틴)**

이사는 크게 3단계로 진행됩니다. 코드가 얼마나 극적으로 짧아지는지 비교해 보세요.

**1단계: 모델(Entity) 갈아 끼우기** 가장 먼저 `.xcdatamodeld` 파일(시각적 에디터)을 열어서 보던 표들을 Swift 코드로 바꿉니다.

- **규칙:** 기존 Core Data의 Entity 이름과 Attribute(속성) 이름을 **토씨 하나 안 틀리고 똑같이** 적어주기만 하면, 시스템이 알아서 예전 데이터를 찾아 연결해 줍니다.

```Swift
// ❌ 기존 Core Data 방식: 에디터에서 그리고, NSManagedObject를 상속받음
@objc(Item)
public class Item: NSManagedObject {
    @NSManaged public var timestamp: Date?
}

// ✅ SwiftData 방식: 깔끔한 순수 Swift 클래스로 변환
import SwiftData

@Model
class Item {
    var timestamp: Date?
    
    init(timestamp: Date? = nil) {
        self.timestamp = timestamp
    }
}
```

_작업이 끝나면 낡은 `.xcdatamodeld` 파일은 과감하게 삭제해도 됩니다!_

**2단계: 컨테이너(Container) 교체하기** 앱이 켜질 때 데이터베이스를 준비시키는 설정 부분입니다. 보통 앱의 최상단(`App` 구조체)에 위치합니다.

```Swift
// ❌ 기존: AppDelegate나 PersistenceController에서 길게 짰던 세팅 코드 삭제

// ✅ SwiftData: 단 한 줄로 끝!
@main
struct MyApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
        // "이제부터 Item이라는 모델을 담는 통(Container)을 쓸게!"
        .modelContainer(for: Item.self) 
    }
}
```

**3단계: 뷰에서 읽고 쓰기 (Context & Fetch)** SwiftUI 뷰 안에서 데이터를 불러오고(Read) 저장하는(Save) 로직을 교체합니다.

```Swift
// ❌ 기존 Core Data
@Environment(\.managedObjectContext) private var viewContext
@FetchRequest(sortDescriptors: [NSSortDescriptor(keyPath: \Item.timestamp, ascending: true)])
private var items: FetchedResults<Item>

// ✅ SwiftData
@Environment(\.modelContext) private var modelContext // 저장/삭제할 때 씀
@Query(sort: \Item.timestamp, order: .forward) // 데이터 불러올 때 씀
private var items: [Item]
```

### 🚨 현업 꿀팁: "두 집 살림(Coexistence)도 가능합니다!"

앱이 너무 거대해서 한 번에 다 바꾸기 무섭다면 어떡할까요? 애플은 **Core Data와 SwiftData가 하나의 데이터베이스 파일을 동시에 공유**할 수 있게 만들어 두었습니다.

- 기존에 짰던 복잡한 UIKit 화면들은 그대로 Core Data를 쓰게 내버려 두고,
    
- 새로 만드는 SwiftUI 화면들만 SwiftData를 써서 같은 데이터를 읽고 쓰게 하는 **'점진적 마이그레이션'**이 가능합니다!

---

**💡 오늘의 요약**

> 마이그레이션의 핵심은 **"기존 모델의 이름과 구조를 유지한 채, 껍데기만 `@Model`과 `@Query`로 바꿔 끼우는 것"**입니다!
