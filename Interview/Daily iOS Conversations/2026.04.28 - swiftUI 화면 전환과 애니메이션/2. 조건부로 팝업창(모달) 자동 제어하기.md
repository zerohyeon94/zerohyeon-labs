**🗣️ 가볍게 시작하기 - 상태를 감시하는 CCTV**

SwiftUI의 팝업창은 오직 `true / false`라는 상태(State) 스위치에 의해서만 켜지고 꺼집니다. 그렇다면 분석 결과가 바뀌는 순간을 **CCTV처럼 계속 지켜보다가, 특정 조건이 만족되는 순간 팝업 스위치를 강제로 꺼버리면(false)** 되겠죠?

이때 CCTV 역할을 해주는 것이 바로 `.onChange`라는 수식어입니다.

---

**🤿 딥하게 들어가기 - 실전 코드 적용**

CoreML 분석 결과(`isFocusing`)가 `true`가 되면, 띄워져 있던 팝업(`showWarningPopup`)을 자동으로 닫는 로직을 작성해 보겠습니다. 앞서 배운 트랜지션 애니메이션까지 곁들여서요!

```Swift
import SwiftUI

struct ZipJoongTimerView: View {
    // 1. 팝업창을 띄울지 말지 결정하는 스위치
    @State private var showWarningPopup = false
    
    // 2. ML 비전 모델의 분석 결과 (예: 집중 중이면 true, 졸면 false)
    @State private var isFocusing = true
    
    var body: some View {
        ZStack {
            // 메인 화면
            VStack(spacing: 30) {
                Text(isFocusing ? "🔥 집중하는 중..." : "💤 졸음 감지됨!")
                    .font(.title)
                
                // (테스트용) 강제로 졸음 상태로 만드는 버튼
                Button("졸음 감지 시뮬레이션") {
                    isFocusing = false
                    withAnimation(.spring()) {
                        showWarningPopup = true // 팝업 띄우기
                    }
                }
                
                // (테스트용) 다시 집중 상태로 돌아가는 버튼
                Button("다시 집중 모드로 전환 (true)") {
                    isFocusing = true
                }
            }
            
            // ⭐️ 팝업창 영역 (showWarningPopup이 true일 때만 나타남)
            if showWarningPopup {
                Color.black.opacity(0.4).ignoresSafeArea() // 어두운 배경
                
                VStack {
                    Text("일어나세요! 집중이 깨졌습니다!")
                        .font(.headline)
                        .padding()
                    Text("다시 화면을 보면 팝업이 자동으로 닫힙니다.")
                        .font(.caption)
                }
                .frame(width: 300, height: 200)
                .background(Color.white.cornerRadius(20))
                .transition(.scale) // 나타나고 사라질 때 뷰가 커졌다 작아지는 효과
            }
        }
        // 3. ⭐️ 핵심 CCTV 로직! isFocusing 값의 변화를 감시합니다.
        // (iOS 17 이상 최신 문법)
        .onChange(of: isFocusing) { oldValue, newValue in
            // 만약 새로운 분석 결과(newValue)가 true(집중 상태)라면?
            if newValue == true {
                // 팝업이 떠있든 말든 강제로 닫아버립니다! (애니메이션과 함께)
                withAnimation(.spring()) {
                    showWarningPopup = false
                }
            }
        }
    }
}
```

---

### 🚨 코드의 핵심 포인트 3가지

1. **상태의 분리** 팝업을 여닫는 변수(`showWarningPopup`)와 분석 결과를 담는 변수(`isFocusing`)를 따로 둡니다.
    
2. **`.onChange` 수식어** 화면의 제일 바깥쪽(`ZStack` 끝 등)에 달아두면, `isFocusing` 값이 변할 때마다 블록 안의 코드가 실행됩니다.
    
3. **`withAnimation` 결합** 팝업을 닫을 때 단순히 `showWarningPopup = false`만 하면 팝업이 뚝 끊기며 사라지지만, `withAnimation`으로 감싸주면 트랜지션 효과가 적용되면서 부드럽게 사라집니다.

---

**💡 오늘의 요약**

> SwiftUI에서 특정 조건에 맞춰 팝업을 자동으로 제어하려면, 팝업 상태 변수와 조건 변수를 분리한 뒤 **`.onChange`**를 사용하여 데이터의 흐름을 감시하고 스위치를 조작하면 됩니다!

이 방식을 응용하면 "네트워크 통신(로딩)이 끝나면 로딩 팝업을 자동으로 닫기", "타이머가 0초가 되면 결과 팝업 띄우기" 등 무궁무진한 자동화 로직을 아주 쉽게 만들 수 있습니다.

UIKit에서는 `Delegate`나 `NotificationCenter`를 써서 복잡하게 알려줘야 했던 작업이 이렇게 선언형으로 깔끔하게 끝납니다.
