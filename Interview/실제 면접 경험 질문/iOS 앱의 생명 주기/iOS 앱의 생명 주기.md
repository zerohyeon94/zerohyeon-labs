## 0) 상태(State) 정의부터 정확히

### Foreground (Active)

- 화면에 보이고, 사용자 입력을 받는 “정상 동작” 상태.

### Foreground (Inactive)

- **포그라운드에 있긴 한데 입력을 못 받는 잠깐의 상태**
    
- 예: 앱 전환 애니메이션 중, 전화/알림/제어센터가 올라오는 순간, FaceID 팝업 등

### Background

- 화면엔 안 보임.
    
- **코드는 ‘일단’ 실행될 수 있음.** 다만 시간 제한이 있고(보통 수 초~수십 초), iOS가 빨리 정리하려고 합니다.
    
- 이때 할 일:
    
    - 작업 정리, 상태 저장, 네트워크/파일 마무리
        
    - (허용된 경우) 짧은 백그라운드 작업 수행 

### Suspended

- 화면엔 안 보임.
    
- **프로세스가 메모리에 “그대로 얼어붙음”**
    
- 코드 실행 **0** (타이머, 네트워크 콜백, RunLoop 모두 멈춤)
    
- OS가 메모리 필요하면 **그냥 종료(terminate)해버릴 수 있음** → “다음에 다시 켜면” 앱이 처음부터 실행될 수도 있음.

> 핵심: **Background = 아직 정리/작업 가능**, **Suspended = 완전 정지(실행 없음)**

---
## Foreground의 설명

- **Foreground + Active**: 화면에 보이고 **입력/이벤트 처리도 정상**(터치 가능)
    
- **Foreground + Inactive**: 화면에 보이긴 하지만 **일시적으로 입력을 못 받는 상태**(터치 거의 안 됨)

그래서 “포그라운드냐?”를 OS 관점으로 크게 보면

- **Foreground = Active 또는 Inactive**
    
- **Background = Background 또는 Suspended(화면 밖)**

### 자주 나오는 예시로 감 잡기

- 앱을 쓰고 있음 → **Active**
    
- 전화/알림 배너 내려오면서 순간 멈칫 → **Inactive**
    
- 배너/전환 끝나고 다시 조작 가능 → **Active**
    
- 홈 버튼/제스처로 나감 → **Inactive → Background → (곧) Suspended**

### 한 줄 정리

“포그라운드는 **사용자에게 보이는 상태**라서 Active와 Inactive를 포함하고, 둘의 차이는 **입력 가능 여부**입니다.”
