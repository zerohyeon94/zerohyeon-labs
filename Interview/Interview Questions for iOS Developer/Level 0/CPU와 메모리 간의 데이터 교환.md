### 🔸 1. **CPU와 메모리 간의 데이터 교환은 어떻게 이루어지나요?**

CPU는 데이터를 **RAM에서 읽어오거나** 연산 결과를 **RAM에 저장**합니다. 이때 두 컴포넌트 간의 데이터 교환은 **버스(Bus)**를 통해 이뤄집니다.

**과정 요약**:
1. CPU가 필요한 데이터의 주소를 명시.
2. 해당 주소의 데이터를 RAM에서 가져옴.
3. 이 데이터는 **데이터 버스**를 통해 CPU로 이동.
4. CPU는 이를 연산하거나 저장합니다.

📌 실제로는 CPU가 **한 번에 데이터를 다 읽는 게 아니라**, 캐시 등을 활용해 효율적으로 교환함.

---
### 🔸 2. **데이터 교환 속도를 높이기 위한 장치들**

1. **캐시 메모리 (Cache)**
    - CPU 안에 내장된 고속 메모리.
    - 자주 쓰이는 데이터/명령어를 저장해서 RAM 접근을 줄임.
2. **DMA (Direct Memory Access)**
    - CPU가 직접 관여하지 않고, 외부 장치가 RAM과 직접 데이터 교환 가능.
    - 예: 네트워크 카드가 수신 데이터를 RAM에 바로 저장.
3. **버스 아키텍처 개선**
    - 예: 하이퍼트랜스포트, QPI 등 빠른 버스 도입.
4. **멀티채널 메모리**
    - RAM 접근을 병렬화해서 대역폭 향상.

---
### 🔸 3. **버스(Bus)란 무엇인가요?**

**버스(Bus)**는 컴퓨터 내부에서 **데이터를 전달하는 통로**입니다. CPU, 메모리, 저장 장치 등 구성 요소들이 서로 정보를 주고받을 수 있도록 함.

**종류**:

|종류|역할|
|---|---|
|**데이터 버스**|실제 데이터를 주고받는 경로|
|**주소 버스**|CPU가 RAM에서 데이터를 찾을 때 사용하는 주소 정보 전달|
|**제어 버스**|읽기/쓰기 등의 명령이나 상태 정보를 전달|

---
### 🔸 4. **버스를 통해 연결되는 다른 장치들**

버스를 통해 연결될 수 있는 장치는 매우 많습니다:
- **저장 장치** (SSD, HDD)
- **GPU (그래픽 처리 장치)**
- **I/O 장치** (마우스, 키보드, USB 등)
- **네트워크 카드**
- **칩셋 및 기타 컨트롤러들**

📌 즉, 버스는 시스템 전체의 **통신 인프라**라고 할 수 있어요.

---
### 🔸 5. **캐시 메모리란? 역할은?**

**캐시 메모리(Cache Memory)**는 CPU 내부(또는 근처)에 위치한 **고속 임시 저장소**로, RAM보다 훨씬 빠릅니다.

**역할**:
- 자주 사용되는 데이터/명령어를 저장.
- CPU가 RAM보다 빠르게 필요한 정보를 가져올 수 있도록 함.
- RAM 접근을 줄여 **처리 속도 향상**에 크게 기여.

**계층 구조**:
- **L1 Cache**: 가장 작고 빠름. CPU 코어당 존재.
- **L2 Cache**: L1보다 크고 느리며, CPU와 RAM 사이 다리 역할.
- **L3 Cache**: 다수의 코어가 공유하는 큰 캐시. L2보다 느리지만 여전히 빠름.

---
### 🔸 6. **캐시 히트(Cache Hit)와 캐시 미스(Cache Miss)**

- **Cache Hit**: CPU가 찾는 데이터가 캐시에 이미 존재 → 매우 빠르게 접근 가능.
- **Cache Miss**: CPU가 캐시에서 데이터를 찾지 못함 → RAM에서 다시 읽어야 하므로 느려짐.

📌 **성능 영향**:
- **Cache Hit율**이 높을수록 전체 시스템 속도 향상.
- Miss가 잦으면 오히려 캐시 관리 비용 + RAM 접근으로 성능 저하 가능.
