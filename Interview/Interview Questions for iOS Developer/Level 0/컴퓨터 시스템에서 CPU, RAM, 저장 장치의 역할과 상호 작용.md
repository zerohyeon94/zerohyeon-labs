### 🔸 1. 컴퓨터 시스템에서 CPU, RAM, 저장 장치의 역할과 상호작용
**CPU (중앙처리장치)**
- 시스템의 '두뇌'. 연산을 처리하고, 명령을 실행.
- 앱의 로직을 실행하며, 연산/판단/명령 제어를 담당.

**RAM (메모리)**
- 실행 중인 데이터와 명령어를 일시적으로 저장.
- CPU가 빠르게 접근할 수 있는 공간.
- 앱을 실행하면 실행파일의 일부가 RAM에 올라감.

**저장 장치 (SSD 등)**
- 데이터를 장기 보관하는 공간. 앱, 사진, 설정 등이 저장됨.
- RAM보다 느리지만, 전원이 꺼져도 데이터가 유지됨.

**상호작용 흐름 예시**
1. 사용자가 앱 아이콘 터치 → iOS가 저장 장치에서 앱의 바이너리를 읽음.
2. 해당 데이터를 RAM으로 적재 → CPU가 명령을 실행하며 앱을 구동.
3. CPU는 계속 RAM에서 데이터를 가져와 처리함 → 사용자 인터페이스 렌더링, 이벤트 처리 등 수행.

### 🔸 2. 앱 실행 시 시스템 구성 요소들의 상호작용
앱이 실행될 때의 **과정 요약**:
1. **App Launch Trigger**  
    사용자가 앱 아이콘을 터치하거나 시스템에서 앱 실행 요청.
2. **저장 장치(SSD)**  
    앱의 실행 파일(.app) 및 리소스(bundle 파일들 등)을 읽어옴.
3. **RAM**  
    iOS는 앱 바이너리, 리소스, 필요한 시스템 라이브러리를 RAM에 적재.  
    런타임에 필요한 데이터(뷰 상태, 변수 등)도 이곳에 저장.
4. **CPU**  
    적재된 코드를 실행하며 앱의 초기화 진행. `main()` 함수 → `@UIApplicationMain` → AppDelegate의 lifecycle 메서드 실행.
5. **UIKit / SwiftUI**  
    화면 구성 요소를 그리기 시작. 렌더링은 GPU와 협력하여 처리.
6. **사용자 화면 표시**  
    최종적으로 렌더된 UI가 디스플레이에 보여짐.

📌 **중요 포인트**:  
저장 장치는 느리기 때문에, 실행에 필요한 데이터만 메모리에 올리고, CPU는 메모리(RAM)에만 접근하면서 속도를 높임.

### 🔸 3. RAM이 부족하면 iOS는 어떻게 동작할까?
**iOS의 메모리 관리 전략**:
1. **백그라운드 앱 종료**  
    우선 백그라운드에 있는 앱들을 종료시킴. 메모리 확보 목적.
2. **메모리 경고 (Memory Warning)**  
    포그라운드 앱에게 메모리 경고(`didReceiveMemoryWarning`) 전달. 개발자는 이 때 캐시 등 비필수 데이터를 해제해야 함.
3. **앱 강제 종료**  
    메모리 사용량이 계속 높으면 현재 앱도 종료시킴 (crash).

**개발 시 중요한 이유**:
- **메모리 누수(Leak)** 방지 필요: 앱이 장시간 실행되면 점점 느려지거나 죽을 수 있음.
- **이미지/데이터 캐시 관리** 중요: 큰 리소스는 필요 시 로딩하고 사용 후 해제하는 전략 필요.
- 앱이 자주 죽는다면 사용자 경험 저하 + App Store 리젝 가능성도 있음.

### 🔸 4. CPU, RAM, 저장 장치 중 '체감 속도'에 가장 큰 영향 요소는?
✅ **대부분의 경우: "저장 장치 속도" (특히 앱 시작 시)**

**설명**:
 
| 요소           | 영향 설명                                                                         |
| ------------ | ----------------------------------------------------------------------------- |
| CPU 속도       | 계산이나 반복 작업에 영향. 하지만 일반적인 앱에선 병목이 CPU보단 I/O에 있음.                               |
| **저장 장치 속도** | 앱 실행 시, 이미지/DB 파일/리소스를 로딩할 때 I/O 속도가 느리면 '버벅이는 느낌' 발생. 특히 앱 실행 첫 화면 속도에 영향 큼. |
| RAM 용량       | 앱이 메모리에 모두 올라가지 못하면 캐시 삭제나 리로딩이 빈번해져 체감 속도 저하. 단, 일정 수준 이상일 때는 영향 작음.         |

📌 예: 사진 편집 앱에서 고해상도 이미지 여러 개 로딩할 때 → 저장 장치 I/O가 느리면 느리게 느껴짐.  
하지만 게임에서 실시간 렌더링 성능 문제라면 → CPU와 GPU의 영향이 더 큼.
