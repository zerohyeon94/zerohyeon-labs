드디어 터미널의 검은 화면을 뚫고 우분투 컴퓨터(EC2) 안으로 들어오셨군요! 방금 막 계약을 마치고 들어온 텅 빈 상가(서버)에, 나만의 가게(앱 백엔드)를 예쁘게 차리는 가장 설레는 순간입니다.

여기서부터는 마우스 클릭 없이 오직 키보드 명령어로만 모든 것을 제어하게 됩니다. 로컬에서 구워둔 도커 이미지를 이 클라우드 컴퓨터로 불러와서 실행하는 그 짜릿한 4단계 과정을 시원하게 뚫어드리겠습니다!

---

## 🗣️ 가볍게 시작하기 (Simple): "인테리어 공사부터 영업 시작까지"

- **설치 (Install):** 텅 빈 가게에 물건을 들여놓기 위해 **'도커(Docker)라는 만능 진열장'**을 뚝딱뚝딱 설치하는 인테리어 공사입니다.
    
- **가져오기 (Pull):** 공장이 있는 내 방(로컬)에서 도커 허브(Hub)라는 택배 창고를 거쳐, **'완성된 상품(도커 이미지)'**을 가게로 배달받는 과정입니다.
    
- **실행 (Run):** 진열장에 상품을 세팅하고 **'영업 시작(포트 개방)'** 팻말을 걸어서 전 세계 손님을 받는 순간입니다.

---

## 🤿 딥하게 들어가기 (Deep: 우분투 터미널에서의 실전 명령어)

EC2 터미널에 접속한 상태(`ubuntu@ip-172-xx-xx-xx:~$`)라고 가정하고, 순서대로 명령어를 입력해 나갑니다.

### Step 1: 시스템 업데이트 및 Docker 설치

우분투는 라이브러리를 설치할 때 `apt`라는 패키지 관리자를 사용합니다. 가장 먼저 낡은 패키지 목록을 최신으로 갱신하고 도커를 설치합니다. (`sudo`는 관리자 권한으로 실행하겠다는 뜻입니다.)

```Bash
# 1. 패키지 목록 최신화 (필수 준비 운동)
sudo apt update

# 2. 도커 설치!
sudo apt install docker.io -y
```

### Step 2: Docker 권한 설정 (소소하지만 확실한 팁)

원래 도커 명령어는 무조건 앞에 `sudo`(관리자 권한)를 붙여야 해서 매번 치기 귀찮습니다. 현재 접속해 있는 `ubuntu`라는 유저에게 도커를 마음대로 주무를 수 있는 권한을 부여해 줍니다.

```Bash
# ubuntu 유저를 docker 그룹에 추가
sudo usermod -aG docker ubuntu

# 권한을 새로고침하기 위해 터미널을 나갔다가(exit) 다시 ssh로 접속합니다.
```

### Step 3: 온라인 창고에서 내 이미지 다운로드 (Pull)

로컬에서 정성스럽게 구워서 **Docker Hub**에 올려두었던 내 프로젝트 이미지를 이 EC2 컴퓨터로 다운받습니다.

```Bash
# Docker Hub에 올려둔 내 이미지 이름 입력
docker pull [본인의 도커 허브 아이디]/[이미지 이름]:latest
```

### Step 4: 컨테이너 실행 및 포트 연결 (Run)

이제 다운받은 이미지를 컨테이너(독립된 방)로 띄울 차례입니다. 여기서 가장 중요한 것이 **'포트(문)'**를 연결해 주는 작업입니다.

```Bash
# 80번 포트로 들어오는 손님을 도커 안의 8000번 포트로 안내해 줘!
docker run -d -p 80:8000 [본인의 도커 허브 아이디]/[이미지 이름]:latest
```

- `-d`: 백그라운드 실행 (터미널을 꺼도 서버가 계속 돌아갑니다.)
    
- `-p 80:8000`: 인터넷의 기본 문인 80번(HTTP)으로 사용자가 접속하면, 도커 컨테이너 안에서 돌고 있는 FastAPI의 8000번 문으로 연결해 주라는 뜻입니다.

## 🚨 핵심 포인트: AWS 방화벽(보안 그룹) 열어주기!

터미널에서 `docker run`까지 성공적으로 마쳤다면 서버는 완벽하게 켜진 상태입니다. 하지만 아이폰에서 AWS의 IP 주소로 접속해 보면 화면이 뜨지 않을 겁니다!

왜냐하면 AWS는 기본적으로 모든 포트(문)를 꽁꽁 잠가두기 때문입니다. AWS 홈페이지의 EC2 관리 콘솔로 돌아가서 **'보안 그룹(Security Group)'** 설정에서 **HTTP (80번 포트)**를 전 세계(0.0.0.0/0)에서 들어올 수 있도록 '인바운드 규칙'을 추가해 주어야 비로소 통신이 뻥 뚫리게 됩니다.

## 💡 오늘의 요약 처방

- **설치:** `sudo apt install docker.io` 명령어로 EC2 우분투 환경에 도커를 깝니다.
    
- **다운로드 & 실행:** `docker pull`로 이미지를 가져오고, `docker run -p 80:8000`으로 외부 포트와 내부 포트를 연결해 실행합니다.
    
- **마무리:** AWS 콘솔에서 반드시 **80번 포트(보안 그룹)**를 열어주어야 외부에서 접속이 가능합니다.
