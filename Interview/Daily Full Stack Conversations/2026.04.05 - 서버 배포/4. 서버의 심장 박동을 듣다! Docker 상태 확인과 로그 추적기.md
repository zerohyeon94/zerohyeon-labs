서버를 처음 띄웠을 때의 쾌감을 맛보셨다면, 이제는 그 서버가 건강하게 잘 살아있는지, 혹은 뒤에서 혼자 에러를 뿜으며 아파하고 있지는 않은지 '청진기'를 대어볼 차례입니다!

화면이 없는 우분투 터미널 환경에서는 오직 명령어로만 서버의 상태를 파악해야 합니다. 실무 백엔드 개발자들이 숨 쉬듯이 매일 두드리는 핵심 모니터링 명령어들을 속 시원하게 분해해 드리겠습니다.

## 🗣️ 가볍게 시작하기 (Simple): "출근부 확인" vs "CCTV 실시간 녹화본"

- **`docker ps` (출근부):** "지금 가게(EC2)에서 정상적으로 살아서 일하고 있는 알바생(컨테이너)은 누구누구야?" 하고 현재 켜져 있는 방들의 목록을 확인하는 명부입니다.
    
- **`docker logs` (CCTV):** 특정 알바생이 "손님이 8000번 문으로 들어왔습니다!", "결제 처리 중 에러가 났습니다!"라고 떠드는 모든 기록을 들여다보는 관찰 카메라입니다.

## 🤿 딥하게 들어가기 (Deep: 실전 모니터링 명령어 4대장)

EC2 터미널에 접속한 상태에서 아래의 명령어들을 통해 컨테이너를 완벽하게 통제할 수 있습니다.

### Step 1: 누가 살아서 돌아가고 있나? (상태 확인)

가장 기본이 되는 상태 점검 명령어입니다.

```Bash
# 1. 현재 '실행 중'인 컨테이너 목록 보기
docker ps
```

이 명령어를 치면 표 형태로 `CONTAINER ID`, `IMAGE`, `STATUS`(예: Up 3 hours), `PORTS` 등이 출력됩니다. 여기서 내 FastAPI 서버가 `Up` 상태로 잘 떠 있는지, 포트는 잘 열려 있는지 확인합니다.

```Bash
# 2. '꺼진 것까지 포함'해서 모든 컨테이너 목록 보기
docker ps -a
```

만약 `docker run`을 했는데 `docker ps` 목록에 아무것도 없다면? 서버가 켜지자마자 에러가 나서 죽어버린 것입니다. 이때 `-a` (all) 옵션을 붙여서 치면, `Exited` (종료됨) 상태로 죽어있는 컨테이너의 흔적을 찾아낼 수 있습니다.

### Step 2: 도대체 왜 죽었을까? 무슨 일이 있나? (로그 확인)

`docker ps`를 통해 확인한 **`CONTAINER ID` (예: a1b2c3d4e5f6)** 또는 **`NAMES`**를 활용하여 컨테이너 안에서 무슨 일이 벌어지는지 확인합니다.

```Bash
# 3. 해당 컨테이너의 지금까지의 로그 전체 출력
docker logs [컨테이너 ID]

# 예시:
docker logs a1b2c3d4e5f6
```

이 명령어를 치면 파이썬 서버가 남긴 모든 `print()` 내용과 접속 기록, 그리고 가장 중요한 **'에러 메시지(Traceback)'**가 터미널에 쫙 출력됩니다. 서버가 죽은 이유를 여기서 찾아냅니다.

```Bash
# 4. (현업 필수) 실시간으로 로그 훔쳐보기
docker logs -f [컨테이너 ID]
```

`-f` (follow) 옵션을 붙이는 것이 실무의 핵심입니다! 이 명령어를 치면 화면이 멈춘 상태로 대기하는데, 이때 아이폰이나 웹 브라우저로 API 주소에 접속해 보면 터미널 화면에 실시간으로 로그가 촤르륵 올라오는 마법을 볼 수 있습니다. (빠져나올 때는 `Ctrl + C`를 누릅니다.)

---

## 🚨 핵심 포인트: 컨테이너 ID는 '앞 3자리'만 쳐도 됩니다!

터미널에서 `docker ps`를 쳤을 때 나오는 `CONTAINER ID`는 보통 12자리의 복잡한 영문과 숫자의 조합(예: `9f8a7b6c5d4e`)입니다. 로그를 볼 때 이걸 마우스로 드래그해서 복사할 필요 없이, 다른 컨테이너와 겹치지만 않는다면 **앞의 2~3자리만 입력해도 도커가 알아서 찰떡같이 찾아냅니다.**

- (정석) `docker logs -f 9f8a7b6c5d4e`
    
- (실무 꿀팁) `docker logs -f 9f8`

이 소소한 팁 하나가 개발 속도를 엄청나게 끌어올려 줍니다!

## 💡 오늘의 요약 처방

- **`docker ps`**: 현재 살아있는 컨테이너 목록과 상태(포트, 실행 시간 등)를 확인합니다.
    
- **`docker ps -a`**: 에러가 나서 죽어버린 컨테이너까지 모두 찾아냅니다.
    
- **`docker logs [ID]`**: 컨테이너가 뿜어낸 과거의 모든 텍스트(에러 포함)를 확인합니다.
    
- **`docker logs -f [ID]`**: 모니터에 CCTV를 틀어놓듯, 들어오는 요청과 에러를 실시간으로 추적합니다.
