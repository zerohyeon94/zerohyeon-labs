🗣️ 가볍게 시작하기 (Simple): 거대한 전용 창고 vs 내 방 서랍장 속 장부

PostgreSQL (실전용 DB): 별도의 건물을 임대해서 경비원(서버 프로그램)을 세워두고 관리하는 거대한 전용 창고입니다. Mac 시스템에 직접 설치하고 항상 실행시켜 두어야 합니다. 
SQLite (개발용 내장 DB): 내 프로젝트 폴더 안에 쏙 들어가는 엑셀 파일 하나입니다. 따로 프로그램을 설치하거나 서버를 켤 필요 없이, 파이썬 코드가 내 방 서랍장을 열어서 바로 읽고 씁니다!

🤿 딥하게 들어가기 (Deep: 파일 하나를 어떻게 생성하고 연결할까요?)

앞서 말씀드린 SQLAlchemy라는 강력한 통역사를 이용해, 이 작은 파일 장부(SQLite)를 만들고 백엔드 프로젝트(FastAPI)와 연결해 보겠습니다. 파이썬 파일 하나(예: database.py)를 만들고 아래의 순서대로 코드를 채워 넣는 것이 정석입니다.

Step 1: 데이터베이스 주소(URL) 설정하기 가장 먼저 해야 할 일은 이 파일이 프로젝트 폴더 어디에, 어떤 이름으로 생길지 주소를 적어주는 것입니다. 코드로 작성할 때는 sqlite:///./zipjoong_timer.db 와 같은 형태의 문자열로 적어줍니다. 이것은 내 현재 폴더 안에 zipjoong_timer.db라는 이름의 파일로 데이터베이스를 만들겠다는 뜻입니다.

Step 2: 통역사가 지나갈 다리(Engine) 놓기 주소를 정했다면, SQLAlchemy에게 이 파일과 통신할 수 있는 튼튼한 다리를 만들라고 명령해야 합니다. 파이썬 코드에서 create_engine 이라는 도구를 사용해 엔진을 만듭니다. 이때 SQLite는 여러 요청이 동시에 들어오면 헷갈려 할 수 있어서, check_same_thread 옵션을 False로 꺼주는 작은 팁이 필요합니다.

Step 3: 데이터 조작을 위한 작업장(Session) 만들기 다리가 연결되었다고 끝이 아닙니다! 데이터를 읽고 쓸 때마다 다리 위에 서서 작업할 수는 없으니, sessionmaker를 사용해 전용 임시 작업장을 만들어줍니다. 앞으로 FastAPI 웨이터가 앱에서 데이터를 달라는 주문을 받을 때마다 이 작업장을 하나씩 열어서 기록을 남기고 닫게 됩니다.

Step 4: 스키마의 뼈대(Base) 준비하기 마지막으로, 우리가 파이썬 클래스로 데이터베이스 테이블을 설계할 때 상속받을 가장 기초적인 뼈대인 declarative_base를 만들어 줍니다. 이 뼈대를 바탕으로 User 정보나 FocusSession 기록 같은 클래스를 만들어나가게 됩니다.

🚨 핵심 포인트: 연결 코드를 작성하고 실행하는 순간 파일이 뿅!

이 4단계의 설정을 마치고, FastAPI 서버를 터미널에서 처음 실행하는 순간! 프로젝트 폴더 안에 아까 적어둔 이름(zipjoong_timer.db)의 파일이 마법처럼 뿅 하고 생성됩니다. 즉, 거창하게 터미널에서 DB 서버 프로그램을 다운받고 설치할 필요 없이, 파이썬 코드 몇 줄만으로 프로젝트 폴더 안에 완벽한 데이터베이스 파일이 하나 뚝딱 만들어지고 연결까지 끝나는 것입니다!

💡 오늘의 요약 처방

SQLite는 별도의 서버 프로그램 설치 없이 프로젝트 폴더 안에 파일 형태로 생성되는 아주 가벼운 내장 데이터베이스입니다. 연결 과정은 파일의 주소를 적고, SQLAlchemy의 엔진(통신 다리)과 세션(임시 작업장)을 만들어주는 단계로 아주 간단하게 끝납니다.
