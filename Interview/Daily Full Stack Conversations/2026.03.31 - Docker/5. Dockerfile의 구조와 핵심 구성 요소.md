Dockerfile은 도커 이미지를 생성하기 위해 작성하는 텍스트 파일로, 시스템 환경을 구축하는 명령어들의 집합입니다. 위에서 아래로 순차적으로 실행되며, 각 명령어는 이미지를 구성하는 하나의 층(Layer)을 형성합니다.

## 1. 기본 뼈대 설정 (FROM)

모든 Dockerfile은 운영체제나 실행 환경의 베이스 이미지를 지정하는 것으로 시작합니다.

- 컨테이너의 바탕이 되는 리눅스 환경이나 특정 언어가 미리 설치된 이미지를 불러옵니다.

## 2. 작업 공간 지정 (WORKDIR)

컨테이너 내부에서 이후 명령어들이 실행될 기본 폴더 경로를 설정합니다.

- 이 명령어를 선언한 후의 모든 작업은 해당 폴더 안에서 이루어집니다.

## 3. 파일 복사 (COPY)

로컬 컴퓨터에 있는 소스 코드, 설정 파일 등을 컨테이너 내부의 작업 공간으로 복사합니다.

- 효율적인 캐시(Cache) 활용을 위해 변경이 잦지 않은 라이브러리 목록 파일 등을 먼저 복사하고, 자주 수정되는 소스 코드는 나중에 복사하는 구조를 주로 사용합니다.

## 4. 시스템 명령어 실행 (RUN)

이미지를 생성하는 과정(빌드 단계)에서 패키지를 다운로드하거나 시스템 설정을 변경할 때 사용합니다.

- 로컬 터미널에서 패키지 설치 명령어를 입력하는 것과 동일한 역할을 수행합니다.

## 5. 컨테이너 실행 명령 (CMD)

이미지 생성이 완료되고, 컨테이너가 실제로 시작될 때 단 한 번 실행되는 최종 명령어를 지정합니다.

- 메인 서버 프로그램을 구동시키거나 애플리케이션을 실행하는 명령이 들어갑니다.

### 핵심 구조 예시

```Dockerfile
FROM ubuntu:20.04
WORKDIR /app
COPY . .
RUN apt-get update
CMD ["bash"]
```

위와 같은 명시적인 명령어들이 단계별로 조합되어, 어떠한 환경에서도 동일하게 동작하는 독립적인 소프트웨어 실행 환경을 구축하게 됩니다.
