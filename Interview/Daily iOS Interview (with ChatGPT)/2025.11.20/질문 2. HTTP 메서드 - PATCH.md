**PATCH**는 HTTP 메서드로, **리소스의 “부분 수정(partial update)”**을 의미합니다.

## PATCH vs PUT vs POST 한눈 비교

- **GET**: 조회. _안전(safe)_, _멱등(idempotent)_.
    
- **PUT**: 리소스 **전체를 대체**(전체 스냅샷 업서트). 보통 _멱등_으로 설계.
    
- **PATCH**: 리소스의 **일부 필드만 변경**. **멱등이 “보장되지 않음”**(API 설계에 따라 멱등이 될 수도/아닐 수도).
    
- **POST**: 생성/비정형 처리. 보통 _비멱등_.

> 즉, **PATCH는 기본적으로 비멱등 취급**이 안전합니다. 같은 PATCH를 여러 번 보내면 결과가 달라질 수 있어요(예: “카운터 증가”). 다만 “이 필드를 값 X로 설정”처럼 정의하면 멱등이 되기도 합니다.

## 언제 PATCH를 쓰나요?

- 일부 필드만 바꾸고 싶고, 전체 리소스(=PUT)를 보내는 게 비효율일 때.
    
- 서버가 **Partial Update**를 명시적으로 지원할 때(예: `/users/123`의 `nickname`만 수정).

## 멱등/재시도 관점

- **기본 정책**: PATCH는 **재시도 대상에서 제외**(네트워크 오류라도).
    
- 예외: API 문서가 “이 PATCH는 멱등”이라 명시하고, **조건부 갱신**을 함께 쓰는 경우.
    
- **조건부 갱신 권장**: `If-Match: <ETag>` 헤더로 **낙관적 동시성 제어**(리소스 버전이 같을 때만 수정). 실패 시 412(Precondition Failed).
    
    - 이렇게 하면 중복 PATCH/경쟁 업데이트로 인한 꼬임을 줄일 수 있음.

## 캐싱/안전성

- PATCH 응답은 일반적으로 캐시하지 않습니다.
    
- 서버는 실패/부분적 성공 케이스를 명확히 상태코드(2xx/4xx/5xx)로 구분해야 합니다.

## iOS에서 쓰는 방법

### URLSession

```swift
var req = URLRequest(url: URL(string: "https://api.example.com/users/123")!)
req.httpMethod = "PATCH"
req.addValue("application/json", forHTTPHeaderField: "Content-Type")
req.addValue("application/json", forHTTPHeaderField: "Accept")
req.httpBody = try JSONSerialization.data(withJSONObject: ["nickname": "NewName"])

/// 조건부 갱신(ETag가 있다면)
if let etag = currentETag {
    req.addValue(etag, forHTTPHeaderField: "If-Match")
}

let task = URLSession.shared.dataTask(with: req) { data, resp, err in
    // 200/204 처리, 412(Precondition Failed) 처리 등
}
task.resume()
```

### Alamofire

```swift
let url = "https://api.example.com/users/123"
let params = ["nickname": "NewName"]
var headers: HTTPHeaders = ["Accept": "application/json",
                            "Content-Type": "application/json"]
if let etag = currentETag { headers.add(name: "If-Match", value: etag) }

AF.request(url, method: .patch, parameters: params, encoding: JSONEncoding.default, headers: headers)
  .validate()
  .responseData { resp in
    // 상태코드/에러 처리
  }
```

## 요약(면접용 한 줄)

> “**PATCH는 부분 수정 메서드**이고 기본적으로 **멱등 보장이 없습니다**. 재시도 정책에선 **비멱등 취급**이 안전하고, 안정성을 위해 `If-Match`(ETag) 같은 **조건부 갱신**을 함께 써서 중복/경쟁 업데이트를 제어합니다. 전체 대체가 필요하면 **PUT**, 생성은 **POST**, 조회는 **GET**을 사용합니다.”
