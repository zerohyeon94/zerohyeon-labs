## 학습 시스템

- **도구**: Obsidian (Daily Note) + Codex CLI/데스크톱
- **생활/출퇴근 기준**: 거주지는 경기도 용인시 수지구, 직장/출근 기준 위치는 신사역 근방이다. Daily Note에는 아침에 두 위치의 날씨 확인 칸과 짧은 파이팅 메시지를 함께 둔다.
- **루틴**: 오전 집중 1시간 → 출근 지하철 30-40분 복습 → 평일 근무(10:00-17:00) → 퇴근 지하철 30-40분 정리 → 퇴근 후 운동 전 1시간 산출물 → 운동 후 하루 마무리 → 일요일 주간 회고
- **영어회화 운영**: 현재 Daily 기본 과제에서는 OPIc 자격증 준비를 제외하고, `듀오링고`와 `스픽`을 매일 사용한다. 듀오링고는 연속 학습 유지용으로 하루 1세트 이상, 스픽은 실전 회화용으로 지하철에서는 듣기·표현 저장, 집에서는 운동 후 10-20분 실제 발화 연습을 기본값으로 둔다.
- **자격증 운영**: 정보처리기사는 평일 핵심 학습을 침범하지 않는 선에서 주 2-3회 선택 과제 또는 주말 단기 목표로 배치한다.
- **업무 스택 적응**: 2026-06-10부터 Backend Go와 Frontend React를 Daily 멘토링 기본 축으로 둔다.
- **모바일 개발 유지**: 영어/자격증을 추가하더라도 iOS/Swift 학습과 모바일 프로젝트 감각 유지를 주기적으로 유지한다.
- **입사 전 전환 학습**: 2026-07-01 크랜베리 입사 전까지 하루업/Jday 배포 수습을 우선하고, Go/React/Python/DB를 보강한다. ZipJoong은 [[Projects/FocusOps/개요|ZipJoong (구 FocusOps)]] 기준으로 Go/React 서버 배포 실험을 준비하되 아직 본격 프로젝트로 크게 열지 않는다.
- **Daily 생성 원칙**: 전날 학습 내용을 바탕으로 `Conversations/`에 오늘 학습 파일을 먼저 만들고, Daily Note의 수행 목록 체크박스로도 연결
- **아침 자동화**: Codex 데스크톱 반복 자동화가 매일 오전 6시(KST)에 오늘 Daily Note, `Conversations/` 학습 파일, `Today.md`를 생성하거나 보강
- **저녁 회고**: 하루 정리를 작성한 뒤 "오늘 내용을 피드백해줘"를 수동 요청하여 피드백 품질을 유지
- **간단히 보기**: 맥과 모바일에서 [[Today|오늘 계획]]을 고정 진입점으로 사용
- **운영 노트**: [[Schedule/근무 후 학습 루틴]]
