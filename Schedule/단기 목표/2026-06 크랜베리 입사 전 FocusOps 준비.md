## 목표

크랜베리 입사 전 Go 백엔드, React 프론트엔드, Python/DB 학습을 실무형 프로젝트 감각으로 연결한다.
ZipJoong은 아직 실제 프로젝트가 시작되지 않았으므로, 당장은 하루업/Jday 배포 과제를 우선 처리하고 이후 Go/React 서버 배포 실험으로 연결한다.

## 우선순위

1. 하루업 백엔드 API 정상 동작 확인 및 iOS 앱 수정 후 빠른 배포
2. Jday 리젝 대응 후 2026-07 전 재배포 확인
3. Go 백엔드와 React 프론트엔드 핵심 흐름 파악
4. Python/DB 실무 감각 보강
5. ZipJoong의 첫 Go/React 서버 배포 범위 정리
6. iOS/Swift 주차 멘토링으로 모바일 감각 유지

## 체크포인트

- D1: Jday iOS 리젝 사유 원문 확인 및 수정 항목 3개 이하로 분해
- D3: 하루업 API/iOS 배포를 위해 정상 동작 API와 수정 필요 항목 분리
- D7: Go/React 학습 노트 3개 이상을 업무 적응 또는 ZipJoong 실행 범위와 연결
- D14: ZipJoong의 Go API + PostgreSQL + React 대시보드 첫 실행 범위 작성
- D21: 크랜베리 입사 전 남은 학습 공백 정리

## Daily 반영 방식

평일 Daily에는 아래 중 최대 2개만 넣는다.

- [ ] Backend Go 또는 Frontend React 학습 파일 1개 읽기
- [ ] Python/DB 실무 개념 1개 정리
- [ ] 하루업 API 정상 동작 확인 또는 iOS 수정 범위 정리
- [ ] Jday 심사 이슈 수습 항목 1개 처리
- [ ] ZipJoong Go/React/DB 실행 범위 중 1개 정리
- [ ] iOS/Swift 주차 멘토링 1개 진행 또는 복습

## 완료 기준

- 하루업 API와 iOS 수정 범위가 빠른 배포 기준으로 정리되어 있다.
- Jday iOS 심사 이슈가 재제출 가능한 체크리스트로 정리되어 있다.
- ZipJoong의 첫 Go/React 서버 배포 범위가 문서화되어 있다.
- Go/React/Python/DB 학습이 추상 공부가 아니라 ZipJoong 또는 크랜베리 업무 적응과 연결되어 있다. 하루업과 Jday는 배포 수습/QA 체크리스트로 별도 관리한다.
