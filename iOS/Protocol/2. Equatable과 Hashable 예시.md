### 1. 상황: 아무런 자격증이 없을 때 😱

Swift는 구조체(`struct`)를 만들면 기본적으로 멍청합니다. 우리가 알려주기 전까지는 비교하는 법도, 저장하는 법도 모릅니다.

```Swift
struct Point {
    var x: Int
    var y: Int
}

let p1 = Point(x: 1, y: 2)
let p2 = Point(x: 1, y: 2)

// [에러 발생!] 
// 컴파일러: "이 두 개가 같은지(==) 내가 어떻게 알아? x만 같으면 같은 거야? y도 같아야 해?"
// if p1 == p2 { print("같다") } 

// [에러 발생!]
// 컴파일러: "Set에 넣으려면 고유한 번호(Hash)가 필요한데, 얘는 그게 없어서 못 넣어!"
// var visited = Set<Point>() 
```

---

### 2. Equatable 채택: "이제 비교(==)는 할 수 있어!"

`Equatable` 프로토콜을 채택하면, Swift가 **"아, 안에 있는 x랑 y가 전부 다 똑같으면 같은 걸로 칠게!"** 하고 자동으로 비교 기능을 만들어줍니다.

```Swift
// ✅ Equatable 채택
struct Point: Equatable {
    var x: Int
    var y: Int
}

let p1 = Point(x: 1, y: 2)
let p2 = Point(x: 1, y: 2)

// 성공! 이제 비교가 됩니다.
if p1 == p2 {
    print("두 점은 같습니다!") // 출력됨
}

// contains도 Equatable 덕분에 가능합니다.
let path = [p1, Point(x: 3, y: 4)]
print(path.contains(p2)) // true
```

> **하지만 아직 Set에는 못 넣습니다.** 비교는 할 줄 알지만, 빠르게 찾기 위한 '주소(Hash)'를 만드는 법은 모르니까요.

---

### 3. Hashable 채택: "이제 Set과 Dictionary에도 들어갈 수 있어!"

`Hashable`을 채택하면 **"나를 고유한 숫자(해시값)로 변환해서 Set에 저장해줘!"** 라고 말하는 것과 같습니다. (참고: `Hashable`은 `Equatable`을 포함하므로, `Equatable`을 따로 안 써도 됩니다.)

```Swift
// ✅ Hashable 채택 (Equatable은 자동으로 포함됨)
struct Point: Hashable {
    var x: Int
    var y: Int
}

let p1 = Point(x: 1, y: 2)
let p2 = Point(x: 1, y: 2)

// 1. 비교 당연히 됨 (Equatable 기능)
if p1 == p2 { print("같음") }

// 2. Set 저장 가능! (Hashable 기능)
var visited = Set<Point>()
visited.insert(p1)

// p2는 p1과 값이 같으므로, 이미 방문한 것으로 처리됨 (중복 제거)
if visited.contains(p2) {
    print("이미 방문한 곳입니다.") // 출력됨
}

// 3. Dictionary 키로 사용 가능!
var pointScores: [Point: Int] = [:]
pointScores[p1] = 100
```
