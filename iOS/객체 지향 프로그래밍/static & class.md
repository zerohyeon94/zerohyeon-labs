두 키워드의 결정적인 차이는 바로 **"자식 클래스가 부모의 기능을 자기 마음대로 뜯어고칠 수 있느냐(Overriding, 재정의)"**에 있습니다.

---

### 1. static (본사의 절대 철칙: 변경 불가!)

- **의미:** 본사에서 내려온 "절대 바꿀 수 없는 규칙"입니다.
    
- **특징:** 자식 클래스(가맹점)에서 이 메서드를 **재정의(Override)할 수 없습니다.**
    
- **비유:** 프랜차이즈의 '브랜드 로고'나 '상호명'. 강남점이든 부산점이든 점장 마음대로 간판 이름을 바꿀 수는 없죠? 이런 게 `static`입니다.

### 2. class (본사의 권장 사항: 자식들이 변경 가능!)

- **의미:** 본사에서 기본 가이드는 주지만, "각 지점 상황에 맞게 알아서 바꿔 써도 돼~" 하는 기능입니다.
    
- **특징:** 자식 클래스에서 `override` 키워드를 써서 자기 입맛에 맞게 **내용을 바꿀 수 있습니다.**
    
- **비유:** 매장의 '인사말'. 본사 매뉴얼은 "안녕하세요!"지만, 부산점에서는 "어서오이소!"라고 재정의(Override)해서 쓸 수 있는 것과 같습니다.

---

### 코드로 보는 확실한 차이

부모 클래스인 `Animal(동물)`과, 이를 상속받는 자식 클래스 `Dog(강아지)`를 만들어 보겠습니다.

**부모 클래스 설계 (Animal)**

```swift
class Animal {
    // 1. static: 모든 동물의 공통 규칙 (자식이 못 바꿈)
    static func breathe() {
        print("산소를 마십니다. (스읍-하)")
    }
    
    // 2. class: 동물마다 다를 수 있는 규칙 (자식이 바꿀 수 있음)
    class func makeSound() {
        print("동물의 기본 소리입니다.")
    }
}
```

**자식 클래스 설계 (Dog)** 강아지는 동물이니까 `Animal`을 상속받습니다. 이때 부모의 기능을 고쳐 써볼게요.

```swift
class Dog: Animal {
    // ❌ 에러 발생! static은 자식이 건드릴 수 없습니다.
    // override static func breathe() {
    //     print("강아지는 헥헥거리며 숨을 쉽니다.") 
    // }
    
    // ✅ 성공! class는 자식이 마음대로 바꿀 수 있습니다.
    override class func makeSound() {
        print("멍멍! 왈왈!")
    }
}
```

**실제로 사용해 보면?**

> `Dog.breathe()` // 출력: 산소를 마십니다. (스읍-하) -> 부모 것 그대로 씀 `Dog.makeSound()` // 출력: 멍멍! 왈왈! -> 강아지 전용으로 바꾼 것 씀

---

### 💡 실전 코딩 테스트 / 앱 개발 팁

1. **구조체(Struct)에는 class가 없습니다.** 구조체나 열거형(Enum)은 애초에 '상속'이라는 기능 자체가 없습니다. 자식이 없으니 물려주고 자시고 할 게 없죠. 그래서 구조체에서는 무조건 `static`만 사용할 수 있습니다. `class` 키워드는 오직 클래스 안에서만 씁니다!
    
2. **기본은 static을 쓰세요.** 현업에서는 실수로 누군가 부모의 중요한 로직을 건드려서 버그가 나는 것을 막기 위해, 기본적으로 `static`을 씁니다. 그러다 "아, 이 기능은 나중에 다른 클래스가 물려받아서 자기들 마음대로 수정하게 해줘야겠다" 싶을 때만 `class`로 열어주는 것이 아주 좋은 습관입니다.
