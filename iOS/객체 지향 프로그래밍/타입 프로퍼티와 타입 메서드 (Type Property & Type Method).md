우리가 평소에 만들던 변수나 함수는 보통 **'인스턴스(Instance)'**에 속하는데, 이것들은 인스턴스가 아닌 **'타입(Type) 자체'**에 속한다는 것이 핵심입니다.

---

### 1. 인스턴스 (Instance) = 생산된 자동차 🚗

- **인스턴스 프로퍼티:** 내 차의 색상, 내 차의 현재 속도. (차마다 다름)
    
- **인스턴스 메서드:** 내 차 엑셀 밟기, 내 차 브레이크 밟기.
    
- **사용법:** 차를 먼저 만들고 나서 사용합니다. `myCar.color = "Red"`

### 2. 타입 (Type) = 자동차 공장 (설계도) 🏭

- **타입 프로퍼티:** 이 공장에서 지금까지 만들어낸 **총 자동차 대수**. (모든 차가 공유하는 정보)
    
- **타입 메서드:** 공장 가동 중지하기, 공장 주소 확인하기.
    
- **사용법:** 차를 만들 필요 없이, 공장 이름(타입 이름)으로 바로 부릅니다. `CarFactory.totalCars`

---

### 실전 코드 비교

Swift에서는 앞에 **`static`**이라는 키워드를 붙여서 타입 프로퍼티와 타입 메서드를 만듭니다.

```swift
class Car {
    // 1. 타입 프로퍼티 (공장 소속: 모든 Car가 공유함)
    static var totalCarsProduced = 0
    
    // 2. 인스턴스 프로퍼티 (개별 자동차 소속: 차마다 다름)
    var color: String
    
    init(color: String) {
        self.color = color
        // 차가 한 대 만들어질 때마다 공장의 총생산량을 1씩 증가
        Car.totalCarsProduced += 1 
    }
    
    // 3. 타입 메서드 (공장 소속)
    static func printFactoryInfo() {
        print("이 공장에서는 총 \(totalCarsProduced)대의 차를 만들었습니다.")
    }
}
```

#### 어떻게 사용할까요?

```swift
// ❌ 인스턴스(개별 차)로는 타입 프로퍼티/메서드에 접근할 수 없습니다.
let myCar = Car(color: "Blue")
// myCar.totalCarsProduced // 에러 발생!

// ✅ 반드시 '타입의 이름(Car)'을 직접 적어서 사용해야 합니다.
let myCar2 = Car(color: "Red")
print(Car.totalCarsProduced) // 출력: 2

Car.printFactoryInfo() // 출력: 이 공장에서는 총 2대의 차를 만들었습니다.
```

### 우리가 이미 써본 타입 프로퍼티와 메서드들

사실 코딩 테스트나 앱 개발을 하시면서 이미 이 녀석들을 엄청나게 많이 써보셨을 겁니다!

- `Int.max` : Int가 가질 수 있는 최댓값 (타입 프로퍼티)
    
- `Double.pi` : 파이(원주율) 값 (타입 프로퍼티)
    
- `Int.random(in: 1...10)` : 랜덤 숫자 뽑기 (타입 메서드)

이것들은 `let myInt = Int()`처럼 굳이 숫자를 하나 만들지 않아도, **`Int`나 `Double`이라는 타입 이름 자체에 점(`.`)을 찍어서 바로 사용**하죠? 이게 바로 타입 프로퍼티와 타입 메서드입니다!

---

**💡 핵심 요약**

- **`static`**이 안 붙어 있으면 -> 붕어빵(인스턴스)을 직접 만들고 나서 써야 함.
    
- **`static`**이 붙어 있으면 -> 붕어빵 틀(타입)에 대고 바로 부를 수 있음. 모든 붕어빵이 공유함.
