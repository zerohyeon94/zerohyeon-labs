**비동기(Asynchronous)가 꼭 필요한 이유**는 현실 세계에서 “동시에 여러 가지 일”을 효율적으로 처리하기 위해서입니다.

### 1. **메인 스레드(=UI)가 멈추지 않게 하기 위해서**

iOS 앱, 웹 앱, 거의 모든 앱에서  
사용자에게 **빠른 반응성**을 제공하는 게 정말 중요합니다.

#### 예시

- 사용자가 버튼을 눌렀을 때,  
    만약 서버에서 데이터를 받아올 때까지 **UI가 멈춘다면**  
    앱이 “멈췄다”라고 느끼고,  
    심지어 **앱이 강제 종료(crash)** 되거나 “앱이 응답하지 않음” 경고를 볼 수도 있어요.

### 2. **I/O 작업(네트워크, 파일 읽기/쓰기 등)은 오래 걸릴 수 있음**

- 서버 통신, 이미지 다운로드, 파일 저장 등은  
    언제 끝날지 예측할 수 없고, 보통 수백 ms ~ 수 초까지 걸릴 수 있습니다.
    
- 이런 작업을 **동기로** 처리하면  
    그 동안 프로그램의 다른 작업이 전혀 진행되지 못합니다.

### 3. **비동기를 쓰면, 동시에 여러 일을 진행할 수 있음**

- 서버에 데이터 요청
    
- 동시에 UI는 계속 반응
    
- 데이터가 오면 “그때” UI를 업데이트

이렇게 **기다리지 않고** 앱이 계속 동작할 수 있습니다.

### 4. **실제 코드 예시**

#### 동기(Synchronous)

```swift
let data = downloadImage() // 서버에서 이미지 받기 (여기서 멈춰있음)
imageView.image = UIImage(data: data)
```

- 사용자는 앱이 “먹통”이 된 것처럼 느낄 수 있음

#### 비동기(Asynchronous)

```swift
downloadImageAsync { data in
    imageView.image = UIImage(data: data)
}
// 이 사이에 UI는 계속 반응함!
```

- 사용자는 앱이 멈추지 않고,  
    이미지를 다운받는 동안에도 다른 작업을 할 수 있음

---

## ✅ **정리**

- **UI 멈춤 방지**: 앱이 먹통이 되지 않도록
    
- **사용자 경험(UX) 향상**: 빠르고 자연스러운 반응
    
- **효율적인 리소스 사용**: 여러 작업을 동시에, 효율적으로
