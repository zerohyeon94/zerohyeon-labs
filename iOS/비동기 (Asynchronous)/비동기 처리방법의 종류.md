## 1. **콜백(Callback) 패턴**

가장 전통적인 방식  
함수를 실행할 때, 완료 후 실행할 함수를 같이 넘겨줌

```swift
func downloadImage(url: String, completion: @escaping (Data?) -> Void) {
    // 네트워크 요청 후
    completion(imageData)
}

downloadImage(url: "http://image.com") { data in
    print("이미지 다운로드 완료!")
}
```

---

## 2. **클로저(Closure) 활용 (Swift의 콜백과 유사)**

비동기 작업의 완료 시점을 클로저로 전달

```swift
func fetchData(completion: @escaping (Result<MyData, Error>) -> Void) {
    // 비동기 네트워크 요청
    // 작업 끝나면
    completion(.success(myData))
}
```

---

## 3. **GCD(Grand Central Dispatch) / DispatchQueue**

스레드를 직접 제어하지 않고,  
작업을 큐에 넣어서 백그라운드/메인 스레드에서 실행

```swift
DispatchQueue.global().async {
    // 비동기 작업 (예: 네트워크)
    let data = downloadSomething()
    
    DispatchQueue.main.async {
        // UI 업데이트
    }
}
```

---

## 4. **OperationQueue**

더 복잡한 비동기 작업(여러 작업 순서 제어 등)에 사용

```swift
let queue = OperationQueue()
queue.addOperation {
    // 비동기 작업
    OperationQueue.main.addOperation {
        // UI 업데이트
    }
}
```

---

## 5. **RxSwift**

리액티브 프로그래밍 방식  
이벤트 스트림을 구독하고, 비동기 데이터 흐름을 관리

```swift
viewModel.dataObservable
    .subscribe(onNext: { data in
        print("데이터 도착:", data)
    })
    .disposed(by: disposeBag)
```

---

## 6. **async/await (Swift 5.5+ 최신 방식)**

동기 코드처럼 작성하지만 실제로는 비동기로 동작

```swift
func fetchData() async throws -> MyData {
    // 비동기 네트워크 요청
}

Task {
    let data = try await fetchData()
    print("받은 데이터:", data)
}
```

---

## 7. **Delegate 패턴**

비동기 이벤트(예: 다운로드 완료)를 delegate 메서드로 알려줌  
(예: URLSession, UITableView 등)

```swift
protocol DownloadDelegate: AnyObject {
    func didFinishDownload(data: Data)
}
```

# ✅ **정리 (iOS에서 비동기 처리 대표 예시)**

- 콜백 / 클로저
    
- GCD(DispatchQueue)
    
- OperationQueue
    
- RxSwift 등 리액티브
    
- async/await
    
- Delegate
    

각 방법은 상황에 따라 장단점이 다르고,  
요즘은 **async/await**과 **RxSwift**를 많이 사용합니다!
