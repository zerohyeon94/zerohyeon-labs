## 📘 Day 7: `enum`, `struct`, `class` 차이와 실전 활용

Swift의 데이터 모델링 핵심이자, **앱 아키텍처 구성의 근간**이 되는 개념입니다.

---

## ✅ 먼저 3줄 요약부터

|타입|요약 설명|용도|
|---|---|---|
|`enum`|**경우의 수 (상태, 타입)** 표현에 적합|상태, 이벤트, 타입 구분 등|
|`struct`|**값 타입** – 복사됨|모델, DTO, ViewModel|
|`class`|**참조 타입** – 공유됨|뷰컨, 뷰, 싱글톤, 상속 등|

---

## 1️⃣ `enum` – 열거형

> 제한된 **값의 집합**을 표현할 때 사용

```swift
enum DeviceState {
    case idle
    case connecting
    case connected
    case error(String)
}
```

### 사용 예

```swift
let state: DeviceState = .connected

switch state {
case .idle: print("대기 중")
case .error(let message): print("오류 발생: \(message)")
default: break
}
```

> ✅ `enum`은 case마다 연관값을 가질 수 있어 상태 관리에 유리

---

## 2️⃣ `struct` – 구조체

> **값을 복사**하여 전달. 기본 타입(Int, String 등)도 전부 struct

```swift
struct User {
    var name: String
    var age: Int
}

var a = User(name: "Jo", age: 28)
var b = a
b.name = "Kim"
print(a.name) // "Jo" → 복사되었기 때문
```

> ✅ `struct`는 값 타입이므로 데이터 일관성이 높고 메모리 안정성 ↑

---

## 3️⃣ `class` – 클래스

> **참조 타입**, 같은 인스턴스를 여러 곳에서 공유

```swift
class Profile {
    var nickname: String = "Guest"
}

let a = Profile()
let b = a
b.nickname = "Jo"
print(a.nickname) // "Jo" → 같은 인스턴스를 참조
```

> ✅ 뷰 컨트롤러, 뷰 계층처럼 **상태 공유/동기화가 필요한 구조**에 사용

---

## 📊 차이 요약 표

| 항목           | `struct`    | `class`          |
| ------------ | ----------- | ---------------- |
| 타입           | 값 타입 (copy) | 참조 타입 (공유)       |
| 상속           | ❌ 불가        | ✅ 가능             |
| ARC 관리       | ❌ 없음        | ✅ 있음 (메모리 관리 필요) |
| mutating 키워드 | ✅ 필요        | ❌ 불필요            |
| 추천 사용처       | 데이터 모델      | UI, 네트워크 매니저 등   |

---

## 🧠 시니어 팁

- ✅ **struct가 기본!** 필요할 때만 class 사용
    
- ✅ `mutating func`은 struct 안에서 프로퍼티를 바꿀 수 있게 해줌
    
- ✅ `enum`은 상태 패턴, API 응답 타입 구분 등에 매우 유용
    
- ✅ RxSwift의 `Driver`, `Observable`, MVVM의 ViewModel 등도 대부분 `struct` 기반 설계
