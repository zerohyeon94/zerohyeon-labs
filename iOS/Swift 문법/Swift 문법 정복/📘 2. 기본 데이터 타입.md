## 📘 1단계 Day 2: 기본 데이터 타입

### 🔹 목표

- `String`, `Int`, `Double`, `Bool` 이해
- `Array`, `Dictionary`, `Set` 기초 사용
- 타입 안전성과 Swift 컨벤션 체득

---

### 🔹 핵심 개념 요약

|타입|설명|예시|
|---|---|---|
|`String`|문자열|`"Hello"`|
|`Int`|정수|`42`|
|`Double` / `Float`|실수 (64bit / 32bit)|`3.14`, `3.0`|
|`Bool`|참/거짓|`true`, `false`|
|`Array<T>`|순서 있는 목록|`[1, 2, 3]`|
|`Dictionary<K, V>`|키-값 쌍|`["name": "Jo"]`|
|`Set<T>`|중복 없는 집합|`Set([1, 2, 3])`|

> 💡 **Swift는 타입 안전(type safety)** 언어입니다. 값의 타입이 잘못되면 컴파일 오류로 알려줍니다.

---

### 🔹 예제 코드

```swift
// 기본 타입
let name: String = "Yeonghyeon"
let age: Int = 28
let height: Double = 174.5
let isDeveloper: Bool = true

// 배열
let favoriteNumbers: [Int] = [3, 7, 42]
print(favoriteNumbers[1])  // 7

// 딕셔너리
let userInfo: [String: Any] = [
    "name": name,
    "age": age,
    "isDeveloper": isDeveloper
]
print(userInfo["name"] ?? "")

// 셋 (중복 제거)
let numbers = Set([1, 2, 2, 3])
print(numbers) // {1, 2, 3}
```

---

### 🧠 시니어 개발자 꿀팁

- ✅ **Array vs Set 선택 기준**
    - 정렬된 순서가 중요 → `Array`
    - 중복 제거, 포함 여부 빠르게 검사 → `Set`
- ✅ **Dictionary에서 타입 불확실할 때 Any 대신 enum으로 모델링 권장**
	```swift
	enum UserKey: String {
	    case name, age, isDeveloper
	}
	
	let user: [UserKey: String] = [.name: "Jo"]
	
	```
- ✅ **`isEmpty`, `contains`, `count` 자주 쓰는 컬렉션 메서드 숙지**
