## 📘 클로저(Closure)란?

> 클로저는 **이름이 없는 함수** 또는 **코드 블럭**이며, 다른 변수처럼 **값으로 전달하거나 저장**할 수 있습니다.

Swift에서 클로저는 다음과 같은 용도로 쓰입니다:

1. **변수/상수에 저장하는 함수**
2. **함수의 인자로 전달되는 콜백**
3. **컬렉션 고차함수(map, filter 등)에 전달**
4. **비동기 작업의 완료 처리 (completion handler)**

---

### 🔹 클로저 문법 구조

```swift
{ (매개변수) -> 반환타입 in
    실행할 코드
}
```

예시:
```swift
let greeting = { (name: String) -> String in
    return "Hello, \(name)"
}

print(greeting("Jo")) // 출력: Hello, Jo
```

---

### 🔹 클로저 vs 함수

|항목|함수 (`func`)|클로저 (`{ ... }`)|
|---|---|---|
|이름|있음|없음 (익명)|
|선언 위치|파일 내 어디든 가능|변수/상수, 인자로 전달|
|캡처|가능|가능|
|용도|명확한 기능 정의|콜백, 전달용 코드 블록 등|

---

### 🔹 클로저의 **축약 문법**

Swift에서는 클로저를 더 짧게 쓰도록 여러 축약 규칙이 존재합니다.

#### ✅ 기본 형태

```swift
let add = { (a: Int, b: Int) -> Int in
    return a + b
}
```

#### ✅ 타입 추론

```swift
let add: (Int, Int) -> Int = { a, b in
    return a + b
}
```

#### ✅ 암시적 반환

```swift
let add: (Int, Int) -> Int = { $0 + $1 }
```

#### ✅ 고차함수 예

```swift
let numbers = [1, 2, 3, 4]
let doubled = numbers.map { $0 * 2 }  // [2, 4, 6, 8]
```

---

### 🔹 클로저 캡처 (캡처 리스트)

클로저는 **자신이 선언된 외부의 변수나 상수**를 **캡처(기억)** 합니다.  
이 특성을 이해하는 것이 중요합니다.

```swift
func makeCounter() -> () -> Int {
    var count = 0
    return {
        count += 1
        return count
    }
}

let counter = makeCounter()
print(counter()) // 1
print(counter()) // 2
```

- 위 클로저는 `count`를 **캡처**해서 계속 값을 유지합니다.

---

### 🧠 시니어 꿀팁

- ✅ 클로저는 메모리 순환 참조(Retain Cycle)의 원인이 될 수 있습니다. → `capture list`에서 `[weak self]`, `[unowned self]` 사용
- ✅ RxSwift, URLSession, UIView.animate 등 대부분의 비동기 처리에 클로저가 사용됨
- ✅ 클로저는 **지연 실행되는 코드 블록**이라고 이해하면 용도 파악이 쉬움

---

## 🎯 예시 목표

- 클로저를 함수 내부 변수로 선언
- 클로저에서 `self`를 참조함
- `[weak self]`를 사용해서 순환 참조 방지

---

## 🧪 예시 코드

```swift
class ProfileManager {
    var name: String

    init(name: String) {
        self.name = name
    }

    func updateNameWithClosure() {
        print("🔹 updateNameWithClosure() 진입")

        // 클로저 변수 선언
        let update: () -> Void = { [weak self] in
            guard let self = self else {
                print("❗️self가 이미 해제됨")
                return
            }
            self.name = "UpdatedName"
            print("✅ 이름이 업데이트 되었습니다: \(self.name)")
        }

        // 클로저 실행
        update()
    }

    deinit {
        print("💀 ProfileManager 메모리 해제됨")
    }
}
```

---

### ✅ 사용 예

```swift
func test() {
    var manager: ProfileManager? = ProfileManager(name: "Yeonghyeon")
    manager?.updateNameWithClosure()
    manager = nil  // 이 시점에 deinit 호출됨
}

test()
```

---

### 🧠 설명 포인트

|요소|설명|
|---|---|
|`let update: () -> Void`|클로저를 함수 내 변수로 선언|
|`[weak self]`|클로저가 `self`를 강하게 참조하지 않도록 함|
|`manager = nil`|외부에서 참조를 끊었을 때 `deinit`이 정상 호출됨|

---

### 🧼 순환 참조 방지 이유

- 클로저는 기본적으로 자신이 참조하는 객체(`self`)를 **강하게 캡처**합니다.
    
- 클래스 인스턴스가 클로저를 갖고 있고, 그 클로저가 다시 인스턴스를 참조하면 → **순환 참조 발생 → 메모리 해제 안됨**
    
- 따라서 `self`를 `[weak self]` 또는 `[unowned self]`로 캡처해서 해결

---

## ✅ 클로저를 매개변수로 받는 함수 예시

```swift
class ProfileManager {
    var name: String

    init(name: String) {
        self.name = name
    }

    func updateNameWithClosure(_ completion: (() -> Void)?) {
        print("🔹 이름 업데이트 중...")

        self.name = "UpdatedName"

        // 전달받은 클로저 호출
        completion?()
    }

    deinit {
        print("💀 ProfileManager 해제됨")
    }
}
```

---

## 🧪 사용 예: 클로저를 전달하면서 호출

```swift
func test() {
    var manager: ProfileManager? = ProfileManager(name: "Yeonghyeon")

    manager?.updateNameWithClosure { [weak manager] in
        guard let manager = manager else { return }
        print("✅ 이름이 성공적으로 업데이트 되었습니다: \(manager.name)")
    }

    manager = nil  // 클로저에서 weak로 캡처했기 때문에 메모리 누수 없음
}
```

---

## 🧠 시니어 꿀팁

|팁|설명|
|---|---|
|`[weak self]`를 클로저 안에서 사용|캡처 순환 방지|
|`@escaping` 키워드|함수 밖에서 나중에 실행되는 클로저인 경우 필요 (아래 참고)|
|옵셔널 클로저 처리|`completion?()` 형태로 안전하게 호출 가능|

---

## 💡 추가: `@escaping`이 필요한 경우

```swift
func doSomethingLater(completion: @escaping () -> Void) {
    DispatchQueue.main.asyncAfter(deadline: .now() + 1) {
        completion()
    }
}
```

- 클로저가 **함수 종료 이후에도 실행될 수 있다면** `@escaping`이 필요합니다.
- 예: 네트워크 응답, 타이머, 애니메이션 콜백 등

---

### 📌 정리

```swift
func updateNameWithClosure(_ completion: (() -> Void)?) { ... }
```

- ✅ 클로저를 매개변수로 받을 수 있다
- ✅ 내부에서 `completion?()`으로 실행 가능
- ✅ 필요하면 `[weak self]`, `@escaping`을 함께 고려해야 함
