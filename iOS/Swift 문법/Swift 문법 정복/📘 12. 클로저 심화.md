클로저는 Swift에서 **일급 시민 함수**로 취급되며, 콜백, 비동기 처리, UI 구성, 고차함수 등에 매우 자주 사용됩니다.

---

### 📚 이론 정리

#### 1. 클로저 기본 형식

```swift
{ (매개변수) -> 반환형 in
    실행코드
}
```

> 매개변수(parameter): 함수나 클로저가 호출될 때 외부에서 전달받는 값(인자, argument)을 받아들이기 위해 선언한 변수

#### 2. 클로저 캡처(Capture)

```swift
func makeIncrementer(amount: Int) -> () -> Int {
    var total = 0
    return {
        total += amount
        return total
    }
}

let incrementByTwo = makeIncrementer(amount: 2)
print(incrementByTwo()) // 2
print(incrementByTwo()) // 4
```

- 클로저는 주변 컨텍스트의 변수를 **캡처하고 유지**할 수 있음

#### 3. 후행 클로저 (Trailing Closure)

```swift
UIView.animate(withDuration: 0.3) {
    view.alpha = 0
}
```

- 마지막 인자가 클로저일 때 괄호 바깥으로 뺄 수 있음

#### 4. 탈출 클로저 (@escaping)

```swift
func fetchData(completion: @escaping (String) -> Void) {
    DispatchQueue.global().async {
        sleep(1)
        completion("데이터 완료")
    }
}
```

- 클로저가 함수의 **스코프 밖에서 호출**될 수 있을 때 사용

#### 5. 클로저와 메모리 누수

```swift
class MyClass {
    var value = 10
    
    func doSomething() {
        someAsync {
            [weak self] in
            print(self?.value ?? 0)
        }
    }
}
```

- `self`를 캡처할 때는 `[weak self]`, `[unowned self]` 사용 권장
---
### 📌 정리

| 구분        | 설명                                       |
| --------- | ---------------------------------------- |
| 캡처        | 주변 변수 값을 저장하고 유지                         |
| 후행 클로저    | 마지막 인자가 클로저일 때 사용                        |
| @escaping | 비동기 함수 등에서 나중에 실행될 때 사용                  |
| 고차함수      | `map`, `filter`, `reduce` 등에서 클로저 자주 사용됨 |
| 메모리 누수 주의 | 클로저 내부 `self` 캡처 시 `[weak self]` 필요      |
