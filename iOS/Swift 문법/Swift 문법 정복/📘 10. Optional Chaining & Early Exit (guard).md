## 핵심 주제

1. ✅ **옵셔널 체이닝 (Optional Chaining)**
    
2. ✅ **Early Exit를 위한 `guard` 구문**

이 둘은 Swift에서 **안정성과 가독성**을 동시에 높여주는 대표적인 기능입니다.

---

### ✅ 1. Optional Chaining 이란?

옵셔널 체이닝은 **옵셔널 값이 `nil`일 경우에도 안전하게 호출**을 이어가도록 도와주는 문법입니다.

```swift
let name: String? = "YeongHyeon"
print(name?.count) // Optional(9)
```

### ⛔️ 옵셔널을 안전하게 호출하지 않으면?

```swift
let name: String? = nil
print(name!.count) // 💥 런타임 크래시 발생!
```

> `name?.count`는 `name`이 nil이면 `nil` 반환하고,  
> `name!.count`는 강제 언래핑으로 위험

---

### 📌 실전 예시 (옵셔널 체이닝)

```swift
class User {
    var profile: Profile?
}

class Profile {
    var nickname: String?
}

let user = User()
let nickname = user.profile?.nickname // nil
```

→ 중간에 하나라도 `nil`이면 전체가 `nil` 반환됨

## 🎯 실제 앱에서 어디서 많이 쓰일까?

|상황|사용 여부|
|---|---|
|IBOutlet UI 연결 여부|✅ `label?.text = ...`|
|중첩된 데이터 접근 (JSON, 모델 객체 등)|✅ `user.profile?.nickname`|
|RxSwift Optional Operator 구성 (ex. compactMap, map)|✅|
|뷰컨트롤러 전환 시 특정 값 설정 여부|✅ `navigationController?.pushViewController(...)`|

### ✅ 2. `guard`를 활용한 Early Exit

Swift에서는 `guard`를 통해 **조건이 안 맞으면 빠르게 빠져나가고**,  
조건이 만족하면 로직을 계속 이어가는 패턴을 추천합니다.

```swift
func greet(name: String?) {
    guard let name = name else {
        print("이름이 없습니다.")
        return
    }
    print("안녕하세요, \(name)님!")
}
```

> `guard`는 **조건을 만족하지 않으면 바로 exit**합니다  
> **`if let`보다 가독성과 안정성이 좋음**

---

### ✅ `guard` vs `if let`

|조건 처리 방식|설명|
|---|---|
|`if let`|조건 만족 시 내부 블록에서만 사용 가능|
|`guard let`|조건 만족 시 **함수 전체 범위**에서 사용 가능 (반드시 `else { return }` 필요)|
