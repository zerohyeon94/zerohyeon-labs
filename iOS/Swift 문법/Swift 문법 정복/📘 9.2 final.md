## ✅ `final` 키워드란?

> `final`은 **상속 또는 override를 금지**하는 키워드입니다.

즉, **이 클래스나 함수는 더 이상 하위 클래스에서 재정의(override)하거나 상속받을 수 없다**는 뜻입니다.

---

## 🧱 사용 위치 및 의미

|위치|예시|의미|
|---|---|---|
|`final class`|`final class APIService {}`|이 클래스를 상속 불가|
|`final func`|`final func doSomething()`|이 메서드를 override 불가|
|`final var`|`final var name: String`|이 프로퍼티는 override 불가 (서브클래스에서 재정의 X)|

---

## 🧠 왜 사용하는가? (장점)

|목적|설명|
|---|---|
|🔒 **불필요한 상속 방지**|의도치 않은 오용을 막아 설계를 고정할 수 있음|
|🚀 **컴파일 최적화**|Swift는 `final`이 붙은 메서드는 동적 디스패치 대신 **정적 디스패치(static dispatch)** 사용 → 성능 향상|
|✅ **안정성 확보**|ViewModel, Singleton 같은 구조는 변경되면 안되므로 `final`로 고정|

---

## ✅ 예시: 왜 `final class`를 쓰는가?

```swift
final class APIService {
    static let shared = APIService()
    private init() {} // 외부에서 생성 불가
}
```

🔍 이 경우 `APIService`는:

- 다른 곳에서 상속받아 변형되면 안 됨
    
- 싱글톤 패턴 보장을 위해 `final`로 봉인함

---

## ⛔ 만약 `final`을 안 쓴다면?

```swift
class APIService { ... }
class HackedService: APIService {
    override func fetchUserProfile(...) {
        // API 결과를 조작하거나 의도와 다르게 사용할 수도 있음
    }
}
```

→ 예상치 못한 오버라이드로 안정성이 무너질 수 있습니다.

---

## 📌 정리

| 상황            | `final` 사용 추천                     |
| ------------- | --------------------------------- |
| ViewModel     | ✅ (ViewModel은 override 대상이 아님)    |
| Singleton 클래스 | ✅ (상속 방지 목적)                      |
| 핵심 메서드        | ✅ (불변 함수로 동작 보장)                  |
| 테스트나 Mock 용   | ❌ (오히려 상속이 필요한 경우 `final` 제거해야 함) |
