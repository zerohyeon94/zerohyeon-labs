## ✅ Swift에서의 프로퍼티란?

> 클래스나 구조체 내부에 선언된 **변수 또는 상수**

Swift에는 총 4가지 주요 프로퍼티가 있습니다:

1. **Stored Property** (저장 프로퍼티)
    
2. **Computed Property** (연산 프로퍼티)
    
3. **Property Observers** (`willSet`, `didSet`)
    
4. **Lazy Stored Property** (`lazy`)

---

## 1️⃣ Stored Property – 값을 저장하는 변수

```swift
struct User {
    var name: String     // stored
    let age: Int         // stored
}
```

- 실제 메모리에 값을 저장
    
- `struct`에서는 반드시 초기화 필요

---

## 2️⃣ Computed Property – 연산된 값을 반환 (저장 안 함)

```swift
struct Rectangle {
    var width: Double
    var height: Double

    var area: Double {
        return width * height
    }
}
```

- `area`는 실제로 저장되는 값이 아니라, `get`만 존재하는 연산 결과

```swift
var halfWidth: Double {
    get { width / 2 }
    set { width = newValue * 2 }
}
```

---

## 3️⃣ Property Observers – 변경 감지용

```swift
var score: Int = 0 {
    willSet {
        print("이제 \(newValue)점으로 바뀌려 합니다")
    }
    didSet {
        print("기존: \(oldValue), 새 값: \(score)")
    }
}
```

> ✅ 주로 **UI 업데이트, 로깅, 알림 전송**에 사용됩니다

---

## 4️⃣ Lazy Stored Property – 느리게 초기화

```swift
struct DataManager {
    lazy var data: [String] = loadHeavyData()

    func loadHeavyData() -> [String] {
        print("📦 데이터 로딩 중...")
        return ["A", "B", "C"]
    }
}
```

- `lazy`는 **처음 접근될 때까지 메모리에 로딩하지 않음**
    
- `var`만 가능 (`let`에는 lazy 사용 불가)
    
- 주로 **무거운 연산, 네트워크 캐시, 뷰 생성 지연** 등에 사용

---

## 📦 정리 표

|타입|저장 여부|특징|예시|
|---|---|---|---|
|Stored|✅|값 그대로 저장|`var name: String`|
|Computed|❌|연산 결과 반환|`var area: Double { width * height }`|
|Observers|✅|변경 전/후 감지|`willSet`, `didSet`|
|Lazy|✅ (지연)|처음 접근 시 초기화|`lazy var image = load()`|

---

## 🧠 시니어 팁

|상황|추천|
|---|---|
|연산 기반 결과가 필요할 때|`computed` 사용|
|무거운 작업을 나중에|`lazy` 사용|
|값 변경 감지 (UI 갱신 등)|`didSet` 활용|
|`lazy`는 종종 **뷰 또는 매니저 객체 생성 시점 지연**용으로도 쓰임||
