## ✅ 언제 어떤 컬렉션을 사용할까?

### 1. **Array - 순서가 중요할 때**

> **💬 예시: 채팅 앱의 메시지 목록**

```swift
let messages: [String] = ["안녕", "잘 지내?", "내일 보자!"]

for message in messages {
    print(message)
}
```

- 순서가 중요하기 때문에 `Array`가 적합
    
- index 접근, `.first`, `.last`, `.map`, `.filter` 등 고차함수와 잘 어울림   

---

### 2. **Set - 중복 제거가 필요할 때**

> **💬 예시: 최근 검색어 저장소**

```swift
var recentKeywords: Set<String> = []

recentKeywords.insert("iOS")
recentKeywords.insert("Swift")
recentKeywords.insert("iOS") // 무시됨

print(recentKeywords) // ["Swift", "iOS"]
```

- 중복을 자동으로 제거해주므로 `Set`이 적합
    
- 특히 빠른 탐색이 장점 (`O(1)` 성능)
    

---

### 3. **Dictionary - 키 기반 접근이 필요할 때**

> **💬 예시: 사용자 정보 저장**

```swift
let user: [String: String] = [
    "id": "123",
    "name": "YeongHyeon",
    "role": "iOS Developer"
]

print(user["name"] ?? "") // "YeongHyeon"
```

- `"name"`으로 바로 접근 가능
    
- 키 기반 조회가 핵심일 때 매우 강력
    

---

## 🧩 JSON 파싱에서의 활용

### ✅ 서버에서 오는 JSON:

```json
{
  "id": 123,
  "name": "YeongHyeon",
  "skills": ["Swift", "RxSwift", "UIKit"]
}
```

### Swift로 디코딩 시:

```swift
struct User: Decodable {
    let id: Int
    let name: String
    let skills: [String]  // ✅ 배열 사용
}
```

> 여기에 사용된 컬렉션:

- `skills`는 순서 있는 목록이므로 `Array`
    
- 딕셔너리 전체 구조는 `[String: Any]` 형태로 디코딩도 가능함

---

### 예시: JSON을 Dictionary로 파싱할 때

```swift
if let path = Bundle.main.path(forResource: "user", ofType: "json"),
   let data = try? Data(contentsOf: URL(fileURLWithPath: path)),
   let json = try? JSONSerialization.jsonObject(with: data) as? [String: Any] {
    
    print(json["name"] ?? "") // Dictionary 기반 접근
}
```

---

## 🧠 핵심 정리

|컬렉션|실전 사용 예시|이유|
|---|---|---|
|`Array`|리스트, 메시지, 순서 있는 피드|순서 있음, index 접근|
|`Set`|태그, 최근 검색어, 중복 제거용 필터|중복 제거, 빠른 탐색|
|`Dictionary`|JSON 응답 파싱, 사용자 속성|키-값 쌍으로 빠른 조회|
