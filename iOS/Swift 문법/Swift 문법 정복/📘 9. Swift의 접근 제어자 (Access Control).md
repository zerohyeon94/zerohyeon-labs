## 🧠 접근 제어자란?

> Swift에서 **클래스, 함수, 변수 등에 접근할 수 있는 범위**를 제한하는 기능입니다.

---

## ✅ Swift의 접근 제어 종류

|제어자|설명|사용 위치|
|---|---|---|
|`open`|다른 모듈에서 **상속 및 override 가능**|Framework SDK 외부|
|`public`|다른 모듈에서 **접근만 가능**, 상속/override 불가|공용 인터페이스|
|`internal` (기본값)|**같은 모듈 내에서만** 접근 가능|대부분의 일반 코드|
|`fileprivate`|**같은 Swift 파일 내에서만** 접근 가능|테스트 유닛, 파일 내부 공유|
|`private`|**같은 스코프 내에서만** 접근 가능|클래스/struct 내부 숨김|

---

## 🎯 실전 예제

```swift
public class PublicAnimal {
    public var name: String
    public init(name: String) {
        self.name = name
    }
}

class InternalZoo {
    var animals: [PublicAnimal] = [] // internal

    fileprivate func feedAll() {
        print("🍽 모든 동물에게 먹이를 줍니다")
    }

    private func secretCleanUp() {
        print("🧹 직원 전용 청소")
    }
}
```

|위치|접근 가능 여부|
|---|---|
|같은 모듈 외부|PublicAnimal은 접근 가능, InternalZoo는 불가능|
|다른 파일|feedAll()은 가능 (`fileprivate`), secretCleanUp은 불가능|

---

## 💡 언제 어떤 제어자를 써야 할까?

|상황|추천 제어자|
|---|---|
|외부 모듈에서 사용될 클래스/메서드|`public`, `open`|
|모듈 내부 공유|`internal`|
|파일 내 extension / 테스트와 공유|`fileprivate`|
|내부 구현 감춤 (View 내부 등)|`private`|

---

## 🧪 실습 코드: 접근 제어자 예제

```swift
class BankAccount {
    private var balance: Int = 0
    fileprivate let owner: String

    init(owner: String) {
        self.owner = owner
    }

    public func deposit(_ amount: Int) {
        balance += amount
    }

    public func showBalance() -> Int {
        return balance
    }
}
```

### 사용:

```swift
let myAccount = BankAccount(owner: "영현")
myAccount.deposit(1000)
print(myAccount.showBalance()) // 1000
// myAccount.balance → ❌ private 접근 불가
```

---

## ✅ 마무리 요약

- 대부분의 앱 개발에선 `internal`, `private`, `fileprivate` 위주 사용
    
- 프레임워크, 라이브러리 제작 시 `public`, `open` 필수 고려
    
- 접근 제어자는 **캡슐화와 책임 분리의 핵심**
