### 1. 강한 순환 참조 (Strong Retain Cycle)와 메모리 누수

스위프트는 ARC(Automatic Reference Counting)를 통해 메모리를 관리합니다. 객체를 참조(사용)하는 횟수를 세어서 0이 되면 메모리에서 해제하는 방식입니다.

클로저 내부에서 클래스의 프로퍼티나 메서드에 접근하려면 `self`를 캡처(Capture)해야 합니다. 기본적으로 클로저는 외부의 값을 **강하게(Strong)** 캡처합니다.

문제는 **클래스(self)가 @escaping 클로저를 소유하고, 그 클로저가 다시 클래스(self)를 강하게 캡처할 때** 발생합니다.

서로가 서로를 꽉 잡고 놓아주지 않기 때문에, 어느 쪽도 메모리에서 해제되지 못하고 영원히 남아있는 **메모리 누수(Memory Leak)** 가 발생합니다.

**문제 상황 코드:**

```Swift
class ProfileViewController {
    var name: String = "사용자"
    
    // 네트워크 요청 클로저를 프로퍼티로 저장 (클래스가 클로저를 소유)
    var completionHandler: (() -> Void)?
    
    func fetchProfile() {
        completionHandler = {
            // 클로저가 self를 강하게 캡처 (클로저가 클래스를 소유)
            print("이름 업데이트: \(self.name)")
        }
    }
    
    deinit {
        print("ProfileViewController 메모리 해제됨") 
        // ⚠️ 순환 참조로 인해 이 코드는 영원히 호출되지 않습니다!
    }
}
```

### 2. [weak self]를 통한 해결

이러한 악순환의 고리를 끊기 위해 캡처 리스트(Capture List)인 `[weak self]`를 사용합니다.

`weak` 키워드를 사용하면 클로저가 `self`를 참조하긴 하지만, **ARC 참조 횟수를 증가시키지 않습니다(약한 참조).** 따라서 클래스는 필요할 때 언제든 정상적으로 메모리에서 해제될 수 있습니다.

**해결 코드:**

```Swift
class ProfileViewController {
    var name: String = "사용자"
    var completionHandler: (() -> Void)?
    
    func fetchProfile() {
        // [weak self]를 캡처 리스트에 추가
        completionHandler = { [weak self] in
            // self가 약한 참조이므로, 클로저 실행 시점에 self가 메모리에서 해제되었을 수 있음
            // 따라서 self는 항상 옵셔널(Optional) 타입이 됨
            
            // 옵셔널 바인딩을 통해 self가 살아있을 때만 안전하게 실행
            guard let self = self else { return }
            print("이름 업데이트: \(self.name)")
        }
    }
    
    deinit {
        print("ProfileViewController 메모리 해제됨") 
        // ✅ 뷰 컨트롤러가 닫히면 정상적으로 해제되고 이 문구가 출력됩니다!
    }
}
```

> **핵심 요약:** `@escaping` 클로저 내에서 `self`를 사용할 때, 클래스와 클로저 간의 강한 순환 참조(Retain Cycle)로 인한 메모리 누수를 방지하기 위해 `[weak self]`를 사용하여 약한 참조로 만들어줍니다. 이때 `self`는 옵셔널이 되므로 `guard let` 구문으로 안전하게 언래핑하여 사용하는 것이 일반적입니다.
