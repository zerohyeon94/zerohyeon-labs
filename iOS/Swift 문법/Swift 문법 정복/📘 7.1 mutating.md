## 🎯 한 줄 요약

> `mutating`은 구조체(struct)나 열거형(enum)의 **내부 속성을 변경할 수 있게 해주는 키워드**입니다.

---

## ✅ 왜 필요한가요?

- Swift의 `struct`와 `enum`은 **값 타입(value type)** 입니다.
    
- 이 말은 **함수 내부에서 속성을 바꾸면 복사본만 바뀌고, 원본은 그대로**라는 의미입니다.
    
- 따라서 Swift는 `struct`/`enum`이 자신의 프로퍼티를 바꾸려면 개발자가 **명확히 선언적으로 "이 메서드는 값을 바꿀 거야"라고 표시하도록 강제**합니다.

→ 그 표시가 바로 `mutating`

---

## 🧪 예시: struct에서 `mutating` 사용

```swift
struct Counter {
    var count = 0

    mutating func increment() {
        count += 1  // ✅ 내부 값을 변경하기 때문에 mutating 필요
    }
}
```

사용:

```swift
var counter = Counter()
counter.increment()
print(counter.count)  // 1
```

> 만약 `mutating`을 빼면 컴파일 에러:

```swift
Cannot assign to property: 'self' is immutable
```

---

## ❌ class에서는 왜 필요 없나요?

- `class`는 **참조 타입(reference type)** 이기 때문에,  
    내부 프로퍼티를 바꿔도 이미 메모리 주소로 접근하고 있어서 `mutating` 없이도 가능합니다.

```swift
class CounterClass {
    var count = 0

    func increment() {
        count += 1  // ✅ OK, mutating 필요 없음
    }
}
```

---

## 🧠 `mutating`이 필요한 다른 경우

|상황|설명|
|---|---|
|`struct` 내부에서 프로퍼티 변경|✅ 반드시 `mutating` 필요|
|`enum`에서 `self =`으로 케이스 변경|✅ `mutating func` 안에서만 가능|
|클래스에서|❌ 필요 없음|

---

## 📦 예: enum에서 `mutating` 사용

```swift
enum Direction {
    case north, south, east, west

    mutating func turnRight() {
        switch self {
        case .north: self = .east
        case .east: self = .south
        case .south: self = .west
        case .west: self = .north
        }
    }
}
```

---

## ✅ 정리

|항목|설명|
|---|---|
|`mutating`이란?|값 타입(struct/enum)이 내부 값을 바꾸는 걸 허용하는 키워드|
|언제 필요?|속성 변경, `self` 재할당, enum 케이스 변경 등|
|class에서는?|필요 없음 (참조 타입)|
