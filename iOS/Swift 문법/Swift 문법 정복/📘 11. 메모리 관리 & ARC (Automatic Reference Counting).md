## ✅ 메모리 관리 & ARC

Swift는 **Automatic Reference Counting (ARC)** 를 통해 메모리를 자동으로 관리합니다.  
그러나 개발자가 참조 순환(Retain Cycle)을 이해하지 못하면 **메모리 누수**가 발생할 수 있습니다.

---

### 📌 1. ARC란?

Swift는 객체가 더 이상 사용되지 않을 때 자동으로 메모리를 해제합니다.  
이때 사용되는 것이 **Reference Count(참조 횟수)** 입니다.

```swift
class Person {
    var name: String
    init(name: String) {
        self.name = name
        print("👶 \(name) 초기화됨")
    }
    deinit {
        print("💀 \(name) 메모리 해제됨")
    }
}
```

```swift
var person1: Person? = Person(name: "YeongHyeon")
person1 = nil // 💀 deinit 호출 → ARC에 의해 메모리 해제됨
```

---

### 📌 2. 강한 참조 순환 (Strong Reference Cycle)

두 객체가 서로를 강하게 참조하면 ARC가 참조 수를 0으로 만들지 못해 메모리 누수가 발생합니다.

```swift
class Person {
    var pet: Pet?
}

class Pet {
    var owner: Person?
}
```

```swift
var john: Person? = Person()
var puppy: Pet? = Pet()

john?.pet = puppy
puppy?.owner = john

john = nil
puppy = nil // ❌ 둘 다 메모리 해제되지 않음
```

---

### 📌 3. 해결 방법: `weak`과 `unowned`

|키워드|설명|
|---|---|
|`weak`|참조 대상이 **Optional**이며, 참조 해제 시 자동으로 nil|
|`unowned`|참조 대상이 **Non-Optional**, 객체가 먼저 해제되면 크래시 발생|

```swift
class Pet {
    weak var owner: Person?
}
```

```swift
class Pet {
    unowned var owner: Person // ⚠️ owner가 먼저 해제되면 크래시
}
```

---

### 📌 4. 클로저에서의 캡처와 순환 참조

클로저는 기본적으로 자신이 정의된 범위의 변수들을 **강하게 캡처**합니다.

```swift
class ViewController {
    var name = "VC"
    
    func setup() {
        DispatchQueue.main.async {
            print(self.name) // 🔁 self를 강하게 캡처 → 순환 참조 가능
        }
    }
}
```

✅ 해결 방법: **[weak self] 또는 [unowned self] 캡처 리스트 사용**

```swift
DispatchQueue.main.async { [weak self] in
    guard let self = self else { return }
    print(self.name)
}
```
