## 🔍 `for-in` vs `forEach` 차이

|항목|`for-in`|`forEach`|
|---|---|---|
|문법|전통적인 for문|클로저 기반 메서드|
|흐름 제어 (`break`, `continue`)|✅ 가능|❌ 불가능|
|사용 시점|간단하고 유연한 반복|고차함수 스타일 반복|
|클로저 캡처|없음|`self` 캡처 가능성 있음|

---

### 📌 예제 비교

#### ✅ `for-in`: 흐름 제어 가능, 직관적

```swift
for i in 1...5 {
    if i == 3 { continue }
    print(i)
}
// 출력: 1, 2, 4, 5
```

#### ❌ `forEach`: break, continue 불가 → 주의!

```swift
(1...5).forEach { i in
    if i == 3 { return }  // 이 return은 forEach 클로저 "탈출"만 함
    print(i)
}
// 출력: 1, 2, 4, 5 가 아닌 → 1, 2 만 출력되고 종료 (헷갈림 주의!)
```


---

### 🧠 시니어 팁: 언제 뭘 쓰면 좋을까?

| 상황                     | 추천 방식     | 이유                    |
| ---------------------- | --------- | --------------------- |
| 조건 분기, 중단 필요한 경우       | `for-in`  | break/continue 가능     |
| 간단한 일괄 처리              | `forEach` | map, filter와 조합하면 깔끔함 |
| UI 코드에서 self 캡처에 민감할 때 | `for-in`  | 클로저에서의 캡처 방지          |

---

### 💬 결론

> `for-in`은 직관적이고 흐름 제어가 가능하기 때문에 대부분의 반복에서 더 안전하고 유연합니다.  
> `forEach`는 **"모든 항목을 무조건 다 순회"** 하며 클로저 스타일로 쓸 수 있지만, 제어 흐름을 바꾸는 데는 적합하지 않습니다.
