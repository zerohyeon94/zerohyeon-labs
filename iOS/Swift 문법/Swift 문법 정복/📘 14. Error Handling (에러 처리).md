Swift에서 에러 처리는 앱이 예상치 못한 상황(네트워크 끊김, 데이터 오류 등)에서 **비정상 종료(Crash)되지 않고 안전하게 대처**하기 위해 필수적입니다.

|**키워드**|**설명**|
|---|---|
|`Error`|에러 타입을 정의할 때 채택하는 프로토콜|
|`throw`|에러를 발생시킴 (던짐)|
|`throws`|함수가 에러를 발생시킬 수 있음을 표시|
|`do-catch`|에러를 잡아서 처리하는 구문|
|`try`|에러가 발생할 수 있는 함수를 실행할 때 사용|

---

### 1. **에러 정의하기 (Error Protocol)**

Swift에서는 주로 `enum`(열거형)을 사용하여 에러 종류를 정의합니다.

```swift
// 에러 프로토콜 채택
enum LoginError: Error {
    case invalidId           // 아이디 불일치
    case shortPassword       // 비밀번호 너무 짧음
    case serverError(code: Int) // 서버 에러 (연관 값으로 정보 전달 가능)
}
```

- **Error 프로토콜 채택 필수**
    
- `enum`을 사용하면 에러 종류를 명확하게 그룹화할 수 있음

---

### 2. **에러 발생시키기 (throw & throws)**

함수 내부에서 조건에 맞지 않을 때 에러를 던집니다.

```swift
// 함수 뒤에 'throws'를 붙여야 에러를 던질 수 있음
func login(id: String, pw: String) throws {
    if id != "admin" {
        throw LoginError.invalidId // 에러 발생(던지기)
    }
    
    if pw.count < 6 {
        throw LoginError.shortPassword
    }
    
    print("로그인 성공!")
}
```

- **throws**: 이 함수는 에러가 날 수 있다고 명시 (반환 타입 화살표 `->` 앞에 작성)
    
- **throw**: 실제로 에러를 발생시키는 키워드 (`return`과 유사하게 함수가 종료됨)

---

### 3. **에러 처리하기 (do-catch)**

에러를 던지는 함수는 반드시 `try`와 함께 `do-catch` 문으로 감싸서 실행해야 합니다.

```swift
do {
    // try를 사용하여 함수 실행
    try login(id: "user", pw: "1234")
    
} catch LoginError.invalidId {
    print("아이디를 확인해주세요.")
    
} catch LoginError.shortPassword {
    print("비밀번호는 6자리 이상이어야 합니다.")
    
} catch {
    print("알 수 없는 에러: \(error)") // 그 외 모든 에러 처리
}
```

- **do 블록**: 정상 실행 시도
    
- **catch 블록**: 에러 발생 시 실행될 코드 (Switch문처럼 분기 처리 가능)

---

### 4. **간단한 에러 처리 (try? vs try!)**

`do-catch` 없이 간단하게 처리하고 싶을 때 사용합니다.

```swift
// 1. try? (Optional 반환)
// 에러가 나면 nil, 성공하면 Optional 값 반환
let result = try? login(id: "wrong", pw: "123") 
// result는 nil이 됨 (앱이 죽지 않음)


// 2. try! (Forced Unwrapping)
// 에러가 절대 안 난다고 확신할 때 사용. 에러 나면 앱 강제 종료(Crash) 💥
try! login(id: "admin", pw: "longpassword123") 
```

- **try?**: 실패해도 괜찮을 때 유용 (가장 많이 사용)
    
- **try!**: 테스트 코드나, 100% 확신할 때만 사용 (위험함)

---

### 5. **에러 전달 (rethrows)**

함수 자체가 에러를 던지는 것이 아니라, **"매개변수로 받은 클로저가 에러를 던질 때만"** 덩달아 에러를 던지도록 할 때 사용합니다. Swift의 `map`, `filter` 등이 이렇게 설계되어 있습니다.

```swift
// 함수 자체는 에러를 안 던지지만, callback이 던지면 나도 던짐(rethrows)
func performAction(action: () throws -> Void) rethrows {
    try action()
}

// 1. 에러 안 던지는 클로저 전달 -> try 불필요
performAction { print("Hello") }

// 2. 에러 던지는 클로저 전달 -> try 필요
do {
    try performAction { throw NetworkError.timeout }
} catch {
    print("에러 발생")
}
```

- **특징**: 입력받은 함수(클로저)의 성격(Throwing 여부)에 따라 내 함수도 유연하게 반응함.

---

### 6. **비동기 에러 처리 (async throws)**

Swift 5.5부터 도입된 Concurrency(`async/await`) 환경에서 에러를 처리하는 방식입니다.

```swift
func fetchUserData() async throws -> String {
    // 네트워크 통신 가정
    throw NetworkError.serverError(code: 500)
}

func load() async {
    do {
        // try가 await보다 앞에 옴
        let data = try await fetchUserData()
        print(data)
    } catch {
        print("비동기 에러 처리: \(error)")
    }
}
```

- **주의**: 키워드 순서는 항상 `try`가 먼저, `await`가 나중입니다. (`try await`)

---

### 7. **사용자 에러 메시지 (LocalizedError)**

단순히 에러 타입(`enum`)만으로는 사용자에게 "서버 연결 실패" 같은 친절한 메시지를 보여주기 어렵습니다. `LocalizedError`를 채택하면 이를 해결할 수 있습니다.

```swift
enum PaymentError: Error, LocalizedError {
    case insufficientFunds
    case limitExceeded
    
    // 이 프로퍼티를 구현하면 alert 메시지로 바로 사용 가능
    var errorDescription: String? {
        switch self {
        case .insufficientFunds:
            return "잔액이 부족합니다. 충전 후 다시 시도해주세요."
        case .limitExceeded:
            return "일일 결제 한도를 초과했습니다."
        }
    }
}

// 사용 예시
let error: PaymentError = .insufficientFunds
print(error.localizedDescription) // "잔액이 부족합니다. 충전 후 다시 시도해주세요."
```

- **실무 팁**: `alert.message = error.localizedDescription` 형태로 바로 UI에 바인딩할 때 유용합니다.

### 8. **결과 타입 (Result)**

성공(`Success`)과 실패(`Failure`)를 하나의 `enum` 값으로 포장해서 반환하는 타입입니다. 비동기 처리나 컴플리션 핸들러(Completion Handler)에서 자주 사용됩니다.

```swift
// Result<성공타입, 에러타입>
func requestData() -> Result<String, NetworkError> {
    let isSuccess = true
    
    if isSuccess {
        return .success("데이터 로드 완료")
    } else {
        return .failure(.timeout)
    }
}

// switch문으로 깔끔한 분기 처리 가능
switch requestData() {
case .success(let data):
    print(data)
case .failure(let error):
    print("에러 발생: \(error)")
}
```

- **장점**: 에러 처리를 `do-catch` 없이 값(Value)처럼 주고받을 수 있음.

### 9. **Swift 6: Typed Throws (타입 명시 에러)**

Swift 6 이전에는 `catch` 블록의 `error`가 무조건 `any Error`(범용 에러) 타입이라서, 구체적인 타입을 알기 위해 타입 캐스팅(`as?`)을 해야 했습니다. **Typed Throws**는 던지는 에러의 타입을 명확히 지정합니다.

```swift
// Swift 5.x (기존) -> 에러 타입 명시 불가
func oldFunc() throws { ... }

// Swift 6 (신규) -> NetworkError만 던진다고 명시!
func newFunc() throws(NetworkError) {
    throw .timeout
}

do {
    try newFunc()
} catch {
    // 여기서 error는 자동으로 'NetworkError' 타입임 (타입 캐스팅 불필요)
    switch error {
    case .timeout: print("타임아웃")
    case .serverError: print("서버 에러")
    // LoginError 같은 엉뚱한 에러는 고려할 필요 없음
    }
}
```

- **장점**: `catch` 블록에서 어떤 에러가 올지 명확하므로 코드가 훨씬 안전해지고 간결해짐.

## 🎯 요약
| **구분**     | **키워드 / 타입**      | **특징**                    | **사용 상황**                    |
| ---------- | ----------------- | ------------------------- | ---------------------------- |
| **기본 정의**  | `enum : Error`    | 에러 종류 나열                  | 커스텀 에러 타입을 만들 때              |
| **메시지 정의** | `LocalizedError`  | `errorDescription` 구현     | Alert에 띄울 친절한 메시지가 필요할 때     |
| **기본 발생**  | `throws`, `throw` | 함수가 실패 가능함을 알림            | 유효성 검사 등 로직 실패 시             |
| **조건부 발생** | `rethrows`        | 인자(클로저)가 에러 낼 때만 던짐       | `map`, `filter` 같은 고차함수 구현 시 |
| **타입 명시**  | `throws(Type)`    | 던지는 에러 타입을 확정 (Swift 6)   | 에러 타입 캐스팅 없이 명확히 처리할 때       |
| **비동기 처리** | `async throws`    | 비동기 + 에러 처리 (`try await`) | 네트워크 통신 등 시간이 걸리는 작업         |
| **정석 처리**  | `do-catch`        | 에러 종류별로 분기 처리             | 사용자에게 에러 상황을 알리고 대처할 때       |
| **간편 처리**  | `try?`            | 실패 시 `nil` 반환             | 에러 내용보다 성공/실패 여부만 중요할 때      |
| **강제 처리**  | `try!`            | 실패 시 `Crash` (앱 종료)       | 테스트 코드 등 100% 성공을 확신할 때      |
| **값으로 처리** | `Result<T, E>`    | 성공/실패를 값(Value)으로 포장      | 비동기 콜백이나 에러를 변수로 저장할 때       |
