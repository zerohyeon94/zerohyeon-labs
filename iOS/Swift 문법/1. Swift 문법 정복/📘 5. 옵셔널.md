## 📘 Day 5: 옵셔널(Optional) 완전 정복

---

## ✅ 옵셔널이란?

> 옵셔널은 **"값이 있을 수도 있고, 없을 수도 있는" 타입**입니다.  
> Swift는 **`nil`을 사용할 수 있는 변수는 반드시 옵셔널로 선언**하게 강제합니다.

```swift
var name: String? = nil  // ✅ 가능
var age: Int = nil       // ❌ 컴파일 에러 (Int는 옵셔널 아님)
```

---

## 🔸 옵셔널 선언 방법

```swift
var name: String? = "Jo"
var score: Int? = nil
```

- `String?`는 `Optional<String>`의 축약형
- 옵셔널은 값이 없을 수 있으므로 안전하게 다뤄야 함

---

## 🔍 옵셔널 처리 방법들

### 1. **Force Unwrap** – ❌ 위험

```swift
let name: String? = "Jo"
print(name!)  // 강제 언래핑 → 위험 (nil이면 Crash)
```

### 2. **Optional Binding (`if let`)**

```swift
if let name = name {
    print("안녕하세요, \(name)")
} else {
    print("이름이 없습니다")
}
```

### 3. **guard let (함수 안에서 조기탈출)**

```swift
func greet(name: String?) {
    guard let name = name else {
        print("이름이 없음")
        return
    }
    print("안녕하세요, \(name)")
}
```

> ✅ `guard`는 **초기 검증 실패 시 빠르게 리턴**하는 코드 스타일에 적합합니다.

---

### 4. **Nil-Coalescing 연산자 (`??`)**

```swift
let nickname: String? = nil
let displayName = nickname ?? "사용자"
print(displayName)  // "사용자"
```


> ✅ 옵셔널이 nil이면 기본값 사용

---

### 5. **Optional Chaining**

```swift
struct User {
    var profile: Profile?
}
struct Profile {
    var bio: String
}

let user = User(profile: nil)
let bio = user.profile?.bio  // bio는 String?
```

> ✅ 중간에 nil이 있으면 전체 결과가 nil이 됩니다. (안전!)

---

## 🧠 시니어 꿀팁

- 옵셔널을 unwrap할 땐 항상 **안전하게** (`if let`, `guard let`)
- `??`는 매우 자주 쓰이는 패턴 (기본값 처리)
- RxSwift에서도 `.map { $0?.name }`, `.filter { $0 != nil }` 등 자주 등장
- 모델 초기화 시에도 옵셔널 여부가 매우 중요 (`init?`, `init(from:)`)
