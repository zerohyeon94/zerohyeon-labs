### 1. **Coordinator의 생명주기(메모리 해제) 문제**

- **LoginCoordinator**가 지역 변수로만 생성되고 강하게 참조되지 않으면, 화면 전환 전에 사라져서 **finishFlow 클로저가 호출되지 않음**
    
- **AppCoordinator**가 하위 Coordinator를 프로퍼티로 **강하게(Strong) 참조**해야 함

#### (예시: 잘못된 경우)

```swift
func showLoginFlow() {
    let loginCoordinator = LoginCoordinator(navigationController: navigationController)
    loginCoordinator.finishFlow = { ... }
    loginCoordinator.start()
    // loginCoordinator가 함수 끝나면 사라짐(ARC에 의해)
}
```

- `LoginCoordinator`가 `AppCoordinator`의 프로퍼티로 **강하게(Strong) 참조**되지 않고  
    함수 내부에서만 생성(local let)된 경우,  
    **`viewModel.loginSuccess.subscribe…`가 호출되기 전에**  
    이미 `LoginCoordinator` 객체가 ARC(자동 참조 카운팅)로 메모리에서 해제됨
    
- 이 경우 `self`는 이미 nil이므로,  
    아무리 로그인에 성공해도 `self?.finishFlow?()`에서 아무 동작도 하지 않습니다.
    
- → 클로저가 동작 안하거나, 화면 전환 안 됨

#### (해결: 프로퍼티로 보관)

```swift
var loginCoordinator: LoginCoordinator? // AppCoordinator 프로퍼티로!

func showLoginFlow() {
    loginCoordinator = LoginCoordinator(navigationController: navigationController)
    loginCoordinator?.finishFlow = { [weak self] in
        self?.showMainFlow()
        self?.loginCoordinator = nil // 필요 시 해제
    }
    loginCoordinator?.start()
}
```

## 추가적으로 MainCoordinator 이후의 동작을 할 경우에도 프로퍼티로 들고 있어야한다.
## 🔑 **왜 MainCoordinator도 프로퍼티로 들고 있어야 하나?**

- `let mainCoordinator = MainCoordinator(...)` **로만 선언하면**  
    `showMainFlow()` 함수가 끝나자마자  
    **mainCoordinator는 ARC(자동 참조 카운팅)**에 의해 바로 메모리에서 해제됩니다.
    
- 만약 MainCoordinator가
    
    - 하위 Coordinator(다른 화면 흐름)
        
    - 또는 딥링크/화면 분기 등에서 “이벤트 클로저”를 사용한다면  
        **중간에 mainCoordinator가 사라져서 흐름이 끊길 수 있습니다.**
        
- **로그인 플로우와 똑같이**  
    `var mainCoordinator: MainCoordinator?` **로 프로퍼티로 선언**해야  
    “흐름 유지”와 “메모리 릭 방지” 모두 잡을 수 있습니다.
    

---

## 🛠️ **올바른 구조 예시**

```swift
class AppCoordinator: Coordinator {
    var navigationController: UINavigationController
    var loginCoordinator: LoginCoordinator?
    var mainCoordinator: MainCoordinator?      // <- 추가!

    // ...생략

    func showMainFlow() {
        mainCoordinator = MainCoordinator(navigationController: navigationController)
        // mainCoordinator?.finishFlow = { ... } // 필요하다면
        mainCoordinator?.start()
    }
}
```

---

## 💡 **정리**

- **Coordinator 패턴에서 “화면 흐름을 제어하는 객체”는 반드시 프로퍼티로 들고 있어야 함**
    
    - **로컬 변수(let/var mainCoordinator = …)**는 함수 리턴 시 해제됨 → 비정상 동작
        
    - **프로퍼티(var mainCoordinator: …)**는 객체가 Coordinator에 붙어 있는 동안 유지

---

### 🚩 **실무 팁**

- 여러 개의 Coordinator를 계층적으로 쓸 경우  
    (예: AppCoordinator → MainCoordinator → DetailCoordinator)  
    각 단계마다 프로퍼티로 들고 있게 설계
    
- 화면 플로우가 끝나면 **nil 할당**해서 메모리 릭 방지
