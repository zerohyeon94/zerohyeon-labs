## 1. 핵심 개념 비교

### 명령형 프로그래밍 (UIKit)
- **접근 방식**: **"어떻게(How)"** 할 것인가를 서술.
- **특징**: 개발자가 UI 객체의 상태 변경을 하나하나 직접 제어해야 함.
- **예시**: "라벨의 텍스트를 'A'로 바꿔라", "버튼의 색을 빨강으로 칠해라".
- **단점**: 상태가 복잡해지면 어떤 코드가 화면을 바꿨는지 추적하기 어려워짐 (버그 발생 원인).

### 선언형 프로그래밍 (SwiftUI)
- **접근 방식**: **"무엇(What)"**이 되어야 하는가를 서술.
- **특징**: 화면과 데이터의 관계(Rule)만 정의해 두면, 데이터 변경 시 시스템이 알아서 화면을 갱신함.
- **예시**: "이 라벨은 `userName` 변수의 값을 보여주는 것이다".
- **장점**: 코드의 가독성이 높고, 상태와 UI의 불일치 문제(Sync 문제)가 사라짐.

## 2. 공식: UI = f(State)
선언형 UI의 본질은 **"뷰(UI)는 상태(State)의 함수(Function)"**라는 것이다.

- **State(데이터)**가 입력되면 -> **View(화면)**가 출력된다.
- 데이터가 바뀌면 함수를 다시 실행해서 새로운 화면을 만든다. (Re-rendering)

## 3. 코드 비교

| 구분          | UIKit (명령형)                          | SwiftUI (선언형)                         |
| :---------- | :----------------------------------- | :------------------------------------ |
| **상태 변경 시** | `label.text = newValue` <br> (직접 수정) | `state = newValue` <br> (데이터만 수정하면 끝) |
| **흐름**      | 이벤트 → 핸들러 → **UI 객체 직접 조작**          | 이벤트 → **상태 변경** → 뷰 자동 갱신             |

## 설먕
### 1. 차이점 직관적으로 이해하기

#### 👨‍✈️ 명령형 (UIKit): "과정"을 지시함

주인공(개발자)이 뷰에게 직접 명령을 내립니다.

> "야, 텍스트 라벨 가져와. 걔한테 '안녕하세요'라고 적어. 그리고 배경색을 빨간색으로 칠해. 만약 버튼이 눌리면 텍스트를 '반갑습니다'로 고쳐 써."

#### 🧑‍🎨 선언형 (SwiftUI): "결과(규칙)"를 정의함

주인공(개발자)이 설계도를 그려서 시스템에 줍니다.

> "여기 텍스트 뷰가 하나 있는데, 얘는 항상 `message`라는 변수의 내용을 보여주는 애야. 그리고 배경색은 `isRed`가 참이면 빨강, 아니면 파랑이야. 끝. (시스템아, 데이터 바뀌면 네가 알아서 그려)"

### 2. 코드로 보는 표현 방식의 차이

사용자가 버튼을 눌러서 **"로그인 상태"**로 바꾸는 상황을 예로 들어보겠습니다.

#### [UIKit] 명령형 (Imperative)

값이 변경되면, **내가 직접 UI를 찾아서 고쳐야 합니다.**

```swift
// 1. 상태 변수 변경
self.isLoggedIn = true

// 2. "명령"을 내려서 화면 갱신 (개발자가 하나하나 챙겨야 함)
func updateUI() {
    if isLoggedIn {
        statusLabel.text = "로그인 완료" // 👈 텍스트 바꿔!
        loginButton.isHidden = true      // 👈 버튼 숨겨!
        logoutButton.isHidden = false    // 👈 버튼 보여줘!
    } else {
        statusLabel.text = "로그인 해주세요"
        // ... 반대 상황도 일일이 작성
    }
}
```

#### [SwiftUI] 선언형 (Declarative)

값이 변경되면, **UI는 자동으로 따라옵니다.** 나는 "이 변수랑 이 화면은 한 몸이야"라고 선언만 해두면 됩니다.

```swift
struct LoginView: View {
    // 1. 상태 선언 (@State)
    @State var isLoggedIn: Bool = false
    
    // 2. 화면 정의 (규칙 선언)
    var body: some View {
        VStack {
            // "텍스트는 isLoggedIn 값에 따라 이거 아니면 저거다"라고 선언
            Text(isLoggedIn ? "로그인 완료" : "로그인 해주세요")
            
            if isLoggedIn {
                Button("로그아웃") { isLoggedIn = false }
            } else {
                Button("로그인") { isLoggedIn = true } // 👈 값만 바꾸면 끝!
            }
        }
    }
}
```

**차이점:** SwiftUI에서는 버튼을 누를 때 `label.text = ...` 처럼 뷰를 직접 건드리는 코드가 **전혀 없습니다.** 그저 `isLoggedIn`이라는 **데이터(State)**만 바꿨을 뿐인데, 시스템이 알아서 `body`를 다시 읽고 화면을 업데이트합니다.

---

### 3. 핵심 공식: UI = f(State)

선언형 프로그래밍을 가장 잘 설명하는 수학 공식입니다.

- **UI**: 현재 화면의 모습
    
- **State**: 데이터 (예: 월급, 로그인 여부, 토글 상태)
    
- **f (Function)**: 뷰(View) 구조체

**"화면(UI)은 데이터(State)를 넣으면 나오는 결과물(Function)이다."**

명령형처럼 "이거 고쳐, 저거 지워"라고 수정하는 게 아니라, **"데이터가 바뀌었으니, 새로운 데이터로 화면을 통째로 다시 그려(Render)"**라고 하는 것입니다. (이게 가능한 이유는 앞서 배운 `Struct`가 매우 가볍기 때문입니다!)
