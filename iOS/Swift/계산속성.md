## ✅ 계산 속성(computed property) 도입의 장점

### 1. **코드의 재사용성과 가독성 향상**

- **계산 로직을 한 곳에 모아둬서**  
    동일한 계산이 필요할 때마다 매번 새로 작성할 필요 없이  
    속성만 불러오면 됨
    
- 예시:
```swift
var totalPrice: Int {     items.reduce(0) { $0 + $1.price } }
```

### 2. **자동 업데이트 및 일관성**

- **데이터가 변경될 때마다 항상 최신 결과**를 반환
    
- 값이 변할 때마다 별도로 수동으로 업데이트해줄 필요 없음
    
- 예시:
```swift
var fullName: String {     "\(firstName) \(lastName)" }
```    

### 3. **불필요한 저장 공간 사용 방지**

- 계산 결과만 필요하고, 값을 저장할 필요가 없을 때  
    **메모리 절약**이 가능
    
- 중간 결과를 따로 저장하지 않고,  
    필요할 때마다 계산해서 반환

### 4. **의도와 동작이 명확해짐**

- 계산이 필요한 값임을 타입 인터페이스에서 바로 알 수 있음  
    (단순 저장 vs. 동적 계산의 차이가 드러남)
    
- Swift에서는 `var`로 표기되지만, getter만 구현하면  
    외부에서는 변수처럼 사용 가능

### 5. **의존성 관리 용이**

- 여러 속성에 걸쳐 계산이 필요할 때,  
    계산식을 변경하면 한 곳만 고치면 됨

---

## 📄 예시: CoreData 모델에서의 계산 속성

```swift
class DailyBudgetModel {
    var spendingRecords: [SpendingRecord]
    // 계산 속성
    var totalSpent: Int {
        spendingRecords.reduce(0) { $0 + $1.amount }
    }
}
```

- `totalSpent`는 항상 최신의 `spendingRecords` 상태에 따라 자동으로 업데이트됨

|장점|설명|
|---|---|
|코드 중복 감소/가독성 향상|속성으로 계산, 어디서든 재사용 가능|
|항상 최신 데이터|데이터 변화에 따라 자동 갱신|
|메모리 절약|저장 필요 없는 값은 저장하지 않음|
|의도 명확화|코드만 보고도 계산 결과임을 알 수 있음|
|유지보수 용이|계산식 변경 시 한 곳만 수정하면 됨|
