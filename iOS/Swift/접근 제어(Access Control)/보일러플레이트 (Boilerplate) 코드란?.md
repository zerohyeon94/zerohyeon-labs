## 1. 정의
프로그래밍에서 **"최소한의 변경으로 여러 곳에서 재사용되며, 반복적으로 비슷한 형태를 띠는 코드"**를 말한다.
쉽게 말해 **"상용구"** 혹은 **"찍어내는 틀"** 같은 코드다.

* **어원:** 신문 인쇄소에서 매일 똑같이 찍어내는 광고나 안내문 철판(Boilerplate)에서 유래했다.
* **특징:**
    * 로직(Logic)이 복잡하지 않다.
    * 단순히 언어의 문법이나 프레임워크의 규칙을 지키기 위해 어쩔 수 없이 작성한다.
    * 코드가 길어지고 가독성을 해친다.

## 2. 예시로 이해하기

### A. 일상 생활 비유
* **계약서 양식:** "본 계약은 갑과 을 사이의..." 처럼 내용은 똑같고 이름만 바꾸는 부분.
* **편지 인사말:** "안녕하세요, 날씨가 춥네요..." 처럼 매번 쓰는 서두.

### B. 코딩 예시 (Swift)

>[!WARNING] 보일러플레이트가 많은 코드
변수 하나를 `private(set)`으로 만들기 위해, 매번 3~4줄의 코드를 반복해서 써야 한다면 그것이 바로 보일러플레이트다.

```swift
// 변수 1개 만들 때마다 이 짓을 해야 함 (지루하고 반복적임)
private var _name: String
var name: String {
    get { return _name }
    set { _name = newValue }
}

private var _age: Int
var age: Int {
    get { return _age }
    set { _age = newValue }
}
```

>[!Good] 보일러플레이트 제거 (Swift 문법 활용)
>Swift는 이런 반복을 줄이기 위해 `private(set)` 같은 문법적 설탕(Syntactic Sugar)을 제공한다.

```swift
// 깔끔하고 핵심만 남음
private(set) var name: String
private(set) var age: Int
```

## 3. 결론

좋은 프로그래머는 이 **"보일러플레이트 코드"를 최대한 줄여서**, 핵심 로직만 눈에 잘 들어오게 만드는 것을 목표로 한다.

### 💡 요약

* **Boilerplate = "지루한 반복 코드"**
* 개발자들은 이걸 아주 싫어합니다. (시간 낭비 + 코드 읽기 힘듦)
* 아까 질문하신 `private(set)`은 바로 이 **보일러플레이트를 제거**해서 코드를 예쁘게 만들어주는 아주 좋은 기능입니다.
