## 1. 단축 평가 (Short-circuit Evaluation)

논리 연산자(`||`, `&&`)를 사용할 때, **앞의 조건만으로 전체 결과가 확정되면 뒤의 조건은 아예 무시(실행조차 하지 않음)**하는 프로그래밍 규칙입니다.

- **`||` (OR 연산자): `A || B`**
    
    - A가 `true`이면? 뒤의 B는 보지도 않고 전체를 `true`로 넘깁니다.
        
    - **효과:** `if stack.isEmpty || stack.removeLast() != expectedOpen`에서 스택이 비어있을 때(`true`), 뒤의 `removeLast()`가 아예 실행되지 않으므로 에러(크래시)를 완벽하게 막아줍니다.
        
- **`&&` (AND 연산자): `A && B`**
    
    - A가 `false`이면? 뒤의 B는 보지도 않고 전체를 `false`로 넘깁니다.

## 2. 조건문 안에서의 함수 실행 (실제 동작)

조건문(`if`) 안에 함수를 넣었다고 해서 '가짜로 결과만 예상'해보는 것이 절대 아닙니다.

- 조건문 안에서 `stack.removeLast()`를 호출하면, **그 즉시 실제로 스택의 마지막 요소가 삭제**됩니다.
    
- 삭제되면서 튀어나온(반환된) 그 값을 가지고 바로 뒤의 `!= expectedOpen`과 비교를 수행하는 것입니다.
    
- 즉, **"데이터 삭제(수정)"**와 **"조건 비교"**가 한 줄에서 동시에 일어나는 매우 효율적인 동작입니다.

## 3. 배운 것 써먹기: `popLast()`로 코드 업그레이드

제일 처음에 질문하셨던 `popLast()`의 특징을 떠올려 볼까요? "배열이 비어있으면 크래시 대신 `nil`을 반환한다."

이 특징을 방금 배운 원리와 결합하면, 복잡했던 방어 코드를 아주 우아하게 바꿀 수 있습니다.

**[수정 전]** `isEmpty`로 먼저 방어하고, `removeLast()`로 빼내기

> `if stack.isEmpty || stack.removeLast() != expectedOpen { return false }`

**[수정 후]** `popLast()` 하나로 안전하게 처리하기

> `if stack.popLast() != expectedOpen { return false }`

**왜 작동할까요?**

1. 스택이 비어있을 때: `popLast()`가 `nil`을 뱉습니다. `nil`과 `expectedOpen`(문자)은 다르니까(`!=`) 조건이 `true`가 되어 `return false`로 안전하게 종료됩니다.
    
2. 스택에 값이 있을 때: 진짜로 값을 하나 빼고(삭제) 괄호 모양이 맞는지 비교합니다.
