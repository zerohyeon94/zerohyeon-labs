## 1. 간단정의

|연산자|의미|포인트|
|---|---|---|
|`==`|**값이 같은지** 비교 (동등성)|값/내용이 같은가?|
|`===`|**같은 인스턴스인지** 비교 (식별성)|메모리 상의 같은 객체인가?|
## 2. 사용 방식

### `==` (동등 연산자)

- **값 타입 + 참조 타입 둘 다 가능**
    
- 전제: 그 타입이 `Equatable`을 채택하거나, `==` 가 구현되어 있어야 함

```swift
struct User: Equatable {
    let id: Int
}

let u1 = User(id: 1)
let u2 = User(id: 1)

u1 == u2   // ✅ true (값이 같아서)
```

### `===` (식별 연산자)

- **클래스(참조 타입)에서만 가능**
    
- `AnyObject`에만 사용 가능

```swift
class Person {
    let name: String
    init(name: String) { self.name = name }
}

let p1 = Person(name: "Tom")
let p2 = p1
let p3 = Person(name: "Tom")

p1 === p2  // ✅ true  (같은 인스턴스)
p1 === p3  // ❌ false (이름은 같지만 다른 인스턴스)
```

---

## 3. “값이 같다” vs “같은 놈이다”

### 값이 같은데, 인스턴스는 다를 수 있다 → `==`만 true

```swift
struct Point: Equatable {
    let x: Int
    let y: Int
}

let a = Point(x: 1, y: 2)
let b = Point(x: 1, y: 2)

a == b    // true (값 동일)
/// === 는 struct에 못 씀 (참조 타입이 아님)
```

### 클래스에서는 “이 사람이냐?”를 보고 싶을 때 → `===`

```swift
class Coordinator {}

let c1 = Coordinator()
let c2 = c1
let c3 = Coordinator()

c1 === c2   // true
c1 === c3   // false
```

## 4. 실제 코드에서의 사용 목적

### ✅ `==` 를 쓰는 상황

- “내용이 같은지” 보고 싶을 때
    
    - 예: 아이디가 같은 User인지, 두 Date가 같은 시각인지
        
- 컬렉션에서 값 비교
    
    - `array.contains(someValue)`
        
    - `filter { $0 == something }`

---

### ✅ `===` 를 쓰는 상황

- **“이 인스턴스 그 자체인지” 확인해야 할 때**
    
- 대표적인 예: **Coordinator 패턴**

```swift
func removeCoordinator(_ coordinatorToRemove: Coordinator) {
    childCoordinators.removeAll { $0 === coordinatorToRemove }
}
```

의미:

> childCoordinators 배열 중에서  
> **지금 종료된 이 코디네이터 인스턴스와 동일한 객체만 제거해라**

값 기반 비교가 아니라,  
**딱 이 인스턴스 한 놈만 제거**하는 게 중요하기 때문에 `===` 사용.

---

## 5. 요약 문장으로 외우기

- `==`
    
    > “**내용이 같냐?**” → 값 비교 (동등성)
    
- `===`
    
    > “**진짜 그 놈이냐?**” → 같은 인스턴스인지 비교 (식별성, 참조 동일성)
