## **Swift의 인스턴스 프로퍼티 초기화 순서**

### **1. 프로퍼티 초기화 규칙**

- 클래스(또는 struct)의 인스턴스를 만들 때  
    **모든 저장 프로퍼티(stored property)**는  
    **self가 생성되기 전에 반드시 값이 할당되어야** 합니다.

---

### **2. Swift의 초기화 순서(과정)**

#### **(1) 프로퍼티 선언부에서 기본값이 있다면, 그 값으로 우선 할당**

```swift
class User {
    var name: String = "Unknown" // 선언부에서 바로 할당
    var age: Int                 // 선언부에서 값 없으면…
    init(age: Int) {
        self.age = age           // 초기화 구문에서 반드시 할당
    }
}
```

- **선언부에서 기본값**이 있으면 먼저 할당
    
- **없으면 init에서 반드시 할당** (모든 저장 프로퍼티는 초기화가 필수)

#### **(2) 프로퍼티 선언부는 "아직 self가 완전히 생성되기 전"**

- 이 시점에서는 `self`로 다른 프로퍼티에 접근할 수 없음  
    → **자신 이외의 인스턴스 멤버 접근 불가**

---

### **3. 클래스 초기화 전체 순서**

1. **상위 클래스의 프로퍼티부터 초기화**
    
2. **하위 클래스의 프로퍼티가 순서대로 초기화**
    
3. **초기화가 완료되면, self가 완전히 생성됨**

---

### **4. Swift가 보장하는 것**

- 인스턴스가 완전히 생성되기 전에는  
    **어떠한 인스턴스 멤버(self의 프로퍼티/메서드)**도  
    **다른 프로퍼티에 의존해서는 안 됨**
- 즉,
    
    - 프로퍼티 초기화 중에는 self의 다른 프로퍼티 접근 불가
        
    - 이는 “초기화 순서 버그”를 막기 위한 언어적 안전장치

---

## **실무 예시**

### **잘못된 예 (컴파일 에러)**

```swift
class MyVC: UIViewController {
    let a = 10
    let b = a + 5      // ❌ 'a'를 이 시점에 참조 불가 (self.a와 같음)
}
```

### **올바른 예**

```swift
class MyVC: UIViewController {
    let a = 10
    lazy var b = a + 5 // lazy는 self가 생성된 후 실제로 처음 접근될 때 초기화되므로 OK
}
```

### **초기화 구문에서 할당**

```swift
class MyClass {
    let x: Int
    let y: Int
    init(value: Int) {
        self.x = value
        self.y = x + 2 // 이 시점은 self가 생성되는 중이므로 OK!
    }
}
```

---

## **요약**

- **Swift의 저장 프로퍼티는**
    
    - 선언부(=property initializer)에서 값 할당
        
    - 아니면 반드시 init에서 값 할당
        
- **self가 완전히 만들어지기 전(=프로퍼티 초기화 단계)에는**
    
    - 자기 외의 다른 프로퍼티에 접근 불가
        
    - “초기화 순서 버그”를 미연에 방지
