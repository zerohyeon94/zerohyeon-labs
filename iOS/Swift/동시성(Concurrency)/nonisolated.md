액터(Actor) 안에 있지만, 줄 서지 않고 바로 실행해도 되는 함수
### 1. 배경: 액터(Actor)는 '은행 창구'입니다.

Swift에서 `actor`는 데이터를 안전하게 보호하기 위해 **한 번에 한 작업만** 처리합니다.

- 은행 창구(Actor)에는 직원이 한 명뿐입니다.
    
- 사람들(Task)이 돈을 입금하거나 출금하려면 **줄을 서서 기다려야(`await`)** 합니다.
    
- 이걸 **"액터 격리(Actor Isolation)"**라고 부릅니다.

### 2. 문제점: "그냥 시계만 보고 싶은데요?"

그런데 은행 창구 안에 벽시계가 하나 걸려 있다고 상상해 보세요. 손님은 단지 **"지금 몇 시인가요?"**라고 묻고 싶을 뿐입니다.

- 이건 장부(돈)를 고치는 위험한 작업이 아닙니다.
    
- 그냥 벽에 걸린 시계(변하지 않는 정보)를 보는 것뿐입니다.
    
- 그런데도 은행 규칙(Actor) 때문에 번호표를 뽑고 30분을 기다렸다가 직원을 통해서 시간을 물어봐야 한다면 너무 비효율적이죠?

### 3. 해결책: nonisolated (격리 해제)

이때 사용하는 것이 바로 `nonisolated`입니다.

함수 앞에 `nonisolated`를 붙이면 이렇게 말하는 것과 같습니다. **"이 함수는 은행 장부(돈)를 건드리지 않으니, 번호표 뽑지 말고(await 없이) 그냥 아무나 바로 실행하세요!"**

---

### 실제 코드 상황으로 비교

**상황:** `User`라는 액터가 있고, `name`(변하지 않음)과 `nickname`(변할 수 있음)을 가지고 있습니다.

**1. 일반적인 액터 함수 (격리됨)**

- `nickname`을 변경하는 함수는 반드시 액터의 보호를 받아야 합니다.
    
- 그래서 외부에서 호출할 때 `await`를 써서 기다려야 합니다.

**2. nonisolated 함수 (격리 안 됨)**

- `name`은 `let`(상수)이라서 변하지 않거나, 단순히 "안녕하세요" 같은 인사를 리턴하는 함수입니다.
    
- 이런 건 굳이 기다릴 필요가 없습니다.
    
- `nonisolated`를 붙이면 외부에서 `await` 없이 일반 함수처럼 즉시 호출할 수 있습니다.

---

### 요약: 언제 쓰나요?

1. **Actor** 내부의 코드를 작성할 때.
    
2. 그 함수가 **Actor의 중요한 데이터(var)를 수정하거나 건드리지 않을 때.**
    
3. 외부에서 **await 없이 빠르게 호출**하게 만들고 싶을 때.

**주의할 점:** `nonisolated` 함수는 액터의 보호막 밖으로 나가는 것이기 때문에, 그 함수 안에서는 액터의 **변수(var)에 접근할 수 없습니다.** (안전하지 않으니까요!) 오직 상수(let)나 파라미터로 받은 값만 다룰 수 있습니다.

---

### 예시
### 상황 설정: 은행 계좌 (BankAccount)

- **계좌번호(let):** 한번 정해지면 절대 안 바뀜. (안전함)
    
- **잔고(var):** 돈을 넣고 빼니까 계속 바뀜. (위험함, 보호 필요)

```swift
actor BankAccount {
    // 1. 안전한 데이터 (상수)
    let accountNumber: String = "123-456-7890"
    
    // 2. 보호가 필요한 데이터 (변수)
    var balance: Int = 1000
    
    // ---------------------------------------------------
    
    // [상황 A] 입금하기 (일반 메서드)
    // - 잔고(balance)를 건드려야 함.
    // - 앞사람이 끝날 때까지 기다렸다가(await) 실행해야 함.
    func deposit(amount: Int) {
        self.balance += amount
    }
    
    // [상황 B] 계좌번호 확인하기 (nonisolated 메서드)
    // - 잔고는 안 건드리고, 그냥 계좌번호(상수)만 알려주면 됨.
    // - 굳이 번호표 뽑고 기다릴 필요 없음! 바로 실행!
    nonisolated func getInfo() -> String {
        return "제 계좌번호는 \(accountNumber) 입니다."
    }
}
```

---
### 실제 사용해보기 (여기가 중요!)

이 `BankAccount`를 밖에서 사용할 때 차이가 확연히 드러납니다.

```swift
let myAccount = BankAccount()

// 1. 일반 메서드 호출 (입금)
// "잠깐만요, 앞사람 업무 끝나면 처리해드릴게요."
// -> 반드시 await(기다림) 키워드가 필요함
await myAccount.deposit(amount: 500) 


// 2. nonisolated 메서드 호출 (정보 확인)
// "아, 계좌번호요? 저기 벽에 써진 거 그냥 바로 보세요."
// -> 기다릴 필요 없음. await 없이 바로 실행!
print(myAccount.getInfo())
```

### 핵심 요약

1. **`deposit` (입금):** `balance`(변수)를 바꾸는 위험한 작업이라서 **`await`**로 줄을 서야 합니다.
    
2. **`getInfo` (정보):** `accountNumber`(상수)만 읽는 안전한 작업이라서 **`nonisolated`**를 붙였고, 덕분에 **줄을 안 서고 바로(`await` 없이)** 실행됩니다.

**주의할 점:** 만약 `nonisolated`라고 써놓고 함수 안에서 슬쩍 `balance`(변수)를 건드리려고 하면? Swift 컴파일러가 **"어허! 줄 안 서고 금고(변수)를 건드리면 안 되지!"** 하고 바로 빨간 줄(에러)을 띄웁니다.
