# View Controller Containment (부모-자식 관계) 
UIKit에서 뷰 컨트롤러를 다른 컨트롤러 안에 넣을 때(Embed) 반드시 지켜야 하는 **5단계 공식**입니다. 
## 1. 5단계 공식 (The Standard Procedure) 
1. **addChild(_:)**: 부모-자식 관계 선언 (계통 등록) 
2. **view.addSubview(_:)**: 자식의 뷰를 부모 뷰 계층에 추가 (실제 화면 배치) 
3. **Auto Layout 설정**: 자식 뷰의 크기와 위치 결정 
4. **didMove(toParent:)**: 관계 설정 완료 보고 (입주 완료) 
## 2. 주요 메서드 설명 
- `addChild`: 부모가 자식의 생명주기(Appearance, Rotation 등)를 관리하도록 연결합니다. 
- `didMove(toParent:)`: 자식 컨트롤러에게 부모에게 완전히 포함되었음을 알려, 내부 로직이 정상 작동하게 합니다. 
## 3. 주의사항
단순히 `addSubview`만 할 경우, 화면 회전이나 `viewWillAppear` 같은 이벤트가 자식에게 전달되지 않아 버그가 발생할 수 있습니다.
