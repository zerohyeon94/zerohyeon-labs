## ✅ 핵심 개념: 앱이 꺼져 있어도 알림이 뜨는 이유는?

👉 **알림은 iOS 시스템 레벨에서 처리되기 때문입니다.**

즉, 앱이 백그라운드든, 꺼졌든 관계없이 **“미리 등록된 알림 요청”**은 시스템이 알아서 실행합니다.

---

## 🔔 알림 종류에 따른 동작 차이

|종류|앱 실행 여부 필요?|누가 띄움?|예시|
|---|---|---|---|
|**로컬 알림**|❌ 앱이 꺼져 있어도 됨|**iOS 시스템**|“하루 공부 알림” 등|
|**푸시 알림 (APNs)**|❌ 앱이 꺼져 있어도 됨|**iOS 시스템**|카톡 메시지, 은행 알림 등|

---

## 1. 📅 로컬 알림 (Local Notification)

### 🧠 어떻게 작동하나요?

- 앱이 **앞에서 시간/조건을 지정해서 미리 등록**해둡니다.
    
- iOS가 **스케줄을 저장하고**, 조건이 되면 앱 없이도 띄움

### ✅ 예시 코드

```swift
let content = UNMutableNotificationContent()
content.title = "공부할 시간이에요!"
content.body = "지금 10분만 집중해볼까요?"

let trigger = UNTimeIntervalNotificationTrigger(timeInterval: 60, repeats: false)

let request = UNNotificationRequest(identifier: "studyReminder", content: content, trigger: trigger)

UNUserNotificationCenter.current().add(request)
```

➡ 이 코드는 “1분 뒤에 알림을 띄워줘”라는 요청을 시스템에 넘깁니다.  
➡ **알림은 앱이 종료되어도 시스템이 띄워줍니다.**

---

## 2. 📡 푸시 알림 (Remote Notification, APNs)

### 🧠 어떻게 작동하나요?

- 서버가 **Apple Push Notification Server (APNs)**로 알림 전송
    
- APNs가 사용자 기기에서 앱을 실행 중이든 꺼져있든 간에 **시스템이 알림 띄움**

### ✅ iOS 앱에서는 이렇게 구현

- 앱이 백그라운드/종료 상태일 때도 푸시를 받도록 설정
    
- 시스템이 알림을 띄워주고, 사용자가 누르면 앱이 실행됨

---

## 🔐 주의할 점: 사용자 동의가 반드시 필요

```swift
UNUserNotificationCenter.current().requestAuthorization(
    options: [.alert, .sound, .badge]
) { granted, error in
    print("권한 granted: \(granted)")
}
```

➡ 이 요청 없이 알림은 **보낼 수 없습니다.**

|질문|답변|
|---|---|
|앱이 꺼졌는데도 알림이 뜨는 이유는?|**iOS 시스템이 대신 띄워줌 (등록된 로컬/푸시 알림)**|

|앱이 알림을 띄우는 게 아니라...|시스템이 “앱 이름으로” 대신 띄우는 것임|
