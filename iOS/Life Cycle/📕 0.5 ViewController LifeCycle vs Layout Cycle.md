# ViewController LifeCycle (화면 “등장/사라짐”의 수명 이벤트)

## 최초 표시 (push/present 기준)

1. `init(...)`
    
2. `loadView()`
    
    - 뷰 트리 생성(스토리보드면 nib 로드)
        
3. `viewDidLoad()`
    
    - 1회성 셋업: 바인딩, 데이터 소스 연결, 스타일 적용
        
4. `viewWillAppear(_:)`
    
    - 화면에 **곧 나타나기 직전**, 상태 싱크(네비/탭바 숨김, 트래킹 시작 등)
        
5. (레이아웃 사이클가 동반될 수 있음: 아래 참조)
    
6. `viewDidAppear(_:)`
    
    - 화면에 **완전히 나타난 직후**, 애니메이션 시작, 네트워크 트리거, 측정 등

## 다른 화면으로 이동/가려질 때

7. `viewWillDisappear(_:)`
    
    - 떠나기 직전, 타이머/관찰 중단 등
        
8. (레이아웃 사이클 동반 가능)
    
9. `viewDidDisappear(_:)`
    
    - 완전히 사라진 뒤, 무거운 자원 해제 등
        
10. `deinit` (해당 VC가 더 이상 참조되지 않으면 해제)

> **포인트**
> 
> - 위 메서드들은 **화면 전환**(등장/사라짐)이 있어야 호출됩니다.
>     
> - 데이터만 바뀌어도(텍스트/제약 변경) **appear/disappear는 다시 호출되지 않음**.

# Layout Cycle (뷰의 “배치 계산” 루프: Auto Layout/프레임 산출)

## 트리거(아무거나 하나가 발생하면 레이아웃 필요 상태가 됨)

- 제약 변경(상수, 활성/비활성), 뷰 추가/삭제/정렬 변경
    
- 텍스트 변경으로 **intrinsicContentSize** 변동
    
- `isHidden`, `contentInset`, `safeAreaInsets` 변화
    
- 회전/사이즈 클래스 변경, Dynamic Type 크기 변경
    
- 명시 호출: `setNeedsLayout()`(지연), `layoutIfNeeded()`(즉시)

## 한 번의 레이아웃 패스(간단 모형)

1. **Constraints pass**
    
    - (필요 시) `updateViewConstraints`(VC) / `updateConstraints()`(UIView)
        
2. **Layout pass**
    
    - VC: `viewWillLayoutSubviews()`
        
    - 각 뷰: `layoutSubviews()`에서 프레임 계산
        
    - VC: `viewDidLayoutSubviews()`

3. **Display pass(그리기)**
    
    - (필요 시) `setNeedsDisplay()` → `draw(_:)`
        
    - Core Animation이 RunLoop 턴 말에 트랜잭션 커밋


> **포인트**
> 
> - `setNeedsLayout()`은 “다음 RunLoop에서 레이아웃해” (지연, 배치에 유리)
>     
> - `layoutIfNeeded()`는 “**지금** 당장 레이아웃해” (애니메이션 블록 안에서 자주 사용)
>     
> - 레이아웃 사이클은 **VC 라이프사이클과 독립**적으로 여러 번 돌 수 있음.

# 실제 동작 시나리오 (타임라인 예시)

## A) 최초 진입 후, 버튼으로 데이터 로드 → UI 갱신

1. `init → loadView → viewDidLoad → viewWillAppear → (layout) → viewDidAppear`
    
2. 사용자가 버튼 탭(메인 스레드) → **네트워크 시작**(백그라운드 권장)
    
3. 응답 도착 → **메인 스레드에서 상태/뷰 속성 변경**(텍스트/숨김/제약 등)
    
4. 시스템이 레이아웃 필요로 표시
    
    - 다음 RunLoop에서 자동 레이아웃 **또는** 코드에서 `layoutIfNeeded()`로 즉시 반영
        
5. (필요 시) 디스플레이 패스 → 화면에 변화 렌더링
    

> 여기서 **VC의 appear 계열은 다시 호출되지 않습니다.**  
> 바뀌는 것은 **레이아웃/디스플레이 사이클**뿐.

## B) 결과에 따라 화면 전환(예: 메인 탭으로 라우팅)

1. 현재 VC: `viewWillDisappear → (layout) → viewDidDisappear`
    
2. 다음 VC: `viewWillAppear → (layout) → viewDidAppear`
