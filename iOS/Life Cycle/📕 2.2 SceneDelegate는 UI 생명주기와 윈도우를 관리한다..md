## ✅ 1. "UI 관련 생명주기"란?

> **앱의 UI가 화면에 나타나고 사라지는 전체 흐름을 Scene 단위로 관리하는 것**을 말합니다.

### Scene 생명주기 관련 메서드들:

```swift
func sceneDidBecomeActive(_ scene: UIScene)
func sceneWillResignActive(_ scene: UIScene)
func sceneWillEnterForeground(_ scene: UIScene)
func sceneDidEnterBackground(_ scene: UIScene)
```

### 각 메서드가 의미하는 동작 예시:

|메서드|상황|예시|
|---|---|---|
|`sceneDidBecomeActive`|사용자가 앱을 포그라운드에서 다시 탭함|게임 재개, 타이머 다시 시작|
|`sceneWillResignActive`|전화 수신, 알림 등으로 일시 정지됨|게임 일시정지|
|`sceneWillEnterForeground`|백그라운드 → 포그라운드 전환 직전|UI 업데이트 준비|
|`sceneDidEnterBackground`|홈버튼, 스와이프 등으로 백그라운드 진입|데이터 저장, 작업 중단|

📌 **즉, SceneDelegate는 “UI가 보이는 상태에서 보이지 않는 상태로 전환되는 것”을 감지하고 대응하는 곳입니다.**

## ✅ 2. "윈도우 관리"란?

> iOS에서 UI는 `UIWindow` 위에 표시되며, SceneDelegate가 이 **윈도우 객체를 생성하고 루트 뷰 컨트롤러를 연결하는 역할**을 합니다.

### 기본 SceneDelegate 코드:

```swift
func scene(_ scene: UIScene,
           willConnectTo session: UISceneSession,
           options connectionOptions: UIScene.ConnectionOptions) {
    
    guard let windowScene = (scene as? UIWindowScene) else { return }
    let window = UIWindow(windowScene: windowScene)
    window.rootViewController = MainViewController()
    self.window = window
    window.makeKeyAndVisible()
}
```

### 여기서 무슨 일이 일어나나?

- 시스템이 UIWindowScene을 전달해 줌
    
- SceneDelegate가 **직접 UIWindow를 생성**
    
- 이 창에 표시할 첫 번째 화면(예: `MainViewController`)을 설정
    
- 윈도우를 화면에 표시 (`makeKeyAndVisible()`)

---

### 🤔 왜 AppDelegate가 아닌 SceneDelegate에서 UI 설정을 하나요?

- **iOS 13부터 하나의 앱에 여러 개의 UI 창(Scene)을 동시에 가질 수 있게 되었기 때문**입니다.
    
- 예: iPad에서 멀티 윈도우, Stage Manager

→ AppDelegate는 앱 전체의 설정  
→ SceneDelegate는 **각 윈도우/UI의 독립적인 생명주기와 구성 관리**

| 기능          | 설명                             | 담당                           |
| ----------- | ------------------------------ | ---------------------------- |
| UI 생명주기     | 앱 화면의 활성/비활성 전환 감지             | `sceneDidBecomeActive`, etc. |
| 윈도우 생성      | UIWindow와 루트 ViewController 구성 | `scene(_:willConnectTo:)`    |
| Scene 개별 관리 | 멀티 윈도우 지원, 각각 생명주기             | SceneDelegate에서 분리 관리        |
