## ✅ 전체 코드 이해

```swift
guard let windowScene = (scene as? UIWindowScene) else { return }

window = UIWindow(windowScene: windowScene)
window?.rootViewController = UINavigationController(rootViewController: ViewController())
window?.makeKeyAndVisible()
```

---

## 1. `guard let windowScene = (scene as? UIWindowScene) else { return }`

- `UIScene`은 앱이 실행될 때마다 만들어지는 UI 환경입니다 (iOS 13부터 도입된 멀티윈도우 지원).
    
- 그 중에서도 `UIWindowScene`만이 **화면을 그릴 수 있는** Scene이기 때문에 **타입캐스팅**합니다.
    
- 실패 시 `return`으로 앱이 중단되지 않도록 방지합니다.

---

## 2. `window = UIWindow(windowScene: windowScene)`

- 코드 기반 UI에서는 `UIWindow`를 직접 생성해줘야 합니다.
    
- `UIWindowScene`을 연결해서 이 창이 어떤 Scene에 나타나야 할지 지정합니다.

---

## 3. `window?.rootViewController = UINavigationController(rootViewController: ViewController())`

### ⬅️ 여기서 궁금해하신 **`ViewController()` 괄호**의 의미:

- `ViewController`는 **클래스 이름**이고,
    
- `ViewController()`는 **인스턴스 생성**입니다.

즉, `ViewController`라는 화면을 **초기화해서 하나의 화면 객체로 만든다**는 뜻입니다.

`UINavigationController`는 여러 화면을 스택 구조로 관리하는 컨트롤러입니다.  
여기선 앱의 첫 화면을 `ViewController()`로 설정하고 있습니다.

---

## 4. `window?.makeKeyAndVisible()`

- **makeKey**: 이 `UIWindow`를 **현재 앱의 주요 창(Key Window)**으로 설정합니다.
    
- **makeVisible**: 실제 화면에 보여지게 만듭니다.

> 즉, 이 한 줄로 만든 창이 사용자에게 보여지고, 이벤트를 받을 수 있게 됩니다.

---

## 🧾 요약

| 코드                            | 의미                           |
| ----------------------------- | ---------------------------- |
| `guard let windowScene = ...` | 현재 Scene이 실제 화면을 가질 수 있는지 확인 |
| `window = UIWindow(...)`      | 코드로 UIWindow 생성              |
| `rootViewController = ...`    | 앱 첫 화면 설정                    |
| `ViewController()`            | ViewController 인스턴스 생성       |
| `makeKeyAndVisible()`         | 사용자에게 창을 보여주고 이벤트를 받게 함      |
