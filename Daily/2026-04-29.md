### 💬 기술 피드백

**Ruff 린트 — 오늘 제대로 부딪혔다**
린트가 "스타일 교정"이라고만 알고 있었다면, 오늘 경험이 그 인식을 바꿨을 것이다. 함수 중복, 정의되지 않은 이름, 과도한 함수 복잡도 — 이건 스타일이 아니라 **실제 버그와 유지보수 문제**다. Ruff가 잡아낸 것들은 다 이유가 있다.

trailing comma 수정(`"message": "...",`)도 마찬가지다. 마지막 항목에 쉼표를 붙이는 것은 Ruff의 취향이 아니라 **git diff를 깔끔하게 유지하기 위한 관행**이다. 나중에 새 항목 추가 시 기존 줄이 변경되지 않는다.

**EC2 Security Group 22번 포트 — 실무에서 반드시 만나는 문제**
SCP/SSH 접속이 CI/CD에서 막힌 원인이 Security Group이었다는 것을 직접 추적해서 찾아낸 것이 핵심이다. 이 경험이 있는 사람과 없는 사람의 차이는 크다. 앞으로는 CD 배포 워크플로우 설계 시 **Security Group 22번 오픈 여부를 체크리스트 첫 번째**로 기억해 두면 된다.

**nginx reload vs restart**
`reload`는 기존 연결을 유지한 채 설정만 다시 읽는다. `restart`는 프로세스 자체를 재시작한다. CD 파이프라인에서는 `restart`도 문제없다 — 어차피 배포 자체가 다운타임을 전제로 하는 경우가 많기 때문. 하지만 무중단 배포가 필요하다면 `reload`로 돌아와야 한다. 지금 선택은 상황에 맞다.

**헬스체크 404 + SSL 도메인 문제**
nginx에 `/health` 경로를 라우팅하지 않은 것은 흔한 실수다. SSL 인증서가 IP가 아닌 도메인 기준으로 발급된다는 것도 직접 부딪혀서 이해한 것이 중요하다. 이 두 가지는 서버 배포 경험이 없으면 이론으로는 잘 와닿지 않는 부분이다.

**미완료 항목에 대해**
블로그, ZipJoong, Git 학습이 밀렸다. CI/CD 작업이 예상보다 훨씬 길어진 것이 이유인데 — 이건 실제 문제를 해결하느라 시간이 간 것이므로 손해가 아니다. 오늘 해결한 EC2/nginx/Ruff 경험이 내일 블로그나 면접에서 꺼낼 수 있는 실제 이야기가 됐다.

**Zero-Alpha-Beta 방향에 대해**
iOS 개발자를 목표로 하면서 AI와 백엔드를 학습하는 흐름이 "섞인다"는 느낌이 드는 것은 자연스러운 불안이다. 하지만 Zero-Alpha-Beta를 macOS Electron 앱으로 만드는 선택은 오히려 좋은 방향이다 — iOS가 아닌 환경에서 Claude API를 직접 다루면서 **AI 연동 경험을 빠르게 쌓을 수 있고**, 이게 나중에 iOS 앱에 AI를 붙일 때 바탕이 된다. 분산이 아니라 쌓이는 것이다.
