### 💬 기술 피드백

**Vision Body Pose → ZipJoong 고도화 판단 — 정확한 직관이다**

"새 프로젝트보다 ZipJoong에 먼저 적용하는 게 낫겠다"는 판단이 맞다. 이유가 있다:
- Body Pose를 처음 쓸 때 가장 많이 막히는 부분은 API 사용법이 아니라 **좌표계 변환**과 **카메라 방향(orientation)** 처리다. ZipJoong은 이미 AVCaptureSession이 구현되어 있어서 이 진입 비용을 절반으로 줄여준다.
- 새 프로젝트에서 처음부터 하면 카메라 세팅 → Body Pose → UI → 각도 계산을 동시에 잡아야 한다. ZipJoong에서 먼저 Body Pose 하나만 붙여보면, 포트폴리오 앱에서는 UI와 각도 계산 개선에 집중할 수 있다.

**AVCaptureSession sessionQueue — 핵심을 잡았다**

"세션을 어느 시점에, 어떤 스레드에서 실행하느냐"가 ZipJoong 로딩 지연의 뿌리다. 구체적으로:
- `startRunning()` / `stopRunning()`은 **블로킹 호출**이다. 메인 스레드에서 부르면 그 시간 동안 UI가 얼어붙는다.
- sessionQueue 패턴은 전용 직렬 큐(`DispatchQueue(label: "sessionQueue")`)를 만들어 모든 세션 조작을 여기서만 실행한다. 메인 스레드는 UI만 담당.
- 내일 CameraService를 열면 `startRunning()`이 `DispatchQueue.main` 또는 `DispatchQueue.global()` 위에서 불리는지 먼저 확인하면 된다. 그게 원인이다.

**Natural Language Framework 관심 — 의료 용어 앱과 연결된다**

NLLanguageRecognizer, NLTagger(감정 분석)에 흥미가 생겼다는 것이 의미 있다. 5번 후보(의료 용어 AI 해설 앱)에서 Vision OCR로 텍스트를 추출한 뒤 NL Framework로 의학 용어를 태깅하면 **서버 없이** 핵심 단어를 분리할 수 있다. 두 관심이 연결된다는 걸 기억해두면 좋다.

**하루업 백엔드 참여 — 내일 저녁 전까지 구조 파악이 목표**

`mission_chatbot_test` 브랜치가 최신이라는 것을 이미 확인했다. 내일 약속 전에 파악해야 할 것:
1. 백엔드 기술 스택 (FastAPI인지, Django인지, Spring Boot인지)
2. `mission_chatbot` 관련 라우터/서비스 파일 위치
3. 현재 어디까지 구현됐고 어디가 막혀 있는지

깊게 이해하려 하지 말고, **"무슨 역할을 내가 맡게 되는가"를 파악하는 수준**이면 내일 대화에 충분하다.
