### 💬 기술 피드백

**On-device AI**
MobileNetV3-Small 선택 이유를 스스로 짚어낸 것이 좋음. "배터리 및 용량 고려"는 정확한 판단이고, 이 선택이 곧 포트폴리오에서 "왜 이 모델을 썼나"에 대한 답이 된다. 면접에서 바로 쓸 수 있는 언어다.

궁금증("Vision 없이 CoreML만으로 졸음 판단이 가능한가?")에 대한 답: 가능하다. 현재 Vision이 계산해주는 EAR(눈 종횡비)을 대신해 직접 얼굴 랜드마크 좌표를 CoreML 분류 모델의 입력으로 넣어 "졸음/집중" 이진 분류를 학습시킬 수 있다. 모델 정확도가 올라가면 CoreML 비율을 30%에서 더 높일 수 있다.

**CI/CD**
ci.yml 수정 결과를 보면 개념을 정확히 이해한 수준으로 구현했다. MySQL 서비스 컨테이너, uv 전환, Python 3.13, ruff 린트+포맷 분리까지 한 번에 처리한 것은 단순히 "버전 숫자 바꾸기"가 아니라 프로젝트 전체 패키지 관리 흐름을 이해하고 맞춘 것이다. 이 수준이면 내일 CD(자동 배포) 워크플로우도 충분히 작성 가능하다.

"CI/CD 구현 시 테스트는 어떤 형태인가?" → 내일 pytest 기본 패턴 학습 파일 준비해둘 예정.

**보완할 부분**
ZipJoong을 오늘 확인하지 못했다. 뽀모도로 이슈는 코드를 봐도 알 수 있지만, 직접 실행해서 현상을 재현해보는 것이 먼저다. 내일 반드시 앱을 켜서 뽀모도로 버튼을 눌러보고 어떤 상황인지 적어두는 것이 필요하다.
