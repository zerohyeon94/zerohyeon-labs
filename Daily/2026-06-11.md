## 🌙 오늘 하루 피드백 (2026-06-11)

### ✅ 완료한 것

- 하루업 프로젝트 회의를 완료하고, iOS 변경 사항과 Backend/API 방향성을 정리했다.
- iOS에서는 직업/목표/관심사 UI를 우선 숨기고 핵심 기능 테스트를 먼저 진행하는 방향이 잡혔다.
- 오늘의 추천 미션은 최근 코드 반영 후 화면 표시를 확인했다.
- 나루 애니메이션 적용과 구 큐레이션 미사용 파일 정리가 완료되었다.
- 다음 주부터 Go/React 기반 프로젝트를 진행할 가능성을 확인했고, 그 전에 개발환경을 빠르게 점검해야 한다는 과제를 잡았다.

### 💬 기술 피드백

오늘 회의의 핵심은 "무엇을 더 만들까"보다 "지금 테스트 가능한 핵심 기능만 남기자"에 가깝다. 직업, 목표, 관심사 설정을 우선 숨기는 결정은 기능을 포기한 것이 아니라 QA 범위를 줄이는 결정이다. 지금 단계에서는 화면을 풍부하게 만드는 것보다 기존 서비스가 안정적으로 동작하는지, 신규 챗봇 서비스와 무엇이 달라지는지 비교 가능한 상태를 만드는 것이 더 중요하다.

Backend 쪽은 `오늘의 추천 미션`이 사용 확정 API로 올라왔고, `나의 기록에서 완료한 미션 정보 표시`, `다른 미션 추천 기능`, `프롬프트 앱 설치 추천 문구 제거`가 이번 주의 실제 변경 후보로 남았다. 이 세 가지는 서로 성격이 다르다. 완료 미션 정보는 조회 API 계약 문제이고, 다른 미션 추천은 추천 로직 또는 데이터 선택 문제이며, 앱 설치 추천 문구 제거는 프롬프트/응답 후처리 문제에 가깝다. 한 번에 "백엔드 수정"으로 묶지 말고 원인 계층을 나눠보면 작업 난이도가 훨씬 선명해진다.

수면은 오늘 꽤 중요한 데이터가 나왔다. 6시 알람을 맞췄지만 실제 씻기 시작이 8시라면, 현재 계획은 "6시에 일어난다"가 아니라 "6시에 깨지만 몸은 8시에 움직인다"에 가깝다. 이걸 실패로만 보면 매일 같은 피로감이 반복된다. 다음 주에는 기상 목표를 6시로 고정하기보다, 7시 30분에 씻기 시작하는 것을 1차 성공 기준으로 잡는 편이 현실적이다.
