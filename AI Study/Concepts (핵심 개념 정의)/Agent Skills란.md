> [!ABSTRACT] 핵심 요약
>
에이전트가 "무엇을 할 수 있는가"를 정의한 실행 가능한 기능의 집합.
### **1) 정의 (Definition)**

- **Skill (또는 Tool)**은 에이전트가 외부 세계와 상호작용하기 위해 사용하는 **함수(Function)**나 **API**입니다.
    
- 단순한 코드 조각이 아니라, **"언제, 어떻게 사용해야 하는지"에 대한 설명(Description)**이 포함되어 있어 LLM이 문맥을 파악하고 스스로 호출할 수 있습니다.

### **2) Skill의 구성 요소**

프로그래밍의 함수 선언과 매우 유사합니다.

- **Name (이름):** 기능의 명칭 (예: `search_web`, `calculate_sum`)
    
- **Description (설명):** _가장 중요함._ LLM에게 이 도구가 무엇이고 언제 써야 하는지 자연어로 설명한 것. (예: "최신 주식 정보를 웹에서 검색할 때 사용하세요.")
    
- **Parameters (매개변수):** 함수 실행에 필요한 입력값 (스키마 정의).

### **3) 작동 예시**

사용자가 "내일 서울 날씨 어때?"라고 물었을 때:

1. **Agent:** 질문 분석 -> '날씨 정보'가 필요함을 인지.
    
2. **Skill 검색:** 보유한 Skill 중 `get_weather(location, date)`가 있음을 확인.
    
3. **Execution:** `get_weather("Seoul", "tomorrow")`를 실행.
    
4. **Result:** 결과를 받아 사용자에게 답변 생성.
