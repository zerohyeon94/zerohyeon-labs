Swift의 `Array`는 **연속된 메모리 공간**에 데이터를 저장하는 구조체이다. 인덱스로 접근하는 속도는 빠르지만, 중간에 데이터를 끼워 넣거나 빼는 작업은 비용이 많이 든다.

## 1. 시간 복잡도 요약표

| 연산 종류           | 메서드 예시                          | 평균 시간 복잡도            | 설명                             |
| :-------------- | :------------------------------ | :------------------- | :----------------------------- |
| **접근 (Access)** | `array[i]`, `first`, `last`     | $O(1)$               | 인덱스로 즉시 접근 (Random Access)     |
| **추가 (End)**    | `append()`, `+=`                | **Amortized $O(1)$** | 끝에 추가. 용량이 꽉 차면 재할당 비용 발생      |
| **추가 (Middle)** | `insert(at:)`                   | $O(n)$               | 해당 인덱스 뒤의 모든 요소를 한 칸씩 뒤로 미룸    |
| **삭제 (End)**    | `removeLast()`, `popLast()`     | $O(1)$               | 끝 요소만 제거하면 되므로 빠름              |
| **삭제 (Middle)** | `remove(at:)`, `removeFirst()`  | $O(n)$               | 삭제 후 빈 공간을 채우기 위해 당겨옴          |
| **검색 (Search)** | `contains()`, `firstIndex(of:)` | $O(n)$               | 찾을 때까지 처음부터 순회 (Linear Search) |
| **정렬 (Sort)**   | `sort()`, `sorted()`            | $O(n \log n)$        | 팀소트(TimSort) 혹은 인트로소트 기반       |
| **속성**          | `count`, `isEmpty`              | $O(1)$               | 개수를 매번 세지 않고 저장된 값을 반환         |

## 2. 주요 메서드 상세 분석

### A. 인덱스 접근 (Random Access): $O(1)$
배열은 메모리 상에 연속적으로 위치하므로, 시작 주소와 인덱스만 알면 해당 데이터의 위치를 바로 계산할 수 있다.

```swift
let value = board[2][1] // 매우 빠름
````

### B. 끝에 추가하기 (Append): Amortized $O(1)$

일반적으로 $O(1)$이지만, **'Amortized(분할 상환)'**라는 단어가 붙는 이유:

- 배열은 처음에 일정 크기(Capacity)를 예약해 둔다.
    
- 예약된 공간이 남았다면 $O(1)$이다.
    
- **공간이 꽉 찼는데 추가하려면?**
    
    1. 더 큰 메모리 공간을 새로 할당한다 (보통 기존의 2배).
        
    2. 기존 데이터를 모두 새 공간으로 복사한다 ($O(n)$).
        
    3. 새 데이터를 추가한다.
        
- 이 복사 비용이 가끔 발생하므로, 평균적으로는 $O(1)$이라고 표현한다.

### C. 중간/앞 삽입 및 삭제 (Insert/Remove): $O(n)$

코딩 테스트에서 가장 주의해야 할 부분이다.

- `removeFirst()`나 `insert(_:at: 0)`을 사용하면, **배열의 모든 요소(N개)를 한 칸씩 이동(Shift)**시켜야 한다.
    
- 반복문 안에서 `removeFirst()`를 쓰면 전체 시간 복잡도는 $O(n^2)$이 되어 **시간 초과**가 날 확률이 매우 높다.

> [!TIP] 성능 팁
> 
> 앞쪽 요소를 자주 지워야 한다면(Queue 처럼), 배열 대신 **두 개의 포인터(index)**를 사용하거나, Foundation 없이 직접 구현한 Queue 자료구조를 사용하는 것이 좋다.

## 3. 코딩 테스트 실전 팁

### 1. `reserveCapacity(_:)` 활용

데이터의 개수를 대략 알고 있다면 미리 공간을 예약하여 재할당(Reallocation) 비용을 아낄 수 있다.

```Swift
var results = [Int]()
results.reserveCapacity(board.count) // 미리 공간 확보 -> append 속도 향상
```

### 2. `removeFirst()` 대신 `reverse()` 활용 고려

배열의 앞에서부터 하나씩 빼야 하는 로직이라면, 차라리 배열을 뒤집은(`reversed()`) 후 `popLast()`를 쓰는 것이 훨씬 빠르다.

- `removeFirst()`: $O(n)$
    
- `popLast()`: $O(1)$

```Swift
// [Bad] 앞에서부터 삭제
while !queue.isEmpty {
    let first = queue.removeFirst() // 매번 O(n) 발생
}

// [Good] 뒤집어서 끝에서 삭제 (스택처럼 사용)
var stack = Array(queue.reversed())
while !stack.isEmpty {
    let last = stack.popLast() // 매번 O(1)
}
```

### 3. `contains`는 $O(n)$이다

배열에 특정 값이 있는지 확인할 때, 배열의 크기가 크다면 `Set`으로 변환하여 확인하는 것이 좋다.

- Array `contains`: $O(n)$
    
- Set `contains`: $O(1)$

---
### 💡 요약 및 제안

* **가장 빠름 ($O(1)$):** 인덱스 조회, 끝에 추가/삭제 (`append`, `popLast`)
* **느림 ($O(n)$):** 중간이나 앞에 추가/삭제 (`insert`, `removeFirst`), 값 검색 (`contains`)
* **주의:** 반복문 안에서 `removeFirst()`를 사용하는 것은 피하세요.
