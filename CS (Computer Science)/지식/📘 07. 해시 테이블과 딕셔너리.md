---

## 1. 🧠 해시 테이블이란?

> **키를 해시 함수를 통해 인덱스로 변환하여 데이터를 저장하는 자료구조**입니다.

- 일반 배열처럼 순차 인덱스(0, 1, 2...)를 사용하지 않음
- 대신 `키(key)`를 **해시 함수**로 변환 → **배열의 인덱스 위치**에 저장

```swift
var dict = ["name": "Yeonghyeon", "job": "iOS Developer"]
```

📌 내부적으로는 `"name"` → 해시 함수 → 배열 인덱스 3번 같은 방식

---

## 2. ✅ 해시 테이블의 특징

|특징|설명|
|---|---|
|평균 탐색 속도|**O(1)** (한 번에 접근)|
|키 기반 저장|키는 고유해야 함|
|내부 충돌 가능|해시값이 같을 수도 있음 (→ 충돌 처리 필요)|

---

## 3. 🧪 예제

```swift
var phoneBook: [String: String] = [:]

phoneBook["Alice"] = "010-1234-5678"
phoneBook["Bob"] = "010-9876-5432"

print(phoneBook["Alice"] ?? "없음") // O(1)로 바로 접근
```

---

## 4. 🔥 해시 충돌이란?

> **서로 다른 키가 같은 해시값을 가지는 상황**

### 예시:

- `"name"`과 `"mane"`가 해시 충돌 발생 가능 (이론상)

---

### 🔧 충돌 처리 방법

1. **체이닝(Chaining)**
    
    - 같은 해시 인덱스에 여러 값을 **리스트로 저장**

2. **오픈 어드레싱(Open Addressing)**
    
    - 빈 공간을 찾아 **다른 인덱스로 저장**

> Swift의 Dictionary는 **체이닝** 기반에 최적화를 가미한 구조로 알려져 있음

---

## 5. 🚀 Dictionary가 O(1)인 이유

```swift
let result = dict["key"] // 해시 함수 → 배열 index 접근
```

- 배열의 index 접근은 O(1)
    
- 해시 함수는 연산이 거의 일정 (빠름)
    
- 따라서 평균적으로 매우 빠름

📌 단, **해시 충돌이 많아지면 O(n)에 가까워질 수 있음**

## ✅ 1. Dictionary는 어떻게 데이터를 빠르게 찾을 수 있나요?

### ✍️ 당신의 답변:

> Dictionary는 key와 value로 각각 가지는데, 고유한 key값을 통해서 value를 저장하게 됩니다. 그래서 key값을 알고 있다면 빠르게 value를 조회할 수 있습니다.

### 👍 잘한 점:

- 핵심 아이디어인 "key를 통해 value를 빠르게 찾는다"는 부분을 잘 짚으셨어요.
    
- 실용적인 설명이라 초심자도 이해하기 쉬운 표현입니다.

### ✨ 개선 제안:

조금 더 기술적인 깊이를 더하면 좋습니다.

#### 🔧 개선된 예시:

> Dictionary는 내부적으로 **해시 테이블(Hash Table)**을 사용합니다.  
> key는 `Hashable` 프로토콜을 따르며, 해시 함수에 의해 정수로 변환된 후 해당 인덱스 위치에 value가 저장됩니다.  
> 이 때문에 key를 알면 평균적으로 **O(1)** 시간 안에 value에 바로 접근할 수 있습니다.

## ✅ 2. 해시 충돌이 발생하면 어떤 문제가 생기고, 어떻게 해결할 수 있을까요?

### ✍️ 당신의 답변:

> 해시 충돌은 서로다른 key가 같은 value를 가졌을 때 내부 해시 알고리즘에 따라서 같은 결과가 나올 수 있고 그에따라 충돌이 발생할 수 있습니다. 그리하여 이를 막기 위해서는 같은 해시 인덱스에 여러 값을 저장하는 리스트로 만들거나, 빈공간을 찾아 다른 인덱스로 저장하는 오픈 어드레싱으로 해결합니다.

### 👍 잘한 점:

- 충돌의 원인을 짚고, 해결 방식(체이닝, 오픈 어드레싱)까지 언급하신 점이 좋아요.
    
- 전반적인 설명 흐름이 잘 구성되어 있습니다.

### 🛠️ 개선 제안:

- `"같은 value"` → `"같은 해시값"`으로 바꾸면 정확도가 더 올라갑니다.
    
- 오픈 어드레싱과 체이닝을 **한 줄씩 예로 설명**해주면 이해도가 더 좋아요.

#### 🔧 개선된 예시:

> 해시 충돌은 서로 다른 key가 **같은 해시값**을 갖는 경우 발생합니다.  
> 이때 값들이 같은 인덱스에 저장되기 때문에 탐색 성능이 떨어질 수 있습니다.  
> 해결 방법으로는,
> 
> - **체이닝(Chaining)**: 같은 인덱스에 여러 값을 리스트로 저장
>     
> - **오픈 어드레싱(Open Addressing)**: 충돌 시 다음 빈 인덱스를 찾아 저장  
> 
> Swift의 Dictionary는 내부적으로 체이닝을 사용합니다.
