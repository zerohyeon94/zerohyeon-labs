## 1. 🔍 이진 탐색이란?

> **정렬된 배열**에서 특정 값을 **반으로 나누며** 빠르게 찾는 알고리즘입니다.  
> 시간 복잡도는 **O(log n)** → 매우 빠름!

### 🪜 동작 원리

1. 배열의 중앙값을 확인
    
2. 찾는 값이 작으면 왼쪽만 탐색
    
3. 크면 오른쪽만 탐색
    
4. 반복 또는 재귀

---

## 2. ✅ Swift 구현 (반복문)

```swift
func binarySearch(_ array: [Int], target: Int) -> Int? {
    var low = 0
    var high = array.count - 1

    while low <= high {
        let mid = (low + high) / 2
        if array[mid] == target {
            return mid
        } else if array[mid] < target {
            low = mid + 1
        } else {
            high = mid - 1
        }
    }

    return nil
}
```

---

## 3. 🔁 Swift 구현 (재귀함수)

```swift
func recursiveBinarySearch(_ array: [Int], target: Int, low: Int, high: Int) -> Int? {
    if low > high {
        return nil
    }

    let mid = (low + high) / 2
    if array[mid] == target {
        return mid
    } else if array[mid] < target {
        return recursiveBinarySearch(array, target: target, low: mid + 1, high: high)
    } else {
        return recursiveBinarySearch(array, target: target, low: low, high: mid - 1)
    }
}
```

## 4. 🔂 재귀 함수란?

> 함수가 자기 자신을 호출하는 함수.  
> 주로 **반복적인 분할/축소 문제** 해결에 유리.

- 예시: 이진 탐색, 피보나치 수열, 팩토리얼, DFS 등
    
- 스택 구조처럼 함수 호출이 쌓였다가 반환됨

---

## 💡 핵심 요약

- 이진 탐색 = 정렬된 배열 + 반씩 줄여가며 찾기
    
- 재귀 함수는 스택처럼 호출됨
    
- 반복보다 코드가 깔끔하지만 호출 오버헤드 있음

## ✅ Q1. 이진 탐색을 사용할 수 없는 경우는 어떤 상황인가요?

### 🟢 사용자 답변 요약:

> 이진 탐색은 정렬된 배열에서 사용할 수 있다.

### ✅ 피드백:

- 핵심을 잘 짚으셨어요!
    
- 하지만 “평균값을 구할 수 있을 때”라는 표현은 이진 탐색의 전제와는 다소 관련이 없어요.  
    → 이진 탐색은 **정렬**만 되어 있으면 되고, 평균값은 계산하지 않습니다.

---

### ✅ 개선된 모범답안:

> 이진 탐색은 **정렬된 배열에서만 사용할 수 있는 탐색 알고리즘**입니다.  
> 만약 배열이 **정렬되어 있지 않거나, 삽입/삭제가 빈번해서 정렬 상태를 유지할 수 없는 경우**, 이진 탐색은 사용할 수 없습니다.  
> 또한 연결 리스트나 비정형 데이터에서는 중간값 접근이 어려워 이진 탐색이 부적절합니다.

---

### ✅ 점수: **80점**

→ 전반적인 이해는 맞고, “정렬”이라는 전제를 잘 알고 있음.  
→ 평균값 관련 설명은 제거하면 더 좋아요.

---

## ✅ Q2. 반복문보다 재귀 함수를 사용하는 장점과 단점은?

### 🟢 사용자 요약:

> 자기 자신을 호출하기 때문에 어떤 기능인지 파악하기 쉽다.  
> 반복문을 써도 되지 않나? 라는 의문이 든다.

### ✅ 피드백:

- 정말 **좋은 문제의식**이에요! 👍
    
- 실제로 많은 경우 재귀보다 반복문이 성능/가독성 면에서 유리합니다.
    
- 하지만 **재귀가 유리한 경우도 분명히 있습니다.**

---

### ✅ 재귀 함수의 **장점**

1. **자연스럽고 간결한 표현**이 가능함
    
    - 예: DFS, 트리 순회, 하노이탑, 분할 정복 알고리즘
        
    - 예시를 반복문으로 바꾸면 코드가 훨씬 복잡해짐

2. **상태 추적이 쉬움**
    
    - 재귀는 함수 호출 스택으로 자연스럽게 상태가 쌓임
        
    - 예: 백트래킹, DFS, 재귀적 구조 문제

---

### ❗ 단점

- 함수 호출마다 **스택 프레임이 쌓임** → 메모리 사용 증가
    
- 종료 조건을 잘못 설정하면 **스택 오버플로우 발생**
    
- 반복문보다 **일반적으로 느릴 수 있음**

---

### ✅ 재귀가 더 유리한 예

```swift
// 트리 구조 탐색 (재귀가 훨씬 깔끔)
func traverse(_ node: TreeNode?) {
    guard let node = node else { return }
    traverse(node.left)
    traverse(node.right)
}
```

> 위 코드를 반복문으로 바꾸려면 스택을 직접 구현해야 하므로 훨씬 복잡해집니다.

---

### ✅ 개선된 모범답안:

> 재귀 함수는 **문제를 더 작게 나누는 방식**으로 접근할 수 있을 때 유리합니다.  
> 반복문보다 **코드가 간결해지고, 논리적으로 문제 구조에 잘 맞는 경우**가 많습니다.  
> 특히 트리나 그래프 순회, 백트래킹 문제에서는 반복문보다 훨씬 명료합니다.  
> 하지만 성능 측면에서는 호출 비용과 메모리 사용이 더 크기 때문에,  
> 반복문이 더 단순하고 효율적일 때는 반복문이 더 적절한 선택이 될 수 있습니다.

---
