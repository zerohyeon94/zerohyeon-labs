## ⏱️ O(1), O(n) 등은 무엇인가요?

이 표기법은 **시간 복잡도(Time Complexity)**라고 불리고,  
어떤 코드나 알고리즘이 **입력 크기에 따라 얼마나 빠르거나 느린지**를 나타냅니다.

### ✅ 이름: **빅오 표기법(Big-O Notation)**

> 알고리즘이 처리하는 데 걸리는 시간이나 공간을, 입력 크기 `n`에 대한 함수로 나타냅니다.

---

## 📏 예시로 이해해보기

|알고리즘/코드|시간 복잡도|의미|
|---|---|---|
|`arr[0]`|`O(1)`|→ 항상 한 번에 조회 가능|
|`for i in 0..<n`|`O(n)`|→ n번 반복됨|
|이중 반복문|`O(n²)`|→ n * n 만큼 반복|
|이진 탐색|`O(log n)`|→ 매번 반으로 나눔|

---

## 📘 각 시간 복잡도 설명

### 🟢 `O(1)` – 상수 시간 (Constant Time)

```swift
let value = array[3]
```

- 배열의 인덱스 접근처럼 **입력 크기와 상관없이 항상 일정한 시간**이 걸리는 작업

---

### 🟡 `O(n)` – 선형 시간 (Linear Time)

```swift
for value in array {
    print(value)
}
```

- 데이터가 n개라면 n번 반복됨
    
- **입력이 커질수록 시간도 비례해서 증가**

---

### 🔴 `O(n²)` – 이차 시간 (Quadratic Time)

```swift
for i in array {
    for j in array {
        print(i, j)
    }
}
```

- 이중 루프 → 배열 크기가 커질수록 실행 시간 **제곱으로 증가**

---

### 🔵 `O(log n)` – 로그 시간

```swift
func binarySearch(_ arr: [Int], target: Int) -> Int? {
    var low = 0, high = arr.count - 1
    while low <= high {
        let mid = (low + high) / 2
        if arr[mid] == target { return mid }
        if arr[mid] < target {
            low = mid + 1
        } else {
            high = mid - 1
        }
    }
    return nil
}
```

- **이진 탐색처럼 매번 탐색 범위를 절반으로 줄일 때**
    
- 매우 빠름!

---

## 🧠 왜 중요한가요?

- 단순히 “동작한다”가 아니라, **“얼마나 빠르게 동작하는가”**를 생각해야 진짜 실력입니다.
    
- 성능이 중요한 앱이나 대용량 데이터 처리 시 필수로 고려해야 해요.

---

## 💡 핵심 요약

- Big-O는 코드의 효율성을 측정하는 단위
    
- 작은 차이가 나중엔 큰 성능 차이를 만듦

---

## ✅ Q1. O(n)과 O(1)의 차이 + 예시

### 🟢 사용자 요약:

> O(1)은 한 번에 조회, O(n)은 데이터 수만큼 반복  
> array[0]은 O(1), `for i in array`는 O(n)

### ✅ 피드백:

- **핵심 개념은 정확**합니다.
    
- 예시도 잘 골랐어요.
    
- 다만 `"1과 n의 차이"`라는 말은 조금 두루뭉술하게 들릴 수 있고,  
    “작업 횟수”가 **입력 크기(n)에 따라 달라진다**는 점을 강조하면 더 명확합니다.

### ✅ 개선된 모범답안:

> O(1)은 **입력 크기에 상관없이 항상 일정한 시간**이 걸리는 작업입니다.  
> 예를 들어 배열의 특정 인덱스를 조회하는 `array[3]` 같은 작업은  
> 어떤 크기의 배열이든 한 번의 연산만으로 결과를 얻을 수 있으므로 O(1)입니다.
> 
> 반면, O(n)은 **입력 크기 n에 비례하여 연산 시간이 증가**하는 작업입니다.  
> 예를 들어 `for i in array { print(i) }`처럼 배열 전체를 순회하는 경우,  
> 배열의 크기가 커질수록 반복 횟수도 증가하므로 O(n)입니다.

### ✅ 점수: **90점**

→ 내용은 맞고 예시도 적절했어요. 단, 용어 표현을 조금 더 명확하게 다듬으면 더 좋아요.

---

## ✅ Q2. 이진 탐색이 O(log n)인 이유

### 🟢 사용자 요약:

> 한 번 반복할 때마다 탐색 범위가 절반으로 줄어들기 때문에 시간도 줄어든다.

### ✅ 피드백:

- 매우 핵심을 정확히 이해하셨어요!
    
- 단순하고 명료한 설명이 좋습니다.
    
- 다만, **“몇 번에 끝나는지”**를 수치적으로 한 줄만 더 보완하면 완벽합니다.
    

### ✅ 개선된 모범답안:

> 이진 탐색은 정렬된 배열에서 중간 값을 기준으로 탐색 범위를 절반씩 줄여나가기 때문에  
> 입력 크기 `n`이 클수록 효율적입니다.  
> 예를 들어, `n = 16`이면 최대 4번, `n = 1,000,000`이어도 약 20번만 비교하면 됩니다.  
> 이처럼 탐색 횟수가 `log₂(n)`에 비례하므로 시간 복잡도는 O(log n)입니다.

### ✅ 점수: **95점**

→ 정확한 개념과 깔끔한 설명! 수치적 예시만 추가되면 완벽!
