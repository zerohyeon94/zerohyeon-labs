## ✅ 해시 충돌 복습

> 서로 다른 키들이 같은 해시값을 가질 때 → 충돌 발생  
> 예: `"cat"`과 `"tac"`이 우연히 같은 해시값을 가지면  
> 같은 인덱스에 둘 다 저장해야 하는 상황

---

## 🔗 체이닝 (Chaining) 예시

> 같은 인덱스에 **배열(혹은 연결 리스트)** 를 두고 값을 여러 개 저장하는 방식

### 🧠 그림으로 보면:

```swift
[0] → empty  
[1] → [("cat", 3), ("tac", 5)] ← 🔥 해시 충돌 발생 but 저장 OK  
[2] → empty
```

```swift
index:     0              1             2
         ┌───┬────────────────────────┬────┐
bucket:  │   │ [("cat",3), ("tac",5)] │    │
         └───┴────────────────────────┴────┘
```

### 🧪 Swift 스타일 예시 코드 (간단한 해시 테이블 구현):

```swift
struct HashTable {
    private var buckets: [[(String, Int)]] = Array(repeating: [], count: 5)

    private func index(for key: String) -> Int {
        return abs(key.hashValue) % buckets.count
    }

    mutating func insert(key: String, value: Int) {
        let i = index(for: key)
        // 이미 있는 키면 값 덮어쓰기
        if let existingIndex = buckets[i].firstIndex(where: { $0.0 == key }) {
            buckets[i][existingIndex].1 = value
        } else {
            buckets[i].append((key, value)) // 체이닝 방식 저장
        }
    }

    func get(key: String) -> Int? {
        let i = index(for: key)
        return buckets[i].first(where: { $0.0 == key })?.1
    }

    func printBuckets() {
        for (i, bucket) in buckets.enumerated() {
            print("[\(i)]: \(bucket)")
        }
    }
}

var table = HashTable()
table.insert(key: "cat", value: 1)
table.insert(key: "tac", value: 2) // 일부러 충돌 유도
table.printBuckets()
```

🔎 `cat`, `tac`이 같은 index로 가도 리스트로 나란히 저장되므로 안전!

---
## 🔍 이 코드의 흐름을 단계별로 설명

### 예: `"cat"`, `"tac"` 두 키를 입력할 때

#### 📌 1. `"cat"`을 삽입

1. `hashValue` 계산 → 예를 들어 `231238`
    
2. `index = hashValue % 5 = 3`
    
3. `buckets[3]`은 현재 `[]` → 여기에 `("cat", 3)` 저장

#### 📌 2. `"tac"`을 삽입

1. `hashValue` 계산 → 우연히 같은 값 `231238` 발생했다고 가정
    
2. `index = 3` → 충돌 발생!
    
3. `buckets[3]`에 `"cat"`이 있음 → 새로 `"tac"` 추가

🔽 저장 구조는 이렇게 됨:

```swift
buckets[3] = [("cat", 3), ("tac", 5)]
```

> 이게 바로 **체이닝 구조**입니다!  
> 충돌해도 한 인덱스에 여러 키-값을 리스트로 넣어 해결

---

## 📤 검색할 때는?

```swift
get("tac")
```

1. 해시 계산 후 인덱스 → 3
    
2. `buckets[3]` 리스트를 순회하며 key가 `"tac"`인 값 검색
    
3. 찾으면 value 반환

> 리스트 길이는 짧으니 성능 거의 O(1)에 가까움

## ✅ 체이닝 방식의 장점

|장점|설명|
|---|---|
|충돌에 강함|여러 개 값을 한 인덱스에 저장 가능|
|확장 쉬움|각 슬롯만 리스트로 확장하면 끝|
|구현 직관적|배열 안의 배열 or 연결 리스트 구조로 구현 가능|

---

## ❗ 단점

|단점|설명|
|---|---|
|리스트가 너무 길어지면 느려짐|극단적 충돌 시 성능이 O(n)에 가까워질 수도|
|메모리 단편화|각 리스트가 별도 메모리를 차지할 수 있음|

---

## 🧠 요약

- 체이닝 = **한 인덱스에 여러 값을 리스트로 저장**
    
- Swift의 `Dictionary`도 이런 체이닝 구조를 기본으로 사용
    
- 구현이 쉬우며 성능도 좋아 실제로 가장 많이 쓰이는 해시 충돌 해결 방식입니다

## 🧭 오픈 어드레싱 (Open Addressing) 예시

> 해시 충돌이 발생했을 때, **다음 빈 슬롯을 찾아** 저장하는 방식입니다.
    - 체이닝은 **하나의 버킷에 여러 값을 리스트로 저장**하지만,
    - 오픈 어드레싱은 **빈 슬롯을 찾아 한 줄 배열에 저장**합니다.

### 🧠 그림으로 보면:

```swift
해시 인덱스 = 2
[0] → nil  
[1] → nil  
[2] → ("cat", 3)  
[3] → ("tac", 5) ← cat과 충돌! → 다음 빈 칸에 저장  
[4] → nil  
```

## 🧪 Swift Playground 예제

```swift
struct OpenAddressingHashTable {
    private var buckets: [(String, Int)?] = Array(repeating: nil, count: 5)

    private func index(for key: String) -> Int {
        return abs(key.hashValue) % buckets.count
    }

    mutating func insert(key: String, value: Int) {
        var i = index(for: key)
        let startIndex = i
        while true {
            // 비어 있거나 동일 키면 삽입/덮어쓰기
            if buckets[i] == nil || buckets[i]?.0 == key {
                buckets[i] = (key, value)
                return
            }
            i = (i + 1) % buckets.count
            if i == startIndex {
                print("⚠️ 테이블이 가득 찼습니다!")
                return
            }
        }
    }

    func get(key: String) -> Int? {
        var i = index(for: key)
        let startIndex = i
        while true {
            guard let pair = buckets[i] else { return nil }
            if pair.0 == key {
                return pair.1
            }
            i = (i + 1) % buckets.count
            if i == startIndex {
                return nil // 전체 한 바퀴 돌았음
            }
        }
    }

    func printTable() {
        for (i, bucket) in buckets.enumerated() {
            if let bucket = bucket {
                print("[\(i)] → \(bucket)")
            } else {
                print("[\(i)] → nil")
            }
        }
    }
}
```

---

## 🔍 사용 예시

```swift
var table = OpenAddressingHashTable()
table.insert(key: "cat", value: 3)
table.insert(key: "tac", value: 5) // cat과 충돌 유도
table.insert(key: "act", value: 7) // 또 충돌 유도
table.printTable()

print("tac: \(table.get(key: "tac") ?? -1)")
```

## 🔄 동작 과정

1. `"cat"`의 해시 인덱스가 2 → [2]에 저장
    
2. `"tac"`이 같은 해시 인덱스를 가짐 → [2]는 이미 차 있음 → [3]에 저장
    
3. `"act"`도 충돌 → [2] → [3] → [4]에 저장

## ✅ 장단점 정리

|장점|설명|
|---|---|
|메모리 효율|리스트 필요 없이 하나의 배열로 충분|
|캐시 성능 우수|데이터가 연속된 메모리에 있어 캐시 히트율 높음|

|단점|설명|
|---|---|
|삭제 어려움|값을 삭제하면 “빈칸”인지 “중간값이 지워졌는지” 모호함 (삭제 마커 필요)|
|성능 저하|충돌 많아지면 선형 탐색으로 느려짐 (O(n))|
|리사이징|테이블이 거의 찰 경우 반드시 리사이징 필요|

## 📦 요약

- 오픈 어드레싱은 **충돌이 일어날 때 다음 칸으로 이동해서 저장**
    
- Swift `Dictionary`는 이 방식을 쓰지 않음 (체이닝을 사용)
    
- **삭제 처리가 복잡**하고, **테이블이 거의 찼을 때 성능이 떨어짐**

### ❗ 단점

- 삽입/삭제가 복잡해짐
    
- 테이블이 거의 꽉 차면 성능 급감

## 🧬 Swift Dictionary는 왜 체이닝을 사용하나요?

Swift의 `Dictionary`는 내부적으로 **open addressing + chaining 혼합 구조**입니다만,  
**사실상 체이닝 기반**이라고 보는 이유는:

|이유|설명|
|---|---|
|성능|평균적으로 빠름 (탐색 시 1~2번 비교)|
|구조|버킷마다 여러 엔트리 저장 가능 (Array or Linked List 형태)|
|구현|Swift의 Dictionary 구현 논문/소스 코드 분석 결과 대부분 체이닝 기반|
|안정성|충돌 많을 때도 성능 유지됨|

> [🔗 Apple Swift open-source source code (stdlib/public/core/Dictionary.swift)](https://github.com/swiftlang/swift/blob/main/stdlib/public/core/DictionaryStorage.swift) 참고

## ✅ 비교 요약

| 항목               | 체이닝 (Chaining)        | 오픈 어드레싱 (Open Addressing) |
| ---------------- | --------------------- | ------------------------- |
| 충돌 처리            | 같은 인덱스에 여러 값 저장 (리스트) | 다른 빈 인덱스에 저장              |
| 성능               | 충돌 많아도 비교적 안정적        | 충돌 많으면 성능 급감              |
| 구현               | 상대적으로 단순, 유연          | 구현 복잡, 삭제 어렵              |
| Swift Dictionary | ✅ 기본적으로 이 방식          | ❌ 전체 구조에는 적합하지 않음         |
