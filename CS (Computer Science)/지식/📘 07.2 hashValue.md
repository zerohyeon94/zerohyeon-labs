## ✅ 1. hashValue란?

```swift
let value = "name".hashValue
print(value)  // 어떤 정수(Int) 값이 출력됨
```

> `hashValue`는 Swift에서 `Hashable` 프로토콜을 따르는 타입이 제공하는 **고유한 정수 값**입니다.  
> 이 값은 **해시 테이블의 인덱스를 계산할 때 사용되는 숫자**입니다.

### 📌 쉽게 말하면:

- `"name"` 같은 문자열을 → **정수 하나로 바꿔주는 것**
    
- 이 정수는 **배열의 인덱스를 계산하거나, 값의 고유성을 판별할 때 사용**됩니다

---

## ✅ 2. 왜 `hashValue`가 충돌할 수 있나요?

### 🎯 핵심 개념:

> 무한히 많은 문자열 → 유한한 `Int` 범위로 압축해야 하므로  
> 서로 다른 문자열인데도 `hashValue`가 **같아질 가능성**이 존재합니다.

---

### 🔍 예시

```swift
print("name".hashValue)
print("mane".hashValue)
```

두 문자열은 서로 다른 문자열이지만, 내부 해시 알고리즘에 따라 계산된 결과가 이론상 **같을 수 있다**는 것이 핵심입니다.

하지만 직접 실행해보면 보통은 다릅니다:

```swift
print("name".hashValue)  // 예: 3200921347169123412
print("mane".hashValue)  // 예: -1023848123841938475
```

> 🎯 즉, 충돌은 실제로 "자주 일어나진 않지만" **발생할 수 있는 문제**입니다.  
> 이런 충돌이 생겨도 Swift는 key 값을 한 번 더 비교해서 구분합니다.

---

## ✅ 3. 왜 충돌이 무서운가요?

- Dictionary나 Set에서는 키 충돌이 발생하면 **성능이 떨어질 수 있습니다.**
    
- 내부적으로 **같은 인덱스에 여러 데이터를 저장**하게 되면,
    
    - 선형 탐색 또는 체이닝 구조로 동작하면서 **탐색 속도가 O(1) → O(n)**으로 떨어질 수 있음

---

## ✅ 비유로 쉽게 이해해볼까요?

### 🏫 학생 사물함 예시

- 학생 이름을 사물함 번호(해시값)로 바꿔서 물건을 넣는다.
    
- `"name"` → 사물함 17번
    
- `"mane"` → 원래는 사물함 21번이어야 하는데, **우연히 같은 17번에 배정**

→ **두 사람이 같은 사물함을 배정받으면 충돌 발생!**  
→ 이 경우, 뒤에 온 사람은 같은 사물함에 "붙여서" 넣거나, 다른 빈 사물함으로 밀려남  
→ 이게 바로 체이닝 or 오픈 어드레싱

---

## ✅ Playground 실습 예시

```swift
let word1 = "name"
let word2 = "mane"

print("🔢 \(word1).hashValue: \(word1.hashValue)")
print("🔢 \(word2).hashValue: \(word2.hashValue)")

if word1.hashValue == word2.hashValue {
    print("❗ 해시 충돌 발생!")
} else {
    print("✅ 해시 충돌 없음")
}
```

👉 이 코드는 실제 충돌이 나는지 확인할 수 있고,  
**“충돌이 일어날 수 있다는 사실 자체”가 중요한 개념**입니다.
