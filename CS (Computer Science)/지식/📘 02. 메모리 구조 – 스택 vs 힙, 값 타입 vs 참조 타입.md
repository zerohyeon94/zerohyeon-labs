---

### 1. 📦 스택(Stack)과 힙(Heap) 메모리

| 구분       | 스택(Stack)       | 힙(Heap)             |
| -------- | --------------- | ------------------- |
| 구조       | 선입후출(LIFO)      | 동적 메모리              |
| 속도       | 빠름              | 느림                  |
| 저장되는 데이터 | 지역 변수, 함수 호출 정보 | 객체(클래스 인스턴스 등)      |
| 관리       | 컴파일러가 자동 관리     | 개발자가 참조로 관리 (ARC 등) |

> 함수가 호출될 때마다 스택 프레임이 쌓이고, 함수가 끝나면 사라집니다.

---

### 2. 🧾 값 타입 vs 참조 타입

|항목|값 타입 (struct, enum)|참조 타입 (class)|
|---|---|---|
|저장 위치|주로 스택|힙|
|복사|복사 시 **값이 복제**됨|복사 시 **참조가 복사**됨|
|대표 예|`Int`, `Double`, `String`, `Array`, `struct`|`UIView`, `NSObject`, 사용자 정의 class|
|Swift 특징|**기본은 값 타입 중심 설계**|사용 시 메모리 관리 주의 필요|

📌 **예시 코드:**

```swift
struct A {
    var value = 1
}
class B {
    var value = 1
}

var a1 = A()
var a2 = a1
a2.value = 2
print(a1.value) // 1 (값 타입 복사)

var b1 = B()
var b2 = b1
b2.value = 2
print(b1.value) // 2 (참조 타입 공유)
```

### 3. 💣 ARC와 메모리 누수

- ARC(Automatic Reference Counting)는 **참조 카운트 기반 자동 메모리 관리** 시스템 
  (클래스의 메모리를 자동 관리함.)
- 모든 참조가 사라지면 힙에 할당된 메모리 자동 해제
- 순환 참조가 생기면 ARC도 메모리를 해제하지 못함.
- 예시) 다음과 같은 상황에서 **메모리 누수** 발생:

```swift
class Person {
    var friend: Person?
}

let a = Person()
let b = Person()
a.friend = b
b.friend = a // 서로가 서로를 참조 → ARC로도 해제 불가 (순환 참조)
```

➡ 해결법: `weak`, `unowned` 키워드 사용

## 💡 핵심 요약
- 값 타입은 복사되며 스택에 저장되는 경향이 있음
- 참조 타입은 힙에 저장되며 참조만 전달됨
- Swift에서는 기본적으로 struct 사용을 권장함

### 🧠 오늘의 질문
## ✅ Q1. class와 struct 중 어떤 것을 기본으로 사용해야 할까요?

### 🟢 사용자 답변:

> 값 타입인 struct를 기본으로 사용해야된다고 생각합니다.  
> 그 이유는 주로 스택에 저장이 되며 그에따라 속도가 빠르게 동작하게됩니다.

### ✅ 검토 및 개선안:

**전반적으로 맞습니다**, struct는 값 타입이며 복사가 간단하고 안전한 방식으로 설계되어 있어서 Swift에서 권장하는 방식이에요.  
다만, **“스택에 저장되니까 무조건 빠르다”**는 표현은 조금 일반화된 면이 있습니다.

### ✅ 개선된 답변:

> Swift에서는 기본적으로 **값 타입인 `struct`를 우선적으로 사용하는 것이 좋습니다.**  
> 그 이유는 값 타입은 복사 시 새로운 인스턴스를 생성하기 때문에 **데이터의 불변성을 보장**하고,  
> 참조 타입과 달리 메모리 누수나 순환 참조 문제에서도 더 안전합니다.  
> 또한 간단한 데이터 모델에서는 성능상 이점도 있을 수 있습니다.  
> 다만, 인스턴스가 크거나 상속, 공유가 필요한 경우에는 `class`를 사용합니다.

---

## ✅ Q2. 메모리 누수가 발생하는 이유는 무엇인가요?

### 🟢 사용자 답변:

> 참조를 사용할 때 2개의 변수가 서로를 참조하게되면 순환 참조가 되어서 해제가 불가능하기 때문에 메모리 누수가 발생합니다.

### ✅ 검토 및 개선안:

이해도와 표현이 아주 정확해요! 다만, 한 문장에 많은 의미가 담겨 있으므로 **구조를 나눠서 설명하면 더 명확**해질 수 있어요.

### ✅ 개선된 답변:

> 메모리 누수는 **참조 타입(`class`)의 인스턴스들이 서로를 강하게 참조(strong reference)할 때 발생합니다.  
> 예를 들어, 두 객체가 서로를 참조하고 있는 경우, 어느 쪽도 먼저 해제되지 않아 ARC가 메모리를 회수하지 못합니다.**  
> 이런 상황을 **순환 참조(circular reference)라고 하며, 이로 인해 메모리 누수가 발생합니다.  
> 이를 방지하기 위해 `weak` 또는 `unowned` 참조를 사용해 참조 카운트를 증가시키지 않도록 해야 합니다.
