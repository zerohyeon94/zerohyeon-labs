## ✅ 해시 함수는 “압축 함수”입니다

### 📌 해시 함수란?

- 임의의 크기의 데이터를 → 고정된 크기의 값으로 변환하는 함수
    
- Swift에서는 `Hashable`을 따르는 모든 값에 대해 내부적으로 해시 함수를 호출해 **Int** 같은 값으로 변환합니다.

```swift
print("name".hashValue)
print("mane".hashValue)
```

> 이 두 값이 대부분 다르지만, **극히 드물게 같을 수도 있습니다.**  
> 이걸 **해시 충돌(hash collision)**이라고 합니다.

---

## 🤔 왜 충돌이 발생 가능한가요?

해시 함수는 **무한한 입력 공간 → 유한한 출력 공간**으로 변환합니다.

### 예시로 설명:

- 입력 가능한 문자열: `"a"`, `"aa"`, `"aaa"`, ... → **사실상 무한**
    
- 출력 가능한 해시값: 예를 들어 `Int64`라면 약 2^64 = 약 18경 개 → **유한**

> 🎯 즉, 무한한 입력을 유한한 출력으로 대응하려면 **중복되는 출력(충돌)**이 생길 수밖에 없습니다.  
> 이건 **수학적으로 필연**입니다. (→ 비둘기집 원리)

---

## 🧪 예: "name"과 "mane"

- 둘은 스펠링이 비슷하지만 해시 값은 대부분 다릅니다.
    
- 하지만 내부 해시 함수가 비트 연산, XOR 등을 통해 계산하다 보면  
    **우연히 같은 해시값이 나올 가능성은 존재합니다.**
    
- 특히 **해시 공간이 제한되어 있거나 데이터가 많을수록 충돌 확률이 높아집니다.**

---

## ✅ Swift에서는 어떻게 막나요?

- Swift의 `Dictionary`는 해시값이 **같더라도 키 값이 다른지**를 추가로 비교합니다.

```swift
if hash1 == hash2 {
   if key1 == key2 {
       // 진짜 같은 키
   } else {
       // 해시 충돌이지만 키는 다름 → 따로 저장
   }
}
```

## 🧠 정리

| 항목           | 설명                              |
| ------------ | ------------------------------- |
| 해시 충돌이란?     | 서로 다른 입력이 같은 해시값을 갖는 것          |
| 왜 발생하나요?     | 무한한 입력 → 유한한 출력이기 때문 (수학적으로 필연) |
| name vs mane | 실제로 충돌이 발생하진 않지만, **이론적으로 가능함** |
| Swift는?      | 해시 충돌이 생겨도 **키 비교로 정확히 구분** 처리  |
