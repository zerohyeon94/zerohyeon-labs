## 1. 컴퓨터의 구성 요소
- CPU: 연산 및 명령 실행
- RAM: 휘발성 메모리 (작업 중 데이터 저장)
- 저장장치: SSD/HDD
- I/O 장치: 키보드, 마우스, 디스플레이 등

## 2. 운영체제란?
- 컴퓨터 자원을 효율적으로 관리
- 사용자와 하드웨어 사이의 중개자

## 3. 프로세스와 스레드
| 구분     | 프로세스                | 스레드                        |
|----------|-------------------------|-------------------------------|
| 정의     | 실행 중인 프로그램 인스턴스 | 프로세스 내의 실행 단위         |
| 메모리   | 각자 독립된 메모리 사용     | 동일한 메모리 공간 공유         |
| 예시     | 여러 앱 실행              | 앱 내에서 동시에 이미지 다운로드, UI 렌더링

---

## 💡 핵심 요약
> 운영체제는 컴퓨터 자원을 관리하는 핵심 프로그램이며, 프로세스와 스레드를 통해 여러 작업을 동시에 수행할 수 있게 한다.

---

### 🧠 오늘의 질문

### ❓ **1. 앱이 실행된다는 건 컴퓨터 안에서 어떤 일이 일어나는 걸까요?**

#### ✅ 답변:

앱이 실행된다는 것은 OS가 앱을 **하나의 프로세스로 생성**하고, 그 앱의 실행 파일을 메모리에 로드하여 **CPU가 명령어를 하나씩 실행**하는 것을 의미합니다.

구체적인 과정:

1. 사용자가 앱 아이콘을 터치
2. 운영체제(iOS)가 해당 앱의 **실행 파일(binary)**을 찾고, 메모리에 로딩
3. OS가 새로운 **프로세스를 생성**하고 해당 메모리를 할당
4. CPU가 앱의 진입점(Main 함수 또는 SceneDelegate 등)을 실행
5. 메인 스레드(UI Thread)가 화면을 그리기 시작

> 즉, 앱 실행은 **OS의 프로세스 생성 → 메모리 적재 → CPU 명령어 실행**의 흐름입니다.

### ❓ **2. 하나의 앱이 여러 작업(스크롤, 다운로드)을 동시에 처리하려면 어떤 기술이 필요할까요?**

#### ✅ 답변:

동시 작업을 처리하려면 **멀티스레딩** 또는 **비동기 처리**가 필요합니다.

iOS에서는 보통 다음 기술을 사용합니다:

- **GCD (Grand Central Dispatch)**: 비동기 큐로 여러 작업을 동시에 처리
- **OperationQueue**: 우선순위와 의존성 설정이 가능한 스레드 풀
- **RxSwift/Combine**: 반응형 프로그래밍으로 데이터 흐름 처리
- **Async/Await**: Swift의 최신 비동기 문법

예:
- 메인 스레드는 UI를 담당
- 네트워크 요청은 백그라운드 스레드에서 처리
- 다운로드 완료 후 메인 스레드로 복귀하여 UI 업데이트

> iOS는 기본적으로 **멀티스레드 기반**이며, 앱 성능과 사용자 경험을 위해 스레드 분리가 중요합니다.
