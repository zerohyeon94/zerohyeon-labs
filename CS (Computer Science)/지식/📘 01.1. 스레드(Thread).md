### ✅ 정의:

> **스레드는 프로세스 안에서 실제로 일을 처리하는 실행 단위**입니다.

- 하나의 **프로세스**(예: 당신의 iOS 앱)는 **최소 하나 이상의 스레드**로 구성되어 있고,
- 그 중 **메인 스레드(Main Thread)**는 UI를 담당합니다.
- 그 외의 스레드는 **백그라운드 작업**(네트워크, 이미지 다운로드 등)을 수행할 수 있어요.

> [!success] 간단 정의 
> 1. 스레드는 프로세스 안에서 실제로 실행되는 최소 단위
> 2. 하나의 앱(프로세스)은 여러 개의 스레드를 가질 수 있다

---

### 📦 비유: 회사와 직원

- **프로세스 = 회사**
- **스레드 = 회사에서 일하는 직원들**

회사(앱)가 여러 명의 직원을 두면 더 많은 일을 동시에 할 수 있는 것처럼, 프로세스도 여러 스레드를 두어 동시에 다양한 작업을 수행할 수 있어요.

---

### 💡 왜 스레드가 중요할까?

#### iOS 앱에서의 예시:

- **메인 스레드**는 UI를 담당하므로 무조건 **빠르게** 돌아야 합니다.
- 네트워크 요청을 메인 스레드에서 하면, 요청이 끝날 때까지 **화면이 멈춥니다.**
- 그래서 우리는 **백그라운드 스레드**를 사용해 네트워크를 처리하고, 결과만 UI에 전달합니다.

### 메인 스레드 vs 백그라운드 스레드
| 항목  | 메인 스레드                        | 백그라운드 스레드        |
| --- | ----------------------------- | ---------------- |
| 역할  | UI 렌더링                        | 네트워크, 계산 등       |
| 중요성 | 매우 높음 (UI가 멈추면 앱이 멈춘 것처럼 느껴짐) | 비동기 처리 담당        |
| 예시  | 버튼 클릭, 화면 전환                  | API 요청, 이미지 다운로드 |
## 3. 예시 코드 (GCD 활용)

```swift
DispatchQueue.global().async {
    // 백그라운드 작업
    let data = downloadImage()

    DispatchQueue.main.async {
        // UI 업데이트
        imageView.image = data
    }
}
```

### 🧠 핵심 요약

|개념|설명|
|---|---|
|프로세스|실행 중인 앱 (메모리와 리소스를 독립적으로 가짐)|
|스레드|프로세스 내에서 작업을 처리하는 실행 단위|
|메인 스레드|UI를 그리는 스레드 (절대 막히면 안 됨)|
|백그라운드 스레드|무거운 작업, 네트워크 요청 등을 처리|

---

## 💡 정리

- 스레드는 앱 내부에서 병렬로 작업을 처리하기 위한 실행 단위
- UI는 메인 스레드에서만 업데이트 가능
