## 1️⃣ Bubble Sort (버블 정렬)

> **인접한 두 값을 비교하여 큰 값을 뒤로 보내는 정렬**

```swift
let array = [5, 3, 8, 1]
```

### 과정:

- 1회전:
    - 5 > 3 → 교환 → [3, 5, 8, 1]
        
    - 5 < 8 → 그대로
        
    - 8 > 1 → 교환 → [3, 5, 1, 8]
- 2회전:
    - 3 < 5 → 그대로
        
    - 5 > 1 → 교환 → [3, 1, 5, 8]
- 3회전:
    
    - 3 > 1 → 교환 → [1, 3, 5, 8]

✅ 최종 결과: `[1, 3, 5, 8]`

---

## 2️⃣ Selection Sort (선택 정렬)

> **가장 작은 값을 찾아서 앞에 가져다 놓는 정렬**

```swift
let array = [5, 3, 8, 1]
```

### 과정:

- 1회전: 전체 중 가장 작은 `1` → [**1**, 3, 8, **5**]
    
- 2회전: 나머지 중 가장 작은 `3` → [1, **3**, 8, **5**]
    
- 3회전: 나머지 중 가장 작은 `5` → [1, 3, **5**, **8**]

✅ 최종 결과: `[1, 3, 5, 8]`

---

## 3️⃣ Insertion Sort (삽입 정렬)

> **정렬된 영역에 새 값을 끼워 넣는 정렬**

```swift
let array = [5, 3, 8, 1]
```

### 과정:

- 시작 상태: [5]
    
- `3`을 `5` 앞에 삽입 → [3, 5]
    
- `8`은 그대로 뒤에 삽입 → [3, 5, 8]
    
- `1`은 제일 앞으로 이동 → [1, 3, 5, 8]

✅ 최종 결과: `[1, 3, 5, 8]`

---

## 4️⃣ Merge Sort (병합 정렬)

> **배열을 나누고 병합하면서 정렬하는 방식**

```swift
let array = [5, 3, 8, 1]
```

### 과정:

1. 반으로 나누기 → [5, 3], [8, 1]
    
2. 각각 또 나누기 → [5], [3], [8], [1]
    
3. 작은 단위부터 정렬하여 병합:
    
    - [5] + [3] → [3, 5]
        
    - [8] + [1] → [1, 8]
        
4. 최종 병합 → [3, 5] + [1, 8] → [1, 3, 5, 8]

✅ 최종 결과: `[1, 3, 5, 8]`

## ⚡ Quick Sort (퀵 정렬)

> **“기준값(Pivot)을 잡고, 그것보다 작은 값은 왼쪽, 큰 값은 오른쪽으로 나눠 정렬”**

### 🔁 동작 순서:

1. 기준(Pivot) 값을 하나 선택
    
2. 배열을 pivot보다 작은 값과 큰 값으로 **두 그룹**으로 나눔
    
3. 각 그룹에 대해 다시 **퀵 정렬을 재귀적으로 호출**
    
4. 정렬된 그룹을 합침

### 📦 예시:

```swift
let array = [5, 3, 8, 1]

pivot = 5
→ 왼쪽 그룹: [3, 1]
→ 오른쪽 그룹: [8]

재귀적으로 다시 정렬 → [1, 3] + [5] + [8] → [1, 3, 5, 8]
```

### ✅ 장점

- 평균적으로 **빠름 (O(n log n))**
    
- 실제 소트 라이브러리에서도 많이 사용됨

### ⚠️ 단점

- **최악의 경우**: pivot이 항상 최솟값이나 최댓값이면 성능이 O(n²)

## 🏔 Heap Sort (힙 정렬)

> **최대값(또는 최소값)을 빠르게 꺼낼 수 있는 힙 자료구조를 이용한 정렬**

### 🔁 동작 순서:

1. 주어진 배열을 **최대 힙(Max Heap)**으로 구성
    
    - 부모가 자식보다 항상 크도록
        
2. 힙에서 가장 큰 값(루트)을 꺼내서 배열의 **끝으로 이동**
    
3. 힙 크기를 줄이고 다시 힙을 구성 (heapify)
    
4. 반복

### 📦 예시:

```swift
[5, 3, 8, 1] → 최대 힙 구성 → [8, 5, 3, 1]

1. 8 꺼내서 배열 끝으로 → [1, 5, 3] + [8]
2. 다시 최대 힙 구성 → [5, 1, 3]
3. 5 꺼내서 끝으로 → [3, 1] + [5, 8]
...

최종 결과: [1, 3, 5, 8]
```

### ✅ 장점

- **시간 복잡도가 항상 O(n log n)** (최악도 안정적)
    
- 정렬 안정성은 없지만, **추출 속도 매우 빠름**

### ⚠️ 단점

- 구현 복잡도 높음
    
- 메모리 접근이 불규칙해 캐시 성능 떨어짐

## ✍️ 요약 비교

|정렬|동작 방식|예시 진행 핵심|
|---|---|---|
|**Bubble**|인접 비교, 큰 값 뒤로|여러 번 반복하며 버블처럼 큰 수가 밀려감|
|**Selection**|가장 작은 값 선택|전체에서 최소값 찾아 교환|
|**Insertion**|정렬된 영역에 삽입|정렬된 곳에 값 하나씩 끼워넣음|
|**Merge**|분할 → 병합 정렬|나누고 나누고 합치면서 정렬|
|**Quick**|Pivot 기준으로 나눔|작은 그룹, 큰 그룹 나누고 재귀 정렬

| 정렬             | 설명               | 특징                     |
| -------------- | ---------------- | ---------------------- |
| **Quick Sort** | Pivot 기준으로 나누기   | 빠르지만 최악 O(n²)          |
| **Heap Sort**  | 최대 힙 구성해서 하나씩 꺼냄 | 항상 O(n log n), 느릴 수 있음 |
