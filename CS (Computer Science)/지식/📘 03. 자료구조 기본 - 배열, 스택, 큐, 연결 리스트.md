### ✅ 오늘 배울 네 가지

|자료구조|특징|사용 예|
|---|---|---|
|배열 (Array)|인덱스로 접근 가능, 고정 순서|리스트, 컬렉션뷰, 테이블뷰|

|연결 리스트 (Linked List)|노드 연결 구조|일부 내부 프레임워크에서 사용|

---

## 1. 📋 배열 (Array)

- **연속된 메모리 공간**에 값을 저장
    
- 인덱스로 접근 가능 → `O(1)` 시간 복잡도
    
- Swift에서는 `Array` 타입 사용

```swift
let numbers = [1, 2, 3]
print(numbers[0]) // 1
```

#### 장점

- 빠른 조회 (`O(1)`)

#### 단점

- 중간 삽입/삭제 느림 (`O(n)`)

---

## 2. 📦 스택 (Stack)

- **LIFO: Last In, First Out**
    
- `push` (삽입), `pop` (제거)

```swift
var stack = [Int]()
stack.append(1) // push
let last = stack.removeLast() // pop
```

#### 사용 예

- 뷰 컨트롤러 push/pop
    
- 괄호 검사, 되돌리기

---

## 3. 🚪 큐 (Queue)

- **FIFO: First In, First Out**
    
- `enqueue`, `dequeue`

```swift
struct Queue<T> {
    private var elements: [T] = []
    mutating func enqueue(_ element: T) { elements.append(element) }
    mutating func dequeue() -> T? { elements.isEmpty ? nil : elements.removeFirst() }
}
```

#### 사용 예

- 작업 처리 대기열
    
- BFS(너비 우선 탐색)

---

## 4. 🔗 연결 리스트 (Linked List)

- 노드가 `값 + 다음 노드`를 가지고 연결된 구조

```swift
class Node {
    var value: Int
    var next: Node?

    init(value: Int) {
        self.value = value
    }
}
```

#### 장점

- 삽입/삭제는 빠름 (`O(1)` 위치에서 가능)

#### 단점

- 검색이 느림 (`O(n)`)

## ✅ Q1. 배열과 연결 리스트의 가장 큰 차이점은 무엇인가요?

### 🟢 사용자 답변 요약:

> 배열은 인덱스를 통해 빠르게 조회 가능하지만,  
> 연결 리스트는 노드를 하나하나 거쳐야 해서 느리다.

### ✅ 피드백:

- 핵심인 “조회 성능 차이”를 잘 설명하셨어요!
    
- 배열의 장점과 연결 리스트의 단점에 집중된 건 좋습니다.
    
- **보완할 점:** 연결 리스트의 장점(삽입/삭제 성능)을 한 줄이라도 언급하면 균형 잡힌 답이 됩니다.

### ✅ 개선된 모범답안:

> 배열은 **연속된 메모리 공간에 저장되며 인덱스로 직접 접근이 가능**하기 때문에 조회 속도가 매우 빠릅니다(O(1)).  
> 반면 연결 리스트는 각 요소가 다음 노드를 가리키는 방식이라, 원하는 값을 찾기 위해 처음부터 순차적으로 탐색해야 하므로 **조회는 느립니다(O(n))**.  
> 하지만 반대로, 연결 리스트는 **중간 삽입과 삭제가 빠르다는 장점**이 있어 상황에 따라 적절히 선택해야 합니다.

### ✅ 점수: **92점** 🎯

→ 명확하고 핵심을 짚었어요. 연결 리스트의 장점도 추가했다면 거의 100점!

## ✅ Q2. 큐와 스택은 어떤 문제에서 각각 유리하게 쓰일 수 있을까요?

### 🟢 사용자 답변 요약:

> 스택은 화면 전환 같은 UI 흐름에서,  
> 큐는 작업 우선순위 처리에서 유리하다.

### ✅ 피드백:

- 스택: “화면 이동/되돌리기”라는 현실적 예시가 좋습니다.
    
- 큐: “우선적으로 동작해야 하는 것”이라는 표현은 약간 애매모호해요.
    
    - → **"순서대로 처리"** 혹은 **"선입선출 작업"**이라는 말로 더 명확히 표현 가능!

### ✅ 개선된 모범답안:

> **스택은 후입선출(LIFO) 구조이기 때문에**, 화면 전환의 뒤로 가기(back), undo 기능처럼 **가장 최근 작업부터 되돌려야 하는 상황에 유리**합니다.  
> **큐는 선입선출(FIFO) 구조라서**, 작업 요청이나 이벤트 처리, 프린터 작업 대기열 등 **먼저 들어온 요청부터 순서대로 처리해야 하는 문제**에 적합합니다.

### ✅ 점수: **88점**

→ 실무 감각을 바탕으로 잘 설명하셨고, 큐 부분만 약간 명확히 다듬으면 완벽해요!
