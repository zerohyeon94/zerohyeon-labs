## ✅ 질문 요약

- 재귀 함수가 연속으로 호출되면 **이전 함수가 끝나지 않은 상태에서 새로운 함수가 호출되는 구조** 아닌가요?
    
- 그래서 **호출된 함수 개수만큼 메모리를 소비**하고 있는 건가요?
    

→ **정답: 네, 정확히 맞습니다!** 🎯  
재귀 함수는 **각 호출이 스택에 쌓이는 구조**이고,  
이 스택이 바로 **호출 스택(Call Stack)** 입니다.

---

## 🧠 재귀 함수 호출 구조 = 스택(Stack)

### 예시: `dfsRecursive(1)` → graph는 다음과 같다고 가정합니다.

```swift
let graph: [Int: [Int]] = [
    1: [2, 3],
    2: [4],
    3: [],
    4: []
]
```

### 호출 흐름:

```text
dfsRecursive(1)
├─ dfsRecursive(2)
│  └─ dfsRecursive(4)
├─ dfsRecursive(3)
```

### 호출 스택 구조:

1. `dfsRecursive(1)` 호출됨 → 스택에 쌓임
    
2. 1번 실행 중 → `dfsRecursive(2)` 호출됨 → 또 스택에 쌓임
    
3. 2번 실행 중 → `dfsRecursive(4)` 호출됨 → 또 쌓임
    
4. 이제 4번은 종료 조건에 도달 → `return` → 스택에서 제거됨
    
5. 2번도 더 이상 할 일이 없음 → `return` → 제거
    
6. 1번의 남은 이웃인 `dfsRecursive(3)` 실행됨
    
7. ...
    

---

## 📦 메모리 관점 설명

|함수 호출|스택 상태 (Top → Bottom)|
|---|---|
|`dfsRecursive(4)`|[4, 2, 1]|
|`dfsRecursive(2)`|[2, 1]|
|`dfsRecursive(1)`|[1]|

→ 따라서 함수가 중첩될수록, 그 개수만큼 **메모리가 호출 스택에 쌓입니다.**

→ 이것이 재귀 함수에서 **스택 오버플로우**가 발생할 수 있는 이유입니다.  
예: 피보나치 재귀로 `fibonacci(10000)`을 호출하면 무한히 스택이 쌓이다가 앱이 크래시납니다.

---

## ✅ 핵심 요약

|항목|설명|
|---|---|
|함수 호출 구조|이전 함수가 끝나기 전에 다음 함수가 호출됨|
|메모리 구조|호출될 때마다 **Stack Frame(지역 변수, 매개변수 등)**이 메모리에 쌓임|
|종료 조건|더 이상 재귀가 없거나 조건에 맞으면 **return**하면서 스택에서 빠짐|
|위험|재귀 깊이가 너무 깊으면 **스택 오버플로우(메모리 초과) 위험**|

## ✅ 시각적으로 이해하면 더 쉬워요

```text
스택 메모리 (Top ↓)

dfsRecursive(4)
dfsRecursive(2)
dfsRecursive(1)
```
