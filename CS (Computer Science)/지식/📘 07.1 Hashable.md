## ✅ `Hashable`이란?

> **값을 해시(hash)할 수 있도록 해주는 프로토콜**입니다.  
> 즉, 어떤 타입이 `Hashable`을 채택하면, 그 값을 고유한 **정수(hashValue)**로 변환할 수 있게 해줍니다.

```swift
struct Person: Hashable {
    let name: String
    let age: Int
}
```

이제 `Person`은 Dictionary나 Set에서 **Key**로 사용할 수 있습니다.

---

## 🧠 왜 Hashable이 필요할까요?

### Swift에서 다음과 같은 동작을 하려면 반드시 필요합니다:

|자료구조|요구사항|
|---|---|
|`Dictionary<Key, Value>`|`Key`는 반드시 `Hashable`이어야 함|
|`Set<Element>`|`Element`는 반드시 `Hashable`이어야 함|

### 이유?

👉 내부적으로 키를 `hashValue`로 바꿔서 인덱스를 계산하므로,  
**고유한 정수값을 생성할 수 있어야 하기 때문입니다.**

---

## ✅ 실제 예시

```swift
let dict: [String: Int] = ["apple": 3, "banana": 2]
```

여기서 `"apple"`은 `String` 타입인데,  
`String`은 내부적으로 이미 `Hashable`을 채택하고 있어서 **Key로 사용 가능**한 거예요!

---

## 📦 Hashable이 하는 일

`Hashable`은 내부적으로 두 가지 기능을 제공합니다:

|기능|의미|
|---|---|
|`hash(into:)`|값 → 고유 정수값으로 변환|
|`==` 비교|두 값이 같은지 비교 (Equatable 상속 포함됨)|

> Swift에서는 `Equatable`도 자동 채택되므로, `==` 연산도 같이 사용 가능해요.

---

## ✅ 커스텀 타입에 Hashable을 붙이면?

```swift
struct Pet: Hashable {
    let name: String
}

let a = Pet(name: "Mong")
let b = Pet(name: "Mong")

print(a == b)               // true
print(a.hashValue == b.hashValue) // true
```

이제 `Pet`을 Dictionary나 Set에서 쓸 수 있게 됩니다:

```swift
var set: Set<Pet> = []
set.insert(Pet(name: "Mong"))  // 가능!
```

## 🎯 요약

|항목|설명|
|---|---|
|`Hashable`이란?|값을 고유한 정수(hash)로 바꿔주는 프로토콜|
|언제 필요?|Dictionary의 Key, Set의 Element에 사용될 때|
|Swift 내장 타입|대부분 (Int, String, Bool, Array 등)은 이미 Hashable|
|커스텀 타입|직접 Hashable 채택해야 사용 가능|

## 📌 추가 팁

- Swift 4.1 이상에서는 `Hashable`을 자동으로 채택해줍니다 (속성이 전부 Hashable이면)
    
- 직접 구현도 가능:

```swift
struct CustomKey: Hashable {
    let id: Int
    func hash(into hasher: inout Hasher) {
        hasher.combine(id)
    }
}
```
