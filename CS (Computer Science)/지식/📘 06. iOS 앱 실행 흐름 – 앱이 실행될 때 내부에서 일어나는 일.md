## 1. 🔄 앱 실행 전반 흐름 요약

```text
앱 아이콘 터치 → iOS 시스템이 프로세스 생성 → 런타임 초기화 → AppDelegate → SceneDelegate → 첫 화면 표시
```

---

## 2. 📦 주요 클래스/파일 설명

### ✅ `AppDelegate.swift`

- 앱 실행의 진입점
    
- 앱 초기 설정을 담당 (예: Firebase, 알림 권한 등)

```swift
func application(_:didFinishLaunchingWithOptions:) -> Bool
```

> 앱이 실행될 때 가장 먼저 호출됨 (시작 시 1번만 호출됨)

---

### ✅ `SceneDelegate.swift`

- iOS 13부터 등장 (다중 윈도우 지원)
    
- **UI 생명주기** 담당
    
- 첫 ViewController를 window에 연결하는 역할

```swift
func scene(_:willConnectTo:options:)
```

---

## 3. 🧠 앱의 상태(Life Cycle)

| 상태          | 설명                 | 관련 메서드                    |
| ----------- | ------------------ | ------------------------- |
| Not Running | 앱이 실행되지 않음         | -                         |
| Inactive    | 실행 중이지만 이벤트 수신 안 함 | `sceneWillResignActive`   |
| Active      | 실행 중이고 이벤트 수신      | `sceneDidBecomeActive`    |
| Background  | 화면에는 없지만 코드 실행 가능  | `sceneDidEnterBackground` |
| Suspended   | 백그라운드에서 대기 중 (중지됨) | -                         |

---

## 4. ✅ 실행 흐름 예시 (최초 실행 기준)

```text
1. AppDelegate → didFinishLaunchingWithOptions
2. SceneDelegate → willConnectTo
3. SceneDelegate → sceneDidBecomeActive
4. 뷰가 화면에 표시됨
```

## ✅ Q1. AppDelegate와 SceneDelegate의 역할은 어떻게 다르고, 왜 둘 다 필요한가요?

### 🟢 사용자 요약:

> AppDelegate는 권한 요청 등 설정,  
> SceneDelegate는 첫 화면 UI 설정

### ✅ 피드백:

- **핵심은 잘 짚었어요.**
    
- 하지만 ‘권한 요청’은 AppDelegate의 일부 사례일 뿐이고,  
    **AppDelegate는 앱 전체의 초기화/상태 관리**를,  
    **SceneDelegate는 각 UI Scene의 생명주기/윈도우 관리**를 담당하는 더 큰 의미가 있습니다.

---

### ✅ 개선된 모범답안:

> `AppDelegate`는 앱 전반적인 초기화 작업을 담당하는 곳으로, 앱이 실행될 때 **단 한 번만** 생성됩니다.  
> 예: 앱 전역 설정, 푸시 알림 등록, Firebase 초기화, 권한 요청 등
> 
> 반면, `SceneDelegate`는 **UI 관련 생명주기를 관리**하며, 앱이 여러 개의 윈도우(Scene)를 가질 수 있도록 도와줍니다.  
> 예: 첫 화면의 ViewController 설정, UI 상태 전이 감지
> 
> iOS 13 이후 **다중 창 지원(iPad)**을 위해 SceneDelegate가 도입되었고, 이 둘을 분리함으로써 **앱 로직과 UI 로직을 명확히 분리할 수 있습니다.**

---

### ✅ 점수: **80점**

→ 핵심은 맞고 예시는 괜찮았지만, 역할 설명이 좁고 단편적이었음  
→ "왜 둘 다 필요한가?"에 대한 답변은 보완 필요

## ✅ Q2. 앱이 백그라운드로 전환될 때 호출되는 메서드는 무엇이고, 어떤 일을 처리해야 할까요?

### 🟢 사용자 요약:

> sceneDidEnterBackground 메서드에서 화면에는 없지만 코드가 실행 가능하다고 했다.

### ✅ 피드백:

- `sceneDidEnterBackground`를 정확히 기억하신 점은 아주 좋아요!
    
- 다만 **“어떤 일을 처리해야 할까?”**에 대해 구체적인 설명이 부족합니다.

---

### ✅ 개선된 모범답안:

> 앱이 백그라운드로 전환될 때 `sceneDidEnterBackground`가 호출됩니다.  
> 이 시점에서는 앱이 화면에서 사라지지만, **일정 시간 동안 코드를 실행할 수 있습니다.**
> 
> 이 메서드에서는 다음과 같은 작업을 처리하는 것이 일반적입니다:
> 
> - 데이터 저장 (예: CoreData, UserDefaults)
>     
> - 네트워크 연결 정리
>     
> - 타이머 중지, 리소스 정리
>     
> - 중요한 상태를 백업하여 앱 종료 시 복원 가능하도록 함
