# 📘 Day 8 – 정렬 알고리즘의 이해

정렬은 자료 구조와 알고리즘에서 **기초이자 핵심 주제**입니다.  
대부분의 검색 알고리즘, UI 정렬, 서버 통신 응답 정리 등에 사용됩니다.

---

## ✅ 오늘 배울 개념

### 1. 정렬(Sorting)이란?

- 데이터를 특정 기준(오름차순, 내림차순 등)에 따라 **순서대로 정렬**하는 과정

---

### 2. 주요 정렬 알고리즘 비교

| 알고리즘                   | 평균 시간복잡도   | 안정성 | 특징           |
| ---------------------- | ---------- | --- | ------------ |
| 버블 정렬 (Bubble Sort)    | O(n²)      | ✅   | 느리지만 구현 쉬움   |
| 선택 정렬 (Selection Sort) | O(n²)      | ❌   | 가장 작은 값부터 선택 |
| 삽입 정렬 (Insertion Sort) | O(n²)      | ✅   | 이미 정렬된 경우 빠름 |
| 병합 정렬 (Merge Sort)     | O(n log n) | ✅   | 안정성 있고 빠름    |
| 퀵 정렬 (Quick Sort)      | O(n log n) | ❌   | 가장 널리 사용, 빠름 |
| 힙 정렬 (Heap Sort)       | O(n log n) | ❌   | 힙 자료구조 활용    |

> ✅ *"안정성"*이란: 같은 값의 원소가 **정렬 후에도 순서 유지**되는지 여부

---

### 3. Swift에서 사용하는 정렬 방식

```swift
let array = [3, 1, 4, 2]
let sorted = array.sorted() // 기본 오름차순
let reversed = array.sorted(by: >) // 내림차순
```

Swift의 `sorted()`는 내부적으로 **Introsort**라는 하이브리드 정렬 알고리즘을 사용합니다.  
(Introsort = QuickSort + HeapSort + InsertionSort 혼합)

---
### 4. 알고리즘별 시각적 차이 (이미지 상상용)

- 🔁 **Bubble Sort**: 인접한 두 수를 계속 바꿔가며 끝까지 비교
    
- 🔍 **Selection Sort**: 현재 위치에서 가장 작은 값을 찾아서 앞으로 이동
    
- ⬇️ **Insertion Sort**: 정렬된 부분에 새로운 수를 끼워 넣음
    
- 🧩 **Merge Sort**: 반으로 나누고 합치며 정렬
    
- ⚡ **Quick Sort**: 기준값(Pivot)을 잡고 좌우로 나눠 정렬

# 📚 주요 정렬 알고리즘 6가지 Swift 예시

## 1️⃣ 버블 정렬 (Bubble Sort)

> 인접한 두 수를 비교해가며 큰 값을 뒤로 보내는 방식 (거품처럼 올라감)

```swift
func bubbleSort(_ array: inout [Int]) {
    for i in 0..<array.count {
        for j in 0..<array.count - i - 1 {
            if array[j] > array[j + 1] {
                array.swapAt(j, j + 1)
            }
        }
    }
}

var a1 = [5, 3, 8, 1]
bubbleSort(&a1)
print(a1) // [1, 3, 5, 8]
```

---

## 2️⃣ 선택 정렬 (Selection Sort)

> 가장 작은 값을 선택해 앞으로 보냄

```swift
func selectionSort(_ array: inout [Int]) {
    for i in 0..<array.count {
        var minIndex = i
        for j in i+1..<array.count {
            if array[j] < array[minIndex] {
                minIndex = j
            }
        }
        array.swapAt(i, minIndex)
    }
}

var a2 = [5, 3, 8, 1]
selectionSort(&a2)
print(a2) // [1, 3, 5, 8]
```

---

## 3️⃣ 삽입 정렬 (Insertion Sort)

> 정렬된 부분에 새 값을 적절히 끼워 넣음

```swift
func insertionSort(_ array: inout [Int]) {
    for i in 1..<array.count {
        var j = i
        while j > 0 && array[j] < array[j - 1] {
            array.swapAt(j, j - 1)
            j -= 1
        }
    }
}

var a3 = [5, 3, 8, 1]
insertionSort(&a3)
print(a3) // [1, 3, 5, 8]
```

---

## 4️⃣ 병합 정렬 (Merge Sort)

> 반으로 나눈 후 각각 정렬하고 병합 (재귀 사용)

```swift
func mergeSort(_ array: [Int]) -> [Int] {
    guard array.count > 1 else { return array }

    let mid = array.count / 2
    let left = mergeSort(Array(array[0..<mid]))
    let right = mergeSort(Array(array[mid...]))

    return merge(left, right)
}

func merge(_ left: [Int], _ right: [Int]) -> [Int] {
    var result = [Int]()
    var l = 0, r = 0

    while l < left.count && r < right.count {
        if left[l] < right[r] {
            result.append(left[l])
            l += 1
        } else {
            result.append(right[r])
            r += 1
        }
    }
    return result + left[l...] + right[r...]
}

let a4 = mergeSort([5, 3, 8, 1])
print(a4) // [1, 3, 5, 8]
```

---

## 5️⃣ 퀵 정렬 (Quick Sort)

> Pivot(기준점)을 기준으로 작고 큰 값을 나누고 재귀 정렬

```swift
func quickSort(_ array: [Int]) -> [Int] {
    guard array.count > 1 else { return array }

    let pivot = array[array.count / 2]
    let less = array.filter { $0 < pivot }
    let equal = array.filter { $0 == pivot }
    let greater = array.filter { $0 > pivot }

    return quickSort(less) + equal + quickSort(greater)
}

let a5 = quickSort([5, 3, 8, 1])
print(a5) // [1, 3, 5, 8]
```

---

## 6️⃣ 힙 정렬 (Heap Sort)

> 최대 힙(또는 최소 힙)을 구성해 정렬하는 방식

```swift
func heapSort(_ array: [Int]) -> [Int] {
    var heap = array
    func heapify(_ n: Int, _ i: Int) {
        var largest = i
        let left = 2 * i + 1
        let right = 2 * i + 2

        if left < n && heap[left] > heap[largest] {
            largest = left
        }
        if right < n && heap[right] > heap[largest] {
            largest = right
        }

        if largest != i {
            heap.swapAt(i, largest)
            heapify(n, largest)
        }
    }

    let n = heap.count
    for i in stride(from: n / 2 - 1, through: 0, by: -1) {
        heapify(n, i)
    }
    for i in stride(from: n - 1, through: 1, by: -1) {
        heap.swapAt(0, i)
        heapify(i, 0)
    }

    return heap
}

let a6 = heapSort([5, 3, 8, 1])
print(a6) // [1, 3, 5, 8]
```

## ✅ Q1. 버블 정렬과 삽입 정렬은 어떤 상황에서 적합한가요?

### 📝 사용자 답변:

> 버블 정렬은 간단하게 크기가 작은 배열을 정렬할 때 적합합니다. 삽입 정렬의 경우에는 이미 정렬된 배열에 맞춰서 넣기에 가장 적합합니다.

### 👍 평가:

- 핵심 개념을 잘 짚었습니다!
    
- **버블 정렬은 단순하고 구현이 쉬운 구조에서**, **삽입 정렬은 정렬된 데이터에 조금씩 추가할 때 유리**하다는 관점은 정확합니다.

### ✅ 개선 제안:

조금 더 구체적인 조건을 추가해보면 더 좋습니다:

```swift
- 버블 정렬은 데이터가 거의 섞여 있는 소규모 배열에서 단순 구현이 필요할 때 적합합니다.
- 삽입 정렬은 "거의 정렬된 데이터(예: 한두 개만 순서가 바뀐 경우)"를 빠르게 정렬하는 데 효율적입니다.
```

## ✅ Q2. 퀵 정렬은 평균적으로 빠르지만, 최악의 경우 성능이 떨어지는 이유는 무엇인가요?

### 📝 사용자 답변:

> 기준값을 어떻게 정하느냐에 따라서 성능 차이가 발생할 수 있습니다.

### 👍 평가:

- 아주 **핵심을 정확히** 짚으셨습니다! 👏
    
- Pivot 선택이 퀵 정렬의 성능을 좌우한다는 개념을 잘 이해하셨습니다.

### ✅ 개선 제안:

아래처럼 구체적 예시를 추가하면 더 완성도 높은 설명이 됩니다:

```swift
퀵 정렬은 평균적으로 O(n log n)이지만, 이미 정렬된 배열에 대해 최솟값 또는 최댓값을 Pivot으로 고르면
한쪽으로만 계속 분할되어 O(n²) 성능이 됩니다. 
→ 이를 방지하기 위해 무작위(Pivot Randomization) 또는 중앙값 선택 전략을 사용합니다.
```
