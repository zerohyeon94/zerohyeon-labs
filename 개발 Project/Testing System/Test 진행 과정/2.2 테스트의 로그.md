## **로그별 결과 상세 해설**

### **① Test Suite 로그**

```sql
Test Suite 'All tests' started at ...
```

- **설명:**  
    Xcode에서 Unit Test가 시작되고, 각 Test Suite(테스트 그룹), Test Case(각 함수 단위)가 차례로 실행됩니다.
    
    - 순서: All tests > GagaeSsiTests.xctest > CoreDataManagerTests > testCreateBudgetConfig

### **② CoreData Warning/Error 로그**

```sql
CoreData: warning: Multiple NSEntityDescriptions claim the NSManagedObject subclass 'BudgetConfig' so +entity is unable to disambiguate.
CoreData: error: +[BudgetConfig entity] Failed to find a unique match for an NSEntityDescription to a managed object subclass
```

- **설명:**
    
    - **경고(warning)**:  
        `BudgetConfig`라는 NSManagedObject subclass(엔티티 클래스)를 여러 군데서 중복 정의하거나,  
        두 개의 NSManagedObjectModel에서 같은 이름의 엔티티를 찾았다는 뜻입니다.
        
        - 예) `GagaeSsi.xcdatamodeld`가 여러 번 로딩되었거나, 앱 타겟과 테스트 타겟 각각에서 model이 중복될 때 발생
            
    - **에러(error)**:  
        위 경고의 심화 버전. BudgetConfig에 대해 어떤 모델에 매칭해야 할지 결정하지 못해, CoreData가 올바르게 동작하지 않을 수 있다는 메시지입니다.
        
- **실무 대응법:**
    
    1. **테스트 타겟에 .xcdatamodeld가 중복 추가되어 있지 않은지 확인**
        
    2. **CoreDataManager 생성자에서 modelName이 두 번 이상 로드되지 않도록 설계**
        
    3. **테스트 타겟의 Build Phases > Compile Sources에 model 클래스가 중복되지 않도록 확인**
        
    
    > 대부분 **테스트 환경에서 NSPersistentContainer의 모델이 중복으로 등록**될 때 발생합니다.
    > 
    > _프로젝트 타겟과 테스트 타겟 각각에서 `BudgetConfig.swift` 등 엔티티 클래스가 중복되어 있거나, model 파일이 여러 번 로딩된 경우가 많음._
    
- **실제 데이터 저장/조회에 치명적 문제는 없었기 때문에, 테스트는 성공**  
    → 하지만, _이 경고/에러는 꼭 수정해야 함!_  
    → 장기적으로 더 큰 문제(데이터 불일치, 예외 등)로 이어질 수 있음

## **리드 개발자 관점에서 주니어 대응/학습 조언**

### ① **단계별 학습/대응 팁**

- **CoreData Warning/Error를 그냥 두면 안 된다**  
    테스트가 통과해도, 이런 구조적 경고/에러는 반드시 원인을 찾아서 수정해야 함.  
    → _“테스트 성공=버그 없음”이 아니라, 모든 경고/에러는 왜 발생하는지 원인 분석이 필수_
    
- **테스트 환경 분리/설계 연습**  
    테스트 타겟이 실제 타겟과 데이터 모델(.xcdatamodeld) 및 NSManagedObject subclass를 어떻게 공유/분리하는지 공부  
    → iOS 실무에서 **“테스트 타겟 분리”와 “in-memory store 적용”**은 성장 포인트!
    
- **실행 로그 파악**  
    어떤 로그가 테스트 실행과 관련 있고, 어떤 로그가 앱 구동과 관련된 것인지 구분하는 습관이 필요
    
- **의심가는 부분은 공식문서/Stack Overflow/팀 선임 개발자에게 적극적으로 질문**

### ② **다음 액션 제안**

1. **CoreData 관련 Warning/Error 해결**  
    → _원인 분석 및 해결법 필요하다면 추가로 질문_
    
2. **테스트 타겟의 데이터 모델 및 클래스 구조 점검**  
    → _중복/불필요한 부분 없는지 Xcode의 Target Membership, Build Phases 확인_
    
3. **테스트 성공/실패에만 의존하지 말고, 경고/에러까지 클린하게!**
    
4. **실행 로그에서 의문 나는 부분이 있다면 직접 실행 흐름(순서도 등) 그려보며 파악**
