## 1. **“date == %@”와 “date >= %@ AND date < %@”의 차이**

### ① **date == %@**

- **정확히 “이 날짜(시각 포함)”와 완전히 일치하는 값만** 조회합니다.
    
- 예를 들어, 저장된 값이 `"2025-06-26 00:00:00"`이고,  
    파라미터로 `"2025-06-26 00:00:00"`를 전달하면 조회됨.
    
- 그런데,  
    저장된 값이 `"2025-06-26 15:12:30"`라면,  
    `"2025-06-26 00:00:00"`와 **같지 않으므로 조회되지 않습니다!**
    

---

### ② **date >= %@ AND date < %@**

- **이 구간(하루)의 모든 시각을 포함**  
    (예: `"2025-06-26 00:00:00"`부터 `"2025-06-27 00:00:00"` 직전까지)
    
- 즉,  
    `"2025-06-26 01:00:00"`,  
    `"2025-06-26 12:34:56"`,  
    `"2025-06-26 23:59:59"` 등  
    **모두 해당 날짜의 데이터로 인식되어 조회됨**
    

---

## 2. **왜 “범위” 조건을 쓰는 것이 실무에서 안전한가?**

- **날짜 타입(Date)의 값은 “년/월/일/시/분/초/밀리초”까지 다 들어있음**
    
- 데이터를 저장할 때
    
    - `"2025-06-26 10:22:33"` 등 시간이 반드시 들어가므로
        
    - `date == %@`는 “정확히 00:00:00” 값에만 반응함
        
    - **즉, 대부분의 실제 데이터는 조회가 안 됨!**
        
- **범위(>=, <) 조건은**
    
    - “2025-06-26 하루 전체”에 해당하는 모든 시각을 커버할 수 있음
        
    - **실제 그날에 저장된 모든 SpendingRecord를 빠짐없이 조회**할 수 있음
        

---

## 3. **DailyBudgetModel에서는 “date == %@”만 써도 괜찮은 이유**

- **DailyBudget**은
    
    - 앱에서 “하루당 딱 하나만 생성/저장”하는 구조로 설계한 경우가 많습니다.
        
    - 이럴 때,  
        저장할 때 항상 `date`를 `"해당일 00:00:00"`로 통일해서 저장한다면  
        `date == %@`로 조회해도 정상 동작
	    ```swift
	    let today = Calendar.current.startOfDay(for: Date())
	    // startOfDay로 설정하면 항상 시분초는 00:00:00으로 저장된다.
		```
        
- **SpendingRecord**는
    
    - 하루에 여러 건(“점심”, “저녁”, “간식” 등)이 생기므로
        
    - 각각의 시간까지 다르기 때문에 **범위 조회가 필수**

---

## 4. **실무 Best Practice 요약**

- **여러 개가 발생할 수 있는 데이터(SpendingRecord 등):**  
    → **“date >= %@ AND date < %@”** 조건을 써서 “해당일 전체” 커버
    
- **해당 일자에 반드시 1개만 있는 데이터(DailyBudget 등):**  
    → date == %@ (단, 저장 시 반드시 00:00:00으로 통일할 것!)  
    → 혹시라도 시각이 들어갈 수 있다면, 범위조건이 더 안전
    

---

## 5. **정리**

- SpendingRecord 등 **하루에 여러 건 쌓이는 데이터**는  
    반드시 **“범위 조건”**(>=, <)으로 조회해야  
    “그날의 모든 지출”이 빠짐없이 다 조회됩니다.
    
- DailyBudget 등 **딱 1건만 저장되는 데이터**는  
    `date == %@`로 해도 되지만,  
    저장/조회 포맷이 100% 일치할 때만 안전합니다.
    

---

## 6. **실무자 관점 한 줄 요약**

> **“하루의 데이터를 조회할 때는  
> 대부분 범위 조건(‘date >= %@ AND date < %@’)이 실무에서 안전하다!”**

---

### ⚡️ **추가 팁**

- 실제로 `Calendar.current.startOfDay(for: date)`와  
    `date(byAdding: .day, value: 1, to: startOfDay)`  
    이 두 값을 써서 항상 “자정 ~ 다음날 자정 직전” 범위로 조회하면  
    타임존/서머타임 이슈도 안전하게 막을 수 있습니다!
