### 🔍 UI vs UX 차이

|항목|UI (User Interface)|UX (User Experience)|
|---|---|---|
|정의|사용자가 **보는 것**과 **직접 조작하는 요소** (버튼, 텍스트필드, 색상, 레이아웃 등)|사용자가 **앱을 이용하면서 느끼는 경험 전반** (편의성, 직관성, 만족도 등)|
|예시|텍스트 필드에 숫자 입력 시 포맷이 적용되는 UI 요소|숫자 쉼표 포맷이 적용되어 금액을 쉽게 인지할 수 있다는 사용자 경험|
|초점|시각적 디자인과 조작 방식|흐름, 효율성, 감정적인 만족감|

## 🌞 라이트 & 🌚 다크 모드 색상 조합

|역할|라이트 모드 예시|HEX|다크 모드 매칭 색상|HEX|설명|
|---|---|---|---|---|---|
|**배경색**|Light Gray|`#F8F8F8`|Dark Gray|`#1C1C1E`|iOS 기본 다크 배경 톤과 유사|
|**메인 컬러** (돼지색)|Soft Pink|`#FFAFAF`|Dusty Pink|`#FFBFC0`|붉은빛 유지 + 어두운 배경에서 또렷함|
|**포인트 컬러** (동전색)|Gold Yellow|`#FFD766`|Warm Gold|`#FFDE7D`|채도는 유지하면서도 부드러운 느낌|
|**텍스트 색상**|Charcoal|`#3C2A1E`|Off White|`#F2F2F2`|텍스트 대비 확보|
|**보조 텍스트 색상**|Dark Gray|`#555555`|Medium Gray|`#A1A1A1`|보조 정보로 적절한 명도 유지|

### 📱 다크 모드에서의 디자인 팁

- **Soft Pink**와 같이 밝은 색은 다크 배경에서 _조금 더 진하게 보이므로_ 적절한 채도 조절 필요
    
- 텍스트는 기본적으로 `Label Color` (iOS 시스템 색상)를 활용하면 자동 대응됨
    
- 버튼, 아이콘 배경 등은 투명도를 활용한 블러 효과 (`systemMaterial`)도 고려해볼 수 있습니다
