### 1. 🎯 **앱의 목적**

월급을 받는 직장인 및 아르바이트생이 **고정 지출을 제외한 실사용 가능 금액을 매일 확인하고**, 절약하면 **다음 날로 이월**되는 형식으로 재미있고 실용적으로 소비를 관리할 수 있도록 돕는 앱.

---

### 2. 👥 **타겟 사용자**

- 20~40대 사회 초년생, 직장인, 알바생
- 소비 계획이 필요하지만 어렵게 느끼는 사람
- 간단한 방식으로 지출을 관리하고 싶은 사용자

---

### 3. 🧮 **핵심 기능**

|기능|설명|
|---|---|
|월급 정보 입력|월급일과 월급 금액을 입력 (매월 반복 가능)|
|고정 지출 등록|집세, 보험, 통신비 등 항목별로 입력|
|일일 예산 자동 계산|`(월급 - 고정비) / 다음 월급일까지 남은 일수`|
|예산 이월 기능|당일 절약한 금액은 다음 날 예산에 추가됨|
|오늘의 예산 표시|"오늘 쓸 수 있는 금액", 남은 일수 등 표시|
|소비 기록|사용한 금액을 등록하여 예산에서 차감|
|통계 보기 (옵션)|주간/월간 예산 흐름, 절약 추이 시각화|

---

# 기능 구현 단계
## ✅ 1단계: **기초 데이터 구조 설계 (Model 설계)**

**목표**: 앱의 핵심 데이터 구조를 정의합니다.  
### 👉 필요한 데이터 모델

```swift
struct BudgetConfig {
    var salary: Double
    var payday: Date
    var fixedCosts: [FixedCost]
}

struct FixedCost {
    var name: String
    var amount: Double
}

struct DailyBudget {
    var date: Date
    var availableAmount: Double
    var spentAmount: Double
}
```

➡️ 이걸 토대로 ViewModel과 CoreData/Storage 설계를 시작합니다.

---

## ✅ 2단계: **기본 화면 흐름 잡기 (Presentation Layer)**

**목표**: ViewController를 만들고, 화면 전환 흐름을 연결합니다.

|화면|구성 요소|
|---|---|
|📥 설정 화면 (초기 진입)|월급일, 월급액, 고정지출 입력|
|🏠 홈 화면|오늘의 예산, 이월 금액, 소비 기록 버튼|
|🧾 소비 기록 화면|소비 입력, 내역 확인|

- UIKit 기준으로 `UINavigationController`를 활용한 화면 전환 구조 추천    

---

## ✅ 3단계: **오늘의 예산 계산 로직 구현**

**목표**: Core 로직을 구현하여 화면에 예산 표시

```swift
let days = DateHelper.daysUntilNextPayday(from: today, payday: config.payday)
let daily = (config.salary - totalFixedCost) / Double(days)
```

- 절약된 금액 이월 로직도 설계    

---

## ✅ 4단계: **데이터 저장소 설계 (CoreData 또는 UserDefaults)**

**목표**: 앱 종료 후에도 데이터를 유지하도록 저장 기능 구현

- config: `UserDefaults` 추천
- 소비 기록, DailyBudget: `CoreData` 추천 (복잡성 대비 유지 용이)

---

## ✅ 5단계: **UI/UX 디테일 개선 및 예산 흐름 시각화**

- 소비 내역 추가/삭제
- 오늘의 예산 변동 시 애니메이션 효과
- 절약 이월 금액 시각화 (예: +13,000원 남음)

---
