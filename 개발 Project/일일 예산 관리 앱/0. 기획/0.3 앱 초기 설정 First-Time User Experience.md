초기 사용자 경험(First-Time User Experience, FTUE)을 잘 설계하는 것은 **앱의 이탈률을 줄이고 정착률을 높이는 핵심**입니다.

질문을 정리하자면:

> 앱 첫 실행 시 **월급과 고정비를 입력받는 화면**을 어떻게 구성할지?
> 
> - 각각 별도 화면으로 나눌지?
>     
> - 입력 폼은 어디에 배치할지?

이 부분을 함께 고려해서 답변드릴게요.

---

## ✅ 우선 결론부터

### ✅ 추천 구조:

**두 개의 화면으로 나누되, 하나의 플로우로 연결된 형태**를 추천합니다.

|순서|화면 이름|설명|
|---|---|---|
|1|월급 입력 화면|첫 화면. 핵심 수입 정보 입력|
|2|고정비 입력 화면|다음 화면. 필수 지출 정보 입력|
|3|홈 화면|계산된 결과 제공|

---

## 🧩 왜 나누는 게 좋을까요?

### 장점:

- 한 화면에 너무 많은 정보를 보여주지 않아 사용자가 덜 부담됨
    
- UX 흐름이 **단계적으로 자연스러움 (Onboarding 스타일)**  
    예:
    
    > “먼저 월급을 입력해주세요” → 다음 → “이번 달 고정비를 입력해주세요”
    

### 참고 앱 예시:

- **토스**: 첫 입력을 단계별로 구분
- **뱅크샐러드**: 고정비/소비 입력도 하나씩 나누어 친절하게 유도

---

## 🧱 각 화면 구성안 (와이어프레임 구조 기준)

### 📱 1. 월급 입력 화면 (SalaryInputView)

```swift
[가게씨 아이콘]       ← 앱 친근한 느낌
"월급을 입력해주세요"  ← 타이틀

[ ₩    2,500,000 ]     ← 숫자 키패드 기반 입력 (comma 포함)

[ 월급일 선택: 25일 ▾ ]

[다음으로] 버튼       ← 고정비 입력 화면으로 이동
```

---

### 📱 2. 고정비 입력 화면 (FixedExpenseInputView)

```swift
[가게씨 아이콘]
"이번 달 고정비를 입력해주세요"

[ 고정비 항목 1  ₩ 입력 ]  [+] 추가 버튼

[ 고정비 항목 2  ₩ 입력 ]

[ 완료하고 시작하기 ] 버튼 → 홈으로 이동
```

---

## 🧩 위치 배치 가이드

|요소|위치 제안|이유|
|---|---|---|
|타이틀 텍스트|화면 상단 중앙 or 왼쪽|사용자의 시선 흐름과 맞음|
|금액 입력 필드|중간 강조|메인 액션이므로 화면의 중심에|
|추가 버튼|금액 입력 필드 옆|UX 플로우 자연스럽게 확장 가능|
|하단 버튼|고정비 입력 완료 후 하단 고정|iOS에서 익숙한 위치, 접근성 고려|

---

## ✅ 보너스 UX 팁

- 금액 입력 시 **천 단위 쉼표** 필수!
- 고정비 항목은 **기본 항목 몇 개 제공** (예: 월세, 핸드폰, 구독서비스 등)
- 처음 등록 후에도 설정 화면에서 수정 가능하게 안내

## ✍️ 정리

- 처음 실행 시 2단계 입력 플로우 추천 (월급 → 고정비)
- 각각 **개별 화면으로 구성**하되, 하나의 플로우처럼 자연스럽게 전환
- 금액 입력은 키패드 전용 UI + 쉼표 처리
