## ✅ 두 가지 접근 방식 비교

|전략|설명|장점|단점|추천 상황|
|---|---|---|---|---|
|**1. 월급일에 한 달치 DailyBudget을 미리 생성**|월급일 기준으로 해당 월 전체 날짜에 대한 DailyBudget 생성|- 한 번만 계산  <br>- 이월 포함해서 매일 정해진 예산 확보 가능|- 사용자 미접속 시 처리 누락 가능  <br>- 유연성 떨어짐|**오프라인 중심 / 한 번 생성으로 처리 끝내고 싶을 때**|

## ✅ 현실적인 추천 방식: **2번, 앱 실행 시 필요한 날만 생성**

### ✅ 이유

- iOS 앱은 **백그라운드 동작이 제한**되므로, 자정 자동 생성은 시스템 제약이 큽니다
    
- 대신 **앱이 실행될 때마다 "오늘의 DailyBudget이 없으면 생성"** 방식이 자연스럽고 간단합니다

---

## ✅ 구현 방식

### 1. `CoreDataManager` 또는 `DailyBudgetManager`에 다음 함수 추가:

```swift
func ensureTodayDailyBudgetExists(baseAmount: Int) -> DailyBudget {
    let today = Calendar.current.startOfDay(for: Date())

    if let existing = fetchDailyBudget(on: today) {
        return existing
    }

    // 새로 생성
    let dailyBudget = DailyBudget(context: context)
    dailyBudget.date = today
    dailyBudget.availableAmount = NSDecimalNumber(value: baseAmount)
    dailyBudget.spentAmount = 0
    saveContext()

    return dailyBudget
}
```

---

### 2. HomeViewModel에서 다음처럼 호출:

```swift
func fetchTodayBudget() {
    let todayBudget = CoreDataManager.shared.ensureTodayDailyBudgetExists(baseAmount: 계산된기본금액)
    ...
}
```

---

### 💡 baseAmount는 어떻게 정하나요?

- 월급일과 고정비를 바탕으로 "이번 달 하루 예산"을 계산
    
- 이건 Setup 단계에서 이미 저장된 BudgetConfig에서 유도할 수 있음

## ✅ 한 달치 미리 생성하려면?

```swift
func generateMonthBudget(for startDate: Date, baseAmount: Int) {
    let calendar = Calendar.current
    for offset in 0..<30 {
        guard let date = calendar.date(byAdding: .day, value: offset, to: startDate) else { continue }

        if fetchDailyBudget(on: date) == nil {
            let budget = DailyBudget(context: context)
            budget.date = date
            budget.availableAmount = NSDecimalNumber(value: baseAmount)
            budget.spentAmount = 0
        }
    }
    saveContext()
}
```
