# DailyBudgetModel: 오늘 이용 가능 금액 계산 정책

## 개요

- 일일 예산 관리 및 이월(Carry Over) 금액 처리를 위한 모델 설계 기준 정리
- 별도의 합계 필드 없이, **계산 속성(computed property)**을 통해 실시간 합산 방식으로 구현

## 설계 이유

- 하루 최대 금액(예산/이월/사용)이 30건 이하로 데이터량이 작으므로, 계산속성으로 처리해도 성능 문제 없음
- 별도 합계 필드 관리보다 데이터 일관성과 코드 단순성이 높음

## 모델 예시

```swift
struct DailyBudgetModel {
    var availableAmount: Int              // 오늘의 기본 예산
    var date: Date
    var carryOverSources: [CarryOverSourceModel] // 이월 금액 목록
    var spendingRecords: [SpendingRecordModel]   // 사용 금액 목록

    // 오늘 실제로 쓸 수 있는 총 금액
    var todayAvailable: Int {
        let carry = carryOverSources.map { $0.amount }.reduce(0, +)
        let spent = spendingRecords.map { $0.amount }.reduce(0, +)
        return availableAmount + carry - spent
    }
}
```

---

## 장점

- 데이터 추가/수정/삭제 시 별도 합계 갱신 필요 없음
    
- 데이터/코드 불일치 가능성 최소화
    
- 언제든 최신 값이 자동 반영
