## ✅ 관계 구조 요약

### 1. `DailyBudget` (중심 엔티티)

- 하루 단위로 기록
    
- 아래 두 개의 서브 엔티티를 가짐

|속성명|관계 대상|관계 타입|설명|
|---|---|---|---|
|`spendingRecords`|SpendingRecord|To-Many|해당 날짜의 소비 내역들|
|`carryOverSources`|CarryOverSource|To-Many|이전 날짜로부터 이월된 내역들|

---

### 2. `SpendingRecord`

|속성명|타입|설명|
|---|---|---|
|`dailyBudget`|To-One ← DailyBudget|이 소비 기록이 속한 날짜|

> ✅ inverse: `spendingRecords`

---

### 3. `CarryOverSource`

|속성명|타입|설명|
|---|---|---|
|`targetBudget`|To-One ← DailyBudget|이 금액이 이월된 날짜 (예: 5월 27일)|

> ✅ inverse: `carryOverSources`

---

## 📊 시각적 관계 구조 (도식)

```swift
DailyBudget (2025.05.27)
├── spendingRecords: [SpendingRecord]  ← 오늘 소비 내역들
├── carryOverSources: [CarryOverSource] ← 이전 날짜들의 절약금

SpendingRecord
└── dailyBudget → DailyBudget

CarryOverSource
└── targetBudget → DailyBudget
└── fromDate     → Date (이월 출처)
```

---

## ✅ 정리

|Entity|핵심 관계|
|---|---|
|`DailyBudget`|1 : N → `SpendingRecord` / `CarryOverSource`|
|`SpendingRecord`|N : 1 → `DailyBudget`|
|`CarryOverSource`|N : 1 → `DailyBudget` (`toDate`)|
