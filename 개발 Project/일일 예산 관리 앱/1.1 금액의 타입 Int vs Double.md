## ✅ 금액의 타입을 정할 때 Int vs Double

### 👉 **정수 단위 금액만 다룬다면 `Int`가 더 안전하고 깔끔합니다.**

`Double`은 부동소수점 연산으로 인해 **오차가 발생할 수 있기 때문**입니다.

---

## 🔍 하지만 왜 때로는 `Double`을 쓰기도 하는가?

|이유|설명|
|---|---|
|✅ 금액을 **나누는 연산**이 있을 때|예: `(3000000 - 500000) / 31` → 결과가 소수점 포함됨|
|✅ **하루 예산 계산**은 소수점으로 떨어질 수 있음|하루 예산이 `96,774.193548387`처럼 될 수 있음|
|✅ 통화 단위를 **소수점까지 보여주고 싶을 때**|예: 일본 엔이 아닌 USD 등 → `$1.25`, `₩6.5`|

---

## 🔧 실전에서는 보통 이렇게 처리합니다:

### 💡 금액의 저장은 `Int`로 하고

### 💡 연산에서만 `Double`로 변환 → 결과를 반올림 또는 버림해서 다시 `Int`로 변환

```swift
let total: Int = 3000000
let fixed: Int = 500000
let days: Int = 31

let dailyBudget = Double(total - fixed) / Double(days)
// 80645.16...

let roundedBudget = Int(dailyBudget.rounded(.down))  // 80645원
```

➡️ 이렇게 하면 **오차 없이, UX도 깔끔**하게 유지됩니다.

## ✅ 가계씨 앱에 추천하는 방식

|항목|타입|이유|
|---|---|---|
|`salary`|**Int**|소수점 없는 입력 (₩ 단위)|
|`fixedCost.amount`|**Int**|마찬가지|
|`availableAmount`, `spentAmount`, `carryOverAmount`|**Int**|실제 소비, 이월도 정수 단위가 적절|
|계산식 내부|`Double` → `Int` 변환|연산 정확도를 위한 일시적 사용|

---

## 📌 참고: 실무에서 금액 타입 선택 기준

|상황|권장 타입|
|---|---|
|일반 소비자 대상 앱 (₩ 단위)|✅ `Int`|
|외화 처리, 금융 앱, 수수료 계산|✅ `Decimal` (정확도 높음), `NSDecimalNumber`|
|일일 예산 등 계산 목적|계산은 `Double`, 저장은 `Int`|

---

**요약**:

> `salary`, `amount`, `dailyBudget` 등은 `Int`로 저장하고,  
> 소수점 연산이 필요한 계산 시에만 `Double`을 쓰고 `Int`로 변환하는 게 최적입니다.
