## 🧠 당신의 이해 요약 & 피드백

### ✔️ 1. SceneDelegate에서 CoreData 유무에 따라 화면 분기

> ✅ 맞습니다!  
> `CoreDataManager.shared.fetchBudgetModel()` 등의 결과를 기준으로

- 설정 완료됨 → `MainTabBarController`
    
- 설정 없음 → `Setup` 화면으로

---

### ✔️ 2. 설정 정보가 없을 경우

```swift
let viewModel = SetupViewModel()
let setupVC = SetupSalaryViewController(viewModel: viewModel)
```

> ✅ 정확합니다.  
> 이렇게 하면 `SetupViewModel` 인스턴스를 하나만 만들고,  
> **`SetupSalary → SetupFixedCost` 등 여러 화면에서 동일한 ViewModel을 공유**할 수 있게 됩니다.

---

### ✔️ 3. SetupSalaryViewController는 init으로 전달받은 ViewModel을 갖는다

```swift
init(viewModel: SetupViewModel) { ... }
```

> ✅ 맞습니다!  
> ViewController는 이 ViewModel을 참조하게 되며,  
> **화면 간 ViewModel이 유지되므로 상태(입력값 등)를 공유할 수 있습니다.**

---

### ✔️ 4. 최종 저장은 ViewModel 내부의 model 값을 CoreData에 넘긴다

```swift
CoreDataManager.shared.save(budgetModel: viewModel.model)
```

> ✅ 완벽하게 맞습니다.  
> `viewModel.model`은 `BudgetConfigModel`이며,

- 사용자의 **월급, 월급일, 고정비 정보**가 여기에 담깁니다.
    
- 이걸 `CoreData`에 저장하는 시점은 **SetupFixedCostViewController의 완료 버튼에서** 발생하게 됩니다.

---

## 💯 요약하면

```swift
[SceneDelegate]
 ↓ (CoreData에 BudgetConfig 없으면)
SetupViewModel 생성
 ↓
SetupSalaryViewController(viewModel)
 ↓
사용자 입력 → viewModel.tempSalary 등 → model에 최종 저장
 ↓
SetupFixedCostViewController(viewModel)
 ↓
"저장하기" → CoreDataManager.save(viewModel.model)
 ↓
MainTabBarController로 전환
```
