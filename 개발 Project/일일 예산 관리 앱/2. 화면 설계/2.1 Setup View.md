## ✨ 핵심 UX 포인트

- 월급과 월급일은 **상단에 고정** (필수 입력)
- 고정비 섹션은 **Optional** 표시
    - UITextField + `+` 버튼으로 항목 추가 가능
    - 고정비가 없어도 진행 가능
- 저장 버튼은 **입력 유효성 검사 후만 활성화**

## ✅ 두 단계로 나누는 구조 (Step-by-Step 방식)

### 📄 1페이지: 필수 입력

> **[SetupSalaryViewController]**

- 월급 입력
- 월급일 입력
- **[다음]** 버튼 → 유효성 검증 → 다음 페이지로 이동

---

### 📄 2페이지: 선택 입력

> **[SetupFixedCostViewController]**

- 고정비 목록 (동적으로 추가/삭제)
- **[건너뛰기] / [저장하고 시작하기]** 버튼

---

## 🧠 이 방식이 좋은 이유

|항목|장점|
|---|---|
|📌 중요도에 따라 흐름 유도|필수 항목을 **먼저 입력하도록 설계** 가능|
|📱 UI 깔끔함|각 화면에 정보가 적절히 분산되어 **가독성/입력성** 향상|
|🧼 고정비가 없는 사용자에게 부담 X|"건너뛰기" 버튼만으로 다음으로 넘어갈 수 있음|
|🧩 설계 확장성 ↑|나중에 소비 습관 질문, 테마 선택, 프로필 등 추가 가능|

## 💡 ViewModel은 공유할 수도 있어요

```swift
class SetupViewModel {
    var salary: Int?
    var payday: Date?
    var fixedCosts: [FixedCostModel] = []

    var isValid: Bool {
        return salary != nil && payday != nil
    }
}
```

두 개의 ViewController에서 같은 ViewModel을 주입해서 공유하는 방식으로 연결하면 깔끔하게 상태를 유지할 수 있습니다.
