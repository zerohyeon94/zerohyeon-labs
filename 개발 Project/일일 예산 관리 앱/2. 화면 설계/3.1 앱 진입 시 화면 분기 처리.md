
> ✅ 앱의 진입 시점에서 CoreData에 필요한 설정 정보(월급, 월급일, 고정비)가 있는지를 먼저 판단해서  
> 그에 따라 `SetupView` 또는 `MainTabBar`로 **분기 처리하는 게 UX와 구조상 훨씬 깔끔합니다.**

---

## ✅ 왜 진입 시 분기가 더 좋은가요?

|위치|장점|
|---|---|
|✅ **앱 시작 시 분기 (SceneDelegate)**|홈 뷰 로딩 전에 미리 분기 가능 → **UI 깔끔**  <br>불필요한 초기 뷰 그리기 방지|
|❌ HomeViewController에서 분기|사용자가 HomeView까지 간 뒤 화면이 갑자기 바뀜 → UX 불편  <br>뒤로 가기 버튼 이슈 발생 가능|

---

## ✅ SceneDelegate.swift 예시 (진입 분기 처리)

```swift
func scene(_ scene: UIScene, willConnectTo session: UISceneSession, options connectionOptions: UIScene.ConnectionOptions) {
    guard let windowScene = (scene as? UIWindowScene) else { return }

    let window = UIWindow(windowScene: windowScene)

    let initialVC: UIViewController

    if CoreDataManager.shared.fetchBudgetModel() != nil {
        // 저장된 BudgetConfig가 존재함 → 메인 탭으로 진입
        initialVC = MainTabBarController()
    } else {
        // 설정 정보 없음 → Setup 진행
        let viewModel = SetupViewModel()
        initialVC = UINavigationController(rootViewController: SetupSalaryViewController(viewModel: viewModel))
    }

    window.rootViewController = initialVC
    window.makeKeyAndVisible()
    self.window = window
}
```

---

## ✅ CoreDataManager 측 로직 예시

```swift
func fetchBudgetModel() -> BudgetConfigModel? {
    let request = NSFetchRequest<NSManagedObject>(entityName: "BudgetConfig")
    request.fetchLimit = 1
    guard let config = try? context.fetch(request).first else { return nil }

    // model 변환 생략 가능
    return BudgetConfigModel(salary: ..., payday: ..., fixedCosts: [...])
}
```

## ✅ UX 흐름

```swift
앱 실행
  ↓
CoreData 검사
  ↓
BudgetConfig 있음? → MainTabBar
BudgetConfig 없음 → Setup 시작
```

## 📌 추가 UX 팁

- 설정을 완료한 뒤에는 `UINavigationController.setViewControllers([MainTabBarController()], animated: true)` 로 전환하면 백스택 없이 메인으로 전환할 수 있습니다.

---

💡 이 구조로 구현하면  
**HomeViewController는 언제나 완전한 데이터가 있는 상태로만 진입**하므로,  
UI 로딩 시 복잡한 예외 처리를 하지 않아도 됩니다.
