# 왜 버튼 탭이 안 먹었을까? (가장 흔한 원인 Top 6)

1. **배경 Tap 제스처가 터치를 “먹어버림”**
    
    - 키보드 내리려고 `UITapGestureRecognizer`를 `view`에 올려둔 경우, 기본값 `cancelsTouchesInView = true` 때문에 버튼 터치가 취소됩니다.
        
    - **해결**
        
	    ```swift
	    let tap = UITapGestureRecognizer(target: self, action: #selector(dismissKeyboard))
		tap.cancelsTouchesInView = false   // ⬅️ 버튼 터치 통과
		view.addGestureRecognizer(tap)
        ```
        
2. **버튼이 다른 뷰(스크롤뷰 등) 아래에 깔림**
    
    - `z-order`에서 아래쪽이면 터치가 위의 뷰가 가로챕니다. (특히 `UIScrollView`, `UIVisualEffectView`, dimming view 등)
        
    - **확인/해결**
        
        - Xcode의 **Debug View Hierarchy**로 버튼이 최상단인지 확인
            
        - 필요하면 `view.bringSubviewToFront(signUpButton)`
            
3. **실제로 키보드에 가려져 있음**
    
    - `keyboardLayoutGuide.topAnchor`는 “키보드의 **윗변**”이에요. 오토레이아웃/애니메이션 타이밍에 따라 순간적으로 겹칠 수 있습니다(특히 예측 바/하드웨어 키보드 전환 등).
        
    - **해결 (애니메이션 동기화)**
        
	    ```swift
	    // 레이아웃 변경이 있다면 키보드 애니메이션 블록과 함께
		UIView.animate(withDuration: duration, delay: 0, options: curve) {
		    self.view.layoutIfNeeded()
		}
        ```
        
        (보통은 `keyboardLayoutGuide`가 알아서 맞춰주지만, 레이아웃 업데이트 타이밍이 어긋나면 즉시 반영해주는 편이 안전합니다.)
        
4. **버튼 자체 속성으로 터치 불가 상태**
    
    - `isEnabled == false`, `isUserInteractionEnabled == false`, `alpha < 0.01`, `isHidden == true` 면 터치 안 됩니다.
        
    - Rx 바인딩에서 로딩 중 `isEnabled`를 꺼두는 로직이 있다면 그 상태일 수 있어요.
        
    - **확인**
        
        ```swift
	    print(signUpButton.isEnabled, signUpButton.isUserInteractionEnabled, signUpButton.alpha, signUpButton.isHidden)
        ```
        
5. **상위 뷰가 터치를 막는 상태**
    
    - 상위 뷰(예: overlay container)가 `isUserInteractionEnabled = false` 이거나, 전체를 덮는 투명 뷰가 손가락을 먼저 받는 경우.
        
    - **해결**
        
        - 오버레이 뷰에 `isUserInteractionEnabled = false`
            
        - 필요 시 오버레이에 `hitTest(_:with:)` 커스터마이징
            
6. **오토레이아웃이 만든 실제 프레임이 터치 영역 밖**
    
    - 제약 충돌/우선순위로 버튼의 frame이 예상과 다를 수 있습니다(너무 작거나 off-screen).
        
    - **확인**
        
        ```swift
	    view.layoutIfNeeded()
		print("Button frame:", signUpButton.frame) // 화면 안/크기 OK?
        ```

# 실제 붐빔 앱에서 문제가 되었던 부분

# 무엇이 영향을 줬나 (원인)

- **BaseViewController의 배경 탭 제스처**가 항상 활성화되어 있었어요.
    
    ```swift
	let tap = UITapGestureRecognizer(target: self, action: #selector(dismissKeyboard))
	tap.cancelsTouchesInView = false   // ← 터치는 취소하지 않지만
	tap.delegate = self
	view.addGestureRecognizer(tap)
    ```
    
- 이 제스처는 **버튼 위를 탭해도 인식**되어 `dismissKeyboard()` → `view.endEditing(true)`를 실행.
    
- 키보드가 내려가면서 **keyboardLayoutGuide에 붙은 버튼의 위치가 즉시 위로 이동**했고,  
    탭 수행 중 프레임이 바뀌어 **손가락이 ‘버튼 밖’으로 벗어나며 `touchUpInside`/`rx.tap`이 불발**되는 상황이 발생했습니다.
    
    - 핵심: `cancelsTouchesInView = false`라도 **제스처의 액션은 실행**됩니다. 액션(키보드 내림)이 **레이아웃 변화를 유발**해 버튼 탭이 무효화된 겁니다.

# 무엇이 보였나 (증상)

- 버튼을 눌러도 **키보드만 내려가고 실제 버튼 동작이 안 됨**.
    
- 버튼 코드 안에서 **`view.endEditing(true)`를 먼저 호출**하자 **이후 동작이 정상 실행**됨.

# 언제 특히 잘 재현되나 (재현 조건)

- 버튼이 **`view.keyboardLayoutGuide.topAnchor`** 에 붙어 있고
    
- **배경 탭 제스처가 버튼 영역도 받는 상태**이며
    
- 탭과 동시에 키보드가 내려가 **버튼 프레임이 이동**하는 경우.

# 어떻게 해결했나 (적용한 해결)

- **버튼 탭 처리에서 `view.endEditing(true)`를 먼저 호출**해 키보드를 내려준 뒤, 화면 전환/다음 로직을 실행.
    
    ```swift
	signUpButton.rx.tap
	  .withLatestFrom(output.isSignupEnabled)
	  .filter { $0 }
	  .subscribe(onNext: { [weak self] _ in
	      self?.view.endEditing(true)   // ✅ 먼저 키보드 내림
	      self?.presentTerms()          // 다음 동작
	  })
	  .disposed(by: bag)
    ```

# 더 견고한 권장 수정 (재발 방지)

1. **제스처가 UIControl에서는 아예 동작하지 않게 필터링 (가장 권장)**
    
    ```swift
	extension BaseViewController: UIGestureRecognizerDelegate {
	    func gestureRecognizer(_ g: UIGestureRecognizer, shouldReceive touch: UITouch) -> Bool {
	        if touch.view is UIControl { return false } // 버튼/스위치/텍스트필드 등
	        return true
	    }
	}
    ```
    
2. (대안) **제스처를 ‘배경 전용 컨테이너’에만 부착**하여 버튼 계층과 분리.
    
3. (보조) 키보드 애니메이션과 **레이아웃 동기화**로 중간 상태 최소화.
    
    ```swift
    UIView.animate(withDuration: duration, delay: 0, options: curve) {
	    self.view.layoutIfNeeded()
	}
    ```

## 결론 요약

- **영향 원인:** 배경 탭 제스처가 버튼 탭에서도 동작해 **키보드 내림 → 레이아웃 이동 → 탭 불발**.
    
- **당장 해결:** 버튼 처리에서 **먼저 `endEditing`** 실행.
    
- **근본 해결:** **UIControl은 제스처가 받지 않도록** `UIGestureRecognizerDelegate`로 필터링(+필요시 배경 컨테이너로 분리).
